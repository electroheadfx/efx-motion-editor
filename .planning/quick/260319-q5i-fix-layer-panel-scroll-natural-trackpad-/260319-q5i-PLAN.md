**TimelineInteraction.ts -- rewrite onWheel (lines 634-679):**

Replace the current modifier dispatch with this logic:

1. `if (e.ctrlKey)` -- Zoom at cursor (Ctrl+scroll for mouse, pinch-to-zoom on trackpad also sets ctrlKey on macOS). Keep existing zoom logic (factor from deltaY, frame-under-cursor stabilization).

2. `else if (e.metaKey)` -- Cmd+scroll = vertical scroll. Use deltaY (the dominant axis for mouse wheel). Clamp using `timelineStore.maxScrollY.peek()`. This gives mouse users Cmd+scroll for vertical scrolling.

3. `else if (e.shiftKey)` -- Shift+scroll = vertical scroll (keep existing behavior as fallback). macOS swaps deltaY to deltaX when Shift held, so use whichever axis has a value.

4. `else` (no modifier) -- Natural scrolling:
   - `deltaX !== 0`: horizontal scroll (`scrollX += deltaX`)
   - `deltaY !== 0`: vertical scroll (`scrollY += deltaY`), clamp to `[0, maxScrollY]`

   This is the critical change: bare deltaY now maps to VERTICAL scroll (natural for trackpad), not horizontal. Trackpad horizontal swipe produces deltaX which correctly maps to horizontal scroll.

For all vertical scroll paths (metaKey, shiftKey, bare deltaY), compute maxScrollY the same way the existing Shift branch does: `RULER_HEIGHT + fxCount * FX_TRACK_HEIGHT + contentCount * TRACK_HEIGHT - canvasHeight`.

**timelineStore.ts -- add scrollY auto-clamping:**

Add an `effect()` import from `@preact/signals` and add a self-clamping effect after the computed declarations (around line 41):

```ts
// Auto-clamp scrollY when content shrinks (e.g., FX layers deleted)
effect(() => {
  const max = maxScrollY.value;
  if (scrollY.peek() > max) {
    scrollY.value = max;
  }
});
```

This ensures that when `fxTrackLayouts` changes (FX layer deleted), `totalContentHeight` recomputes, `maxScrollY` recomputes, and if `scrollY` exceeds the new max, it gets clamped immediately.

Also update `setScrollY` to clamp against `maxScrollY`:
```ts
setScrollY(v: number) {
  scrollY.value = Math.max(0, Math.min(maxScrollY.peek(), v));
},
```
