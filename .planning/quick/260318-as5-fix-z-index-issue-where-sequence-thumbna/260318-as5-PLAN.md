  <action>
In TimelineRenderer.ts `draw()` method, add a canvas clip region that prevents any content (thumbnails, placeholders, frame borders, key photo separators) from rendering left of TRACK_HEADER_WIDTH within the content track area.

The root cause: On line ~247, `fx = (range.startFrame + f) * frameWidth - scrollX + TRACK_HEADER_WIDTH` can produce values less than TRACK_HEADER_WIDTH when scrolled. The virtualization check on line ~253 only skips frames ENTIRELY off-screen (`fx + fw < TRACK_HEADER_WIDTH`), but partially-visible frames still draw their full rectangle starting at `fx` which bleeds into the header.

The fix approach -- add a clip rect for the content drawing area. Inside the existing `ctx.save()` / `ctx.restore()` block (the one that clips below the ruler at line ~156-331), after the FX tracks are drawn and before the content track loop begins (around line ~204), add an inner `ctx.save()` with a clip rect:

```
ctx.save();
ctx.beginPath();
ctx.rect(TRACK_HEADER_WIDTH, RULER_HEIGHT - scrollY, w - TRACK_HEADER_WIDTH, h + scrollY);
ctx.clip();
```

Then after the content track thumbnail/frame drawing loop ends (after the key photo separator drawing, around line ~291, just before `trackY += TRACK_HEIGHT` at line ~293), close with `ctx.restore()`.

IMPORTANT: The track header (background + name text) at lines 210-227 must be drawn OUTSIDE this clip so the header remains visible. So the structure should be:

For each track:
1. Draw track background (full width) -- BEFORE clip
2. Draw selection indicator -- BEFORE clip
3. Draw track header bg + name text -- BEFORE clip
4. ctx.save() + clip to right of TRACK_HEADER_WIDTH
5. Draw key photo ranges (thumbnails, placeholders, borders, separators) -- INSIDE clip
6. ctx.restore() to remove clip

This also applies to the FX track range bars -- they already have manual clipping logic (clippedLeft/clippedRight at lines 406-408), but adding a clip region would be more robust. However, since FX tracks already handle clipping manually and the issue is specifically about content sequence thumbnails, focus the clip on the content track thumbnail loop only.

Do NOT change the existing virtualization skip logic (it remains as a performance optimization). The clip rect is a visual safety net for partially-visible frames.
