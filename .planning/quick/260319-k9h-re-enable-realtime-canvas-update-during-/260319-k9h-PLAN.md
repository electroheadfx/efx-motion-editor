Add `import {playbackEngine} from '../../lib/playbackEngine';` at the top (it is already imported, verify).

2. **Playhead drag pointerDown** (lines ~460-466): The initial seek on pointerDown already calls `playbackEngine.seekToFrame(frame)` -- keep it as is.

4. **FX range drag end** (line ~679): Remove the explicit `timelineStore.syncDisplayFrame()` call. Since the render effect no longer checks timelineDragging, the display will already be current after each FX range update (sequences.value changes trigger the render effect). Keep `timelineStore.setTimelineDragging(false)` since the signal is still used for the undo coalescing pattern.

5. **Keyframe drag end** (line ~628): Similarly remove the explicit `timelineStore.syncDisplayFrame()` call. The render effect already fires on sequence data changes during keyframe moves.

Do NOT remove the `timelineDragging` signal itself or the `setTimelineDragging(true/false)` calls -- they are still used to gate other behaviors (undo coalescing start/stop). Only the Preview render gate is removed.
