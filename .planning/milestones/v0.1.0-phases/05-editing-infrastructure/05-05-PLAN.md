**A) Rewrite jklShuttle.ts -- state-only, no rAF loop:**

Remove the entire rAF loop (`shuttleTick`, `rafId`, `lastTime`, `accumulator`, `stopShuttleLoop`, `updatePlayback`). The shuttle module becomes purely a state manager.

New signals to export:
- `shuttleDirection`: signal<1 | -1> -- current direction (1 = forward, -1 = reverse). Default: 1.
- `shuttleSpeedTier`: signal<number> -- current speed tier index (0-3 mapping to SPEED_TIERS [1,2,4,8]). Default: 0 (1x).
- `shuttleSpeed`: computed signal that returns `SPEED_TIERS[shuttleSpeedTier.value]` (always positive, direction is separate).
- `currentSpeedLabel`: signal<string> -- keep existing for SpeedBadge.
- `showSpeedBadge`: signal<boolean> -- keep existing for SpeedBadge.

New behavior for pressL():
- If direction is already forward (1): increment speedTier (cap at SPEED_TIERS.length - 1).
- If direction is reverse (-1): reset speedTier to 0, set direction to forward (1).
- Update speed label and flash badge.
- Does NOT start or stop playback.

New behavior for pressJ():
- If direction is already reverse (-1): increment speedTier (cap at SPEED_TIERS.length - 1).
- If direction is forward (1): reset speedTier to 0, set direction to reverse (-1).
- Update speed label and flash badge.
- Does NOT start or stop playback.

New behavior for pressK():
- Reset speedTier to 0, direction to 1 (forward).
- Update speed label to '' and flash badge.
- Does NOT stop playback (Space owns play/stop).

New behavior for resetShuttle():
- Reset speedTier to 0, direction to 1, label to '', badge hidden. Same cleanup as before but adapted to new signals.

Helper updateLabel():
- Direction forward + tier 0 (1x): label = '' (no badge needed at default)
- Direction forward + tier > 0: label = `${speed}x`
- Direction reverse + tier 0 (1x): label = '-1x'
- Direction reverse + tier > 0: label = `-${speed}x`

Keep flashBadge() and BADGE_DURATION as-is.

**B) Modify PlaybackEngine to read shuttle state and auto-loop:**

1. Import `shuttleDirection`, `shuttleSpeed` from `./jklShuttle`.
2. Modify the `tick()` method:
   - Read `shuttleSpeed.peek()` for the speed multiplier (always >= 1).
   - Read `shuttleDirection.peek()` for direction (1 or -1).
   - Calculate frameDuration as `1000 / (fps * speed)` (faster playback = shorter frame duration).
   - Step frames in the direction: `direction > 0 ? stepForward logic : stepBackward logic`.
   - At boundaries, implement auto-loop:
     - If direction is forward and currentFrame >= maxFrames - 1: seek to frame 0 and continue.
     - If direction is reverse and currentFrame <= 0: seek to maxFrames - 1 and continue.
   - Do NOT stop playback at boundaries -- loop endlessly.

3. Replace the current forward-only step logic in the while loop:
   ```typescript
   while (this.accumulator >= frameDuration) {
     this.accumulator -= frameDuration;
     const currentFrame = timelineStore.currentFrame.peek();
     const direction = shuttleDirection.peek();

     if (direction > 0) {
       // Forward
       if (currentFrame >= maxFrames - 1) {
         timelineStore.seek(0); // auto-loop to start
       } else {
         timelineStore.stepForward();
       }
     } else {
       // Reverse
       if (currentFrame <= 0) {
         timelineStore.seek(maxFrames - 1); // auto-loop to end
       } else {
         timelineStore.stepBackward();
       }
     }
   }
   ```

4. Remove the `if (currentFrame >= maxFrames - 1) { this.stop(); return; }` block -- playback no longer stops at end.

5. The `start()` method remains the same (starts rAF loop, sets isPlaying true).
6. The `stop()` method should also call `resetShuttle()` to clean up shuttle state when playback stops:
   ```typescript
   stop() {
     if (this.rafId !== null) {
       cancelAnimationFrame(this.rafId);
       this.rafId = null;
     }
     timelineStore.setPlaying(false);
     resetShuttle();
   }
   ```
7. The `toggle()` method remains the same (calls start/stop based on isPlaying).
