**1. Fix `projectStore.closeProject()` (INFRA-01, INFRA-02):**

In `projectStore.ts`, update `closeProject()` to call ALL store resets and engine stops in the correct order:

```
closeProject() {
  // 1. Stop engines and timers FIRST (prevents orphaned operations)
  stopAutoSave();           // INFRA-02: stop auto-save timer
  playbackEngine.stop();    // INT-02: stop playback rAF loop

  // 2. Reset all stores
  batch(() => {
    name.value = 'Untitled Project';
    fps.value = 24;
    width.value = 1920;
    height.value = 1080;
    filePath.value = null;
    dirPath.value = null;
    isDirty.value = false;
    isSaving.value = false;
  });
  sequenceStore.reset();
  imageStore.reset();
  uiStore.reset();
  timelineStore.reset();    // INT-02: reset timeline position
  layerStore.reset();       // Future-proofing for Phase 6
  historyStore.stack.value = [];  // Clear undo stack for new project
  historyStore.pointer.value = -1;
}
```

Add imports for `timelineStore`, `layerStore`, `historyStore`, `playbackEngine`, `stopAutoSave`.

Also update `openProject()` to call `closeProject()` at the start (before loading new data) to ensure clean state. After hydration, call `startAutoSave()` to restart auto-save for the new project.

Also update `createProject()` to call `closeProject()` at the start (before setting new state). After setting state, call `startAutoSave()`.

**2. Make `startAutoSave()` idempotent (INFRA-02 safety):**

In `autoSave.ts`, add a guard at the top of `startAutoSave()`: if `disposeEffect` is already set, call `stopAutoSave()` first. This prevents duplicate timer accumulation if called multiple times.

**3. Create `unsavedGuard.ts`:**

Create `Application/src/lib/unsavedGuard.ts` following the pattern from 05-RESEARCH.md:

- Export `type GuardResult = 'saved' | 'discarded' | 'cancelled'`
- Export `async function guardUnsavedChanges(): Promise<GuardResult>`
- If `!projectStore.isDirty.value`, return `'discarded'` immediately (no changes to save)
- Use `message()` from `@tauri-apps/plugin-dialog` with `kind: 'warning'` and `buttons: { yes: 'Save', no: "Don't Save", cancel: 'Cancel' }`
- If result is `'Yes'`: check `projectStore.filePath.value` — if null, open `save()` dialog for Save As; if user cancels the save picker, return `'cancelled'`; otherwise call `saveProjectAs()`; if filePath exists, call `saveProject()`. Return `'saved'`.
- If result is `'No'`: return `'discarded'`
- If result is `'Cancel'` (or any other): return `'cancelled'`

Use `message` and `save` from `@tauri-apps/plugin-dialog`. Use the `buttons` property (NOT the deprecated `okLabel`/`cancelLabel`).
