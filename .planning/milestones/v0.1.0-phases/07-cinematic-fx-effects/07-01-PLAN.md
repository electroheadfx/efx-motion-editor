Create three new files implementing all FX rendering logic.

**fxGenerators.ts** -- Pure drawing functions for procedural effects. Each function signature: `(ctx: CanvasRenderingContext2D, width: number, height: number, params: SourceDataType, frame: number) => void`

Import `Random` from `@efxlab/motion-canvas-core` (path: `@efxlab/motion-canvas-core`; the class is exported from `lib/scenes/Random`). The import is: `import { Random } from '@efxlab/motion-canvas-core';`

If the above import doesn't compile (MC packages can have tricky exports), fall back to implementing a simple Mulberry32 PRNG inline -- same algorithm MC uses, ~10 lines.

All generators use normalized coordinates (0-1) scaled to canvas dimensions for resolution independence (FX-10).

Seed derivation: `const effectiveSeed = params.lockSeed ? params.seed + frame : performance.now() + frame;`

1. `drawGrain(ctx, width, height, params, frame)`:
   - dotCount = floor(width * height * params.density * 0.01)
   - For each dot: random x/y in canvas, random alpha (0 to params.intensity), random brightness (0 or 255)
   - Use `ctx.fillRect(x, y, params.size, params.size)`

2. `drawParticles(ctx, width, height, params, frame)`:
   - For each particle (params.count): random base position, offset by frame * speed for animation
   - Size from rng.nextFloat(sizeMin, sizeMax)
   - Draw white circles with 0.6 alpha: `ctx.arc(x, y, size, 0, Math.PI * 2)`

3. `drawLines(ctx, width, height, params, frame)`:
   - For each line (params.count): random start point, random angle, random length in [lengthMin*maxDim, lengthMax*maxDim]
   - Draw white lines with params.thickness and 0.4 alpha using ctx.beginPath/moveTo/lineTo/stroke

4. `drawDots(ctx, width, height, params, frame)`:
   - For each dot (params.count): random base position, offset by frame * speed for drift animation
   - Size from rng.nextFloat(sizeMin, sizeMax)
   - Draw white filled circles with 0.5 alpha

5. `drawVignette(ctx, width, height, params)` (no frame/seed -- deterministic gradient):
   - cx = width/2, cy = height/2
   - maxRadius = sqrt(cx*cx + cy*cy)
   - innerRadius = maxRadius * params.size
   - createRadialGradient(cx, cy, innerRadius, cx, cy, maxRadius)
   - Stops: 0 -> rgba(0,0,0,0), softness -> rgba(0,0,0, intensity*0.5), 1.0 -> rgba(0,0,0, intensity)
   - fillRect(0, 0, width, height)

**fxColorGrade.ts** -- ImageData pixel manipulation.

Export interface `ColorGradeParams`:
```typescript
export interface ColorGradeParams {
  brightness: number;  // -0.5 to 0.5
  contrast: number;    // -1 to 1
  saturation: number;  // 0 to 2 (1 = normal)
  hue: number;         // -180 to 180 degrees
  fade: number;        // 0 to 1
  tintColor: string;   // hex color like '#D4A574'
}
```

Export `applyColorGrade(ctx, width, height, params)`:
- Get ImageData from ctx (use physical canvas dimensions -- the ctx is already in DPI-scaled coordinate space, but getImageData operates in pixel coords. Call: `ctx.getImageData(0, 0, width, height)` where width/height are the LOGICAL dimensions passed in -- the DPI scaling is handled by the caller in PreviewRenderer)
- IMPORTANT: Since PreviewRenderer applies DPI scaling via ctx.scale(dpr, dpr), and the color grade function receives logical dimensions, we need to save/restore the context transform. The approach: save ctx, reset transform to identity, getImageData(0, 0, canvas.width, canvas.height) using PHYSICAL dimensions, process pixels, putImageData at (0, 0), restore ctx. Expose this as `applyColorGrade(ctx: CanvasRenderingContext2D, canvasPhysicalW: number, canvasPhysicalH: number, params: ColorGradeParams)`.
- Parse tintColor hex to RGB components
- Apply in order: brightness -> contrast -> saturation -> hue rotation -> fade
- Brightness: add params.brightness * 255 to each channel
- Contrast: factor = (1 + params.contrast) / (1.001 - params.contrast); channel = factor * (channel - 128) + 128
- Saturation: lum = 0.299*r + 0.587*g + 0.114*b; channel = lum + params.saturation * (channel - lum)
- Hue rotation: Apply 3x3 rotation matrix in RGB space. cos/sin of params.hue * PI/180.
- Fade: blend each channel toward tint: channel + (tintChannel - channel) * params.fade
- Clamp all values to 0-255
- putImageData back

Add a helper `parseTintHex(hex: string): [number, number, number]` to convert '#RRGGBB' to [r, g, b].
