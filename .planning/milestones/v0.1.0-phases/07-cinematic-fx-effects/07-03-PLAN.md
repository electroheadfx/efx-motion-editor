Extend PropertiesPanel with context-sensitive FX controls per CONTEXT.md decision: "FX-specific controls replace transform section when an FX layer is selected."

   ```typescript
   import { isFxLayer, isGeneratorLayer, isAdjustmentLayer } from '../../types/layer';
   import { COLOR_GRADE_PRESETS, PRESET_NAMES } from '../../lib/fxPresets';
   ```

2. Create a helper to update source data (preserving other source fields):
   ```typescript
   function updateSource(layerId: string, layer: Layer, updates: Record<string, unknown>) {
     layerStore.updateLayer(layerId, {
       source: { ...layer.source, ...updates } as LayerSourceData,
     });
   }
   ```

3. Create FX section sub-components (inline in PropertiesPanel.tsx, following the established pattern of BlendSection/TransformSection/CropSection being in the same file):

   **GrainSection** -- for generator-grain layers:
   - NumericInput: Density (0-1, step 0.01)
   - NumericInput: Size (1-4, step 1)
   - NumericInput: Intensity (0-1, step 0.01)
   - Seed toggle button: "Lock" (lockSeed: true/false). Show small lock/unlock icon. When locked, also show NumericInput for seed value (integer, step 1).

   **ParticlesSection** -- for generator-particles layers:
   - NumericInput: Count (1-500, step 1)
   - NumericInput: Speed (0-5, step 0.1)
   - NumericInput: Min Size (0.5-20, step 0.5)
   - NumericInput: Max Size (0.5-20, step 0.5)
   - Seed toggle + seed value (same pattern as GrainSection)

   **LinesSection** -- for generator-lines layers:
   - NumericInput: Count (1-100, step 1)
   - NumericInput: Thickness (0.5-5, step 0.5)
   - NumericInput: Min Length (0-1, step 0.01)
   - NumericInput: Max Length (0-1, step 0.01)
   - Seed toggle + seed value

   **DotsSection** -- for generator-dots layers:
   - NumericInput: Count (1-200, step 1)
   - NumericInput: Min Size (1-20, step 0.5)
   - NumericInput: Max Size (1-20, step 0.5)
   - NumericInput: Speed (0-5, step 0.1)
   - Seed toggle + seed value

   **VignetteSection** -- for generator-vignette layers:
   - NumericInput: Size (0-1, step 0.01) -- clear zone radius
   - NumericInput: Softness (0-1, step 0.01)
   - NumericInput: Intensity (0-1, step 0.01)
   - (No seed controls -- vignette is deterministic)

   **ColorGradeSection** -- for adjustment-color-grade layers:
   - Preset dropdown: `<select>` with PRESET_NAMES. When changed, load preset values into layer source. Set preset field to selected name.
   - NumericInput: Brightness (-0.5 to 0.5, step 0.01)
   - NumericInput: Contrast (-1 to 1, step 0.01)
   - NumericInput: Saturation (0 to 2, step 0.01)
   - NumericInput: Hue (-180 to 180, step 1)
   - NumericInput: Fade (0 to 1, step 0.01)
   - Tint color input: `<input type="color" value={layer.source.tintColor} />` for the fade-to-tint color
   - When any individual parameter changes, set preset to 'none' (user is customizing)

   All sections use `updateSource(layer.id, layer, { fieldName: newValue })` which calls `layerStore.updateLayer`. This ensures all changes flow through sequenceStore for automatic undo.

4. Modify the main `PropertiesPanel` component's render logic:
   - If `isFxLayer(selectedLayer)`: show BlendSection (blend mode + opacity + visibility), divider, then the appropriate FX section based on source type. Do NOT show TransformSection or CropSection (per CONTEXT.md: FX layers don't need spatial transforms).
   - If NOT FX layer: show existing layout unchanged (BlendSection, TransformSection, CropSection, plus source info).
   - Remove the Phase 7 FX stub placeholder text that currently shows for video layers.

5. For the in/out points: Add an **InOutSection** that shows for ALL FX layers:
   - NumericInput: In Frame (min 0, step 1, value = layer.inFrame ?? 0)
   - NumericInput: Out Frame (min 0, step 1, value = layer.outFrame ?? '') -- empty means "end of sequence"
   - When In changes: `layerStore.updateLayer(layer.id, { inFrame: val || undefined })`
   - When Out changes: `layerStore.updateLayer(layer.id, { outFrame: val || undefined })`
   - Show this section after the FX-specific controls, separated by a divider.

Each section uses the existing SectionLabel component for consistent styling. All inputs use the existing NumericInput component with coalescing support.
