**TimelineInteraction.ts -- Vertical scroll and hit-test adjustments:**

1. **onWheel -- Split horizontal and vertical scroll:**

Change the non-zoom wheel handler from combining deltaX+deltaY into horizontal, to using deltaX for horizontal and deltaY for vertical:

```typescript
} else {
  // Horizontal scroll (deltaX or shift+deltaY)
  if (e.deltaX !== 0) {
    const newScrollX = timelineStore.scrollX.peek() + e.deltaX;
    timelineStore.setScrollX(Math.max(0, newScrollX));
  }
  // Vertical scroll (deltaY without modifier)
  if (e.deltaY !== 0 && !e.shiftKey) {
    const newScrollY = timelineStore.scrollY.peek() + e.deltaY;
    // Compute max scrollY: total content height - visible canvas height
    const fxCount = this.renderer?.getFxTrackCount() ?? 0;
    const contentCount = trackLayouts.peek().length;
    const totalContentH = RULER_HEIGHT + fxCount * FX_TRACK_HEIGHT + contentCount * TRACK_HEIGHT;
    const canvasH = this.canvas?.getBoundingClientRect().height ?? totalContentH;
    const maxScrollY = Math.max(0, totalContentH - canvasH);
    timelineStore.setScrollY(Math.min(maxScrollY, Math.max(0, newScrollY)));
  } else if (e.deltaY !== 0 && e.shiftKey) {
    // Shift+scroll = horizontal scroll (standard macOS pattern)
    const newScrollX = timelineStore.scrollX.peek() + e.deltaY;
    timelineStore.setScrollX(Math.max(0, newScrollX));
  }
}
```

2. **Hit-testing -- Account for scrollY in all Y-based calculations:**

The following methods need scrollY adjustment. Read scrollY from the renderer via `this.renderer?.getScrollY() ?? 0`:

- `trackIndexFromY(clientY)`: Add scrollY to the y calculation (scrolled content maps to higher logical Y):
```typescript
const scrollY = this.renderer?.getScrollY() ?? 0;

- `dropIndexFromY(clientY)`: Same scrollY adjustment.

- `isInFxArea(clientY)`: Adjust for scrollY:

const scrollY = this.renderer?.getScrollY() ?? 0;

```

- `fxTrackIndexFromY(clientY)`: Same scrollY adjustment.

- `isInRuler(clientY)`: NO change needed (ruler is fixed at the top, no scroll).

3. **Cursor hints in onPointerMove**: The FX area and content track area checks already call the methods above, so they automatically get scrollY adjustment through the updated methods.
