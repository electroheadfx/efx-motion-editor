## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| sm | 4px (1 in Tailwind) | Gap between key photo cards in strip (`gap-1`), border-radius on cards, corner icon padding |
| md | 8px (2 in Tailwind) | Strip container padding (`px-2 py-2`), color picker popover internal padding, checkerboard cell size |
| lg | 16px | Section-level gap (not directly used in new components this phase) |

Exceptions:
- Badge inset positioning: 2px (`top-0.5`, `right-0.5`, `bottom-0.5`, `left-0.5`) -- inherited from existing delete badge pattern in KeyPhotoCard. Tailwind `top-0.5` (2px) is a well-established pattern for badge/overlay positioning where 4px would push elements too far from the card edge. Not a named scale token.
- Split button divider gap: 1px solid border between Camera and Square halves -- a visual separator, not a spacing unit. Uses standard 1px border, same pattern as existing list dividers.
- Key photo card: 56px height (existing, `h-14`), min-width 56px -- inherited from existing KeyPhotoCard
- Split add button: 24px wide x 56px tall (existing dimensions from AddKeyPhotoButton, now split vertically into two 24x28px halves)
- Corner icon buttons: 14px (3.5 in Tailwind) width/height -- matches existing delete button (`w-3.5 h-3.5`)
- Color picker popover: 4px margin-bottom above the anchor icon
- Checkerboard cell: 4px on timeline canvas, 8px in CSS (KeyPhotoCard strip)

## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Badge | 9px | 400 | 1.0 | Hold frames badge (`text-[9px]`) -- existing pattern |
| Label / Input | 10px | 400 | 1.2 | Empty state text, placeholder text, hex color text input in color picker popover -- existing pattern |
| Property label | 12px | 500 | 1.4 | Not directly used this phase, but available for any sidebar labels |

**Weights used in this phase:** 400 (regular) and 500 (medium). No bold/semibold (600) needed for new components.

**Rationale for 3-size scale:** The original 11px Input role was collapsed into the 10px Label role. A 1px difference (11px vs 10px) is not perceptually meaningful at this scale, and both serve small-text purposes (hex input field, labels). Three sizes (9, 10, 12) provide cleaner visual separation with a minimum 1px/10% step between each role.

## Interaction Contracts

### 1. Split Add Button

| Interaction | Behavior |
|-------------|----------|
| Click top half (Camera) | Opens imported media view (`uiStore.setEditorMode('imported')`) -- existing behavior |
| Click bottom half (Square) | Immediately creates key solid with #000000, appended to end of keyPhotos array, undo-able |
| Hover (either half) | `brightness-125` filter -- existing pattern |

**Visual spec:**
- Container: 24px wide, 56px tall, `border-radius: 4px`, `overflow: hidden`, `flex-col`
- Top half: 24x28px, Camera icon at 12px, centered
- Bottom half: 24x28px, Square icon at 12px, centered
- Divider: 1px solid #2A2A3A (top border on bottom half)
- Background: #3A3A50 on both halves
- Icon color: `var(--sidebar-text-secondary)`

### 2. Key Solid Card (4-corner layout)

| Corner | Icon | Size | Behavior |
|--------|------|------|----------|
| Top-right | `X` | 10px in 14x14 circle | Delete -- existing pattern, hover-reveal (`opacity-0 group-hover:opacity-100`) |
| Bottom-right | Text `{N}f` | 9px | Hold frames badge -- existing pattern, click to edit inline |
| Bottom-left | `Pipette` | 10px in 14x14 button | Click opens ColorPickerPopover above the icon. Hidden when `isTransparent === true` |
| Top-left | `Square` or `Blend` | 10px in 14x14 button | Toggle solid/transparent mode. `Square` = solid (default), `Blend` = transparent. Always visible on hover |

**Corner button shared style:**
- Size: `w-3.5 h-3.5` (14x14px)
- Background: `#00000080` (idle), `#000000AA` (hover)
- Border-radius: full (`rounded-full`)
- Position: absolute, 2px from respective corner (`top-0.5`/`bottom-0.5`, `left-0.5`/`right-0.5`) -- see Spacing Scale exceptions for 2px justification
- Visibility: `opacity-0 group-hover:opacity-100 transition-opacity`
- Icon color: white, 10px

**Card background by type:**
- Solid: `backgroundColor: kp.solidColor` (the actual color fills the card)
- Transparent: CSS checkerboard pattern (see Checkerboard Pattern Spec below)

### 3. Color Picker Popover

| Property | Value |
|----------|-------|
| Position | Absolute, anchored above bottom-left Pipette icon |
| Offset | `bottom: 100%; left: 0; margin-bottom: 4px` |
| Background | `var(--sidebar-panel-bg)` |
| Border | `1px solid var(--sidebar-border-unselected)` |
| Border radius | 8px (`rounded-lg`) |
| Shadow | `shadow-xl` |
| Padding | 8px (`p-2`) |
| Z-index | 50 |

**Contents (vertical stack, gap 4px):**

1. **Native color input:** `<input type="color">` -- 32x32px (`w-8 h-8`), `rounded`, `cursor-pointer`. Value bound to `kp.solidColor`. On input, calls `sequenceStore.updateKeySolidColor()` with undo.
2. **Hex text input:** `<input type="text">` -- full width, `text-[10px]`, `bg-[var(--sidebar-input-bg)]`, `color: var(--sidebar-text-primary)`, `rounded`, `px-1 py-0.5`, `border: none`, `outline: none`. Displays current hex value (e.g., `#FF0000`). On blur or Enter, validates hex format and applies. On Escape, reverts.

| Interaction | Behavior |
|-------------|----------|
| Open | Click Pipette icon on a key solid card |
| Close | Click outside popover, press Escape, or click Pipette again |
| Color change (native input) | Live preview on canvas while picking (D-14). Undo pushed on final value |
| Color change (hex input) | Applied on blur/Enter. Validated: must match `/^#[0-9a-fA-F]{6}$/` |
| Transparent mode active | Popover is hidden (Pipette icon itself is hidden per D-12) |

### 4. Solid/Transparent Toggle (Top-Left Corner)

| State | Icon | Visual | Behavior on click |
|-------|------|--------|-------------------|
| Solid mode (default) | `Square` | Filled square icon | Switches to transparent mode: `isTransparent = true`. Card shows checkerboard. Pipette icon hides. Undo-able |
| Transparent mode | `Blend` | Blend/mix icon | Switches back to solid mode: `isTransparent = undefined`, `solidColor` restored (defaults to #000000 if none). Card shows solid color. Pipette icon reappears. Undo-able |

### 5. Timeline Rendering

| Entry Type | Timeline Cell Rendering |
|------------|------------------------|
| Key photo | Existing: tiled image thumbnail pattern (no change) |
| Key solid | `ctx.fillStyle = range.solidColor; ctx.fillRect(fx, fy, fw, fh)` -- solid colored rectangle |
| Key transparent | Canvas-drawn checkerboard using `drawCheckerboard(ctx, fx, fy, fw, fh, 4)` -- 4px cells, colors #808080/#B0B0B0 |

Key photo separator lines between ranges: unchanged (#444444, 1.5px).

### 6. Canvas Preview Rendering

| Entry Type | Canvas Rendering |
|------------|-----------------|
| Key photo | Existing: image composited onto canvas (no change) |
| Key solid | `ctx.fillStyle = entry.solidColor; ctx.fillRect(0, 0, logicalW, logicalH)` -- full canvas fill before FX layers |
| Key transparent | No fill -- `clearRect` leaves canvas transparent, app background (black) shows through per D-19 |

### 7. Sidebar Sequence Thumbnail

| First Entry Type | Thumbnail Rendering |
|------------------|---------------------|
| Key photo | Existing: image thumbnail URL as background-image (no change) |
| Key solid | `backgroundColor: firstKp.solidColor` -- color swatch fills thumbnail box |
| Key transparent | CSS checkerboard pattern fills thumbnail box |

## Checkerboard Pattern Spec

### CSS (for KeyPhotoCard and sidebar thumbnail)

```css
background-image:
  linear-gradient(45deg, #808080 25%, transparent 25%),
  linear-gradient(-45deg, #808080 25%, transparent 25%),
  linear-gradient(45deg, transparent 75%, #808080 75%),
  linear-gradient(-45deg, transparent 75%, #808080 75%);
background-size: 8px 8px;
background-position: 0 0, 0 4px, 4px -4px, -4px 0px;
background-color: #B0B0B0;
```

### Canvas (for TimelineRenderer)

- Cell size: 4px
- Light color: #B0B0B0
- Dark color: #808080
- Draw method: `ctx.fillRect` with alternating colors, clipped to range bounds

## State Transitions

### Key Solid Lifecycle

```
[Add Solid] --> Solid (black #000000)
                  |
                  +--> [Pick Color] --> Solid (custom color)
                  |
                  +--> [Toggle] --> Transparent
                  |                    |
                  |                    +--> [Toggle] --> Solid (previous color restored)
                  |
                  +--> [Delete] --> Removed (undo-able)
                  |
                  +--> [Reorder] --> Same entry, new position (SortableJS)
```

### Color Picker Popover State

```
Closed (default)
  |
  +--> [Click Pipette] --> Open
                            |
                            +--> [Pick color / type hex] --> Still Open (live preview)
                            |
                            +--> [Click outside / Escape / Click Pipette] --> Closed
                            |
                            +--> [Toggle to transparent] --> Closed (Pipette hidden)
```
