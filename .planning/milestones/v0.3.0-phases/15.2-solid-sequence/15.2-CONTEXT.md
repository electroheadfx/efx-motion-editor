# Phase 15.2: Solid Sequence - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Extend key photos to support solid color and transparent entries alongside imported images. Any sequence can mix key photos, key solids, and key transparencies. Key solids render as full-resolution colored rectangles; key transparencies render as canvas background. This enables inter-sequence blank/colored space for FX layer stacking. Does NOT include new sequence kinds, new canvas background features, or changes to FX layer behavior.

</domain>

<decisions>
## Implementation Decisions

### Key solid data model
- **D-01:** Key solids are entries in the existing `keyPhotos` array — not a separate list. A key entry is either a photo (has `imageId`) or a solid (has `solidColor` or transparent flag)
- **D-02:** Any sequence can freely mix key photos, key solids, and key transparencies — no separate "solid sequence" kind
- **D-03:** Key solids have hold frames, reorder, and delete — identical to key photos
- **D-04:** Default solid color is black (`#000000`)

### Add-solid flow (KeyPhotoStrip)
- **D-05:** The existing "+" add button in the key photo strip is replaced by a vertically split span with 2 buttons: top `Camera` icon (opens image importer), bottom `Square` icon (adds black solid)
- **D-06:** Clicking `Square` immediately creates a key solid with default black color — no dialog
- **D-07:** Clicking `Camera` opens the image importer as before

### Key solid cell controls (4 corners)
- **D-08:** Top-right: delete button (same as key photo)
- **D-09:** Bottom-right: frame count badge (same as key photo)
- **D-10:** Bottom-left: `Pipette` icon — opens inline color picker popover
- **D-11:** Top-left: solid/transparent toggle — `Square` icon (solid mode) ↔ `Blend` icon (transparent mode)
- **D-12:** Switching to transparent mode hides the color picker (bottom-left Pipette icon)

### Color picker UX
- **D-13:** Clicking Pipette opens a small inline popover with gradient square + hex input
- **D-14:** Color changes apply with live preview on canvas while picking
- **D-15:** No recent/project color memory — each key solid's color is independent

### Timeline rendering
- **D-16:** Key solid frame ranges render as filled rectangles in the solid's color (black solid = black block, red solid = red block)
- **D-17:** Key transparent frame ranges render with a checkerboard pattern (Photoshop-style transparency)

### Canvas preview rendering
- **D-18:** Key solid renders as a full-resolution rectangle of its color on the canvas
- **D-19:** Key transparent renders as the canvas background color (currently black — no transparency grid on canvas)

### Sidebar display
- **D-20:** Sequence thumbnail uses the first entry: color swatch if first entry is a key solid, photo thumbnail if first entry is a key photo
- **D-21:** Sequence info counts all entries uniformly (e.g., "5 keys · 2.4s") — no separate photo/solid counts

### Cross-dissolve & transitions
- **D-22:** Cross-dissolve between key photo and key solid blends normally — standard alpha interpolation between image and solid color

### Claude's Discretion
- KeyPhoto type extension approach (discriminated union vs optional fields)
- Color picker popover component implementation (custom vs lightweight library)
- Checkerboard pattern implementation (CSS pattern vs canvas-drawn)
- Exact gradient square layout in color picker popover
- frameMap / KeyPhotoRange changes to carry solid info to renderers
- Project format version bump if needed (v9 → v10)

</decisions>

<specifics>
## Specific Ideas

- Split add button is vertically divided: Camera on top, Square on bottom — compact within the key photo strip
- Key solid cell background IS the solid color itself (like a color swatch thumbnail)
- Transparent key cell shows checkerboard pattern as its "thumbnail" in the strip
- The 4-corner icon layout mirrors key photo cards: delete and frame count in same positions, color picker and type toggle use the two remaining corners
- Solid/transparent toggle uses Lucide `Square` (solid) and `Blend` (transparent) icons

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- **KeyPhotoStrip badges:** Delete (top-right) and frame count (bottom-right) already implemented — solid cells reuse these in same positions, add two new corners
- **TransitionProperties toggle:** Solid/transparency mode toggle pattern with conditional color picker — same UX concept for key solid type toggle
- **`<input type="color">`:** Used in SidebarFxProperties and TransitionProperties — can serve as base for color picker popover or be enhanced with gradient square
- **SortableJS in KeyPhotoStrip:** Drag-and-drop reorder works on array items — key solids interleave naturally since they share the keyPhotos array
- **FadeMode type:** Already has `'transparency' | 'solid'` discriminator in transitions — similar pattern for key entry type

### Established Patterns
- **Signal store mutations:** All keyPhoto ops use snapshot/restore + pushAction for undo — key solid ops follow same pattern
- **Conditional rendering in KeyPhotoCard:** Badge visibility based on state — extend for color picker and type toggle visibility
- **Theme CSS variables:** Color system with `--color-*` variables — checkerboard pattern uses theme-aware colors
- **Portal-based popovers:** Used in ImportGrid and SequenceList — color picker popover follows same pattern

### Integration Points
- **KeyPhoto type:** Extend with solid/transparent discriminator + optional color field
- **KeyPhotoStrip:** Split add button, new corner controls on solid cells
- **TimelineRenderer.drawLinearTrack():** Render solid color or checkerboard instead of thumbnail tiles
- **previewRenderer.renderFrame():** Render solid rectangle or skip (transparent) instead of loading image
- **frameMap.ts:** FrameEntry needs to carry solid info so renderers know what to draw
- **projectStore:** Serialize/deserialize new key solid fields with backward compat
