  <name>Task 1: Add missing BPM and audio_asset_id fields to Rust MceAudioTrack</name>
  <files>Application/src-tauri/src/models/project.rs</files>
  <action>
    **Root cause:** The Rust `MceAudioTrack` struct in `project.rs` is missing 5 fields that the TypeScript side serializes: `audio_asset_id`, `bpm`, `beat_offset_frames`, `beat_markers`, `show_beat_markers`. When Tauri receives the project JSON via IPC, serde silently drops these unknown fields. On reload, all BPM data is gone.

    **Fix in `Application/src-tauri/src/models/project.rs`:**

    Add the following 5 fields to the `MceAudioTrack` struct, all with `#[serde(default)]` for backward compatibility with older project files that lack them, and with `skip_serializing_if` to keep .mce files clean:

    After the existing `total_frames_in_file` field, add:

    ```rust
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub audio_asset_id: Option<String>,

    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub bpm: Option<f64>,

    #[serde(default)]
    pub beat_offset_frames: i32,

    #[serde(default, skip_serializing_if = "Vec::is_empty")]
    pub beat_markers: Vec<u32>,

    #[serde(default)]
    pub show_beat_markers: bool,
    ```

    **Type choices match TypeScript:**
    - `audio_asset_id`: `Option<String>` — optional for v8 files without asset IDs
    - `bpm`: `Option<f64>` — null means not yet detected, f64 matches JS number
    - `beat_offset_frames`: `i32` with default 0 — signed to match offset_frame/slip_offset pattern, allows negative grid offsets
    - `beat_markers`: `Vec<u32>` — pre-computed frame numbers, empty when no BPM
    - `show_beat_markers`: `bool` with default false — toggle state

    **Why this fixes it:** The Rust struct now has fields matching all the JSON keys that TypeScript sends. Serde can deserialize them on load and serialize them on save. The data survives the full round-trip: TypeScript buildMceProject -> Rust serde Deserialize (save to disk) -> Rust serde Serialize (load from disk) -> TypeScript hydrateFromMce.

    **Backward compatibility:** All new fields use `#[serde(default)]`, so opening a pre-v12 project file (which lacks these fields) will use sensible defaults (None/0/empty/false) instead of failing to parse.
