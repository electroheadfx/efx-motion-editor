    **3. Application/src/lib/assetRemoval.ts**

    Per D-08 through D-14 -- cascade removal coordination module.

    Import from stores (imageStore, sequenceStore, audioStore) and from `../lib/history` (pushAction). Import `batch` from `@preact/signals`. Import `remove as removeFile` from `@tauri-apps/plugin-fs`. Import types from `assetUsage.ts`.

    Export two main functions:

    **cascadeRemoveAsset(assetId: string, assetType: 'image' | 'video' | 'audio', assetName: string, locations: UsageLocation[]): boolean**

    Logic per D-08 through D-11, D-14:
    1. If `locations.length > 0`:
       - Build confirm message per UI-SPEC copywriting: `"${assetName}" is used in ${locations.length} location(s):\n\n${locations.map(l => `- ${l.sequenceName} > ${l.detail}`).join('\n')}\n\nRemove all references and delete from library?`
       - Show `window.confirm(message)`. Return false if cancelled.
    2. If `locations.length === 0`:
       - Show `window.confirm('Remove "${assetName}" from library?')`. Return false if cancelled.
    3. Capture snapshots BEFORE mutations (per D-14, composite undo):
       ```typescript
       const seqSnap = structuredClone(sequenceStore.sequences.peek());
       const seqActive = sequenceStore.activeSequenceId.peek();
       const audioSnap = structuredClone(audioStore.tracks.peek());
       const audioSelected = audioStore.selectedTrackId.peek();
       const imgSnap = structuredClone(imageStore.images.peek());
       const vidSnap = structuredClone(imageStore.videoAssets.peek());
       const audAssetSnap = structuredClone(imageStore.audioAssets.peek());
       ```
    4. Perform mutations inside `batch()`:
       - For image assets (D-09, D-10):
         - Key photos: set `keyPhoto.imageId` to `'__placeholder__'` (sentinel per RESEARCH.md Approach A)
         - Layers (static-image with matching imageId): remove layer from `seq.layers`
         - Layers (image-sequence containing imageId): filter imageId out of `imageIds` array
       - For video assets (D-10):
         - Layers (video with matching videoAssetId): remove layer from `seq.layers`
       - For audio assets (D-11):
         - Audio tracks with matching audioAssetId: remove from `audioStore.tracks`
       - Remove asset from imageStore (images/videoAssets/audioAssets as appropriate)
       - Write mutated sequences back: `sequenceStore.sequences.value = updatedSeqs`
    5. Capture after-snapshot and push SINGLE undo action via `pushAction`:
       ```typescript
       pushAction({
         id: crypto.randomUUID(),
         description: `Remove ${assetType} "${assetName}"`,
         timestamp: Date.now(),
         undo: () => {
           batch(() => {
             sequenceStore.sequences.value = seqSnap;
             sequenceStore.activeSequenceId.value = seqActive;
             audioStore.tracks.value = audioSnap;
             audioStore.selectedTrackId.value = audioSelected;
             imageStore.images.value = imgSnap;
             imageStore.videoAssets.value = vidSnap;
             imageStore.audioAssets.value = audAssetSnap;
           });
         },
         redo: () => {
           batch(() => {
             sequenceStore.sequences.value = afterSeqSnap;
             sequenceStore.activeSequenceId.value = afterSeqActive;
             audioStore.tracks.value = afterAudioSnap;
             audioStore.selectedTrackId.value = afterAudioSelected;
             imageStore.images.value = afterImgSnap;
             imageStore.videoAssets.value = afterVidSnap;
             imageStore.audioAssets.value = afterAudAssetSnap;
           });
         },
       });
       ```
    6. Return true (removal performed).

    **cascadeDeleteFile(assetId: string, assetType: 'image' | 'video' | 'audio', assetName: string, assetPath: string, locations: UsageLocation[], thumbnailPath?: string): Promise<boolean>**

    Logic per D-12, D-13:
    1. If `locations.length > 0` (in-use):
       - First confirm per D-12 step 1: `"${assetName}" is used in ${locations.length} location(s):\n\n${locationList}\n\nThis will remove all references first.`
       - If cancelled, return false.
       - Perform cascade removal (same as cascadeRemoveAsset steps 3-5, but inline -- do NOT call cascadeRemoveAsset because we need to continue to step 2).
       - Second confirm per D-12 step 2: `Permanently delete "${assetName}" from disk? This cannot be undone.`
       - If cancelled, return true (references already removed but file kept -- this is correct behavior).
       - Delete file: `await removeFile(assetPath)`. Wrap in try/catch.
       - If thumbnailPath provided (images), also `await removeFile(thumbnailPath)`. Wrap in try/catch.
    2. If `locations.length === 0` (unused):
       - Single confirm per D-13: `Delete "${assetName}" from disk? This cannot be undone.`
       - If cancelled, return false.
       - Remove from store (same store removal as above, with undo support).
       - Delete file from disk.
    3. Return true.

    **Important:** The cascade mutation logic (steps 3-5) should be extracted into a private helper function `performCascadeRemoval(assetId, assetType)` to avoid duplication between `cascadeRemoveAsset` and `cascadeDeleteFile`.
