## Decisions Made
- Video layers use videoAssetId (references VideoAsset.id in imageStore) instead of videoPath for stable cross-session references
- Audio tracks get audioAssetId field; for v8 files without audio_asset_id, falls back to mat.id (preserving current implicit pattern)
- Project format v9 keeps video_path in serialization for forward-compat so v8 readers can still open files
- Usage scanner is a pure module that takes data as function arguments (not importing stores) for testability
- previewRenderer resolves video source by looking up path from imageStore.videoAssets.peek()
