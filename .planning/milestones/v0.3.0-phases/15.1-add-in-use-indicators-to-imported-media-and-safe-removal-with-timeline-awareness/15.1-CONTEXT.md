# Phase 15.1: Media In-Use Indicators & Safe Removal - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Add visual in-use indicators to all imported media in ImportGrid and implement safe removal with timeline awareness. Check timeline usage across all sequences before removal, warn user with details, show color-coded badge with usage count on every asset. Does NOT include new import flows, audio editing, or timeline changes beyond cascade removal.

</domain>

<decisions>
## Implementation Decisions

### In-use badge appearance
- **D-01:** Every asset in ImportGrid gets a color-coded badge showing usage count
- **D-02:** Color thresholds: green = 0 uses, yellow = low, orange = medium, red = high (exact thresholds at Claude's discretion)
- **D-03:** Badge displays the numeric count of uses (e.g., "0", "3", "7")
- **D-04:** Assets with 0 uses show a green badge (not hidden) — helps users spot orphan media

### Badge interaction
- **D-05:** Clicking the badge opens a popover listing where the asset is used (e.g., "Seq 1 → Key Photo #3", "Seq 2 → Layer 'overlay'")
- **D-06:** The popover includes "Remove Reference" and "Delete File" buttons
- **D-07:** Right-click context menu still works alongside the badge popover (both entry points to removal)

### Removal behavior
- **D-08:** Warning message (native `confirm()`) lists where the asset is used — detailed, not generic
- **D-09:** Key photos referencing a removed image become placeholder/blank frames (not deleted)
- **D-10:** Layers referencing a removed asset are removed entirely from their sequence
- **D-11:** Audio tracks referencing a removed audio asset are removed entirely from the timeline
- **D-12:** "Delete File" for in-use assets requires two-step confirmation: first removes references (with warning), then second confirmation to physically delete the file
- **D-13:** For assets not in use, single native `confirm()` before removal/deletion
- **D-14:** Undo support for removal if complexity is reasonable (Claude's discretion — store already has snapshot/restore)

### Asset reference refactor
- **D-15:** Video layer sources refactored to reference `videoAssetId` instead of `videoPath` string — consistent with image layers using `imageId`
- **D-16:** Audio tracks refactored to reference `audioAssetId` instead of `filePath`/`relativePath` string — consistent ID-based linking
- **D-17:** All three asset types (image, video, audio) use ID-based references throughout the system

### Usage scope
- **D-18:** Usage checking looks across ALL sequences in the project, not just the active sequence
- **D-19:** Unified in-use checking covers all asset types: images (keyPhotos + layers), videos (layers), audio (tracks)

### Claude's Discretion
- Exact color thresholds for yellow/orange/red badge levels
- Popover positioning and styling (consistent with existing portal patterns)
- Placeholder/blank frame visual design for removed key photos
- Undo implementation scope (full removal undo vs reference-only undo)
- Whether the video/audio ID refactor requires a project format version bump (v8 → v9)
- Badge rendering approach for different asset display types (image thumbnails vs video thumbnails vs audio list items)

</decisions>

<specifics>
## Specific Ideas

- Badge color scale mirrors severity: green = safe to delete, red = heavily used across project
- Popover lists exact locations: "Seq 1 → Key Photo #3", "Seq 2 → Layer 'overlay'" — users know exactly what will break
- Two-step delete for in-use files prevents accidental data loss: first cleanup references, then confirm physical deletion
- Key photo placeholders preserve timeline structure (frame count, hold duration) even when source image is removed

</specifics>
