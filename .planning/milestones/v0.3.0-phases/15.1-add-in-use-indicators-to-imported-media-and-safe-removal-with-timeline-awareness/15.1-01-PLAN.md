<objective>
Refactor asset references from path-based to ID-based (video layers + audio tracks), create unified usage scanner, update project format to v9, and add CSS variables for usage badges.

Purpose: This is the foundation for Phase 15.1. Usage scanning depends on consistent ID-based references across all asset types. Without this refactor, video and audio assets cannot be reliably tracked because their IDs are unstable (re-derived during hydration). This plan also creates the assetUsage module that Plan 02 consumes for badge rendering and cascade removal.

Output: Stable ID-based asset references, unified usage scanner module, project format v9 with v8 backward compatibility, and badge CSS variables.
</objective>

    ```typescript
    import type {Sequence} from '../types/sequence';

    export interface UsageLocation {
      sequenceId: string;
      sequenceName: string;
      type: 'key-photo' | 'layer' | 'audio-track';
      detail: string;  // e.g., "Key Photo #3", "Layer 'overlay'", "Track 'Music'"
    }

    export interface AssetUsage {
      assetId: string;
      assetType: 'image' | 'video' | 'audio';
      locations: UsageLocation[];
      count: number;
    }
    ```

    **getImageUsage(imageId: string, sequences: Sequence[]): AssetUsage**
    - Iterate ALL sequences (not just active -- per D-18)

    - Check `seq.layers` for `static-image` type with matching `imageId`
    - Check `seq.layers` for `image-sequence` type where `imageIds` includes the `imageId`

    - Check `seq.layers` for `video` type with matching `videoAssetId`
