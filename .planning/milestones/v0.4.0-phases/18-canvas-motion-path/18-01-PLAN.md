<objective>
Create the motion path visualization overlay that renders an After Effects-style dotted trail with keyframe markers on the canvas. The path shows the spatial trajectory of animated layers, with dots at every interpolated frame position and circle markers at keyframe positions.

Purpose: Users can visually see where a layer moves across frames, with dot spacing naturally revealing easing curves. This is the foundation for the drag interaction added in Plan 02.
Output: MotionPath.tsx component, motionPathHitTest.ts utility, unit tests, CanvasArea integration.
</objective>

**Step A: Create `motionPathHitTest.ts`** at `Application/src/components/canvas/motionPathHitTest.ts`.

Export a single function `hitTestKeyframeCircles`:
```typescript
import type { Point } from './coordinateMapper';

export interface KeyframeCircle {
  x: number;  // project-space X
  y: number;  // project-space Y
  frame: number;  // sequence-local frame offset
}

/**
 * Hit test keyframe circles on the motion path.
 * Returns the index of the hit circle or null.
 * Iterates in reverse order so topmost (last rendered) circle is hit first.
 */
export function hitTestKeyframeCircles(
  point: Point,
  circles: KeyframeCircle[],
  zoom: number,
  hitScreenSize: number = 12,
): number | null {
  const hitRadius = hitScreenSize / zoom;
  const hitRadiusSq = hitRadius * hitRadius;
  for (let i = circles.length - 1; i >= 0; i--) {
    const c = circles[i];
    const dx = point.x - c.x;
    const dy = point.y - c.y;
    if (dx * dx + dy * dy <= hitRadiusSq) {
      return i;
    }
  }
  return null;
}
```

**Step C: Create `MotionPath.tsx`** at `Application/src/components/canvas/MotionPath.tsx`.

This is a Preact functional component. It renders an SVG overlay inside the CSS-transformed canvas wrapper div, so all coordinates are in project space.

Props: none (reads all data from signals).

Behavior:
1. Read signals: `keyframeStore.activeLayerKeyframes.value`, `keyframeStore.selectedKeyframeFrames.value`, `timelineStore.isPlaying.value`, `timelineStore.displayFrame.value`, `canvasStore.zoom.value`, `projectStore.width.value`, `projectStore.height.value`, `layerStore.selectedLayerId.value`.

2. Early returns (render nothing):
   - If `isPlaying` is true (per D-10)
   - If no `selectedLayerId`
   - If `activeLayerKeyframes.length < 2` (need at least 2 keyframes for a path)

3. Check if any keyframe has different x or y values (if all keyframes have the same x,y, there is no spatial motion to show -- skip rendering). Compare first keyframe's x,y against all others.

4. Sample dot positions using `interpolateAt()` from `keyframeEngine.ts`:
   - Get first keyframe frame and last keyframe frame
   - Loop from firstFrame to lastFrame (inclusive)
   - For each frame, call `interpolateAt(keyframes, frame)`
   - Convert offset-from-center to project-space: `projectX = vals.x + canvasW / 2`, `projectY = vals.y + canvasH / 2`
   - Store as `{ x, y, frame }` array

5. Build keyframe circle positions:
   - For each keyframe in `activeLayerKeyframes`, compute project-space position: `x = kf.values.x + canvasW / 2`, `y = kf.values.y + canvasH / 2`
   - Store as `KeyframeCircle[]` (reuse type from motionPathHitTest.ts)

6. Find the current frame's dot:
   - Compute local frame: need `findLayerContext` pattern. Import `sequenceStore`, `trackLayouts` from `../lib/frameMap`. Replicate `findLayerContext` logic locally (it is not exported from keyframeStore):
     ```typescript
     function findLayerStartFrame(layerId: string): number {
       const seqs = sequenceStore.sequences.peek();
       const layouts = trackLayouts.peek();
       for (const seq of seqs) {
         if (seq.layers.some(l => l.id === layerId)) {
           if (seq.kind === 'fx' || seq.kind === 'content-overlay') {
             return seq.inFrame ?? 0;
           }
           const layout = layouts.find(t => t.sequenceId === seq.id);
           return layout?.startFrame ?? 0;
         }
       }
       return 0;
     }
     ```
   - `localFrame = displayFrame - startFrame`
   - Find the dot in the dots array where `dot.frame === localFrame` (if it exists and is within the keyframe range)

7. Render SVG:
   ```tsx
   <svg style={{
     position: 'absolute', inset: 0,
     width: '100%', height: '100%',
     overflow: 'visible',
     pointerEvents: 'none',
   }}>
   ```

   Counter-scaled sizes (per D-03, using CSS variable `var(--color-accent)` which resolves to `#2D5BE3` etc.):
   - `dotRadius = 2 / zoom` (~2px screen)
   - `kfCircleRadius = 5 / zoom` (~5px screen)
   - `kfStrokeWidth = 1.5 / zoom`
   - `highlightRadius = 4 / zoom` (~4px screen, larger than regular dot)

   Render in this order:
   a. **Trail dots** -- one `<circle>` per dot, cx/cy from dot positions, r=dotRadius, fill="var(--color-accent)", opacity=0.35. Use `key={dot.frame}`.
   b. **Current frame highlight** -- if currentDot exists, render a `<circle>` at currentDot position, r=highlightRadius, fill="var(--color-accent)", opacity=0.9.
   c. **Keyframe circles** -- for each keyframe circle position:
      - `selected = selectedKeyframeFrames.has(kf.frame)`
      - If selected: fill="var(--color-accent)", stroke="var(--color-accent)", strokeWidth=kfStrokeWidth
      - If not selected: fill="none", stroke="var(--color-accent)", strokeWidth=kfStrokeWidth
      - r=kfCircleRadius
      - Set `data-kf-frame={kf.frame}` attribute for debugging
      - Do NOT set pointerEvents here (entire SVG is pointerEvents: none; drag handled by TransformOverlay in Plan 02)

   For performance with high frame counts (> 300 frames between first and last keyframe):
   - Instead of individual circle elements, render a `<polyline>` with `stroke-dasharray` to create a dotted line effect
   - Still render individual circles for keyframe positions and current-frame highlight
