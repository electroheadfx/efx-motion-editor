<objective>
Fix the two-part architectural gap where keyframe property edits (sidebar and canvas drag) do not update the canvas preview in real-time.

Purpose: UAT Test 5 reported that modifying keyframe properties requires clicking "Update keyframe" or refreshing the page to see changes on canvas. The root cause is two dead-end signal paths: (1) sidebar between-keyframe edits write to transientOverrides which the renderer never reads, and (2) canvas drag handlers update layer.transform which interpolateLayers() immediately overwrites with keyframe values.

Output: Both edit paths write directly to layer.keyframes, which flows through the existing sequenceStore.sequences signal that Preview.tsx already subscribes to, causing immediate re-render.
</objective>

**Helper pattern:** To avoid repeating the layer lookup, add a small helper at the top of the handler scope:
```typescript
function getLayerById(layerId: string): Layer | undefined {
  const seqs = sequenceStore.sequences.peek();
  for (const seq of seqs) {

    if (l) return l;

  return undefined;
```
