Create `Application/src/components/canvas/PaintOverlay.tsx` (per D-09, D-11). This component captures all pointer events when paint mode is active.

Props interface:
```typescript
interface PaintOverlayProps {
  containerRef: RefObject<HTMLDivElement>;
  isSpaceHeld: RefObject<boolean>;
  onPanStart: (e: PointerEvent) => void;
}
```

The component renders an absolutely-positioned div covering the canvas area with `pointerEvents: 'all'`, `cursor` based on active tool (crosshair for brush/eraser, pointer for eyedropper, crosshair for fill, crosshair for shapes), and `zIndex: 20` (above canvas, same as TransformOverlay).

State management via useRef:
- `isDrawing: RefObject<boolean>` -- true during active stroke
- `currentPoints: RefObject<[number, number, number][]>` -- accumulated points for current stroke
- `shapeStart: RefObject<{x: number; y: number} | null>` -- shape tool start point
- `currentElementId: RefObject<string>` -- id of the element being drawn

Event handlers:

**handlePointerDown(e: PointerEvent):**
1. If `isSpaceHeld.current === true`, delegate to `onPanStart(e)` and return (per D-11: Space+drag pans)
2. If left button (button === 0):
   - Get containerRef.current.getBoundingClientRect()
   - Convert to project coords via `clientToCanvas(e.clientX, e.clientY, rect, canvasStore.zoom.peek(), canvasStore.panX.peek(), canvasStore.panY.peek(), projectStore.width.peek(), projectStore.height.peek())`
   - Get pressure from e.pressure (default 0.5 if 0)
   - If activeTool is 'brush' or 'eraser': start stroke -- set isDrawing=true, init currentPoints with first point [x, y, pressure], generate elementId via crypto.randomUUID()
   - If activeTool is 'line', 'rect', 'ellipse': store shapeStart = {x, y}, generate elementId
   - If activeTool is 'eyedropper': read pixel color from canvas at click point. Get the preview canvas via `containerRef.current.querySelector('canvas')`. Use `canvas.getContext('2d').getImageData(...)` at the clicked project coordinate (scaled by dpr). Set `paintStore.setBrushColor(hex)`.
   - If activeTool is 'fill': will be handled in Plan 06 (no-op for now)
   - Call `e.currentTarget.setPointerCapture(e.pointerId)` for reliable tracking
   - `e.preventDefault()` to prevent text selection

**handlePointerMove(e: PointerEvent):**
1. If not drawing and no shape in progress, return
2. Convert to project coords via clientToCanvas
3. If brush/eraser: push [x, y, pressure] to currentPoints. Request a live preview re-render by calling a `requestAnimationFrame`-gated function that renders the in-progress stroke to a temporary canvas overlay.
4. If shape tool: update shape preview (x2, y2)

**handlePointerUp(e: PointerEvent):**
1. If brush/eraser and isDrawing:
   - Create PaintStroke from accumulated points with current brush settings
   - Get selected paint layer id from `layerStore.selectedLayer.peek()`
   - Get current frame from `timelineStore.currentFrame.peek()`
   - Call `paintStore.addElement(layerId, frame, stroke)` (this handles undo)
   - Clear drawing state
2. If shape tool and shapeStart is set:
   - Create PaintShape from start/end points
   - Call `paintStore.addElement(layerId, frame, shape)`
   - Clear shapeStart
3. Release pointer capture
4. Trigger canvas re-render

The component also renders an in-progress stroke preview. Use a `useRef` to hold a temporary canvas element (same size as the project canvas, positioned absolutely over it) that shows the stroke being drawn in real-time. On pointerMove, clear this temp canvas and render the current stroke points using `strokeToPath` from paintRenderer. On pointerUp, clear the temp canvas (the committed stroke will be rendered by PreviewRenderer in Plan 03).

For the live preview during drawing, use `renderPaintFrame` from paintRenderer on the temporary canvas context with just the single in-progress element.

Cursor styles by tool:
- brush: `crosshair`
- eraser: `crosshair`
- eyedropper: `crosshair` (or use `url(data:image/svg+xml,...)` custom cursor if desired)
- fill: `crosshair`
- line/rect/ellipse: `crosshair`
