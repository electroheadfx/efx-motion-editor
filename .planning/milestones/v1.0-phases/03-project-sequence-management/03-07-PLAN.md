**Fix 2 — forceFallback for SortableJS (closes UAT test 3 gap):**
- Add `forceFallback: true` to the Sortable.create options object (around line 20-33)
- This makes SortableJS use CSS transform + pointer events instead of native HTML5 DnD
- Tauri v2 intercepts native HTML5 drag events (dragDropEnabled defaults to true); forceFallback bypasses this entirely
- Also add `fallbackClass: 'opacity-30'` to style the fallback ghost element (since ghostClass only works with native DnD)

Keep all existing functionality: DOM revert pattern in onEnd, sequences.length useEffect dep, handle: '.seq-drag-handle'.

Do NOT modify tauri.conf.json — forceFallback solves the problem without disabling Tauri's file drop feature which is needed for image import.
