**1. Create `Application/src/components/timeline/ThumbnailCache.ts`:**

A class that lazily loads thumbnail images for use with Canvas `drawImage()`. Design:

- `get(imageId: string, thumbnailUrl: string): HTMLImageElement | null` -- returns cached image if loaded and complete, otherwise starts loading and returns null (caller draws placeholder)
- `onLoad: (() => void) | null` -- callback fired when any image finishes loading (timeline sets this to trigger redraw)
- Private `cache: Map<string, HTMLImageElement>` -- loaded images
- Private `loading: Set<string>` -- currently loading image IDs (prevents duplicate requests)
- `clear()` -- clears cache and loading sets
- When an image loads (`img.onload`), move from loading to cache, call `onLoad` callback
- On `img.onerror`, remove from loading set (don't retry -- thumbnail might be missing)

```typescript
export class ThumbnailCache {
  private cache = new Map<string, HTMLImageElement>();
  private loading = new Set<string>();
  onLoad: (() => void) | null = null;

  get(imageId: string, thumbnailUrl: string): HTMLImageElement | null {
    const cached = this.cache.get(imageId);
    if (cached?.complete) return cached;
    if (!this.loading.has(imageId)) {
      this.loading.add(imageId);
      const img = new Image();
      img.onload = () => {
        this.cache.set(imageId, img);
        this.loading.delete(imageId);
        this.onLoad?.();
      };
      img.onerror = () => {
        this.loading.delete(imageId);
      };
      img.src = thumbnailUrl;
    }
    return null;
  }

  clear() {
    this.cache.clear();
    this.loading.clear();
  }
}
```

**2. Create `Application/src/components/timeline/TimelineRenderer.ts`:**

A pure TypeScript class (no Preact imports) that handles all Canvas 2D drawing. The class receives a canvas element in its constructor and provides a `draw()` method.

Design constants:
- `BASE_FRAME_WIDTH = 60` -- width of one frame at zoom=1
- `TRACK_HEIGHT = 52` -- height of one track row
- `TRACK_HEADER_WIDTH = 80` -- width of the track label column
- `RULER_HEIGHT = 24` -- height of the time ruler at the top
- `PLAYHEAD_COLOR = '#E55A2B'` -- orange-red playhead (matches current mockup)
- `TRACK_BG = '#111111'` -- dark track background
- `TRACK_HEADER_BG = '#0D0D0D'` -- darker header background
- `FRAME_BORDER_COLOR = '#222222'` -- thin border between frames

Constructor:
- Takes `canvas: HTMLCanvasElement`
- Calls `setupCanvas()` for Retina DPI scaling (Pitfall 2 from research)
- Creates `ThumbnailCache` instance
- Sets `thumbnailCache.onLoad` to trigger a redraw via dirty flag

`setupCanvas()` method:
- `const dpr = window.devicePixelRatio || 1`
- `const rect = canvas.getBoundingClientRect()`
- `canvas.width = rect.width * dpr`
- `canvas.height = rect.height * dpr`
- `ctx.scale(dpr, dpr)` -- all subsequent drawing is in CSS pixels
- Store `displayWidth = rect.width` and `displayHeight = rect.height` for hit testing

`resize()` method:
- Called on window resize / container resize via ResizeObserver
- Re-runs `setupCanvas()`

`draw(state)` method takes `{ frame, zoom, scrollX, tracks, imageStore }`:
- Clear entire canvas
- Draw time ruler at top (frame numbers at regular intervals based on zoom)
- For each track in `tracks` (TrackLayout[]):
  - Draw track header (sequence name, left column)
  - For each KeyPhotoRange in the track:
    - Compute x position: `range.startFrame * frameWidth - scrollX + TRACK_HEADER_WIDTH`
    - Compute width: `range.holdFrames * frameWidth`
    - Skip if entirely outside visible area (virtualization -- TIME-01)
    - Draw frame background rectangle with alternating key-photo colors
    - For each individual frame within the range:
      - Get thumbnail from cache: `thumbnailCache.get(range.imageId, imageStore.getDisplayUrl(image))`
      - If available: `ctx.drawImage(img, x, y, frameWidth, TRACK_HEIGHT - 4)` with 2px padding
      - If not available: draw a placeholder rect with the key photo index text
    - Draw key photo border/separator lines between different key photos
- Draw playhead line: vertical line at `frame * frameWidth - scrollX + TRACK_HEADER_WIDTH` spanning full height, using PLAYHEAD_COLOR, 2px wide
- Draw playhead triangle at the top (in the ruler area) for the grab handle

`frameFromX(clientX: number, canvasRect: DOMRect): number` helper:
- `const x = clientX - canvasRect.left - TRACK_HEADER_WIDTH`
- `return Math.floor((x + scrollX) / frameWidth)`
- Clamp to [0, totalFrames - 1]

`destroy()` method:
- Clears thumbnail cache

Key detail: `frameWidth = BASE_FRAME_WIDTH * zoom`. This is the pixel width of one frame at the current zoom level.

The renderer does NOT manage state. It receives all state as parameters to `draw()`. This keeps it pure and testable.

**1. Create `Application/src/components/timeline/TimelineInteraction.ts`:**

A class that attaches mouse/wheel/touch event handlers to the canvas element and translates them into timelineStore actions. Takes a canvas element and a `frameFromX` function.

Event handlers:

**Click-to-seek (TIME-02):**
- `mousedown` on canvas: compute frame from x-coordinate using `frameFromX()`. Check if click is on the playhead (within 5px of playhead x position) -- if so, start drag mode. If not on playhead, just seek: `playbackEngine.seekToFrame(frame)`.

**Playhead scrubbing (TIME-03):**
- On `mousedown` on playhead: set `isDragging = true`, use `setPointerCapture(e.pointerId)` for smooth capture
- On `mousemove` while dragging: compute frame from x, call `playbackEngine.seekToFrame(frame)` continuously
- On `mouseup`: release capture, set `isDragging = false`
- The pointer capture ensures smooth scrubbing even when cursor leaves the canvas

**Zoom (TIME-04):**
- `wheel` event with `ctrlKey || metaKey`: zoom centered on cursor position
  - `const cursorX = e.clientX - canvasRect.left - TRACK_HEADER_WIDTH`
  - `const oldZoom = timelineStore.zoom.peek()`
  - `const factor = e.deltaY > 0 ? 0.9 : 1.1`
  - `const newZoom = clamp(oldZoom * factor, 0.1, 10)`
  - Adjust scrollX to keep the frame under cursor stable:
    `const frameUnderCursor = (timelineStore.scrollX.peek() + cursorX) / (BASE_FRAME_WIDTH * oldZoom)`
    `const newScrollX = frameUnderCursor * BASE_FRAME_WIDTH * newZoom - cursorX`
  - Call `timelineStore.setZoom(newZoom)` and `timelineStore.setScrollX(Math.max(0, newScrollX))`
- `wheel` event without modifier: horizontal scroll
  - `timelineStore.setScrollX(Math.max(0, timelineStore.scrollX.peek() + e.deltaX + e.deltaY))`
- macOS pinch: listen for `gesturechange` event on the canvas. `e.scale` provides the pinch scale factor. Apply same zoom logic as wheel.
- Always `e.preventDefault()` on handled wheel events to prevent page scroll

**Constructor / attach / detach pattern:**
- `attach(canvas: HTMLCanvasElement)` -- adds all event listeners
- `detach()` -- removes all event listeners (called on component unmount)

Import `playbackEngine` from `../../lib/playbackEngine` for seek calls.
Import `timelineStore` from `../../stores/timelineStore` for zoom/scroll.
Import `BASE_FRAME_WIDTH, TRACK_HEADER_WIDTH` constants from TimelineRenderer (export them as named constants).

**2. Create `Application/src/components/timeline/TimelineCanvas.tsx`:**

A thin Preact component that:
- Creates a `<canvas>` element with a ref
- On mount (`useEffect`):
  - Creates `TimelineRenderer` instance with the canvas
  - Creates `TimelineInteraction` instance and attaches to canvas
  - Sets up a `ResizeObserver` on the canvas container to call `renderer.resize()` on container size changes
  - Sets up an `effect()` from `@preact/signals` that subscribes to:
    - `timelineStore.currentFrame.value`
    - `timelineStore.zoom.value`
    - `timelineStore.scrollX.value`
    - `trackLayouts.value`
  - On any change, calls `renderer.draw({ frame, zoom, scrollX, tracks, imageStore })`
- On unmount: disposes effect, detaches interaction, destroys renderer, disconnects ResizeObserver
- The canvas element has `class="w-full h-full"` and the parent div is `flex-1 min-h-0 overflow-hidden`

Use `useRef` for the canvas element. Use `useRef` for the renderer and interaction instances to avoid recreating on re-render.
