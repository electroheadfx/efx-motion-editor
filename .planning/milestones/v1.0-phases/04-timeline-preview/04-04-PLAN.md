<task type="auto">
  <name>Task 1: Add Cache-Control headers to efxasset protocol and cache-buster to image URLs</name>
  <files>Application/src-tauri/src/lib.rs, Application/src/lib/ipc.ts</files>
  <action>
Two changes are needed to break WebKit WebView's aggressive caching of custom protocol responses:

**1. In `Application/src-tauri/src/lib.rs`** — Add cache-busting headers to the efxasset protocol response:

In the `register_uri_scheme_protocol("efxasset", ...)` handler, on the success branch where the Response is built, add these headers BEFORE `.status(200)`:

```rust
.header("Cache-Control", "no-cache, no-store, must-revalidate")
.header("Pragma", "no-cache")
```

The full success response builder becomes:
```rust
tauri::http::Response::builder()
    .header("Content-Type", mime)
    .header("Access-Control-Allow-Origin", "*")
    .header("Cache-Control", "no-cache, no-store, must-revalidate")
    .header("Pragma", "no-cache")
    .status(200)
    .body(data)
    .unwrap()
```

**2. In `Application/src/lib/ipc.ts`** — As a belt-and-suspenders defense, append a cache-busting query parameter to the URL so WebKit sees each imageId as a unique resource even if it ignores Cache-Control on custom protocols:

Change the `assetUrl` function to accept an optional `bustKey` parameter:

```typescript
export function assetUrl(filePath: string, bustKey?: string): string {
  const encoded = encodeURIComponent(filePath)
    .replace(/%2F/g, '/')
    .replace(/%3A/g, ':');
  const bust = bustKey ? `?v=${bustKey}` : '';
  return `efxasset://localhost${encoded}${bust}`;
}
```

This is backward-compatible — all existing callers pass no bustKey and get the same URL as before. Only the preview renderer will pass the imageId as bustKey to guarantee uniqueness.

**Why both changes:** Cache-Control headers may not be respected by WebKit on custom protocols (implementation-defined). The query param ensures each unique image gets a unique URL regardless. The imageId IS the bustKey because each distinct image has a unique imageId in the store.
