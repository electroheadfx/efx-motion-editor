```typescript
private async loadPaperTextures(papers: Array<{ name: string; url: string }>, defaultPaper: string): Promise<void> {
  for (const paper of papers) {
    try {
      const result = await loadPaperTexture(paper.url, this.width, this.height)
      this.paperTextures.set(paper.name, result)
    } catch (e) {
      console.error(`Failed to load paper texture: ${paper.name}`, e)
    }
  }
  if (defaultPaper && this.paperTextures.has(defaultPaper)) {
    this.setPaperGrain(defaultPaper)
  } else if (this.paperTextures.size > 0) {
    const firstKey = this.paperTextures.keys().next().value
    if (firstKey) this.setPaperGrain(firstKey)
  }
  this.bgData = drawBg(this.bgCtx, this.state.bgMode, this.width, this.height, this.paperTextures, this.userPhoto)
  this.dualCanvas.dryCtx.drawImage(this.bgCanvas, 0, 0)
}
```

Per D-04 from 06-CONTEXT.md: Delay onEngineReady callback until after paper textures have loaded.

**Approach:** Add an async `init()` method to EfxPaintEngine that awaits loadPaperTextures, then have the Preact wrapper call init() and fire onEngineReady after it resolves. The constructor stays synchronous (creates buffers, sets up events, starts render loop) but the "ready" signal waits for textures.

1. **In `paint-rebelle-new/src/engine/EfxPaintEngine.ts`**:

   Add a public async `init()` method that does what the constructor currently does fire-and-forget:

   ```typescript
   /**
    * Async initialization: loads paper textures and draws background.
    * Call after construction. onEngineReady should fire after this resolves.
    */
   async init(): Promise<void> {
     await this.loadPaperTextures(this._initPapers, this._initDefaultPaper)
   }
   ```

   In the constructor (~line 234-235):
   - Store config.papers and config.defaultPaper as private fields `_initPapers` and `_initDefaultPaper` so init() can access them.
   - Keep the fire-and-forget `this.loadPaperTextures(...)` call as-is for backward compatibility (direct EfxPaintEngine users who don't call init() still get textures loaded eventually).
   - OR, cleaner: remove the fire-and-forget call from constructor entirely, and let init() be the only path. If init() is never called, textures don't load (acceptable -- consumers must call init()).

   Choose the cleaner approach: remove fire-and-forget from constructor. Store papers config. Add init():

   - Add private fields:
     ```typescript
     private readonly _initPapers: Array<{ name: string; url: string }>
     private readonly _initDefaultPaper: string
     ```
   
   - In constructor, REPLACE line 235 (`this.loadPaperTextures(config.papers, config.defaultPaper || '')`) with:
     ```typescript
     this._initPapers = config.papers || []
     this._initDefaultPaper = config.defaultPaper || ''
     ```
   
   - Add the public async init() method right after the constructor (in the PUBLIC API section):
     ```typescript
     /**
      * Async initialization: loads paper textures and redraws background.
      * Must be called after construction for full engine readiness.
      */
     async init(): Promise<void> {
       await this.loadPaperTextures(this._initPapers, this._initDefaultPaper)
     }
     ```

   - Keep `this.bgData = drawBg(...)` and `this.dualCanvas.dryCtx.drawImage(this.bgCanvas, 0, 0)` in the constructor (lines 240-241) -- this draws the initial background with procedural texture. The loadPaperTextures method already redraws background after loading real textures.

   - Keep `this.render()` call in constructor (line 244) -- render loop should start immediately.
