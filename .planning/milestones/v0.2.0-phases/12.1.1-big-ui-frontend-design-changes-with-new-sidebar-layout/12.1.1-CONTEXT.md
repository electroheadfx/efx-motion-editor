**Gathered:** 2026-03-17

<domain>
## Phase Boundary

Restructure the sidebar into 3 resizable sub-windows (SEQUENCES, LAYERS, PROPERTIES) with improved key photos inline placement, keyframe navigation bar, interpolation mode swap, resizable sidebar width, double-line collapse trigger, and pixel-perfect visual polish matching the .pen design specification.

<decisions>
## Implementation Decisions

### Key Photos Inline Behavior
- Key photos appear **directly under the selected sequence** inside the sequence group card — no separate "KEY PHOTOS" section header
- **Auto-collapse previous:** Selecting a different sequence auto-collapses the previous one's key photos and auto-expands the new one
- **Scrub auto-select:** When scrubbing on the timeline and releasing on a different sequence, that sequence auto-selects (which triggers collapse/expand of key photos)
- **Slide animation:** Key photos expand/collapse with a smooth height transition (~150ms)
- **+ button opens full importer:** Shows ALL imported media (not filtered), same as the Imported toolbar view
- **Card treatment:** Both selected (border `#636382`, bg `#1f1f33`) and unselected (border `#3D3D5A`) sequences get card styling with borders

### Sub-Window Resizing
- **3 panels:** SEQUENCES, LAYERS, PROPERTIES — each independently resizable via ellipsis drag handles between them
- **Collapse to zero via drag:** Dragging the resizer all the way collapses the panel entirely (acts as alternative to chevron toggle)
- **Independent scrolling:** Each sub-window scrolls independently — the sidebar itself never scrolls
- **Persist sizes:** Panel heights saved to config/store, restored on app restart
- **Grab cursor:** Show `cursor: grab` on hover over the ellipsis divider, `cursor: grabbing` while dragging

### Keyframe Navigation + Interpolation Swap
- **Keyframe nav bar** replaces the old `+ Keyframe` button: `< + 🗑 >` layout
- **< > arrows** navigate playhead to previous/next keyframe; disabled (grayed out) when at first/last keyframe
- **+ button** adds a keyframe when playhead is NOT on a keyframe; shows **update icon** when playhead IS on a keyframe (replaces current `◆ Update` text with an icon)
- **Trash button** grayed out unless playhead is on a keyframe — deletes the current keyframe
- **Hover effect:** Light color on mouse over each nav button
- **Interpolation swap trigger:** When a **keyframe diamond is selected on the timeline** (not just playhead position), Blend+Opacity row is replaced by inline interpolation curve controls
- **Interpolation controls:** Same curve presets as the current double-click popover (Linear, Ease In, Ease Out, Ease In-Out, Cubic) rendered inline in the properties panel
- **Remove double-click popover:** The old double-click-to-show-interpolation-menu behavior is removed entirely
- **Deselecting a diamond** brings back Blend+Opacity row

### Sidebar Visual Polish
- **New sidebar-specific CSS variables** with theme variants, stored in config YAML alongside existing theme variables — not hardcoded hex, not reusing existing theme vars
- **Sidebar width resizable:** Default ~300px, draggable to resize horizontally (similar pattern to panel resizers)
- **Double-line collapse trigger:**
  - The decorative double lines are **10% of sidebar height, vertically centered** on the right boundary
  - The **full-height right boundary** is the clickable/hoverable zone — not just the visible lines
  - Subtle glow/brighten on hover over the full-height zone
  - Click to collapse; when collapsed, the double line stays at the left window boundary to click and expand
  - Remove the current `«` chevron collapse button
- **Custom thin scrollbars:** 4px width, dark track, lighter thumb, matches sidebar dark theme, auto-hide when not scrolling
- **Panel backgrounds:** Each sub-window has `#252539` background with `cornerRadius: 8` for contrast against `#2B2B3D` sidebar wrapper

### Claude's Discretion

- Scrollbar thumb/track exact colors (should match sidebar dark theme)
- Animation easing curve for key photo slide transition
- Exact hover glow intensity on collapse trigger

- `uiStore.sidebarWidth` / `uiStore.propertiesPanelWidth` — Signals already exist but are unused, ready for resizer implementation

### Established Patterns
- **Signal-based state:** All UI state uses Preact signals from uiStore
- **Transient overrides:** Keyframe editing between keyframes uses transientOverrides pattern — keyframe nav bar must integrate with this
- **Portal rendering:** Context menus use Preact portals — interpolation popover removal simplifies this
- **Pointer drag pattern:** NumericInput label-drag-to-scrub uses pointermove/pointerup — same pattern for panel resizers

### Integration Points
- `EditorShell.tsx` — Renders `<LeftPanel />` with sidebarCollapsed conditional — sidebar width resizer attaches here
- `uiStore.selectSequence()` — Sequence selection triggers key photo expand — needs to also auto-collapse previous
- `keyframeStore.isOnKeyframe` / `keyframeStore.selectedDiamondId` — Drives interpolation/blend+opacity swap
- `sequenceStore` — Timeline scrub release auto-selects sequence
- Config YAML at `~/.config/efx-motion/builder-config.yaml` — New sidebar CSS variables stored here alongside theme
