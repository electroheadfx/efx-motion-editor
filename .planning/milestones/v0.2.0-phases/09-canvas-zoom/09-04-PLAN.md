<objective>
Fix Cmd+=/Cmd+-/Cmd+0 keyboard shortcuts by adding a View menu to the Tauri native menu that intercepts these accelerators before WKWebView's built-in zoom handlers consume them. Same proven pattern as the existing Edit menu Undo/Redo fix.

Purpose: On macOS, WKWebView has native accelerators for Cmd+=, Cmd+-, and Cmd+0 (page zoom). These intercept keydown events at the Cocoa layer before they reach JavaScript. The existing tinykeys bindings in shortcuts.ts are correct but never fire. Adding custom Tauri menu items with matching accelerators overrides the native handlers and emits events to the frontend.

Output: Working Cmd+=/Cmd+-/Cmd+0 shortcuts via Tauri View menu items + frontend event listeners.
</objective>
