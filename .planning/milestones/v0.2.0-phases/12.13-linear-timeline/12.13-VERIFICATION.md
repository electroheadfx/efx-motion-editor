#### 1. Layout Toggle Visual

**Test:** Open the app, locate the timeline toolbar. Confirm S/L buttons appear between zoom controls and the Add Layer button. Click L.
**Expected:** All content sequences collapse onto one horizontal row. Sequence boundary separators (2px vertical lines) appear between adjacent sequences. Canvas redraws immediately.
**Why human:** Canvas 2D rendering output cannot be verified programmatically.

#### 2. Name Overlay Toggle

**Test:** In linear mode, confirm the display shows "T+N" button. Click it to toggle.
**Expected:** In T+N mode, each sequence segment shows a semi-transparent black label bar at its bottom with the sequence name. In T mode (thumb-only), no label bar appears.
**Why human:** Text overlay rendering and legibility require visual confirmation.

#### 3. X-Position Sequence Selection

**Test:** In linear mode with at least 2 content sequences, click on different sequence segments horizontally.
**Expected:** Each click selects the sequence under the cursor (correct sequence highlighted). No incorrect Y-based selection (should not always select first sequence).
**Why human:** Selection state changes and canvas highlight are runtime state changes.

#### 4. Keyframe Diamond Positioning

**Test:** Select a layer that has keyframes, then switch to linear mode.
**Expected:** Keyframe diamonds appear centered vertically on the single content row, not at a Y position corresponding to the sequence's stacked-mode track index.
**Why human:** Keyframe Y coordinate depends on canvas rendering and the effectiveTrackIndex=0 branching at runtime.

#### 5. Layout Persistence

**Test:** Switch to stacked mode, close the app, reopen it.
**Expected:** Timeline opens in stacked mode (persisted preference). Repeat with linear mode.
**Why human:** Persistence requires Tauri IPC config file write and read at startup — runtime-only behavior.
