# Phase 12.6: Layer Sequence Auto-Selection and Key Photo UX Fixes - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix sequence/layer auto-selection UX: auto-select new sequences with open key photo section, open importer on creation with multi-select confirm flow, auto-select top-most layer on sequence selection, and improve key photo strip sync with timeline.

</domain>

<decisions>
## Implementation Decisions

### New sequence creation flow
- Clicking "+ Add" creates the sequence, auto-selects it in the sidebar (key photos section expands with slide animation), then opens the Imported view (`setEditorMode('imported')`)
- Importer opens in **multi-select mode** with a prominent **Confirm button** — user selects multiple photos, clicks Confirm to add all as key photos
- **Cancel/close behavior:** If the user closes the importer or clicks Cancel without confirming, the newly created sequence is **deleted entirely** (treated as uncommitted)
- This flow only applies to the "+ Add" button — duplicate and other creation paths do NOT trigger the importer flow

### Layer auto-selection on sequence switch
- When switching sequences in the sidebar, **auto-select the top-most layer** (highest index in the layer stack)
- If the sequence has only the base layer, select the base layer
- Auto-selection happens on **every** sequence switch, regardless of layer count
- Auto-selecting the layer also **shows transform overlay handles** on the canvas (full visual feedback, same as manual click)
- Both `layerStore.setSelected()` and `uiStore.selectLayer()` must be called (bidirectional sync pattern from Phase 11-04)

### Empty sequence UX
- For **existing** sequences with no key photos (e.g., user deleted all of them), replace the passive "No key photos yet" text with a **clickable "Add photos" prompt** that opens the Imported view
- This is a normal importer open — closing without adding photos does NOT delete the sequence (cancel-delete only applies to brand new sequences from "+ Add")

### Key photo section behavior
- Clicking a key photo in the strip **selects the top-most layer** (or base layer if only one exists) — shows PROPERTIES panel with controls
- Clicking a key photo **auto-seeks the playhead** to that key photo's frame position in the timeline
- **Single highlight style only** — the key photo under the playhead gets the thick ring; clicking a key photo seeks the playhead there so the same highlight applies
- **Auto-scroll key photo strip** on timeline interactions (scrub, frame navigation, keyframe nav) to keep the active key photo visible — but **NOT during playback**
- When **playback stops**, snap the strip to show the active key photo

### Claude's Discretion
- Exact animation timing for importer open/close transition
- Confirm button styling and placement in the Imported view
- How to track "pending new sequence" state for cancel-delete behavior
- Auto-scroll animation easing for key photo strip
- Whether to debounce strip auto-scroll during rapid scrubbing

</decisions>

## Existing Code Insights

### Reusable Assets
- `sequenceStore.createSequence()` — Already sets activeSequenceId; needs post-creation hook for importer flow
- `uiStore.setEditorMode('imported')` — Existing mechanism to open Imported view
- `playbackEngine.seekToFrame()` — Used by Phase 12.2 for auto-seek; reuse for key photo click-to-seek
- `trackLayouts` from `frameMap.ts` — Frame position lookups for key photo seek targets
- `KeyPhotoStripInline` — Already has horizontal scroll with wheel handler; needs programmatic scrollLeft for auto-scroll

### Established Patterns
- **Bidirectional layer selection:** `layerStore.setSelected(id)` + `uiStore.selectLayer(id)` (Phase 11-04)
- **Auto-seek on selection:** `handleSelect` in SequenceList already seeks playhead via trackLayouts (Phase 12.2)
- **Signal-based state:** All UI state uses Preact signals from uiStore/sequenceStore
- **Editor mode switching:** `editorMode` signal drives WelcomeScreen/ImportedView/SettingsView rendering in EditorShell

### Integration Points
- `LeftPanel.tsx` line 167 — "+ Add" button onClick needs to chain: create → select → open importer
- `ImportGrid.tsx` / `ImportedView.tsx` — Needs multi-select confirm mode and cancel-delete callback
- `SequenceList.tsx` handleSelect — Needs to add layer auto-selection after sequence switch
- `KeyPhotoStrip.tsx` click handler — Needs to add layer selection + playhead seek
- `playbackEngine` stop callback — Needs to trigger strip auto-scroll to active key photo

</code_context>

<specifics>
## Specific Ideas

- The importer confirm button should be prominent and easy to see — not buried in a corner
- Cancel-delete for new sequences means the sequence creation is "transactional" — only committed when photos are confirmed
- Key photo strip auto-scroll should feel smooth but not distracting — sync on timeline interactions and playback stop, not during playback itself

</specifics>
