### Human Verification Required

#### 1. Tile repetition at high zoom

**Test:** Open the app with a sequence containing at least one key photo. Zoom the timeline in to a high zoom level where frame cells are wider than the thumbnail tile width.
**Expected:** Multiple repetitions of the thumbnail image appear side-by-side within the frame cell area. Images are NOT stretched — they repeat at natural proportions. No visible gaps or separators between repeated tiles.
**Why human:** Canvas pattern tile visual correctness cannot be verified statically.

#### 2. Low-zoom solid color fallback

**Test:** Zoom out until frame cells are very small (below approximately 4px wide).
**Expected:** Frame cells switch to solid alternating colors (dark blue `#1A1A2A` and dark green `#1A2A1A`). No sub-pixel tile rendering attempts. The transition from tiles to solid colors should be clean.
**Why human:** The 4px threshold behavior is a runtime visual check.

#### 3. Center-crop for narrow cells (portrait photos)

**Test:** Use portrait-orientation key photos. Zoom to a level where the frame cell width is less than the tile width (one tile width = `naturalWidth * scale` where scale = `(TRACK_HEIGHT - 4) / naturalHeight`).
**Expected:** The center of the image is shown rather than the left edge. The thumbnail appears centered within the cell.
**Why human:** The `offsetX = (fw - tileWidth) / 2` centering is a visual property.

#### 4. Scroll stability

**Test:** With tiles visible at a medium-high zoom level, scroll horizontally through the timeline.
**Expected:** Tile positions remain stable relative to their frame cells during scroll. No tile drift or phase-shift artifacts.
**Why human:** The `ctx.translate(fx + offsetX, fy)` approach resets per frame — scroll correctness is a runtime rendering check.

#### 5. Frame borders and key photo separators (regression check)

**Test:** At any zoom level with tiled thumbnails visible, inspect the frame cell borders and any positions where key photo ranges change.
**Expected:** Thin grey lines (`#222222` via CSS variable) still appear between every frame cell. Thicker lines (`#444444`, 1.5px) appear at key photo range boundaries.
**Why human:** Visual regression check — these drawing calls exist in code (lines 307-309, 313-322) but confirm no z-order or clip regression.
