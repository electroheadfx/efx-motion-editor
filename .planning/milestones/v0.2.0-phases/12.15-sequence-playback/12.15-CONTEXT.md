# Phase 12.15: Sequence Playback - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Multi-select sequence isolation ("solo/mute") for playback. Users can toggle individual sequences into an isolation set from both the timeline name labels and the sidebar accent bar. When any sequences are isolated, playback plays only those sequences in order (skipping non-isolated ones), with an optional loop toggle. Also adds a global loop toggle for both isolated and full-timeline playback.

</domain>

<decisions>
## Implementation Decisions

### Timeline sequence name interaction
- Click target is the existing name label overlay at the bottom of each sequence's frame range (Canvas 2D)
- Hover state: pointer cursor + solid orange background on the text label (text + padding only, not full sequence width)
- First click on any sequence's name label: toggles that sequence INTO the isolation set. Visual: orange border around the sequence's entire frame range + name label background stays solid orange
- Second click: toggles that sequence OUT of the isolation set. Visual: border removed, label returns to normal (black semi-transparent bg)
- Clicking any sequence name label directly isolates it — does NOT require the sequence to be active first. Clicking also makes it the active sequence

### Sidebar accent bar interaction
- The existing accent bar (4px × 24px, left edge of sequence row) becomes clickable
- Only visible on the active/selected sequence (current behavior preserved)
- Click toggles the active sequence in/out of the isolation set
- When isolated: bar turns solid orange
- When not isolated: bar returns to normal color (current accent color)
- To isolate a non-active sequence from sidebar: first select it (click the row), then click the accent bar

### Multi-select isolation (both entry points)
- Isolation is a SET of sequence IDs — multiple sequences can be isolated simultaneously
- Both timeline name labels and sidebar accent bar modify the same isolation set
- Example: sequences 1-2-3, isolate 1 and 3 (from either timeline or sidebar) → playback runs 1 → 3
- Visual states sync across both UIs: isolate from timeline → sidebar bar turns orange (if that sequence is active), and vice versa

### Playback behavior when sequences are isolated
- Playback plays only isolated sequences in their timeline order, skipping non-isolated ones
- Playhead visually jumps over skipped sequences on the timeline (no dimming of skipped sequences)
- FX and content-overlay sequences that overlap isolated sequence frame ranges still render/composite
- When playback stops (loop OFF): playhead returns to the start of the first isolated sequence
- When no sequences are isolated: normal playback behavior (play whole timeline)

### Loop toggle
- New global loop toggle, applies to both isolated playback and full-timeline playback
- Loop ON: seamless loop (isolated sequences loop through their set; full timeline wraps to frame 0)
- Loop OFF: playback stops at end, playhead returns to start (of first isolated sequence, or of full timeline)
- UI: loop icon button (lucide Repeat/Repeat1) in the timeline controls bar, next to play/step buttons
- Persisted as global app preference in ~/.config/efx-motion/builder-config.yaml (not per-project)

### Deselect-all shortcut
- Escape key clears all isolation (removes all sequences from the isolation set)
- Context-aware priority: fullscreen exit > isolation clear > other dismiss behaviors

### Claude's Discretion
- Exact orange color value for border and label (match existing accent or adjust for visibility)
- Orange border line width on timeline
- Hit target size expansion for name label (how much padding around text counts as clickable)
- How the playhead "jump" animation looks when skipping non-isolated sequences
- Loop icon positioning and styling in the timeline controls bar
- .mce format version bump strategy for the new isolation field

</decisions>
