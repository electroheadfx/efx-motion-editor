# Phase 12.1: Remove Bottom Parameters Bar and Relocate to Sidebar Panel - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove the bottom `PropertiesPanel` bar entirely. Relocate all layer/FX property controls into the existing left sidebar. Additionally: move the Imported grid and Settings (FPS, Resolution, Theme) out of the sidebar/toolbar into their own full-window views. Make the sidebar horizontally collapsible. Add a uniform Scale input to Transform controls.

</domain>

<decisions>
## Implementation Decisions

### Sidebar structure (left panel, ~268px)
- Three collapsible sections: SEQUENCES, LAYERS, PROPERTIES
- Each section header has a chevron (▼/▶) to collapse/expand
- SEQUENCES and LAYERS start expanded by default
- PROPERTIES section only appears when a layer is selected; disappears on deselect
- Clicking a layer to deselect it hides the PROPERTIES section
- The entire sidebar is horizontally collapsible via a toggle icon (« / »)
- When collapsed, canvas + timeline take full width; a thin strip with the toggle icon remains for re-opening

### Key Photos placement
- Key Photos appear inline under the selected/active sequence within the SEQUENCES section
- Collapsed sequences show just the sequence name (no key photos)
- Expanding a sequence reveals its KEY PHOTOS strip + [+] button below the sequence row

### Properties section — content layers
- Appears below the LAYERS list when a content layer is selected
- Two-column grid layout for numeric inputs (~100px per input with label)
- Sections within Properties: Keyframe button, Blend, Opacity, Transform, Crop, Blur
- Transform layout (two-column):
  - Row 1: X / Y
  - Row 2: Scale (uniform, adjusts SX+SY together) / Rot
  - Row 3: SX / SY (for non-uniform scaling)
- Crop layout (two-column): T/R on row 1, B/L on row 2
- Opacity: full-width slider with percentage
- Blend mode: full-width dropdown
- Blur radius: single input

### Properties section — FX layers
- When an FX layer is selected from the timeline, the LAYERS section is replaced by an "FX: [Layer Name]" section
- Shows only the selected FX layer's properties (no list of other FX layers)
- Displays: Opacity slider, Visibility toggle, then FX-specific controls (Grain/Vignette/Color Grade/Blur/Particles/Lines/Dots)
- Deselecting the FX layer restores the normal LAYERS section

### Global Scale transform (new feature)
- New uniform "Scale" input that adjusts both SX and SY together
- Existing SX/SY inputs remain for non-uniform scaling
- Scale sits on the same row as Rotation in the two-column grid

### Imported window
- Imported grid is removed from the left sidebar entirely
- New "Imported" button in the toolbar (right side, before Settings and Export)
- Clicking "Imported" replaces the canvas+timeline area with a full Imported grid view
- The sidebar remains visible on the left — user can drag thumbnails from the Imported grid to Key Photos strip or Layers in the sidebar
- Close button (✕) near the Import button returns to editor mode (canvas+timeline)
- The [+ Import] button for adding new files lives inside this view

### Settings window
- Same pattern as Imported — replaces the canvas+timeline area
- New "Settings" button in the toolbar (right side, between Imported and Export)
- Contains: FPS selector (15/24), Resolution dropdown, Theme switcher
- Close button (✕) to return to editor mode
- FPS and Resolution are now per-project only (removes the per-sequence FPS/Res duplication)
- Theme switcher moves from toolbar to Settings window

### Toolbar cleanup
- Remove: FPS toggle (moved to Settings), Theme switcher (moved to Settings)
- Keep: New, Open, Save, dirty indicator, Blur controls (HQ, Bypass)
- Add: Imported button, Settings button (both right-aligned, before Export)

### Bottom bar removal
- The entire `PropertiesPanel` component at the bottom of `EditorShell` is removed
- The divider above it is removed
- All property editing now happens exclusively in the sidebar

### Claude's Discretion
- Sidebar collapse animation (slide vs instant)
- Exact toggle icon design and placement
- Property section divider styling (thin lines vs spacing)
- Scroll behavior when Properties section is taller than available space
- Transition animation between Editor/Imported/Settings modes
- How the Imported button appears (text, icon, or both)
- Settings page layout and styling details

</decisions>

<specifics>
## Specific Ideas

- The sidebar collapse is horizontal — it slides the entire sidebar off-screen to give maximum space to canvas and timeline
- When sidebar is collapsed, only a tiny strip with the « icon remains to re-open it
- The Imported and Settings views follow the same UX pattern: replace main content area, sidebar stays visible, close button to return
- FPS duplication (toolbar + per-sequence) is resolved by making FPS per-project only, living in Settings
- Key Photos are logically grouped under their parent sequence — collapsing a sequence hides its photos

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `NumericInput` component (`PropertiesPanel.tsx`): Drag-to-scrub with coalescing — reuse in vertical layout with minor width adjustments
- `SectionLabel` component (`PropertiesPanel.tsx`): 9px uppercase label — reuse for section headers
- `TransformSection`, `CropSection`, `BlurSection`, `FxSection` (`PropertiesPanel.tsx`): All property sections exist — refactor from horizontal to vertical layout
- `KeyframeButton` component (`PropertiesPanel.tsx`): Reuse as-is in sidebar properties
- `ThemeSwitcher` component (`ThemeSwitcher.tsx`): Move into Settings view
- `ImportGrid` component (`ImportGrid.tsx`): Move from LeftPanel into standalone Imported view
- `SequenceSettings` component (`LeftPanel.tsx`): FPS/Res controls — move to Settings view, simplify to per-project

### Established Patterns
- Collapsible sections: No existing pattern — this is new. Preact Signals can track collapse state in `uiStore`
- Content area switching: `WelcomeScreen` already conditionally renders vs editor — similar pattern for Imported/Settings modes
- SortableJS with `forceFallback:true` for drag-and-drop in sidebar — drag from Imported grid to sidebar targets

### Integration Points
- `EditorShell.tsx`: Remove `<PropertiesPanel />` from bottom, add mode switching (editor/imported/settings)
- `LeftPanel.tsx`: Major refactor — remove ImportGrid, add collapsible sections, add Properties section, restructure Key Photos under sequences
- `Toolbar.tsx`: Remove FPS toggle + ThemeSwitcher, add Imported + Settings buttons
- `uiStore.ts`: Add signals for sidebar collapsed state, active mode (editor/imported/settings), section collapse states
- `sequenceStore.ts`: Remove per-sequence FPS/Res setters (or deprecate in favor of project-level)
- `projectStore.ts`: Ensure FPS and Resolution are project-level (already has `projectStore.fps`)

</code_context>
