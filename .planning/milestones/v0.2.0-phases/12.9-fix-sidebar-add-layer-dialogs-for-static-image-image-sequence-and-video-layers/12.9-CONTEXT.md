# Phase 12.9: Fix Sidebar Add-Layer Dialogs - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current add-layer popover dialogs (thumbnail picker, OS directory picker, video list picker) with proper Imported view flows for all three content layer types: Static Image, Image Sequence, and Video. Each flow opens the Imported view with appropriate filtering and selection mode.

</domain>

<decisions>
## Implementation Decisions

### Static Image flow
- Clicking "Static Image" in the Add menu opens the Imported view in **single-select mode**
- Grid shows **images only** (videos filtered out)
- Single-click on an image creates a static-image layer immediately and **auto-closes** back to editor
- "+ Import" button available in the view to import new images if needed
- Cancel (X button) simply closes the view ŌĆö no layer created, no side effects
- Header: contextual text indicating picking mode (e.g., "Select image for layer")

### Image Sequence flow
- Clicking "Image Sequence" in the Add menu opens the Imported view in **multi-select mode**
- Grid shows **images only** (videos filtered out)
- Header: **"Select images for sequence"** with selected count and Confirm button (same pattern as + Add sequence flow from Phase 12.6)
- Confirm creates one image-sequence layer with the selected images as its imageIds array
- Cancel (X button) simply closes the view ŌĆö no layer created, no side effects

### Video flow
- Clicking "Video" in the Add menu opens the Imported view filtered to **videos only**
- Grid shows video assets with **thumbnail + film icon badge** (video thumbnail generation already exists from quick-9)
- Single-click on a video creates a video layer immediately and **auto-closes** back to editor
- "+ Import" button opens OS file dialog filtered to video extensions (mp4, mov, webm, avi)
- Imported video files are copied to the **project's videos/ subdirectory** (existing pattern)
- Cancel (X button) simply closes the view ŌĆö no layer created, no side effects

### Imported view modes
- The Imported view needs a new concept: **asset filter mode** to control what's shown in the grid
- Three filter contexts: images-only (for Static Image + Image Sequence), videos-only (for Video), all (for normal Imported browsing)
- Selection mode orthogonal to filter: single-select (Static Image, Video) or multi-select (Image Sequence)
- When opened for add-layer purposes, a callback/signal determines what happens on selection (create layer vs add key photo)

### Claude's Discretion
- How to implement the filter mode signal/state (new signal on uiStore vs parameter passing)
- Film icon badge design and positioning on video thumbnails
- Exact header text for each mode
- Whether to show a "No videos imported yet" empty state in video-only mode
- How to integrate video assets into ImportGrid (separate render path or unified grid items)

</decisions>
