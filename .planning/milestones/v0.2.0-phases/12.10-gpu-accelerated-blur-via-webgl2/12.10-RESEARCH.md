# Phase 12.10: GPU-Accelerated Blur via WebGL2 - Research

**Researched:** 2026-03-18
**Domain:** WebGL2 GPU shader programming, Canvas 2D interop, Gaussian blur algorithms
**Confidence:** HIGH

## Summary

This phase replaces two CPU-bound blur algorithms (fast downscale-upscale and HQ StackBlur) with a single GPU-accelerated WebGL2 two-pass separable Gaussian blur. The implementation creates a new `glBlur.ts` module that manages an offscreen WebGL2 context, compiles separable Gaussian blur shaders, and renders via a ping-pong framebuffer pattern (horizontal pass then vertical pass). The result is read back by drawing the GL canvas onto the target Canvas 2D context via `drawImage()`.

The approach is well-established in game and graphics programming. A 2D Gaussian kernel is separable into two 1D passes, reducing complexity from O(N^2) to O(2N) texture lookups per pixel. WebGL2's `texImage2D` accepts an HTMLCanvasElement directly (no `getImageData()` needed), and the linear texture filtering hardware optimization further halves the required samples. WebGL2 is supported in Tauri 2.0's WKWebView on macOS and in WebKitGTK 2.40+ on Linux.

**Primary recommendation:** Implement a self-contained `glBlur.ts` module (~150-200 lines) with lazy WebGL2 context initialization, pre-compiled separable Gaussian shaders, ping-pong framebuffers, and a `drawImage(glCanvas)` readback path. Use RGBA8 texture format, `texStorage2D` for allocation, and `LINEAR` filtering for the linear sampling optimization. Fall back to CPU StackBlur if `getContext('webgl2')` returns null.

### Pattern 1: Lazy WebGL2 Context Initialization
**What:** Create the WebGL2 context on first blur call, not at import time. Cache the context and reuse across frames.
**When to use:** Always -- avoids wasting GPU resources if blur is never used.
**Example:**
```typescript
// Source: WebGL best practices (MDN)
let _gl: WebGL2RenderingContext | null = null;
let _glCanvas: HTMLCanvasElement | null = null;
let _initFailed = false;

function getGL(): WebGL2RenderingContext | null {
  if (_initFailed) return null;
  if (_gl) return _gl;

  _glCanvas = document.createElement('canvas');
  const gl = _glCanvas.getContext('webgl2', {
    // alpha: true is default and recommended (alpha:false can cause
    // emulation overhead on some platforms per MDN best practices)
    premultipliedAlpha: false,
    preserveDrawingBuffer: true, // needed for drawImage readback
    antialias: false,
    depth: false,
    stencil: false,
  });

  if (!gl) {
    _initFailed = true;
    return null;
  }
  _gl = gl;
  return _gl;
}
```

### Pattern 2: Ping-Pong Framebuffer for Two-Pass Blur
**What:** Two framebuffers with attached textures. Pass 1 (horizontal) renders source -> FBO-A. Pass 2 (vertical) renders FBO-A texture -> screen (or FBO-B if needed for readback).
**When to use:** Always -- separable Gaussian blur requires two passes.
**Example:**
```typescript
// Source: webglfundamentals.org ping-pong pattern
interface BlurResources {
  program: WebGLProgram;
  texSource: WebGLTexture;
  texIntermediate: WebGLTexture;
  fbo: WebGLFramebuffer;
  vao: WebGLVertexArrayObject;
  // Uniform locations
  uTexture: WebGLUniformLocation;
  uDirection: WebGLUniformLocation;
  uResolution: WebGLUniformLocation;
  uRadius: WebGLUniformLocation;
  // Tracked dimensions
  width: number;
  height: number;
}
```

### Pattern 3: drawImage Readback (No readPixels)
**What:** After WebGL2 renders the blurred result to its canvas, use Canvas 2D `drawImage(glCanvas, ...)` to composite it. This avoids the expensive `readPixels()` GPU->CPU roundtrip.
**When to use:** Always -- this is the spec-recommended readback for compositing.
**Example:**
```typescript
// After GL render completes:
targetCtx.drawImage(glCanvas, 0, 0, width, height);
// The 2D context composites the GL canvas content directly via GPU path
```

### Pattern 4: preserveAlpha via Shader Output Masking
**What:** For RGB-only blur (generators need it to avoid alpha halos), the fragment shader blurs only RGB channels and writes `alpha = texture(source, texCoord).a` (original alpha from center texel).
**When to use:** When `preserveAlpha` flag is true.
**Recommendation (Claude's Discretion):** Use a `uniform bool u_preserveAlpha` in the fragment shader. When true, sample RGB blur normally but write the unblurred center-pixel alpha. This is simpler than maintaining two shader variants and the branch cost is negligible in a fragment shader.

### Anti-Patterns to Avoid
- **Do NOT use `getImageData()`/`putImageData()`**: This forces a GPU->CPU->GPU roundtrip. Use `texImage2D(canvas)` for upload and `drawImage(glCanvas)` for readback.
- **Do NOT call `gl.getError()` per frame**: This forces a GPU flush and pipeline stall. Only check errors during initialization/setup.
- **Do NOT change FBO attachments in the render loop**: Set up framebuffers once during initialization; rebinding attachments invalidates completeness and stalls the GPU.
- **Do NOT use `texImage2D` for resize**: Use `texStorage2D` for initial allocation; recreate textures only when dimensions change.
- **Do NOT use RGB8 internal format**: RGB formats are emulated as RGBA internally on most GPUs with performance overhead. Always use RGBA8.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Gaussian kernel weights | Manual weight calculation | Standard formula: `w(i) = exp(-i^2 / (2 * sigma^2))`, precomputed at compile time | Kernel weights are pure math; runtime computation is fine for small kernels but precomputed is simpler |
| CPU fallback blur | Custom CPU convolution | `stackblur-canvas` (already in project) | StackBlur is a well-tested O(N) approximation; don't rewrite it |
| WebGL2 context loss recovery | Complex state machine | Simple re-init: set `_gl = null`, next frame lazily recreates | Context loss is rare in desktop Tauri; full state machine is overkill |
| Fullscreen quad geometry | Custom vertex buffer | Two-triangle fullscreen quad via hardcoded vertices | Trivial pattern: 4 vertices, 2 triangles |

**Key insight:** The GL module should be ~150-200 lines because WebGL2 two-pass blur is a well-understood pattern. The complexity is in getting the plumbing right (texture formats, framebuffer setup, resize handling), not in algorithmic novelty.

## Common Pitfalls

### Pitfall 1: preserveDrawingBuffer and drawImage Timing
**What goes wrong:** Without `preserveDrawingBuffer: true`, the WebGL canvas contents may be cleared before `drawImage()` reads them back.
**Why it happens:** WebGL spec allows implementations to clear the drawing buffer after compositing. Without preserve, the buffer is undefined after the current event loop task.
**How to avoid:** Set `preserveDrawingBuffer: true` on context creation. The performance cost is acceptable for an offscreen canvas that is never composited by the browser.
**Warning signs:** Intermittent blank/black output from the blur.

### Pitfall 2: Texture Dimension Mismatch on Resize
**What goes wrong:** Canvas dimensions change (zoom, window resize) but GL textures/framebuffers retain old size, causing stretched or cropped blur output.
**Why it happens:** `texStorage2D` is immutable -- you cannot resize a texture created with it.
**How to avoid:** Track current width/height in the BlurResources struct. On dimension change, delete old textures and framebuffer, recreate with new dimensions.
**Warning signs:** Blurred output doesn't match canvas size after resize.

### Pitfall 3: WebKit premultiplied Alpha
**What goes wrong:** Alpha artifacts (dark fringes, incorrect transparency) when blurring content with partial alpha.
**Why it happens:** WebKit's WebGL implementation and Canvas 2D have different premultiplication expectations. If the GL context uses premultiplied alpha but the 2D canvas expects straight alpha (or vice versa), colors bleed incorrectly.
**How to avoid:** Set `premultipliedAlpha: false` on the WebGL2 context. This matches the Canvas 2D `drawImage` expectation for straight alpha compositing. Generator layers use `preserveAlpha: true` which avoids alpha blur entirely.
**Warning signs:** Dark or light halos around semi-transparent edges.

### Pitfall 4: GLSL Loop Iteration Limits
**What goes wrong:** Shader compilation fails or produces incorrect results with very large blur radii.
**Why it happens:** GLSL ES 3.00 allows dynamic loop bounds in WebGL2, but very large loops (e.g., 500+ iterations) can hit GPU time limits or produce numerical instability.
**How to avoid:** Cap the maximum kernel radius at a reasonable value (e.g., 64 texels) since the linear sampling optimization halves the samples needed. For the project's normalized 0-1 radius range with quadratic mapping (`radius^2 * maxDim * 0.05`), max pixel radius is ~96px at 1920px width, requiring ~48 samples per pass with linear filtering -- well within limits.
**Warning signs:** Shader compilation warnings, visual banding at extreme radii.

### Pitfall 5: WebGL2 Context Not Available
**What goes wrong:** Application crashes or renders nothing when WebGL2 is unavailable.
**Why it happens:** Some Linux configurations, older WebKitGTK versions (<2.40), or GPU driver issues can prevent WebGL2 context creation.
**How to avoid:** The `getContext('webgl2')` call returns `null` on failure. Set `_initFailed = true` and route all subsequent calls to CPU StackBlur fallback. Never throw -- always fall back silently.
**Warning signs:** `getContext('webgl2')` returning null (log this once at init time).

### Pitfall 6: Forgetting to Set Viewport Before Each Pass
**What goes wrong:** Blur renders to wrong portion of framebuffer, producing cropped or offset results.
**Why it happens:** WebGL viewport state persists; if not reset before each draw call targeting a different-sized target, the render area is wrong.
**How to avoid:** Call `gl.viewport(0, 0, width, height)` before each draw call.
**Warning signs:** Partial or offset blur output.

## Code Examples

### Fullscreen Quad Vertex Shader (GLSL ES 3.00)
```glsl
// Source: standard WebGL2 fullscreen triangle/quad pattern
#version 300 es
layout(location = 0) in vec2 a_position;
out vec2 v_texCoord;

void main() {
  gl_Position = vec4(a_position, 0.0, 1.0);
  // Map from clip space [-1,1] to texture space [0,1]
  v_texCoord = a_position * 0.5 + 0.5;
}
```

### Separable Gaussian Fragment Shader (GLSL ES 3.00)
```glsl
// Source: RasterGrid linear sampling technique + songho.ca WebGL blur
#version 300 es
precision highp float;

in vec2 v_texCoord;
out vec4 fragColor;

uniform sampler2D u_texture;
uniform vec2 u_direction;   // (1/w, 0) for horizontal, (0, 1/h) for vertical
uniform float u_radius;     // blur radius in pixels (sigma-related)
uniform bool u_preserveAlpha;

// Compute Gaussian weight for offset i given sigma
float gaussian(float i, float sigma) {
  return exp(-0.5 * (i * i) / (sigma * sigma));
}

void main() {
  float sigma = max(u_radius / 3.0, 1.0); // radius ~= 3*sigma
  int samples = int(ceil(u_radius));
  samples = min(samples, 64); // cap for safety

  float centerAlpha = texture(u_texture, v_texCoord).a;
  vec4 color = texture(u_texture, v_texCoord) * gaussian(0.0, sigma);
  float totalWeight = gaussian(0.0, sigma);

  for (int i = 1; i <= 64; i++) {
    if (i > samples) break;
    float w = gaussian(float(i), sigma);
    vec2 offset = u_direction * float(i);
    color += texture(u_texture, v_texCoord + offset) * w;
    color += texture(u_texture, v_texCoord - offset) * w;
    totalWeight += 2.0 * w;
  }

  color /= totalWeight;

  if (u_preserveAlpha) {
    fragColor = vec4(color.rgb, centerAlpha);
  } else {
    fragColor = color;
  }
}
```

### WebGL2 Blur Module Structure (TypeScript)
```typescript
// Source: MDN WebGL best practices + webglfundamentals.org
// Application/src/lib/glBlur.ts -- conceptual structure

// --- Shader sources (inline template literals) ---
const VERT_SRC = `...`;
const FRAG_SRC = `...`;

// --- Cached GL state ---
let _gl: WebGL2RenderingContext | null = null;
let _glCanvas: HTMLCanvasElement | null = null;
let _resources: BlurResources | null = null;
let _initFailed = false;

// --- Public API ---
export function applyGPUBlur(
  source: HTMLCanvasElement,
  targetCtx: CanvasRenderingContext2D,
  radiusNorm: number,
  width: number,
  height: number,
  preserveAlpha: boolean,
): boolean {
  // Returns false if GPU path unavailable (caller should use CPU fallback)
  const gl = getGL();
  if (!gl) return false;

  const res = ensureResources(gl, width, height);
  if (!res) return false;

  // Upload source canvas as texture
  gl.bindTexture(gl.TEXTURE_2D, res.texSource);
  gl.texSubImage2D(gl.TEXTURE_2D, 0, 0, 0, gl.RGBA, gl.UNSIGNED_BYTE, source);

  const pixelRadius = normalizedToPixelRadius(radiusNorm, Math.max(width, height));
  if (pixelRadius < 1) return false;

  // Pass 1: horizontal blur (source -> FBO intermediate)
  gl.bindFramebuffer(gl.FRAMEBUFFER, res.fbo);
  gl.viewport(0, 0, width, height);
  gl.useProgram(res.program);
  gl.uniform2f(res.uDirection, 1.0 / width, 0.0);
  gl.uniform1f(res.uRadius, pixelRadius);
  gl.uniform1i(res.uPreserveAlpha, preserveAlpha ? 1 : 0);
  gl.bindTexture(gl.TEXTURE_2D, res.texSource);
  gl.bindVertexArray(res.vao);
  gl.drawArrays(gl.TRIANGLE_STRIP, 0, 4);

  // Pass 2: vertical blur (intermediate -> screen)
  gl.bindFramebuffer(gl.FRAMEBUFFER, null);
  gl.viewport(0, 0, width, height);
  gl.uniform2f(res.uDirection, 0.0, 1.0 / height);
  gl.bindTexture(gl.TEXTURE_2D, res.texIntermediate);
  gl.drawArrays(gl.TRIANGLE_STRIP, 0, 4);

  // Read back to target 2D context
  targetCtx.drawImage(_glCanvas!, 0, 0, width, height);
  return true;
}
```

### Updated fxBlur.ts API
```typescript
// Application/src/lib/fxBlur.ts -- simplified public API
import {applyGPUBlur} from './glBlur';
import {canvasRGB, canvasRGBA} from 'stackblur-canvas';

export {normalizedToPixelRadius} from './glBlur'; // or keep in fxBlur

export function applyBlur(
  canvas: HTMLCanvasElement,
  ctx: CanvasRenderingContext2D,
  radius: number,
  width: number,
  height: number,
  preserveAlpha: boolean,
): void {
  if (radius <= 0) return;

  // Try GPU path first
  if (applyGPUBlur(canvas, ctx, radius, width, height, preserveAlpha)) {
    return; // GPU success
  }

  // CPU fallback: StackBlur
  const pixelRadius = normalizedToPixelRadius(radius, Math.max(width, height));
  if (pixelRadius < 1) return;
  if (preserveAlpha) {
    canvasRGB(canvas, 0, 0, width, height, pixelRadius);
  } else {
    canvasRGBA(canvas, 0, 0, width, height, pixelRadius);
  }
}
```

## Claude's Discretion Recommendations

### Gaussian Kernel Generation: Runtime in Shader
**Recommendation:** Compute Gaussian weights at runtime in the fragment shader using `exp(-0.5 * i*i / sigma*sigma)`. The GPU computes this trivially and it avoids maintaining precomputed lookup tables for variable radii. The `sigma = radius / 3.0` mapping gives a good Gaussian approximation where 99.7% of the distribution falls within the kernel radius.
**Confidence:** HIGH -- standard practice in GPU blur implementations.

### GL Error Handling: Init-Only Checking
**Recommendation:** Check `gl.getError()` only during initialization (shader compile, link, FBO setup). Never call it per-frame. Log errors to console with `console.warn` prefix. On any init error, set `_initFailed = true` and fall back to CPU.
**Confidence:** HIGH -- per MDN WebGL best practices, `getError()` causes a GPU pipeline flush.

### Texture Format: RGBA8
**Recommendation:** Use `gl.RGBA8` internal format with `gl.RGBA` / `gl.UNSIGNED_BYTE`. RGB formats are emulated as RGBA on most GPUs with overhead. RGBA8 is universally supported, provides the alpha channel needed for `preserveAlpha` behavior, and avoids format conversion overhead.
**Confidence:** HIGH -- per MDN WebGL best practices.

### Radius-to-Sigma Mapping: sigma = pixelRadius / 3.0
**Recommendation:** Keep the existing `normalizedToPixelRadius()` function (quadratic mapping: `normalized^2 * maxDim * 0.05`). In the shader, map pixel radius to sigma as `sigma = max(pixelRadius / 3.0, 1.0)`. This matches the mathematical property that 99.7% of a Gaussian's energy is within 3 sigma, so `radius ~= 3 * sigma`. The kernel samples from -radius to +radius.
**Confidence:** HIGH -- standard Gaussian blur math.

## Open Questions

1. **Linear sampling optimization: include in initial implementation?**
   - What we know: Linear sampling reduces texture fetches by ~50% by exploiting hardware bilinear filtering. Requires precomputing merged offsets and weights.
   - What's unclear: Whether the additional code complexity is justified given the already-fast GPU path at typical canvas sizes (1920x1080).
   - Recommendation: Start with straightforward per-texel sampling (simpler, easier to debug). The GPU is fast enough for this use case. Optimize with linear sampling later only if profiling shows a bottleneck.

2. **Context loss handling: how robust?**
   - What we know: WebGL contexts can be lost (GPU memory pressure, system sleep). Desktop Tauri apps rarely experience this.
   - What's unclear: Frequency of context loss in Tauri WKWebView on macOS.
   - Recommendation: Listen for `webglcontextlost` event on the GL canvas. On loss, set `_gl = null` and `_resources = null`. Next blur call will lazily re-initialize. No complex state machine needed.

3. **OffscreenCanvas vs regular HTMLCanvasElement for GL context?**
   - What we know: `OffscreenCanvas` is supported in modern browsers and could provide marginally better performance by being fully off-main-thread.
   - What's unclear: OffscreenCanvas WebGL2 support in Tauri WKWebView specifically. WebKit commit 355b4e8 fixed a related issue suggesting evolving support.
   - Recommendation: Use regular `document.createElement('canvas')` for maximum compatibility. The canvas is never added to DOM so it's effectively offscreen.
