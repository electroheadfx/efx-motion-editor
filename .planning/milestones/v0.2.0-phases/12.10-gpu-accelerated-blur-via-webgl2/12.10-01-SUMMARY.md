- Created glBlur.ts: WebGL2 GPU blur module with GLSL ES 3.00 shaders, ping-pong framebuffer, lazy context init, context loss handling, and preserveAlpha support
- Rewrote fxBlur.ts with unified applyBlur() that tries GPU first and falls back to CPU StackBlur
- Simplified previewRenderer.ts: removed all HQ/fast branching, all blur now routes through applyBlur()

- `Application/src/lib/fxBlur.ts` - Unified blur API with GPU-first, CPU-fallback (~63 lines, down from 139)
- `Application/src/lib/previewRenderer.ts` - Simplified blur routing via applyBlur() (no HQ/fast branching)

## Decisions Made
- Inlined normalizedToPixelRadius formula in glBlur.ts to avoid circular import between glBlur and fxBlur
- Used runtime Gaussian weight computation in shader rather than precomputed lookup tables (GPU computes exp() trivially)
- RGBA8 texture format with texStorage2D for immutable allocation (RGB8 is emulated as RGBA on most GPUs)
- sigma = max(pixelRadius / 3.0, 1.0) for proper Gaussian energy distribution within kernel radius
- Capped shader loop at 64 iterations for GPU safety (covers max pixel radius ~96px at 1920px width)
