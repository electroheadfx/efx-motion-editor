# Phase 12.10: GPU-Accelerated Blur via WebGL2 - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning
**Source:** PRD Express Path (SPECS/GPU-Accelerated-Blur-motion-editor.md)

<domain>
## Phase Boundary

Replace both CPU blur algorithms (fast downscale-upscale and HQ StackBlur) with a single WebGL2 two-pass separable Gaussian blur. GPU-accelerated, always high quality, near-constant cost regardless of layer count or radius. Falls back to CPU blur if WebGL2 unavailable.

</domain>

<decisions>
## Implementation Decisions

### New WebGL2 Blur Module
- Create `Application/src/lib/glBlur.ts` — WebGL2 context management, shader compilation, texture handling
- Lazy-init a shared `WebGL2RenderingContext` (offscreen canvas, not visible)
- Vertex shader: fullscreen quad
- Fragment shader: separable Gaussian kernel (configurable radius via uniform)
- Two render passes: horizontal blur → framebuffer → vertical blur → output
- `texImage2D(canvas)` to upload source — no `getImageData()` needed
- Read result back via `drawImage(glCanvas)` onto the target Canvas 2D context

### Update fxBlur.ts
- New `applyGPUBlur()` export that replaces both `applyFastBlur` and `applyHQBlur`
- Same signature: `(canvas, ctx, radius, width, height, preserveAlpha)`
- Falls back to current CPU StackBlur if WebGL2 context creation fails

### Update previewRenderer.ts
- `applyBlurToCanvas()` routes through GPU blur as primary path
- Remove the HQ/fast branching (GPU is always HQ-quality at fast speed)

### Simplify blurStore.ts
- The HQ toggle becomes unnecessary (keep bypass toggle only)

### Constraints (Locked)
- Must work in Tauri 2.0 WKWebView (WebKit) — WebGL2 is supported, WebGPU is not
- Must preserve the `preserveAlpha` behavior (generators need RGB-only blur to avoid alpha halos)
- Must handle canvas resize gracefully (recreate textures/framebuffers on dimension change)
- Fallback to current CPU blur is required — never break rendering if GPU is unavailable
- `stackblur-canvas` dependency can be removed only after GPU path is verified stable

### Claude's Discretion
- Gaussian kernel generation algorithm (precomputed vs runtime)
- GL error handling strategy
- Texture format choices (RGBA8 vs others)
- Unit size for "radius" mapping to shader sigma value

</decisions>
