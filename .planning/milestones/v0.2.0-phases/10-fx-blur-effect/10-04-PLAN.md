**Fix blur persistence by adding missing fields to Rust serde structs (UAT Tests 2 and 3):**

The frontend correctly serializes `blur` (on MceLayer) and `radius` (on MceLayerSource) to JSON, but the Rust structs lack these fields. Serde silently drops them during deserialization, so the .mce file is written without them. On reopen, the frontend falls back to defaults (0.3 for radius, undefined for blur).

**1. Add `blur` to `MceLayer` struct (line 53-66):**
After the `order: u32` field (line 65), add:
```rust

```

**2. Add `radius` to `MceLayerSource` struct (line 82-141):**

```rust
    // Adjustment-blur fields

    pub radius: Option<f64>,
```

Both fields use the same `#[serde(default, skip_serializing_if = "Option::is_none")]` pattern as every other optional field in these structs (e.g., `density`, `size`, `softness`, `brightness`). This ensures:
- Old .mce files without these fields deserialize correctly (Option defaults to None)
- Fields are only written to JSON when present (skip_serializing_if)
- Frontend JSON with these fields is no longer silently dropped

**Important:** Do NOT change any other fields or structs. This is a minimal, targeted fix.

**Fix HQ blur canvas corruption caused by tainted canvas from efxasset:// protocol images (UAT Test 5):**

Root cause: Images loaded from `efxasset://` custom protocol without `crossOrigin='anonymous'` taint the canvas. StackBlur's `canvasRGBA`/`canvasRGB` call `getImageData()` which throws `SecurityError` on tainted canvases. The unhandled exception skips `ctx.restore()` calls, corrupting the 2D context save/restore stack (causing zoom/crop artifacts).

Two fixes needed -- defense in depth:

**Fix A: Set crossOrigin on Image elements (primary fix):**

In the `getImageSource()` method (~line 315), add `img.crossOrigin = 'anonymous'` BEFORE setting `img.src`. The efxasset:// protocol handler already sends `Access-Control-Allow-Origin: *` headers, so this should untaint the canvas:

```typescript
const img = new Image();
img.crossOrigin = 'anonymous';  // Prevent canvas tainting from efxasset:// protocol
img.onload = () => {
    this.loadingImages.delete(imageId);
    this.imageCache.set(imageId, img);
    this.onImageLoaded?.();
};
img.onerror = () => {
    this.loadingImages.delete(imageId);
};
img.src = assetUrl(image.project_path, imageId);
```

**Fix B: Add try-catch around HQ blur calls (safety net):**

Even with Fix A, add try-catch as a defensive measure. If `crossOrigin` doesn't fully resolve tainting (some WebKit builds may still taint for custom protocols), the app gracefully falls back to fast blur instead of corrupting the canvas.

1. In `drawAdjustmentLayer()`, the `adjustment-blur` case (~line 439-455). Wrap the HQ blur call in try-catch:

```typescript
case 'adjustment-blur': {
    if (blurStore.isBypassed()) break;
    const blurSource = layer.source as {type: 'adjustment-blur'; radius: number};
    if (blurSource.radius <= 0) break;

    ctx.save();
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    const effectiveRadius = blurSource.radius * layer.opacity;
    if (blurStore.isHQ()) {
        try {
            applyHQBlur(this.canvas, effectiveRadius, this.canvas.width, this.canvas.height, false);
        } catch {
            // Fallback to fast blur if HQ fails (tainted canvas SecurityError)
            applyFastBlur(this.canvas, ctx, effectiveRadius, this.canvas.width, this.canvas.height);
        }
    } else {
        applyFastBlur(this.canvas, ctx, effectiveRadius, this.canvas.width, this.canvas.height);
    }
    ctx.restore();
    break;
}
```

2. In `applyBlurToCanvas()` (~line 483-497). This method is used for per-layer and per-generator blur via the blur offscreen canvas. Wrap the HQ path in try-catch:

```typescript
private applyBlurToCanvas(
    canvas: HTMLCanvasElement,
    ctx: CanvasRenderingContext2D,
    radius: number,
    w: number,
    h: number,
    preserveAlpha: boolean,
): void {
    if (blurStore.isBypassed() || radius <= 0) return;
    if (blurStore.isHQ()) {
        try {
            applyHQBlur(canvas, radius, w, h, preserveAlpha);
        } catch {
            // Fallback to fast blur if HQ fails (tainted canvas SecurityError)
            applyFastBlur(canvas, ctx, radius, w, h);
        }
    } else {
        applyFastBlur(canvas, ctx, radius, w, h);
    }
}
```

**Important:** The try-catch goes AROUND the applyHQBlur call, not inside it. This ensures ctx.save/restore stack stays balanced (the ctx.restore() in drawAdjustmentLayer always executes). Do NOT catch errors in applyHQBlur itself -- the StackBlur library re-throws internally and we want that propagation to reach our try-catch.
