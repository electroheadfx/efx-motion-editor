<task type="auto">
  <name>Task 2: Create blur algorithm module (fxBlur.ts)</name>
  <files>Application/src/lib/fxBlur.ts</files>
  <action>
Create `Application/src/lib/fxBlur.ts` with two blur functions and cached offscreen canvases.

**Cached canvas pool** (follow fxColorGrade.ts `getOffscreen` pattern):
- Two cached offscreen canvases (`_blurCanvasA`, `_blurCanvasB`) for ping-pong downscale
- `getBlurCanvas(slot: 'A' | 'B', w: number, h: number)` helper that creates or resizes cached canvas

**`normalizedToPixelRadius(normalized: number, canvasMaxDim: number): number`**:
- Quadratic mapping: `normalized * normalized * canvasMaxDim * 0.05`
- Returns `Math.max(0, Math.round(...))`. Returns 0 if normalized <= 0.

**`applyFastBlur(source: HTMLCanvasElement, targetCtx: CanvasRenderingContext2D, radius: number, width: number, height: number): void`**:
- If radius <= 0, return immediately
- Map normalized radius to 1-4 downscale passes: `Math.max(1, Math.min(4, Math.ceil(radius * 4)))`
- Copy source to canvas A via drawImage
- Loop downscale passes: halve dimensions each pass, draw from current to other canvas with `imageSmoothingEnabled = true`, swap references
- After all passes, draw result back to targetCtx at full size (width x height) with `imageSmoothingEnabled = true`
- Important: clear targetCtx before drawing the upscaled result so the old unblurred content is replaced

**`applyHQBlur(canvas: HTMLCanvasElement, radius: number, width: number, height: number, preserveAlpha: boolean): void`**:
- If radius <= 0, return immediately
- Convert normalized radius to pixel radius via `normalizedToPixelRadius(radius, Math.max(width, height))`
- If pixelRadius < 1, return
- If `preserveAlpha`: call `StackBlur.canvasRGB(canvas, 0, 0, width, height, pixelRadius)`
- Else: call `StackBlur.canvasRGBA(canvas, 0, 0, width, height, pixelRadius)`
- Import StackBlur from 'stackblur-canvas'

Export: `applyFastBlur`, `applyHQBlur`, `normalizedToPixelRadius`

Modify `Application/src/lib/previewRenderer.ts` to support blur at all three levels.

**New imports:**

- `import {blurStore} from '../stores/blurStore';`

**New private field:**
- `private blurOffscreen: HTMLCanvasElement | null = null;` -- cached offscreen canvas for per-layer/generator blur rendering (separate from existing `offscreenCanvas` used for video)

**New private helper: `getBlurOffscreen(w: number, h: number): { canvas: HTMLCanvasElement; ctx: CanvasRenderingContext2D } | null`**
- Creates or resizes `this.blurOffscreen`, returns canvas + ctx
- Same pattern as fxColorGrade's `getOffscreen`

**New private helper: `applyBlurToCanvas(canvas: HTMLCanvasElement, ctx: CanvasRenderingContext2D, radius: number, w: number, h: number, preserveAlpha: boolean): void`**
- If `blurStore.isBypassed()` or radius <= 0, return
- If `blurStore.isHQ()`: call `applyHQBlur(canvas, radius, w, h, preserveAlpha)`
- Else: call `applyFastBlur(canvas, ctx, radius, w, h)`

**Modify `renderFrame()` -- layer loop:**
In the main layer loop, for content layers (not generator, not adjustment):
- After resolving the source, check if `layer.blur` exists and is > 0
- If blur > 0 and not bypassed: render the layer to `blurOffscreen` (via getBlurOffscreen), apply blur to it, then composite the blurred offscreen onto the main canvas using the layer's blend mode and opacity
- If blur <= 0 or bypassed: use existing `drawLayer` path unchanged
- The offscreen rendering should: clear offscreen, save/restore ctx, use identity transform (no DPI scaling -- offscreen is at logical size), call drawImage with the same aspect-ratio fitting logic as drawLayer but onto the offscreen canvas, then apply blur, then draw the offscreen onto main canvas with blend mode + opacity

For generator layers:
- Check if `layer.blur` exists and is > 0
- If blur > 0 and not bypassed: render generator to offscreen (via getBlurOffscreen at logical dimensions), apply RGB-only blur (preserveAlpha=true), composite blurred offscreen onto main canvas with blend mode + opacity
- If blur <= 0 or bypassed: use existing `drawGeneratorLayer` path unchanged
- Generator rendering to offscreen: create a temporary ctx on the offscreen, call the appropriate draw function (drawGrain, etc.) on that ctx, then blur, then composite

**Modify `drawAdjustmentLayer():`**
- Add case for `'adjustment-blur'` alongside existing `'adjustment-color-grade'`:
  ```typescript
  case 'adjustment-blur': {
    if (blurStore.isBypassed()) break;
    const blurSource = layer.source as { type: 'adjustment-blur'; radius: number };
    if (blurSource.radius <= 0) break;
    // Reset transform to physical pixel coords (same as color-grade)
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    // Scale radius by layer opacity for partial blur application
    const effectiveRadius = blurSource.radius * layer.opacity;
    if (blurStore.isHQ()) {
      applyHQBlur(this.canvas, effectiveRadius, this.canvas.width, this.canvas.height, false);
    } else {
      applyFastBlur(this.canvas, ctx, effectiveRadius, this.canvas.width, this.canvas.height);
    }
    break;
  }
  ```
- Change the early return `if (layer.source.type !== 'adjustment-color-grade') return;` to a switch statement that handles both `adjustment-color-grade` and `adjustment-blur`

**Modify `renderFrame()` hasDrawable check:**
- In the `clearCanvas` branch, adjustment layers now include adjustment-blur which modifies existing content. The existing logic already continues past adjustments and checks for content/generators. No change needed since adjustment-blur only matters when content is below, same as color-grade.

**Modify `dispose()`:**
- Add `this.blurOffscreen = null;`

**Critical implementation notes:**
- NEVER use `ctx.filter` -- it is broken in Tauri WebKit on tainted canvases
- For layer blur offscreen rendering, the offscreen canvas should be at logical pixel dimensions (logicalW x logicalH), not physical (DPI-scaled) dimensions. The blur is applied in logical space, then the blurred offscreen is drawn onto the main canvas which is already DPI-scaled.
- For standalone adjustment-blur, work in physical pixel coords (canvas.width x canvas.height) since we are modifying the already-rendered composite (same as color-grade).
- Generator blur MUST use preserveAlpha=true (RGB-only blur) to avoid alpha halos when compositing over the scene.
