# Phase 12.12: New Content Layer for Timeline - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add content layer types (static-image, image-sequence, video) as timeline-level sequences that stack and interleave with FX sequences. These content layers span the full timeline with in/out frame controls, loop their content, and have full property controls (opacity, blend mode, position, transform, crop, keyframes). Added via the timeline menu under a new "Content" section. Content overlay layers render above content sequences in the compositing stack.

</domain>

<decisions>
## Implementation Decisions

### Timeline track visuals
- Content overlay tracks interleave freely with FX tracks — user can reorder them together in the same zone (above content sequences)
- Range bars with small thumbnail/preview icon showing the content source inside the bar
- Colors match sidebar dot colors: green (`--sidebar-dot-green`) for static-image, blue (`--sidebar-dot-blue`) for image-sequence, purple (`#8B5CF6`) for video
- Same FX_TRACK_HEIGHT (24px) range bars — consistent visual language

### Menu and button
- Rename button from "+ FX" to "+ Layer"
- Add "CONTENT" section in the menu before GENERATORS, with three items: Static Image, Image Sequence, Video
- Existing GENERATORS and ADJUSTMENTS sections remain below

### Content source selection
- Reuse Phase 12.9 intent dispatch pattern: menu sets `addLayerIntent` signal + switches to imported mode, ImportedView shows filtered media
- For image-sequence: multi-select from importer grid, images become frame sequence ordered by filename
- User can change source after creation via a "Change source" button in sidebar properties

### Looping behavior
- Static images hold indefinitely across the in/out range — no animation or cycling
- Image sequences loop seamlessly: cycle back to first image when reaching end, repeat within in/out range
- Videos loop seamlessly: restart from beginning when reaching end, repeat within in/out range
- Default in/out range: full timeline (0 to totalFrames), same as FX sequences — user can trim with in/out handles

### Compositing order
- Timeline content layers render ABOVE content sequences (key photos) — same stacking zone as FX
- Timeline ordering determines compositing order within the interleaved FX/content overlay zone

### Sidebar property controls
- When a timeline content layer is selected, sidebar PROPERTIES panel shows its controls — reuse existing SidebarProperties pattern
- Full controls: opacity, blend mode, position (x/y), scale (scaleX/scaleY), rotation, crop (top/right/bottom/left)
- Full keyframe animation support — same KeyframeStore pattern with diamond icons on timeline
- Visibility toggle (eye icon) and Delete key removal — same as FX sequence behavior (toggleFxSequenceVisibility pattern)

### Claude's Discretion
- Thumbnail rendering strategy inside range bars (how to get a preview icon)
- How to handle the sequence `kind` field — new kind value vs reusing 'fx' with content layer types
- Exact FX reorder drag interaction when content and FX tracks are interleaved
- How keyframe frame offsets work for timeline-level content layers (global vs in-frame-relative)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `createFxSequence()`: Pattern for creating timeline-level sequences with in/out frames — extend for content layers
- `toggleFxSequenceVisibility()`: Visibility toggle pattern — reuse directly
- `fxTrackLayouts` computed: FX track layout data — extend to include content overlay tracks
- `addLayerIntent` signal + ImportedView: Intent dispatch pattern for source selection from Phase 12.9
- `SidebarFxProperties`: FX sidebar controls pattern — similar pattern for content overlay layer properties
- `TransformSection`: Existing transform controls (position, scale, rotation) for content layers
- `LayerList` dot colors: `--sidebar-dot-green` (static-image), `--sidebar-dot-blue` (image-sequence), `#8B5CF6` (video)

### Established Patterns
- Sequence `kind` discriminator (`'content' | 'fx'`): May need extension or new value for content overlays
- FX sequence rendering: `drawFxTrack()` in TimelineRenderer — pattern for range bar drawing with in/out handles
- FX track interaction: Click selection, in/out handle drag, reorder drag — all in TimelineInteraction
- Keyframe system gates on `seq.kind === 'fx'` — needs updating to support timeline content layers

### Integration Points
- `AddFxMenu.tsx` → rename to `AddLayerMenu.tsx`, add Content section before Generators
- `frameMap.ts fxTrackLayouts` → include content overlay sequences in layout computation
- `previewRenderer.ts` → render content overlay layers above content sequences, below/interleaved with FX
- `TimelineRenderer.ts drawFxTrack()` → draw content overlay range bars with thumbnails
- `TimelineInteraction.ts` → handle click/drag on content overlay tracks
- `keyframeStore.ts` → remove `kind === 'fx'` exclusion, support keyframes on timeline content layers
- `projectStore.ts` → serialize/deserialize content overlay sequences in .mce format

</code_context>
