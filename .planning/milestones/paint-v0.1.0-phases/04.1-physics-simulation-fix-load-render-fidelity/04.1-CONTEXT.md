<decisions>
## Implementation Decisions

### Physics algorithm replacement
- **D-01:** Abandon Rebelle physics entirely. The Rebelle flow/transport approach never produced visible results in the port. Replace `flowStep()` with a new diffusion algorithm adapted from `research/watercolor-canvas.html`
- **D-02:** New algorithm: FBM-displaced darken-flood with self-stabilization (*1.02) and gentle drying. Ping-pong float buffers, precomputed displacement map, `min()` of displaced neighbors per frame
- **D-03:** Displacement driven by FBM noise + paper grain heightmap blend. Paper texture influences diffusion direction alongside organic FBM noise. Different papers = different spreading patterns

### Color handling in diffusion
- **D-04:** Replace flowStep() only — keep existing brush deposit logic (renderPaintStroke, wetBrush, all brush stamping). Only replace the inter-frame diffusion/flow physics. Minimal surgery to existing code
- **D-05:** Color approach is Claude's discretion. Must integrate cleanly with existing wet layer arrays (wetR, wetG, wetB, wetAlpha, wetness). Options: per-channel RGB diffusion or greyscale density + separate color tracking

### Load/replay fidelity
- **D-06:** Loaded paintings must be visually identical to live-painted result. Replay must run enough diffusion passes between strokes to match the organic spreading that happened during live painting
- **D-07:** Replay timing: track diffusion frame count (or elapsed time) between strokes. On load, run that many diffusion passes between each stroke replay

### Tool adaptation
- **D-08:** All physics-dependent tools rethought for the new diffusion model: water, blow, smear, blend, wet, dry. Each tool should interact with the diffusion system rather than the old Rebelle flow arrays
- **D-09:** Water tool = increase wetness/alpha to allow diffusion to re-spread. Blow = directional wetness or displacement bias. Smear/blend = local paint movement via diffusion interaction. Wet/dry = bulk wetness control

### Preserved systems
- **D-10:** Keep existing LUT drying system from Phase 04 (dL/ao tables, drySpeed slider). Drying integrates with the new diffusion — dried paint stops diffusing
- **D-11:** Keep existing brush deposit logic, brush texture mask, all brush types from Phase 03

### Claude's Discretion
- Per-channel RGB diffusion vs greyscale density + color map (D-05)
- FBM octave count and scale parameters for displacement map
- How to blend FBM noise with paper heightmap (additive, multiply, weighted mix)
- Diffusion pass count per physics tick for performance balance
- Half-resolution diffusion option for performance (like the research prototype)
- How blow tool translates directional force into diffusion system interaction
- Exact smear/blend mechanics within the new model

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `dryStep()` (line 1500): LUT-driven drying — works independently, keep as-is
- Wet layer arrays (wetR, wetG, wetB, wetAlpha, wetness): Float32Array per-pixel color storage — diffusion operates on these
- `paperHeight` Float32Array: Paper heightmap from loaded texture — use as displacement input alongside FBM
- `physicsStep()` (line 1692): Calls flowStep + dryStep at 10fps — replace flowStep call with new diffusion
- `replayWithPhysics()` (line 1965): Physics-interleaved replay — adapt to use new diffusion pass count

### Established Patterns
- Physics at 10fps via `setInterval` independent of paint framerate
- Wet layer as Float32Array arrays for per-pixel physics
- Paper heightmap persists independently of background mode
- `directRender` flag switches between wet-layer path and direct canvas path

### Integration Points
- `flowStep()` is called from `physicsStep()` — direct replacement point
- `replayWithPhysics()` runs `physicsStep()` in a loop — needs diffusion frame count tracking
- All 8+1 brush types deposit into wet layer arrays — diffusion reads from same arrays
- Paper texture loading (`setPaperPhysics()`) already produces `paperHeight` Float32Array

### From research/watercolor-canvas.html
- `precomputeDisplacement()`: FBM noise → Float32Array displacement map, computed once at init
- `diffuse()`: Ping-pong buffers, bilinear sample at displaced positions, `min()` flood, *1.02 stabilization
- `sample()`: Bilinear interpolation for sub-pixel displaced reads — essential for smooth diffusion
- Runs per-frame (60fps in research) — may need to run multiple passes per physics tick at 10fps

</code_context>

<specifics>
## Specific Ideas

- The watercolor-canvas.html diffusion produces beautiful organic watercolor bloom visible in the user's screenshot — this specific visual quality is the target
- FBM displacement is precomputed once, making diffusion per-frame extremely fast (milliseconds)
- The darken-flood (`min()` of displaced neighbors) is what creates the characteristic watercolor edge darkening
- Self-stabilization (*1.02) prevents infinite spreading — paint naturally settles
- Paper grain should make paint spread differently on different paper textures (valleys catch paint)

</specifics>
