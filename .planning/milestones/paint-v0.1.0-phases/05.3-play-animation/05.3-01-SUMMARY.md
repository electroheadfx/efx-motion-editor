## Accomplishments
- Created AnimationPlayer class with play/stop/isPlaying/getState public API for frame-based stroke replay
- Implemented timestamp-ratio frame distribution algorithm mapping stroke timestamps to proportional frame ranges
- Added 5 new engine public methods (getStrokes, setInputLocked, renderAllStrokes, setAnimationMode, renderPartialStrokes) for animation control
- Progressive point-by-point rendering across frames with configurable FPS and onFrame callback

- `paint-rebelle-new/src/engine/EfxPaintEngine.ts` - Added inputLocked/animationMode fields, 5 new public methods, pointer input guard, render loop animation guard

## Decisions Made
- Physics (spreading/drying) skipped during animation frames -- follows redrawAll pattern with forceDryAll after each stroke, no physics ticks
- Used requestAnimationFrame with elapsed-time accumulator for frame scheduling (drift-corrected)
- Baseline ~8 points per frame for progressive rendering -- balances smooth visual build with performance
- Deep-copy strokes in getStrokes() so AnimationPlayer cannot mutate engine state
- Kept existing replayAnimated() as-is -- AnimationPlayer is a separate, independent class
