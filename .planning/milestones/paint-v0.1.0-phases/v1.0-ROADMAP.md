# Roadmap: EFX Physics Paint

## Overview

EFX Physics Paint is a TypeScript library that ports natural-media paint physics (watercolor, ink, oil) from Rebelle's Processing.js algorithm to vanilla Canvas 2D. The journey goes from pixel-perfect algorithm port (Phase 1) through rendering pipeline (Phase 2), brush system (Phase 3), drying/persistence (Phase 4), to a polished npm library with demo app (Phase 5).

## Phases

- [ ] **Phase 1: Algorithm Port Foundation** - Core physics engine (wet/dry, flow field, paper texture) with basic demo canvas
- [ ] **Phase 2: Rendering Pipeline** - Multi-layer compositing with paper texture background and density-weighted transparency
- [ ] **Phase 3: Brush System & Tools** - All brush types with tablet support and brush texture mask
- [ ] **Phase 4: Drying & Persistence** - Wet-to-dry drying LUT and stroke serialization/replay
- [ ] **Phase 5: Library & Demo Polish** - npm package export, component wrapper, full UI controls

## Phase Details

### Phase 1: Algorithm Port Foundation
**Goal**: Working wet/dry physics engine matching original Rebelle behavior, with basic demo canvas
**Depends on**: Nothing (first phase)
**Requirements**: PHYS-01, PHYS-02, PHYS-03, CANVAS-01, CANVAS-02, DEMO-01
**Note**: LIB-02 (TypeScript types) is partially addressed via types.ts documentation contract in plan 01-03; full LIB-02 compliance deferred to Phase 5 when HTML is extracted to TypeScript modules.
**Success Criteria** (what must be TRUE):
  1. Dual-layer wet/dry physics produces visually identical output to original Rebelle (wetLayer Float32Array + dryLayer Uint8Array)
  2. Paper texture loads from jpg and correctly influences flow field (512x512 red channel, wrapping coordinates)
  3. Flow field transport creates visible capillary-action paint spreading between strokes
  4. Canvas stride is exactly 902 (bR+2), verified against original
  5. Canvas resizes responsively without breaking paint physics

- [x] 01-02-PLAN.md — Rendering flow redesign: wet-layer-only painting, compositing, drying, flow physics

### Phase 05.1: Physics Gap Fixes from 05-PHYSICS-GAPS.md (INSERTED)

**Goal:** Replace FBM diffusion with Stam's stable fluids solver, fix 6 physics gaps (DENSITY_K, /800 divisor, brush grain, color mixing, diffusion quality, opacity interaction), add paper-height deposit modulation, water-height equalization, edge darkening, and subtractive RYB color mixing
**Requirements**: D-01, D-02, D-03, D-04, D-05, D-06, D-07, D-08, D-09, D-10, D-12, D-13
**Depends on:** Phase 05
**Success Criteria** (what must be TRUE):
  1. Stam stable fluids solver produces organic paint spreading via advect + diffuse + project pipeline
  2. Water-height equalization creates velocity from height differences (Van Laerhoven omega_h=0.06)
  3. Edge darkening pushes pigment to wet-area boundaries (Curtis FlowOutward)
  4. Paper-height deposit modulation creates watercolor stipple texture (no brush_texture.png)
  5. Subtractive RYB color mixing produces green from yellow + blue overlap
  6. Dual DENSITY_K: display=1.5 (compositor), physics=3.5 (drying)
  7. forceDryAll and dryStep both use /800 divisor consistently
  8. savedWet stores full-intensity values; opacity applied only in compositor
  9. Viscosity is parameterizable for future media types
**Plans:** 3/3 plans complete

Plans:
- [x] 05.1-01-PLAN.md — Types + Stam stable fluids solver (fluids.ts with advect/diffuse/project, height equalization, edge darkening)
- [x] 05.1-02-PLAN.md — Deposit modulation + drying/compositor fixes (paper-height deposit, subtractive mixing, /800, dual DENSITY_K, full-intensity savedWet)
- [x] 05.1-03-PLAN.md — Engine integration (wire fluids.ts into EfxPaintEngine, remove brushTexture, viscosity API, visual verification)
