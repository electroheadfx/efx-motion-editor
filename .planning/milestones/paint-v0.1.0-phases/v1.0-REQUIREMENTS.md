# Requirements: EFX Physics Paint

**Defined:** 2026-03-28
**Core Value:** A paint physics engine producing convincing watercolor/wet-media strokes with paper texture interaction, transparency, and tablet/pen pressure support — usable as drop-in library in efx-motion-editor.

## v1 Requirements

Requirements for initial library release. Each maps to roadmap phases.

### Physics Engine

- [x] **PHYS-01**: Dual-layer wet/dry paint physics (JS number arrays for wet channels + canvas ImageData dry layer) producing organic watercolor behavior with Stam stable-fluids solver
- [x] **PHYS-02**: Paper texture loading (512x512 jpg → Float32Array heightmap) influencing brush and flow field — red channel sampling with wrapping coordinates
- [x] **PHYS-03**: Flow field transport (dH array) calculated from brush mask × paper texture heightmap, creating capillary-action spreading
- [x] **PHYS-04**: Drying LUT (two-table system: dL cumulative + ao inverse) for wet-to-dry conversion over time

### Rendering

- [x] **RENDER-01**: Multi-layer compositing pipeline (dryLayer + wetLayer → canvas, paper as background only — NOT double-applied)
- [x] **RENDER-02**: Transparency support via density-weighted blending (NOT globalAlpha — from paint-studio-v9)

### Stroke System

- [x] **STROKE-01**: Stroke data model `{x, y, p, tx, ty, tw, spd}` with [x, y, pressure] minimum — matches efx-motion-editor PaintStroke JSON
- [x] **STROKE-02**: Tablet/Wacom pen support (pressure, tilt X/Y, twist, speed) via PointerEvent coalescing — from efx-motion-editor
- [x] **STROKE-03**: Stroke persistence — serialize/deserialize strokes to JSON

### Brush System

- [x] **BRUSH-01**: Brush types: paint, erase, water, smear, blend, blow, wet, dry
- [x] **BRUSH-02**: Brush parameters: size, opacity, pressure, water amount, dry amount
- [x] **BRUSH-03**: Paper-height deposit modulation — paint amount varies with paper surface texture, creating natural watercolor stipple without a separate brush texture asset

### Canvas & Coordinate System

- [x] **CANVAS-01**: Responsive canvas with correct coordinate mapping (canvas coords ≠ texture coords)
- [x] **CANVAS-02**: Configurable canvas dimensions (default 1000x650) with correct coordinate mapping for pointer events and physics buffers

### Library Infrastructure

- [x] **LIB-01**: Exported as `@efxlab/efx-physic-paint` npm package via tsup (ESM-only output)
- [x] **LIB-02**: TypeScript with full type definitions and no runtime dependencies
- [x] **LIB-03**: Preact/React component wrapper for demo app

### Demo App

- [x] **DEMO-01**: Vite + Preact + TypeScript demo app with working canvas
- [x] **DEMO-02**: UI controls for all brush and canvas parameters (~9 sliders + button groups per D-12)
- [x] **DEMO-03**: Paper texture selector (paper_1.jpg, paper_2.jpg, paper_3.jpg)
- [x] **DEMO-04**: Stroke replay from JSON (verify serialization works)

## v2 Requirements (Deferred)

- **LIB-04**: npm package published to registry (@efxlab/efx-physic-paint)
- **BRUSH-04**: Tilt paper tool (brush follows paper texture heightmap)
- **PERF-01**: OffscreenCanvas + Web Worker for physics computation

## Out of Scope

| Feature | Reason |
|---------|--------|
| WebGL rendering | Processing.js uses Canvas 2D pixel arrays; WebGL requires full shader rewrite |
| Non-destructive FX workflow | Handled by p5.brush in efx-motion-editor integration |
| perfect-freehand integration | Separate system for flat brush strokes in efx-motion-editor |
| Multi-canvas/document support | Single-canvas library focused |
| efx-motion-editor integration | Next milestone; this builds the library |
