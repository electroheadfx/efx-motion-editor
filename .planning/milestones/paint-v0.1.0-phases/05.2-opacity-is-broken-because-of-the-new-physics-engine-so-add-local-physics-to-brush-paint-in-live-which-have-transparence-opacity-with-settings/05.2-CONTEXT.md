# Phase 05.2: Opacity Fix + Local Physics Mode - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix per-stroke opacity layering lost by D-12 (Phase 05.1), add a local physics mode that runs Stam solver scoped to the active stroke area, and unify the render path so all paint goes through the wet layer + compositor. Per-pixel opacity accumulator restores transparency, local physics makes paint spread during painting, and a UI toggle lets users switch between local and global physics.

</domain>

<decisions>
## Implementation Decisions

### Per-stroke opacity model
- **D-01:** Add a per-pixel opacity accumulator buffer (`wet.strokeOpacity`, Float32Array) that tracks intended visual opacity per pixel. Each stroke deposits its opacity value at the pixels it touches
- **D-02:** Overlap compositing uses Porter-Duff "over" operator: `result = existing + new * (1 - existing)`. Two 50% strokes overlap at 75%, three at 87.5%. Never exceeds 1.0
- **D-03:** Physics (Stam solver) still operates on full-intensity `wet.alpha` buffers — D-12 principle preserved. The `strokeOpacity` buffer is display-only, read by the compositor
- **D-04:** Compositor reads `wet.strokeOpacity[i]` per-pixel instead of the global `userOpacity` parameter. `finalAlpha = displayAlpha * paperMod * strokeOpacity[i]`

### Local physics mode
- **D-05:** New physics mode: "local physics" — Stam solver runs scoped to the active stroke's bounding box + margin, using the same global physics settings (viscosity, diffusion rate, etc.)
- **D-06:** Spread distance derived from global physics strength — higher water amount = wider spread margin. Not a fixed pixel radius
- **D-07:** Both local and global physics modes exist. Local = physics runs on stroke footprint during painting. Global = Stam solver runs on entire wet layer (existing behavior). User toggles between them
- **D-08:** Default mode: Local physics

### Opacity settings/UI
- **D-09:** Physics mode toggle in toolbar: Local / Global. Default: Local
- **D-10:** Existing opacity slider (10-100) unchanged — now controls per-stroke opacity via the accumulator instead of global compositor multiplier
- **D-11:** New slider: local physics spread strength — controls how aggressively local physics spreads paint around the stroke. Separate from global viscosity

### Unified render path
- **D-12:** Remove the `directRender` split. ALL paint goes through wet layer + compositor with per-pixel opacity, regardless of `wetPaper` setting
- **D-13:** `wetPaper` toggle affects wet composite blending behavior only, not the render path itself. One code path for all strokes
- **D-14:** This simplifies the codebase — `renderPaintStrokeSingleColor` no longer needs the directRender branch

### Claude's Discretion
- Exact bounding box margin calculation for local physics area
- How to handle strokeOpacity during physics diffusion (whether to advect it alongside paint or keep it static)
- Performance optimization for local Stam solver (subset grid vs full grid with mask)
- Whether wetPaper wet composite blending needs adjustment with unified path
- Transition behavior when switching between local and global modes mid-session

</decisions>

<specifics>
## Specific Ideas

- Local physics uses the "strength of the global physics" — same viscosity, diffusion parameters, just scoped to stroke area
- The per-pixel opacity accumulator preserves D-12's principle (full-intensity physics) while restoring per-stroke visual transparency
- At 100% opacity, strokeOpacity[i] saturates to 1.0 quickly (Porter-Duff), so visual behavior matches current 100% behavior
- At <100% opacity, overlapping strokes build up naturally — matching traditional paint layering expectations

</specifics>
