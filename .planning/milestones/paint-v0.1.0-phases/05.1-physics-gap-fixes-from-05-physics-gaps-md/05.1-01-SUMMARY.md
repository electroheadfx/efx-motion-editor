## Decisions Made
- Used 4 Gauss-Seidel iterations (not 20) for the linSolve function, balancing quality with the 16ms per-tick performance budget. 4 iterations are standard for real-time Stam implementations.
- Canvas-resolution grid (W=canvasW, H=canvasH) for spatial alignment with wet layer buffers. No coarser grid with upsampling -- avoids interpolation artifacts at paint boundaries.
- Premultiplied alpha advection: advect R*A, G*A, B*A, A as four fields, recover R=R_premul/A. Prevents color ghosting per Research Pitfall 4.
- Temporary Stam-sized Float32Arrays allocated per tick for color advection. Can be moved to persistent FluidBuffers allocation if profiling shows GC pressure.
