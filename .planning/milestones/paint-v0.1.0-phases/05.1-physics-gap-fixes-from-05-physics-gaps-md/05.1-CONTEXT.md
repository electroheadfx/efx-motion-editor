# Phase 05.1: Physics Gap Fixes — Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the non-functional FBM diffusion engine with Stam's stable fluids solver, fix 6 physics gaps identified during Phase 05 library extraction, and add paper-height-influenced paint deposit, water-height equalization flow, edge darkening, and subtractive RYB color mixing. The current architecture (typed arrays, functional modules, EfxPaintEngine facade) is preserved — only the physics internals change.

</domain>

<decisions>
## Implementation Decisions

### Diffusion engine replacement
- **D-01:** Replace FBM-displaced darken-flood diffusion with Stam's stable fluids solver (implicit solver, never blows up, parameterizable viscosity)
- **D-02:** Water-height equalization: velocity field enhanced with height differences between neighboring cells (ω_h ≈ 0.06 per Van Laerhoven). Water flows to equalize heights — produces organic spreading and dark-edge effect naturally
- **D-03:** Edge darkening via outward flow: reduce water pressure near wet-area boundaries (Curtis FlowOutward approach). Pigment migrates from interior to edges as paint dries
- **D-04:** Architecture stays Canvas 2D + typed arrays. Stable fluids operates on Float32Array velocity/pressure/water grids. No WebGL required — the papers' GPU shaders translate to per-pixel typed array loops

### Opacity & density model
- **D-05:** Dual DENSITY_K values: DENSITY_K_DISPLAY = 1.5 (for opacity slider response in compositor) and DENSITY_K_PHYSICS = 3.5 (for strong wet layer during diffusion/drying). Compositor uses display K, physics uses physics K
- **D-06:** Restore /800 divisor everywhere: both forceDryAll and dryStep use /800, matching v3 sacred value. Paper-height deposit (D-08) restores the sparse deposits that made /800 work correctly

### Brush grain / paint deposit
- **D-07:** Remove brush_texture.png dependency entirely. Paper-height modulates deposit instead
- **D-08:** Paper-height deposit modulation based on Curtis/Van Laerhoven adsorption equation: deposit influenced by paper height h, granulation γ, and density δ. Pigment settles preferentially in paper valleys, creating watercolor stipple texture naturally
- **D-09:** Deposit equation: Δp_ad = Δt × (p_water × (1 - w_frac) × (1 - h×γ) × δ) — adapted for our wet layer (w_frac = wetness fraction, h = paper height at pixel, γ and δ are tunable constants)

### Color mixing
- **D-10:** Simple subtractive RYB mixing at stroke overlaps. Wire existing `mixSubtractive` from `src/util/color.ts` into wet layer deposit path
- **D-11:** Full Kubelka-Munk pigment compositing deferred to a future "watercolor system" phase. Phase 05.1 uses RGB + subtractive mixing only

### Opacity × physics interaction
- **D-12:** Store savedWet at full intensity (pre-opacity). Apply user opacity only in the compositor, never in the wet layer buffers. This ensures diffusion operates on full-strength values

### Vision / future direction
- **D-13:** The stable fluids solver is chosen specifically because viscosity is parameterizable — low viscosity = watery paint, high viscosity = oil/acrylic. This phase builds the foundation for multi-media paint support in future phases

### Claude's Discretion
- Grid resolution for stable fluids solver (match canvas resolution or use coarser grid with upsampling)
- Exact values for granulation γ and density δ defaults (tunable via sliders later)
- Number of Jacobi iterations for pressure relaxation
- Boundary condition handling at canvas edges
- Whether to keep any FBM noise as secondary perturbation on top of stable fluids

</decisions>

<specifics>
## Specific Ideas

- Stam's stable fluids: the core algorithm is well-documented in "Real-Time Fluid Dynamics for Games" (Stam, 2003). The key operations are: advect, diffuse, project (enforce incompressibility). All operate on 2D grids — maps directly to our Float32Array per-pixel approach
- Van Laerhoven's water-height equalization is a simple additive velocity contribution: v_new = ω_i × v_old + ω_h × v_height (ω_h = 0.06). This alone produces dark edges and organic spreading
- Curtis's pigment adsorption equation gives paper-valley deposit naturally — no brush texture needed. The granulation parameter γ controls how much paper height influences deposit
- The engine should remain viscosity-parameterizable for future acrylic/oil media — don't hardcode watery paint assumptions
- Research papers used GPU fragment shaders but the algorithms are per-pixel operations on grids — direct translation to typed array loops in Canvas 2D

</specifics>
