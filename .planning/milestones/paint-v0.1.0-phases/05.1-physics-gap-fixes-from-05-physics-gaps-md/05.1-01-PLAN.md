    Create `paint-rebelle-new/src/core/fluids.ts` implementing the complete Stam stable fluids solver for watercolor paint simulation. This is a NEW file. Follow the established pattern: no module-level mutable state, all buffers passed as function arguments.

    **File header comment:**
    ```
    // ============================================================
    //  Stam Stable Fluids Solver for Watercolor Physics
    //  Based on Stam 2003 "Real-Time Fluid Dynamics for Games"
    //  With Van Laerhoven water-height equalization and Curtis edge darkening.
    //  No module-level mutable state. No DOM access.
    // ============================================================
    ```

    **Imports:**
    ```typescript
    import type { WetBuffers, FluidBuffers, FluidConfig } from '../types'
    ```

    **Core solver functions (from RESEARCH.md Pattern 1):**

    1. `IX(W: number, i: number, j: number): number` — Grid indexing for (W+2)*(H+2) grids. Returns `i + (W + 2) * j`.

    2. `setBnd(W: number, H: number, b: number, x: Float32Array): void` — Boundary conditions. For b=1, negate X velocity at left/right walls. For b=2, negate Y velocity at top/bottom. For b=0, copy neighbor values. Handle all 4 corners as average of 2 adjacent cells. Use separate W and H (not square N) since our canvas is 1000x650.

    3. `linSolve(W: number, H: number, b: number, x: Float32Array, x0: Float32Array, a: number, c: number, iterations: number): void` — Gauss-Seidel relaxation. Default iterations parameter allows caller to tune (recommendation: 4 for performance, up to 20 for quality). Loop `iterations` times over all interior cells [1..W] x [1..H], computing `x[IX(W,i,j)] = (x0[IX(W,i,j)] + a * (sum of 4 neighbors)) / c`. Call `setBnd` after each iteration.

    4. `diffuse(W: number, H: number, b: number, x: Float32Array, x0: Float32Array, diff: number, dt: number, iterations: number): void` — Implicit diffusion. Compute `a = dt * diff * W * H` (use W*H not N*N since non-square). Call `linSolve(W, H, b, x, x0, a, 1 + 4*a, iterations)`.

    5. `advect(W: number, H: number, b: number, d: Float32Array, d0: Float32Array, u: Float32Array, v: Float32Array, dt: number): void` — Semi-Lagrangian advection with bilinear interpolation. `dt0x = dt * W`, `dt0y = dt * H` (separate scaling for non-square grid). Backtrace position, clamp to [0.5, W+0.5] x [0.5, H+0.5], bilinear interpolate. Call `setBnd` at end.

    6. `project(W: number, H: number, u: Float32Array, v: Float32Array, p: Float32Array, div: Float32Array, iterations: number): void` — Pressure projection for incompressibility. Compute divergence, solve pressure via `linSolve`, subtract pressure gradient from velocity. Use `hx = 1.0/W`, `hy = 1.0/H` for non-square scaling.

    7. `addSource(W: number, H: number, x: Float32Array, s: Float32Array, dt: number): void` — Simple: `x[i] += dt * s[i]` for all cells.

    **High-level pipeline functions (from RESEARCH.md Pattern 2):**

    8. `velStep(W: number, H: number, u: Float32Array, v: Float32Array, u0: Float32Array, v0: Float32Array, visc: number, dt: number, iterations: number): void` — Velocity update: addSource -> swap u/u0 and v/v0 (use array `.set()` for swap) -> diffuse -> project -> advect -> project.

    9. `densStep(W: number, H: number, x: Float32Array, x0: Float32Array, u: Float32Array, v: Float32Array, diff: number, dt: number, iterations: number): void` — Density update: addSource -> swap x/x0 -> diffuse -> advect. Use diff=0.00001 (very low — pigment doesn't self-diffuse much; transport is via advection by velocity field).

    **Watercolor-specific functions (from RESEARCH.md Patterns 3-4):**

    10. `addHeightEqualization(W: number, H: number, u0: Float32Array, v0: Float32Array, waterHeight: Float32Array, omega_h: number): void` — Van Laerhoven D-02: for each interior cell where `waterHeight > 1`, add `omega_h * (waterHeight[left] - waterHeight[right])` to u0 and `omega_h * (waterHeight[up] - waterHeight[down])` to v0. The waterHeight array is the wet.alpha mapped to Stam grid coordinates.

    11. `darkenEdges(W: number, H: number, wetMask: Float32Array, blurMask: Float32Array, u0: Float32Array, v0: Float32Array, darkening: number): void` — Curtis FlowOutward D-03: Build wetMask from wet.alpha (1.0 where alpha > threshold, 0.0 elsewhere). Apply a simple 3x3 box blur to create blurMask (repeat 3 times for approximation of Gaussian). For boundary cells (where wetMask > 0.5 and blurMask < wetMask), compute gradient of blurMask and add outward velocity force scaled by `darkening * edgeFactor`.

    12. `buildWetMaskFromAlpha(W: number, H: number, wetAlpha: Float32Array, canvasW: number, canvasH: number, wetMask: Float32Array, threshold: number): void` — Utility to copy wet.alpha into Stam-sized wetMask grid. Map canvas coordinates (0-based, canvasW*canvasH) to Stam coordinates (1-based interior, (W+2)*(H+2)). Set wetMask[IX(W,i+1,j+1)] = wetAlpha[j*canvasW+i] > threshold ? 1.0 : 0.0.

    13. `boxBlur3x3(W: number, H: number, src: Float32Array, dst: Float32Array, passes: number): void` — Simple box blur for edge darkening mask. For each pass, average each interior cell with its 8 neighbors (divide by 9). Copy dst back to src between passes if passes > 1.

    **Orchestrator function:**

    14. `fluidPhysicsStep(wet: WetBuffers, fluid: FluidBuffers, config: FluidConfig, canvasW: number, canvasH: number, blowDX: Float32Array, blowDY: Float32Array, lastStrokeBounds: { x0: number; y0: number; x1: number; y1: number } | null, physicsMode: 'last' | 'all' | null, sampleHFn: (x: number, y: number) => number): void`

    This replaces `diffuseStep()`. Implementation:
    - `const W = canvasW, H = canvasH` (use canvas resolution per discretion)
    - `const dt = 0.016` (16ms tick)
    - `const gridSize = (W + 2) * (H + 2)`
    - `const iterations = 4` (per discretion — 4 for performance)
    - Clear velocity sources: `fluid.u0.fill(0); fluid.v0.fill(0)`
    - Copy wet.alpha into a Stam-sized temporary for height equalization:
      For each canvas pixel (x,y), set fluid scratch using mapping `IX(W, x+1, y+1)`
    - Call `addHeightEqualization(W, H, fluid.u0, fluid.v0, <mapped waterHeight>, config.omega_h)`
    - Build wet mask and apply edge darkening: `buildWetMaskFromAlpha(W, H, wet.alpha, canvasW, canvasH, fluid.wetMask, 20)`, blur it into `fluid.blurMask`, call `darkenEdges(W, H, fluid.wetMask, fluid.blurMask, fluid.u0, fluid.v0, config.darkening)`
    - Add blow tool forces: map blowDX/blowDY to Stam grid u0/v0
    - Call `velStep(W, H, fluid.u, fluid.v, fluid.u0, fluid.v0, config.viscosity, dt, iterations)`
    - Advect wet layer color channels using velocity field:
      Use premultiplied alpha advection (per RESEARCH.md Pitfall 4):
      - Create temporary Stam-sized arrays for R*A, G*A, B*A, A
      - Copy wet buffers to Stam grid with premultiplication
      - Advect each premultiplied channel + alpha through `advect()`
      - Recover R = R_premul / A (with A > 0.5 check)
      - Copy results back to wet buffers
    - Apply wetness advection: advect wetness field similarly
    - Note: Do NOT advect the velocity field itself (self-advection deferred per CONTEXT.md deferred list — Navier-Stokes self-advection)

    **CRITICAL implementation details:**
    - All Stam grid arrays are sized `(canvasW + 2) * (canvasH + 2)` with 1-cell boundary padding
    - Interior cells are indexed [1..canvasW] x [1..canvasH], boundaries are 0 and W+1/H+1
    - When mapping between canvas (0-based, W*H) and Stam (1-based, (W+2)*(H+2)): canvas pixel (x,y) maps to Stam cell (x+1, y+1)
    - The velocity field (u,v) lives permanently in FluidBuffers and persists between physics ticks
    - Color/alpha advection creates temporary Stam-sized arrays per tick (allocated once as locals or in FluidBuffers if performance becomes an issue)
    - Boundary conditions: use b=1 for u, b=2 for v, b=0 for scalars (standard Stam)

    **Performance guard:** If `physicsMode === 'last'` and `lastStrokeBounds` is non-null, only process the bounded region (+ 50px margin). This means only copying/advecting the relevant subregion of the wet layer. For the initial implementation, process the full grid when in 'all' mode.

    Export all functions for testability. The `fluidPhysicsStep` is the primary entry point.
