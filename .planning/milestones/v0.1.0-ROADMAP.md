## Overview

EFX-Motion Editor v0.1.0 delivers the complete editing foundation through cinematic FX: Tauri scaffold, UI shell, image pipeline, project management, timeline, preview, undo/redo, keyboard shortcuts, multi-layer compositing, and built-in cinematic effects. The build went from zero to a production-capable stop-motion editor in 10 days across 36 plans and 11 quick-task polish iterations.

## Milestone Summary

**Key Decisions:**
- Preact + Preact Signals over React (3KB bundle, fastest rendering)
- Canvas 2D PreviewRenderer over Motion Canvas generator model for compositing
- Custom command-pattern undo over external library (operation-level granularity)
- tinykeys over hotkeys-js for keyboard shortcuts (650B, TypeScript-native)
- FX as timeline-level sequences rather than layer-level effects
- Project format versioned (v1 through v4) with backward compatibility

**Issues Resolved:**
- Data bleed on "New Project" from Toolbar (stores not reset) — fixed in Phase 5
- timelineStore/playbackEngine not reset on project close — fixed in Phase 5
- stopAutoSave() never called — fixed in Phase 5
- Thumbnail dir mismatch on project migration — fixed in Phase 3.1
- NumericInput re-render loop — fixed in Phase 6
- Color grade render bug — fixed in Phase 7

**Technical Debt Carried Forward:**
- Coalescing API (startCoalescing/stopCoalescing) has no UI consumers
- canUndo/canRedo signals exported but no UI disables buttons
- resetShuttle() not called from closeProject()
- 07-11 (Add FX button to timeline) listed but never planned
