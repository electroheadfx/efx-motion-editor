## Integration Findings

### Connected (45/47)

All major cross-phase connections verified:
- Theme signals propagate to canvas drawing and all UI phases
- Canvas zoom drives both transform overlay and CSS layout
- GPU blur routes through shared `applyBlur` API into preview and export
- Keyframe interpolation flows through both preview and export compositing
- Sidebar properties panels wired to all layer types including content overlays
- Transition compositing integrated into shared `renderGlobalFrame`
- Isolation correctly excluded from export path (by design)
- Full-speed/fullscreen mounts separate Preview instance at z-50

### Defects (2)

**INT-01: Content-overlay export image preloading**
- `preloadExportImages` in `exportRenderer.ts` only collects image IDs from the main `frameMap` (content sequences)
- Images from `content-overlay` and FX `image-sequence` layers are not preloaded before the export loop
- Impact: First-frame blank/missing for content-overlay images in export
- Fix: Extend `preloadExportImages` to scan `sequenceStore.sequences` where `kind !== 'content'`

**INT-02: FX generator animation frame offset**
- `exportRenderer.ts` line 224 passes `localFrame` (content-relative) instead of `fxLocalFrame` (FX-relative)
- Impact: Animated generators (grain, particles, lines, dots) use wrong frame offset when FX sequence has `inFrame > 0`
- Fix: Change `localFrame` to `fxLocalFrame` (already computed at line 210)
