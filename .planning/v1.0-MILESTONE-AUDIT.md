## Cross-Phase Integration Issues

### P0 — Thumbnail Directory Name Mismatch

**Files:** `image_pool.rs` line 61 vs `project_io.rs` line 9
**Affected:** IMPT-03, PROJ-02, PROJ-03

`image_pool::process_image()` creates thumbnails at `images/thumbs/` (no dot prefix). `project_io::create_project_dir()` creates `images/.thumbs/` (dot prefix). `project_io::migrate_temp_images()` looks for `.thumbs` subdirectory.

**Impact:** After creating a project (which triggers temp-to-real migration), thumbnails are orphaned in the temp directory because the migration function looks for `images/.thumbs/` but the actual thumbnails are in `images/thumbs/`. Post-migration, `ImportGrid` and `KeyPhotoStrip` show broken thumbnail images.

**Fix:** Change `image_pool.rs` line 61 from `images_dir.join("thumbs")` to `images_dir.join(".thumbs")` to match `project_io.rs`.

### P1 — imageStore Changes Don't Trigger Auto-Save

**File:** `autoSave.ts` effect() subscription
**Affected:** PROJ-04

The `effect()` in `autoSave.ts` subscribes to `projectStore` and `sequenceStore.sequences` signals but not `imageStore.images`. Image imports don't call `markDirty()` or trigger auto-save.

**Fix:** Add `imageStore.images.value` to the effect subscription, or wire `imageStore._setMarkDirtyCallback` similar to sequenceStore.

### P1 — uiStore.selectedSequenceId Not Synced on Hydrate

**File:** `projectStore.ts` `hydrateFromMce()` / `closeProject()`
**Affected:** SEQN-01, PROJ-03

`sequenceStore.activeSequenceId` is set during `hydrateFromMce()` but `uiStore.selectedSequenceId` is not updated. After opening a project, the sequence list visual highlight may point to a stale ID.

**Fix:** Call `uiStore.selectSequence()` alongside `sequenceStore.setActive()` in `hydrateFromMce()` and `closeProject()`.
