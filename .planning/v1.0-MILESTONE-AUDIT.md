## Cross-Phase Integration

### Verified Connections: 32/35

The integration checker verified 32 of 35 cross-phase export connections are correctly wired. Three connections are **partial**:

1. **FOUN-02 (partial):** IPC bridge pattern is sound, but `projectGetDefault` and `imageGetInfo` wrappers have zero production callers (orphaned Phase 1 test artifacts)
2. **PROJ-04 (partial):** Auto-save works correctly but `stopAutoSave()` is exported and never called
3. **PREV-01 (partial):** Preview shows correct frame images via `<img>` overlay; Motion Canvas compositing deferred to Phase 5 by design

### Integration Issues

#### INT-01 (High): Data bleed on New Project from Toolbar

**Path:** Toolbar "New" → `NewProjectDialog.handleCreate()` → `projectStore.createProject()` — does NOT clear `sequenceStore`, `imageStore`, or `uiStore` first. Old sequences/images bleed into new project.

**Fix:** Add `sequenceStore.reset(); imageStore.reset(); uiStore.reset(); timelineStore.reset(); playbackEngine.stop();` at the top of `createProject()`.

**Affected:** PROJ-01, PROJ-02, SEQN-01

#### INT-02 (Medium): timelineStore/playbackEngine not reset on project close

**Path:** `projectStore.closeProject()` resets projectStore/sequenceStore/imageStore/uiStore but not timelineStore or playbackEngine.

**Fix:** Add `timelineStore.reset()` and `playbackEngine.stop()` to `closeProject()`.

**Affected:** PREV-02, PREV-05

#### INT-03 (Medium): stopAutoSave() never called

**Path:** `startAutoSave()` called at app init; `stopAutoSave()` exported but never called. Auto-save effect runs indefinitely. Mitigated by `filePath` guard.

**Affected:** PROJ-04
