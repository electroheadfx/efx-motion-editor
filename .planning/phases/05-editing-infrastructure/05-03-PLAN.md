**1. Create `Application/src/lib/jklShuttle.ts`:**

Implement the DaVinci Resolve-style JKL shuttle controller (per user decision):

```typescript
import { signal } from '@preact/signals';
import { playbackEngine } from './playbackEngine';
import { timelineStore } from '../stores/timelineStore';
import { projectStore } from '../stores/projectStore';
import { totalFrames } from './frameMap';
```

Constants:
- `SPEED_TIERS = [1, 2, 4, 8]` — 4 tiers tuned for stop-motion frame rates (Claude's discretion)
- `BADGE_DURATION = 1200` — ms to show speed badge before fade-out

Signals:
- `currentTier = signal(0)` — 0 = stopped, positive = forward tiers, negative = reverse tiers
- `speedLabel = signal('')` — e.g. "2x", "4x", empty when stopped
- `showSpeedBadge = signal(false)` — drives SpeedBadge visibility

Internal state:
- `rafId: number | null = null` — for the shuttle rAF loop
- `badgeTimeout: ReturnType<typeof setTimeout> | null = null`
- `lastTime: number = 0`
- `accumulator: number = 0`

Exported functions:

- `pressL()`: If currentTier < 0, increment toward zero (decelerate). If currentTier >= 0 and < SPEED_TIERS.length, increment (accelerate forward). Call `updatePlayback()`.

- `pressJ()`: If currentTier > 0, decrement toward zero (decelerate). If currentTier <= 0 and > -SPEED_TIERS.length, decrement (accelerate reverse). Call `updatePlayback()`.

- `pressK()`: Set currentTier to 0. Stop the shuttle rAF loop. Call `playbackEngine.stop()`. Clear speed label. Hide badge.

- `resetShuttle()`: Same as pressK but no playbackEngine.stop() call (used when playback stops naturally). Reset all internal state.

- `currentSpeedLabel` — export the speedLabel signal for SpeedBadge to read
- `showSpeedBadge` — export for SpeedBadge to read

Internal `updatePlayback()`:
1. Calculate speed multiplier from currentTier: if 0, call pressK logic. Otherwise `absIndex = Math.abs(currentTier) - 1`, `speed = SPEED_TIERS[min(absIndex, TIERS.length-1)]`, multiply by sign.
2. Update speedLabel signal: e.g. "2x" for tier 2 forward, "-4x" for tier -3 reverse, empty for tier 0 or ±1
3. Flash speed badge: set showSpeedBadge = true, clear previous badgeTimeout, set new timeout to hide after BADGE_DURATION ms. For tier 0 or ±1, still flash briefly to confirm the action.
4. **Reset accumulator to 0** when speed changes (Pitfall 7 from research: prevents frame burst)
5. Stop existing rAF loop. If speed !== 0, stop playbackEngine's normal playback (if running) and start the shuttle's own rAF loop.

Internal shuttle rAF tick:
- Similar to PlaybackEngine.tick but uses the shuttle speed multiplier for variable-rate:
  - `delta = now - lastTime`, `lastTime = now`
  - `fps = projectStore.fps.peek()`
  - `frameDuration = 1000 / (fps * abs(speedMultiplier))`
  - `accumulator += delta`
  - While accumulator >= frameDuration: drain frames in the correct direction (forward or backward)
  - Clamp to 0 and totalFrames-1 bounds. If hitting bounds, stop shuttle.
  - Request next frame if still shuttling.

**2. Create `Application/src/lib/shortcuts.ts`:**

```typescript
import { tinykeys } from 'tinykeys';
```

Helper function `shouldSuppressShortcut(event: KeyboardEvent): boolean`:
- Check `event.target` as HTMLElement
- Return true if tagName is INPUT, TEXTAREA, or SELECT
- Return true if `target.isContentEditable` is true
- Return false otherwise

Export `mountShortcuts(): () => void` that returns the tinykeys unsubscribe function:

```typescript
return tinykeys(window, {
  // Playback (KEY-01, KEY-02)
  'Space': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault(); // Prevent page scroll (Pitfall 6)
    playbackEngine.toggle();
  },
  'ArrowLeft': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    playbackEngine.stepBackward();
  },
  'ArrowRight': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    playbackEngine.stepForward();
  },

  // JKL Scrub (KEY-03)
  'KeyJ': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    pressJ();
  },
  'KeyK': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    pressK();
  },
  'KeyL': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    pressL();
  },

  // Undo/Redo (KEY-04)
  '$mod+KeyZ': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    undo();
  },
  '$mod+Shift+KeyZ': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    redo();
  },

  // File operations (KEY-05)
  '$mod+KeyS': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    handleSave();  // reuse Toolbar's save logic
  },
  '$mod+KeyN': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    handleNewProject();  // guard + trigger new project flow
  },
  '$mod+KeyO': (e) => {
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    handleOpenProject();  // guard + trigger open flow
  },

  // Delete (KEY-06)
  'Backspace': (e) => {
    if (shouldSuppressShortcut(e)) return;
    handleDelete();
  },
  'Delete': (e) => {
    if (shouldSuppressShortcut(e)) return;
    handleDelete();
  },

  // Shortcuts overlay (KEY-08)
  'Shift+Slash': (e) => {  // ? key
    if (shouldSuppressShortcut(e)) return;
    e.preventDefault();
    uiStore.toggleShortcutsOverlay();
  },
});
```

**Internal handler functions** (defined inside shortcuts.ts or imported):

- `handleSave()`: Import projectStore, call save/saveAs logic (same as Toolbar handleSave). Import `save` from `@tauri-apps/plugin-dialog` for Save As flow.
- `handleNewProject()`: Import `guardUnsavedChanges`. If guard returns 'cancelled', return. Otherwise, trigger new project creation. Since Cmd+N needs to show the NewProjectDialog (which is React state in Toolbar), use a signal approach: add `showNewProjectDialog` signal to uiStore, set it to true from here. Toolbar reads this signal to show the dialog.
- `handleOpenProject()`: Import `guardUnsavedChanges` and `open` from dialog plugin. If guard returns 'cancelled', return. Show file picker, open project if selected.
- `handleDelete()`: Check `uiStore.selectedLayerId` — if set, call `layerStore.remove()`. Otherwise check for selected key photo context (check if there's a way to identify selected key photo — for now, this deletes selected layer only; key photo deletion from keyboard will be refined when key photo selection state is added in Phase 6). If nothing is selected, no-op.
