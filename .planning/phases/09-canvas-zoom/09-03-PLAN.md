**1. Rewrite fitToWindow() -- remove 1.0 cap, compute from project resolution:**

The current implementation has three bugs:
- `Math.min(fitScale, 1.0)` hard-caps zoom at 100% (line 114)
- `natW = Math.min(containerW, 830)` uses a CSS max-width artifact instead of project resolution
- The aspect ratio math is overly complex

Replace with a simple calculation:
```typescript
fitToWindow() {
  const containerW = containerWidth.peek();
  const containerH = containerHeight.peek();
  if (containerW <= 0 || containerH <= 0) return;

  const projW = projectStore.width.peek();
  const projH = projectStore.height.peek();
  if (projW <= 0 || projH <= 0) return;

  // Scale that fits the full project resolution inside the container
  const fitScale = Math.min(containerW / projW, containerH / projH);
  // Clamp to valid zoom range (allow above 1.0 -- no artificial cap)
  zoom.value = Math.max(MIN_ZOOM, Math.min(fitScale, MAX_ZOOM));
  panX.value = 0;
  panY.value = 0;
}
```

This computes the zoom level that makes the full project resolution (e.g. 1920x1080) fit inside the measured container, allowing zoom values above 1.0 when the container is larger than the project resolution.

**2. Add a `fitZoom` computed signal** that other parts can query to know the current fit level (used for grab cursor logic):

```typescript
const fitZoom = computed(() => {
```

**3. Add clampPan() helper** to prevent canvas from escaping viewport:
```typescript
function clampPan() {
  const z = zoom.peek();
  const cW = containerWidth.peek();
  const cH = containerHeight.peek();
  const pW = projectStore.width.peek();
  const pH = projectStore.height.peek();
  if (cW <= 0 || cH <= 0 || pW <= 0 || pH <= 0) return;

  // In the CSS transform `scale(z) translate(px, py)`, the translate
  // is applied in scaled space. The rendered size is pW*z x pH*z (but
  // the translate values are multiplied by z again by the scale transform).
  // Maximum pan = half the overflow in *unscaled* translate units.
  const maxPanX = Math.max(0, (pW - cW / z) / 2);
  const maxPanY = Math.max(0, (pH - cH / z) / 2);

  panX.value = Math.max(-maxPanX, Math.min(maxPanX, panX.peek()));
  panY.value = Math.max(-maxPanY, Math.min(maxPanY, panY.peek()));
}
```

NOTE on the math: With transform `scale(z) translate(px, py)`, the actual pixel offset is `z * px`. The canvas rendered size is `z * pW` (project width scaled). The container is `cW`. The overflow (how much bigger the scaled canvas is than the container) is `z * pW - cW`. Maximum translate that keeps at least some canvas visible = half the overflow, in translate-space (divide by z): `(z * pW - cW) / (2 * z) = (pW - cW/z) / 2`.

**4. Fix setSmoothZoom() cursor-anchored math:**

Current bug: `panX.value = panX.value * scale + cursorX * (1 - scale) / clamped` -- the `/ clamped` is wrong for the transform order.

With CSS `scale(z) translate(px, py)`, the actual pixel position of a point at translate-offset `px` is `z * px`. The cursor anchor formula must keep the cursor's world-space point stationary:

```typescript
setSmoothZoom(newZoom: number, cursorX: number, cursorY: number) {
  const clamped = Math.max(MIN_ZOOM, Math.min(MAX_ZOOM, newZoom));
  const oldZoom = zoom.peek();
  if (Math.abs(clamped - oldZoom) < EPSILON) return;

  // cursorX/Y are in container-space, relative to container center.
  // In translate-space: cursor_translate = cursorX / oldZoom
  // After zoom change, to keep that world point under the cursor:
  //   newPan = oldPan + cursorX/oldZoom - cursorX/clamped
  panX.value = panX.peek() + cursorX / oldZoom - cursorX / clamped;
  panY.value = panY.peek() + cursorY / oldZoom - cursorY / clamped;
  zoom.value = clamped;
  clampPan();
}
```

**5. Fix zoomIn() and zoomOut() pan scaling:**

Current code: `panX.value = panX.value * scale` -- this amplifies any existing offset. For center-anchored (toolbar/keyboard) zoom, simply reset pan to 0:

```typescript
zoomIn() {
  const current = zoom.value;
  let next: number | null = null;
  for (const preset of ZOOM_PRESETS) {
    if (preset > current + EPSILON) { next = preset; break; }
  }
  if (next === null) return;
  zoom.value = next;
  clampPan();
}

zoomOut() {
  const current = zoom.value;
  let next: number | null = null;
  for (let i = ZOOM_PRESETS.length - 1; i >= 0; i--) {
    if (ZOOM_PRESETS[i] < current - EPSILON) { next = ZOOM_PRESETS[i]; break; }
  }
  if (next === null) return;
  zoom.value = next;
  clampPan();
}
```

This keeps existing pan but clamps it to valid bounds, so if the image was offset it snaps back within bounds.

**6. Update setPan() to call clampPan():**

```typescript
setPan(x: number, y: number) {
  panX.value = x;

```
