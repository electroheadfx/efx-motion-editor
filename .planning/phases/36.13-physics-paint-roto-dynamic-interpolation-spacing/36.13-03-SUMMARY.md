# Plan 36.13-03 Summary

## Objective

Make Roto key utilities preserve intentional custom timing when source keys move and remove ambiguous timing when source keys are deleted.

## What Changed

- `buildRotoKeyUtilityTransaction` now accepts existing `segmentSpacingOverrides` and returns the next override set on every key utility transaction.
- Duplicate and Insert rebase override endpoints through moved/copied source-key mappings.
- Delete drops any override touching the deleted source key so the merged segment falls back to global spacing.
- Paste can receive a far-empty resolved source target and replacement previous-segment override.
- `createRotoSession` can pass current override state and far-empty paste target resolution into key transactions.
- `replaceKeys` effects expose `transaction.segmentSpacingOverrides` for later Studio/store persistence.
- Generated-frame copy remains blocked through the existing render-only action availability guard.
