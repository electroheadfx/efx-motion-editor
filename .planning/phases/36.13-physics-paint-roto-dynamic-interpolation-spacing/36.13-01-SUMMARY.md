# Phase 36.13 Plan 01: Dynamic Interpolation Source/Display Contract Summary

Tested source/display contract for durable per-source-segment Roto interpolation spacing, with real source keys promoted as authoritative identity and generated frames kept as derived render-only display output.

## What Changed

- Added RED Vitest coverage for D-01 through D-04 using literal spec examples:
  - compact source keys `0 / 1 / 2` display as real keys `0 / 3 / 6` when global count is 2;
  - custom source keys `0 / 1 / 2 / 6` display as real keys `0 / 3 / 6 / 11` with a `2 -> 6 = 4` override;
  - global count changes affect only non-custom segments;
  - far empty display save at display `11` resolves to source key `6` with a `2 -> 6 = 4` override.
- Added `PhysicPaintRotoSegmentSpacingOverride` and `PhysicPaintRotoInterpolationSettings.segmentSpacingOverrides` for validated persisted settings.
- Added normalization/validation helpers for source-endpoint spacing overrides, rejecting malformed endpoints, duplicate segments, non-integer counts, and counts beyond `PHYSIC_PAINT_MAX_APPLY_FRAMES`.
- Updated workflow-state helpers to consume custom per-segment counts:
  - `normalizeRotoSegmentSpacingOverrides`
  - `inferRotoSegmentSpacingOverrides`
  - `getExpandedRotoRealKeyFrames`
  - `getRotoInterpolationSpanFrames`
  - `getSourceRotoFrameForDisplayFrame`
  - `resolveRotoFarEmptyDisplaySaveTarget`
