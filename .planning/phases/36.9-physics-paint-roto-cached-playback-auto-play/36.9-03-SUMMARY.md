# Phase 36.9 Plan 03: Visible Cached Roto Transport and UAT Summary

**Cached Roto playback is now discoverable in the Roto navigator, supports loop/fps preview controls and Space Play/Stop, and sequences only real cached key frames.**

- **Completed:** 2026-06-26T12:25:00Z

## Accomplishments

- Added visible cached Roto Play/Stop transport in the existing Roto navigator using one icon slot that switches between Play and Stop.
- Added loop replay and fps cadence controls, defaulting from project fps and restarting active playback when cadence changes.
- Added Space Play/Stop shortcut scoped to Roto mode and guarded against editable shortcut targets.
- Fixed cached playback so each tick advances the visible launch frame context while keeping the engine ready during preview playback.
- Fixed playback frame selection to use only real cached Roto keys, preventing empty fallback frames from playing after the cached keys.
- Preserved preview-only/no-mutation behavior: no render/export/save-all action, no AnimationPlayer path, no flush/apply/regenerate/cache rewrite during cached playback.

## Decisions Made

- The Play and Stop controls share the same navigator slot so the transport does not add a separate text button or widen the Roto header.
- Loop uses an icon-only pressed button (`aria-pressed`) to match the user's requested visual treatment without adding checkbox UI.
- Cadence is a small numeric fps input because the requested workflow needs quick low-fps preview such as 2 fps while retaining project-fps default behavior.
- The playback list is derived only from `getRealCachedRotoFrameNumbers(launchContext)` and `findCachedRotoReferenceFrame(appFrame)` so preview playback reflects saved real keys rather than timeline occupancy or transient background placeholders.
