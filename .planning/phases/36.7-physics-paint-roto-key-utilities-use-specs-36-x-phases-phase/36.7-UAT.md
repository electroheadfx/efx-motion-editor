## Tests

### 1. Open Roto Key Utility Surface
expected: Open the already-running app, enter Physics Paint in Roto mode on a layer with at least two real Roto keys, and select a real key. You should see the existing contextual key utility pill in the timeline lane with `Key {frame}`, `Insert`, `Dup`, `Copy`, `Paste`, and `Delete`; there should be no new top-header duplicate controls, shortcuts, modal, toast, tutorial, interpolation, cached playback, or Play/Roto conversion UI.
result: pass

### 2. Insert Clean Blank Real Key
expected: Click `Insert` before a selected real Roto key. The inserted frame should become a clean blank editable real key, current/later real keys should shift forward, the canvas should not show yellow/translucent generated or reference pixels, and the timeline cell should show the clean real/cached Roto key state with `Inserted blank key before frame {frame}.`
result: pass

### 3. Return to Inserted Key and Edit
expected: Navigate away from the inserted key, return to it, and paint on it. The frame should remain a normal clean editable real key and should not restore stale generated/reference/background overlay content.
result: pass

### 4. Duplicate Real Key
expected: Click `Dup` on a selected real Roto key. A next-frame real key should be created, later real keys should shift forward, cache/cell/preview state should stay clean and updated, and status should show `Duplicated to frame {frame}.`
result: pass

### 5. Copy Real Key and Paste to Empty Target
expected: Click `Copy` on a real Roto key, select an empty target frame, then click `Paste`. The target should become a clean editable real key using the copied content, preview/cell state should update, and status should show `Pasted key to frame {frame}.`
result: pass

### 6. Paste to Generated Target
expected: After copying a real Roto key, select a generated/render-only target if available and click `Paste`. The target should become a clean editable real key, generated render-only behavior for that frame should clear, no stale blended overlay should remain, and status should show `Pasted key to frame {frame}.`
result: pass

### 7. Delete Real Key With Later Keys
expected: Click `Delete` on a real Roto key that has later real keys. The selected key should be removed, later real keys should shift backward, the deleted frame should not reappear as non-empty/yellow/translucent after navigating away and back, cells should reflect the new clean real-key layout, and status should show `Deleted key {frame}.`
result: pass

### 8. Delete Last Remaining Real Key
expected: If available, delete the last remaining real Roto key. The layer should be allowed to return to no real Roto keys, and returning to that frame should show no stale deleted canvas/cache.
result: pass

### 9. Generated and Empty Source Guards
expected: Select generated or empty frames. `Dup`, `Insert`, `Copy`, and `Delete` should be unavailable for those source frames; `Paste` should become available only after a real key has been copied and should still be allowed on empty/generated/real targets.
result: pass

### 10. Dirty Save Before Key Action
expected: Dirty the current real key, then trigger `Insert`, `Delete`, `Copy`, or `Dup`. You should see `Saving frame {frame} before {action}...`, all five key tools should disable while saving/actioning, and if save fails the action should cancel with `Could not save frame {frame}; {action} was cancelled.` without mutating cache or losing the dirty visible canvas.
result: pass

### 11. Focused Regression Tests
expected: Run the focused Phase 36.7 regression command. `physicsPaintRotoKeyController`, `PhysicsPaintWorkflowStrip`, and `physicsPaintWorkflowState` tests should pass, covering real-key transforms, source/target eligibility, controller transactions, UI scope, and Studio delegation.
result: pass

### 12. Typecheck
expected: Build the local physics paint workspace package if needed, then run the app TypeScript typecheck. The typecheck should pass without new test config, fallback shims, or unrelated source changes.
result: pass

### 13. User Story Coverage
expected: After the live flow passes, the phase should satisfy the user story: as a stop-motion animator, you can duplicate, insert, delete, copy, and paste real Roto keys to build frame-by-frame animation efficiently, with clean cache/cell/canvas state throughout.
result: pass
