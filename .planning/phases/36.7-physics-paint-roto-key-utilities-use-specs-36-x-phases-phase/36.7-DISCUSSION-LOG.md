## Placement Rules

### Duplicate destination

| Option | Description | Selected |
|--------|-------------|----------|
| Next frame shift | Duplicate the current real key into the next frame and shift later real keys forward. | ✓ |
| Next empty only | Duplicate into the next empty frame without moving later keys. | |
| Ask target frame | Let the animator choose a destination frame. | |
| You decide | Let planning keep the behavior closest to current implementation and spec constraints. | |

**User's choice:** Next frame shift
**Notes:** Locks Duplicate as a splice-like timeline operation.

### Insert behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Blank before current | Insert a new blank/empty real key at the current frame and shift the current/later real keys forward. | ✓ |
| Copy before current | Insert a duplicate of the current key at the current frame and shift current/later keys forward. | |
| Blank after current | Insert a blank real key after the current key and shift later keys forward. | |
| You decide | Let planning choose the simplest behavior that preserves predictable frame ownership. | |

**User's choice:** Blank before current
**Notes:** Insert is a blank cel before the selected real key.

### Delete gap behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Shift backward | Remove the current real key and shift later real keys backward. | ✓ |
| Leave gap | Remove only the current key, leaving later keys at their original frame numbers. | |
| Ask per delete | Prompt the animator to choose shift vs leave gap. | |
| You decide | Let planning preserve coherent behavior across all five actions. | |

**User's choice:** Shift backward
**Notes:** Delete closes gaps, keeping structural actions splice-like.

### Generated frame shifting

| Option | Description | Selected |
|--------|-------------|----------|
| Real keys only | Shift only real Roto keys; generated/interpolated frames are regenerated from real keys afterward. | ✓ |
| Shift all cells | Shift real keys plus generated/cache-derived cells as visible timeline cells. | |
| Disable if generated | Only allow splice actions when no generated/interpolated frames are present. | |
| You decide | Let planning follow the safest existing source/type boundaries. | |

**User's choice:** Real keys only
**Notes:** Generated/interpolated frames are render-only derived state, not editable splice participants.

## Dirty State Handling

### Dirty Duplicate/Copy source

| Option | Description | Selected |
|--------|-------------|----------|
| Save first | Save the current dirty key through the existing Roto save path, then duplicate/copy the newly saved result. | ✓ |
| Cached only | Use the last cached version and ignore unsaved edits. | |
| Block until saved | Disable Duplicate/Copy and tell the animator to Save current first. | |
| You decide | Let planning use the behavior most consistent with Phase 36.6 save-on-leave. | |

**User's choice:** Save first
**Notes:** Preserves what the animator sees.

### Dirty Insert/Delete

| Option | Description | Selected |
|--------|-------------|----------|
| Save first | Save first, then perform the structural insert/delete. | ✓ |
| Block dirty | Disable Insert/Delete until the animator saves or discards. | |
| Discard dirty | Run Insert/Delete against the last cached state and discard unsaved edits. | |
| You decide | Let planning keep behavior consistent across all key utilities. | |

**User's choice:** Save first
**Notes:** Structural actions preserve dirty visible state before shifting/removal.

### Save-before-action failure

| Option | Description | Selected |
|--------|-------------|----------|
| Cancel action | Do not run the key utility, leave the dirty frame visible/in memory, and show failure feedback. | ✓ |
| Use cached fallback | Run the key utility using the last cached frame even though the save failed. | |
| Ask after failure | On failure, ask whether to continue with cached state or cancel. | |
| You decide | Let planning follow the safest failure behavior from save-on-leave. | |

**User's choice:** Cancel action
**Notes:** Dirty in-memory frame remains the unsaved truth until retry succeeds.

### Post-success saved state

| Option | Description | Selected |
|--------|-------------|----------|
| Saved clean | Structural actions update parent cache/timeline/preview immediately and affected frames are clean. | ✓ |
| Dirty until Save | Mark affected keys dirty and require Save current after the utility action. | |
| Mixed status | Only current frame is clean; shifted/copied frames are marked pending/dirty until revisited. | |
| You decide | Let planning choose the simplest state model that satisfies UAT. | |

**User's choice:** Saved clean
**Notes:** The utility action itself is a cache operation.
