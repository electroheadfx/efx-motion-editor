## Decisions Made

- The controller is pure/stateless and does not use `@preact/signals`. Signals were not needed because no persistent shared state moved outside Studio; the boundary receives current cache/local state as data and returns deterministic transaction outputs. This prevents new `useEffect`/ref sprawl by making transaction layout, generated-target cleanup, deleted-frame cleanup, dirty-save-before-action exposure, and active restore intent explicit return values.
- Studio intentionally remains responsible only for true boundaries: snapshot/flush before dirty actions, sending `replace-roto-key-frames`, refreshing launch-context cache metadata, updating local refs/maps from controller output, and loading/clearing the engine canvas according to `transaction.activeRestore`.
