## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between kbd and description in entry rows (`gap-1`), entry row vertical padding (`py-1`) |
| sm | 8px | Tab button vertical padding (`py-2`), content area gap between entry rows (`gap-2`) |
| md | 12px | Tab button horizontal padding (`px-3`) — 12px is 3x4, valid |
| lg | 16px | Gap between tab bar and content area (`pt-4`) |
| xl | 24px | Modal padding (`p-6`) |

Exceptions: none. Entry row internal layout (description + kbd pair) is a preserved element from the existing overlay and is outside this phase's spacing contract scope. All new spacing introduced by this phase uses multiples of 4 only.

## Interaction Contract

### Tab Selection

| Action | Behavior |
|--------|----------|
| Click tab button | Set activeTab to that index; content area shows that group's entries |
| ArrowRight on tab bar | Cycle activeTab forward (wraps from last to first) |
| ArrowLeft on tab bar | Cycle activeTab backward (wraps from first to last) |
| Overlay open | activeTab resets to 0 (Playback) via useState initial value |
| Overlay close + reopen | activeTab is 0 again (component unmounts/remounts) |

### Tab Button States

| State | Text Color | Border | Background |
|-------|-----------|--------|------------|
| Default | `--color-text-muted` | none | transparent |
| Hover | `--color-text-secondary` | none | transparent |
| Active | `--color-text-heading` | 2px bottom `--color-accent` | transparent |

### Keyboard Navigation

| Key | Scope | Action |
|-----|-------|--------|
| Escape | Modal (existing) | Close overlay |
| ArrowLeft | Tab bar focused | Previous tab (wrapping) |
| ArrowRight | Tab bar focused | Next tab (wrapping) |

Note: Arrow key handlers attach to the tab bar container element only (`onKeyDown`), not globally. The modal's `e.stopPropagation()` on the container prevents leakage to global shortcuts (layer nudge).

### Dismiss Behavior (preserved)

| Action | Result |
|--------|--------|
| Escape key | Close overlay |
| Click backdrop | Close overlay |
| Click close button | Close overlay |
| Click inside modal | No action (stopPropagation) |

## Layout Contract

### Modal Structure (top to bottom)

```
+------------------------------------------+
|  Header: "Keyboard Shortcuts"    [X]     |  h: auto, mb-5 (20px)
+------------------------------------------+
|  Tab Bar: [Playback] [File] [Editing]... |  h: auto, border-b
+------------------------------------------+
|                                          |
|  Content Area: single group entries      |  min-h: 320px, pt-4 (16px)
|    Description ............... [Kbd]     |
|    Description ............... [Kbd]     |
|    ...                                   |
|                                          |
+------------------------------------------+
```

### Modal Dimensions

| Property | Value | Rationale |
|----------|-------|-----------|
| Max width | 600px | Preserved from current overlay |
| Horizontal margin | 16px (`mx-4`) | Preserved from current overlay |
| Padding | 24px (`p-6`) | Preserved from current overlay |
| Border radius | 8px (`rounded-lg`) | Preserved from current overlay |
| Content min-height | 320px | Prevents modal height jump between tabs (largest group Canvas has 8 entries at ~36px each = 288px, rounded up for comfort) |
| Content overflow | `overflow-y-auto` | Safety valve if future groups exceed min-height |

### Tab Bar Layout

| Property | Value |
|----------|-------|
| Direction | Horizontal flex row (`flex`) |
| Wrap | `flex-wrap` (in case labels overflow at small widths) |
| Bottom border | 1px solid `--color-border-subtle` |
| Tab padding | 12px horizontal, 8px vertical (`px-3 py-2`) |
| Tab font | 12px regular (`text-xs`) |
| Tab transition | `transition-colors` (color change on hover/active) |
| Tab cursor | `cursor-pointer` |
| Active indicator | 2px bottom border `--color-accent` (via `border-b-2`) |

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Tab semantics | Tab buttons use `<button>` elements (not divs) for native keyboard focus |
| Focus management | Tab bar container receives keyboard events for arrow navigation |
| Active indication | Active tab has visible 2px accent border (not color-only differentiation) |
| Event isolation | Modal container uses `e.stopPropagation()` to prevent arrow keys from leaking to global layer-nudge shortcuts |
| Screen reader | Active tab communicates state via visible text color change + border indicator |
