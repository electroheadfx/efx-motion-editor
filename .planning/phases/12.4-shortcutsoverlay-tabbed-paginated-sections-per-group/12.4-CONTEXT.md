# Phase 12.4: ShortcutsOverlay Tabbed/Paginated Sections per Group - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current flat 2-column grid of all 28 shortcuts in ShortcutsOverlay with a tabbed interface. Each shortcut group becomes its own tab, plus a "Sections" index tab as the first tab. Only one group's entries display at a time. The component remains a single file refactor of `ShortcutsOverlay.tsx` with no new dependencies.

</domain>

<decisions>
## Implementation Decisions

### Tab structure
- 8 tabs total: "Sections" index tab first, then 7 group tabs (Playback, File, Editing, Navigation, Canvas, Blur, Keyframes)
- No "All" tab — per-group tabs only, no flat 2-column view
- Keep strict 1:1 mapping between SHORTCUT_GROUPS and tabs — no merging sparse groups (Blur 2 entries, Keyframes 1 entry stay as separate tabs)
- Future phases will add more shortcuts to sparse groups

### Sections index tab
- First tab (index 0) shows a clickable list of all 7 group names with shortcut counts
- Clicking a group name in the index switches activeTab to that group's tab
- Acts as a table of contents / navigation hub

### Keyboard navigation
- Number keys 1-7: direct jump to a specific group tab from any tab (1=Playback, 2=File, ..., 7=Keyframes)
- Arrow Left/Right: cycle through tabs (wrapping from last to first and first to last)
- Tab key: equivalent to Arrow Right (advance to next tab, wrapping)
- Arrow Up/Down: navigate rows within the Sections index tab, Enter to jump to that group
- All navigation wraps cyclically — reaching the end cycles back to the first tab

### Tab overflow behavior
- `flex-wrap` on the tab bar — tabs wrap to a second row if they don't fit at 600px width
- All 8 tabs always visible, no horizontal scrolling or text shrinking
- Expected split: ~5 tabs on first row, ~3 on second row

### Claude's Discretion
- Exact tab bar padding/gap fine-tuning within the UI-SPEC constraints
- Content area min-height calibration (UI-SPEC says 320px)
- Arrow Up/Down row highlight styling in the Sections tab

</decisions>
