**Part B: Make TimelineRenderer.ts theme-aware**

The TimelineRenderer uses Canvas 2D drawing with hardcoded constants. CSS variables cannot be read from canvas context directly, so implement a cached color lookup:

1. **Remove the hardcoded color constants** (TRACK_BG, TRACK_HEADER_BG, FRAME_BORDER_COLOR, RULER_BG, RULER_TEXT_COLOR, TRACK_NAME_COLOR, FX_TRACK_BG, FX_TRACK_HEADER_BG) and replace them with a cached lookup object.

2. **Add a color cache mechanism** at module level:
```typescript
let cachedColors: Record<string, string> | null = null;

function getThemeColors(): Record<string, string> {
  if (cachedColors) return cachedColors;
  const style = getComputedStyle(document.documentElement);
  cachedColors = {
    trackBg: style.getPropertyValue('--color-timeline-track-bg').trim() || '#111111',
    headerBg: style.getPropertyValue('--color-timeline-header-bg').trim() || '#0D0D0D',
    frameBorder: style.getPropertyValue('--color-timeline-frame-border').trim() || '#222222',
    rulerBg: style.getPropertyValue('--color-timeline-ruler-bg').trim() || '#0A0A0A',
    rulerText: style.getPropertyValue('--color-timeline-ruler-text').trim() || '#666666',
    trackName: style.getPropertyValue('--color-timeline-track-name').trim() || '#999999',
    fxTrackBg: style.getPropertyValue('--color-timeline-fx-track-bg').trim() || '#0D0D0D',
    fxHeaderBg: style.getPropertyValue('--color-timeline-fx-header-bg').trim() || '#0A0A0A',
  };
  return cachedColors;
}

export function invalidateColorCache(): void {
  cachedColors = null;
}
```

3. **Replace constant references in draw methods** with `getThemeColors().trackBg`, `getThemeColors().headerBg`, etc. Call `getThemeColors()` once at the start of each `draw()` call and use the returned object throughout. Do NOT call `getComputedStyle` on every frame -- only when cache is null (after theme change).

4. **Keep PLAYHEAD_COLOR (#E55A2B) and DROP_INDICATOR_COLOR (#4488FF) as hardcoded constants** -- these are functional colors that must remain high-visibility regardless of theme.

5. **Keep PLACEHOLDER_BG_A (#1A1A2A) and PLACEHOLDER_BG_B (#1A2A1A) hardcoded** -- these are subtle tinted backgrounds for empty frame slots. They work across all themes since they are very dark and only visible when no thumbnail is present.

6. **Wire cache invalidation to theme changes.** In the component that manages the TimelineRenderer (likely TimelinePanel.tsx or wherever the renderer is instantiated), add an effect that calls `invalidateColorCache()` when the theme signal changes, then triggers a redraw. Use `currentTheme` from themeManager to detect changes:
```typescript
import { currentTheme } from '../../lib/themeManager';
import { invalidateColorCache } from '../timeline/TimelineRenderer';

// In the component's setup or useEffect:
// Subscribe to theme changes
const unsub = currentTheme.subscribe(() => {
  invalidateColorCache();
  // trigger redraw if renderer exists
});
```

If the TimelinePanel already has a draw loop or requestAnimationFrame, the cache invalidation alone is sufficient -- the next draw will pick up the new colors. If not, explicitly trigger a redraw after invalidation.
