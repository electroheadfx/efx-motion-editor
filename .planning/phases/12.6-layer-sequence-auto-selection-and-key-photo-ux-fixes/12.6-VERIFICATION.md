### Human Verification Required

The following behaviors are correctly wired in code but require a running Tauri application to confirm the user-visible outcomes:

#### 1. Transform Overlay After Sequence Switch

**Test:** With at least two sequences, click from sequence A to sequence B in the sidebar.
**Expected:** Transform handles (scale/rotate corner grips) appear on the canvas around the active layer of sequence B, without needing to click any layer manually.
**Why human:** Transform overlay rendering depends on the live signal subscription in the canvas renderer, which cannot be verified statically.

#### 2. Key Photo Click-to-Seek

**Test:** Click any key photo thumbnail in the inline strip of an active sequence.
**Expected:** Timeline playhead jumps to that key photo's start frame. The active layer selection indicator updates.
**Why human:** `playbackEngine.seekToFrame` drives DOM mutations in the canvas/timeline that cannot be grepped.

#### 3. Strip Auto-Scroll on Scrub (Not During Playback)

**Test:** Drag the playhead across several key photos. Then start playback and observe the strip.
**Expected:** While scrubbing (playhead drag), strip scrolls to keep the highlighted key photo visible. During active playback, strip does NOT scroll. After stopping playback, strip snaps to the photo at the stopped frame.
**Why human:** `timelineStore.isPlaying.peek()` gate requires live playback state; `displayFrame` signal timing depends on the playback tick loop.

#### 4. Frame-Derived Ring Highlight

**Test:** Scrub the playhead through all key photos of a sequence without clicking any.
**Expected:** The ring highlight moves from photo to photo tracking the playhead position. There is no separate "selected" state — the ring is purely positional.
**Why human:** Visual ring class toggling is reactive to `displayFrame` signal; requires live signal propagation to verify the correct photo receives `ring-2`.

#### 5. Empty Sequence + Add Photos Prompt

**Test:** Create a new sequence via a method that leaves it empty (e.g., duplicate and remove all key photos, or confirm + Add with zero images selected — though this is blocked by UI). Then click that sequence in the sidebar.
**Expected:** The expanded strip area shows "+ Add photos" text (not blank). Clicking it opens the Imported view in normal (not multi-select) mode.
**Why human:** Requires live state where a sequence has zero key photos and is active.

#### 6. Transactional + Add Flow — Confirm Path

**Test:** Click + Add. Select 3 images in multi-select mode. Click Confirm.
**Expected:** All 3 images appear as key photos in the new sequence. Importer closes. No pending state remains.
**Why human:** `sequenceStore.addKeyPhoto` mutations require live imageStore state with real image IDs.

#### 7. Transactional + Add Flow — Cancel Path

**Test:** Click + Add. Without selecting any images (or after selecting some), click the X button.
**Expected:** The new sequence that was created is gone. The sequence list is identical to before + Add was clicked.
**Why human:** `sequenceStore.remove` mutation and reactive re-render require a live app.

#### 8. AddKeyPhotoButton Does NOT Trigger Cancel-Delete

**Test:** While a sequence has key photos, click the small + button in the inline strip (AddKeyPhotoButton). The Imported view opens. Close it with X.
**Expected:** No sequence is deleted. The importer closes normally. The sequence remains unchanged.
**Why human:** `pendingNewSequenceId` must NOT be set in this flow — requires verifying that `handleClose` skips the `sequenceStore.remove` branch because `pending` is null.

No gaps found. All 5 automated truths are fully wired. The 9 human verification items cover visual, real-time, and live-state behaviors that are correctly implemented in code but require a running application to confirm end-to-end.
