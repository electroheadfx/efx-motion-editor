1. Create `Application/src/lib/brushP5Adapter.ts` -- the single integration point for p5.brush standalone. This file replaces ALL of the broken custom rendering code. Structure:

```typescript
import * as brush from 'p5.brush/standalone';
import type { PaintStroke, BrushStyle, BrushFxParams } from '../types/paint';
```

**Style mapping** -- map our 6 BrushStyle values to p5.brush preset names:
```typescript
const STYLE_MAP: Record<string, string> = {
  flat: '',              // never reaches p5.brush (handled by Canvas2D)
  watercolor: 'marker',  // marker tip + fill effects for wash
  ink: 'pen',            // fine pen with slight scatter, edge darkening via overlap
  charcoal: 'charcoal',  // heavy scatter, grainy
  pencil: 'HB',          // standard graphite look
  marker: 'marker',      // solid disc, minimal variation
};
```

**Custom brush definitions** -- use `brush.add()` to create tuned variants that better match our expectations:
- `our_ink`: type 'default', weight 0.25, scatter 0.2, sharpness 0.85, grain 0.8, opacity 180, pressure [0.8, 1.2], rotate 'natural'
- `our_charcoal`: type 'default', weight 0.35, scatter 1.5, sharpness 0.68, grain 2.0, opacity 120, rotate 'random'
- `our_pencil`: type 'default', weight 0.3, scatter 0.6, sharpness 0.3, grain 0.7, opacity 170

Register these in an `initCustomBrushes()` function called once after first `brush.load()`. Update STYLE_MAP to use `our_ink`, `our_charcoal`, `our_pencil` instead of built-in names.

**Singleton OffscreenCanvas management:**
```typescript
let _offscreen: OffscreenCanvas | null = null;
let _initialized = false;
let _customBrushesAdded = false;
```

`ensureInitialized(width, height)`:
- If offscreen is null or dimensions changed, create new OffscreenCanvas and call `brush.load(offscreen)`
- Call `brush.seed(42)` for deterministic rendering (export parity per D-12)
- If `!_customBrushesAdded`, call `initCustomBrushes()` and set flag

**Core rendering function** -- `renderStyledStrokes(strokes: PaintStroke[], width: number, height: number): OffscreenCanvas | null`:
- Filter out flat and eraser strokes (safety check)
- If no styled strokes, return null
- Call `ensureInitialized(width, height)`
- `brush.clear()`
- `brush.push()`
- `brush.translate(-width / 2, -height / 2)` -- CRITICAL: p5.brush uses center-origin coords
- For each stroke:
  - Determine brush name from STYLE_MAP
  - Apply FX params to modulate brush behavior:
    - If stroke has `fieldStrength > 0.01`: call `brush.field('curved')` then `brush.wiggle(fieldStrength)`
    - If stroke.brushStyle === 'watercolor': set up fill mode with `brush.fill(stroke.color, Math.round(stroke.opacity * 200))`, `brush.fillBleed(bleed ?? 0.6)`, `brush.fillTexture(grain ?? 0.4, 0.4)`, then draw the stroke shape as a filled ellipse/rectangle that follows the stroke path (use `brush.circle()` at averaged stroke position for wash effect), THEN also draw the stroke line with marker brush for the wet-on-wet look
    - For non-watercolor styles: `brush.set(brushName, stroke.color, stroke.size)` then `brush.spline(stroke.points, 0.5)` if points.length >= 2
  - After each stroke: `brush.noField()` (reset field)
- `brush.pop()`
- `brush.render()` -- MANDATORY, flushes compositing
- Return `_offscreen`

**Watercolor rendering detail:**
For watercolor strokes, use a two-part approach:
1. Draw the stroke path with the marker brush for the core wash: `brush.set('marker', stroke.color, stroke.size * 1.2)` then `brush.spline(stroke.points, 0.5)`
2. Add fill bleed along the stroke by drawing a series of filled circles at sampled points along the stroke path: for every Nth point (e.g., every 5th), call `brush.fill(stroke.color, Math.round(stroke.opacity * 80))`, `brush.fillBleed(bleed ?? 0.6)`, `brush.fillTexture(grain ?? 0.4, 0.4)`, `brush.circle(point[0], point[1], stroke.size * 0.8)` then `brush.noFill()`

This creates the watercolor wash + bleed edge effect that p5.brush excels at.

**Edge darkening for ink:**
Ink's edge darkening effect comes naturally from p5.brush's spectral mixing -- overlapping strokes automatically produce darker edges. The `edgeDarken` param can modulate the brush opacity: `brush.set('our_ink', stroke.color, stroke.size)` with opacity scaled by `(1 + (edgeDarken ?? 0.7) * 0.3)`.

**Scatter for charcoal:**
The custom brush definition already has scatter=1.5. The `scatter` FX param can further modulate this by calling `brush.scaleBrushes(1 + (scatter ?? 0.4))` before drawing charcoal strokes, then restoring with `brush.scaleBrushes(1)` after.

**Grain for pencil/charcoal:**
p5.brush's built-in grain parameter on the brush preset handles this. The `grain` FX param value doesn't need additional code -- it's baked into the custom brush definition.

**Dispose function** -- `disposeBrushFx()`:
- Set `_offscreen = null`, `_initialized = false`

IMPORTANT: The exported function signatures MUST match what paintRenderer.ts currently imports:
- `export function renderStyledStrokes(strokes: PaintStroke[], width: number, height: number): OffscreenCanvas | null`
- `export function disposeBrushFx(): void`

Note: paintRenderer.ts currently expects `HTMLCanvasElement | null` from the old renderer. The new adapter returns `OffscreenCanvas | null`. Both are valid sources for `ctx.drawImage()` -- Canvas2D's drawImage accepts CanvasImageSource which includes OffscreenCanvas. This is a safe change but the Plan 20-09 import path switch will handle the type alignment.
