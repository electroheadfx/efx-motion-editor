2. Add a new exported function `renderPaintFrameWithBg()` that wraps the existing `renderPaintFrame()` with solid background rendering and frame-level FX cache compositing. This handles both the solid background per D-11 and the per-frame FX cache:

```typescript
/**
 * Render paint frame with solid background (per D-11) and frame-level FX cache.
 *
 * Rendering order:
 * 1. Solid background fill (paintBgColor)
 * 2. Flat strokes via Canvas2D (strokeToPath)
 * 3. Frame-level FX cache via single drawImage() if available
 *    (all FX strokes were batch-rendered together by p5.brush for spectral mixing)
 * 4. Falls back to renderStyledStrokes() if no frame cache exists yet
 *
 * @param layerId - Optional layer ID for frame cache lookup
 * @param frameNum - Optional frame number for frame cache lookup
 */
export function renderPaintFrameWithBg(
  ctx: CanvasRenderingContext2D,
  frame: PaintFrame,
  width: number,
  height: number,
  layerId?: string,
  frameNum?: number,
): void {
  // Solid paint background per D-11 (default white, user-configurable)
  const bgColor = paintStore.paintBgColor.peek();
  ctx.save();
  ctx.fillStyle = bgColor;
  ctx.fillRect(0, 0, width, height);
  ctx.restore();

  // Check for frame-level FX cache (all FX strokes pre-rendered together)
  let hasFrameCache = false;
  if (layerId !== undefined && frameNum !== undefined) {
    const fxCache = paintStore.getFrameFxCache(layerId, frameNum);
    if (fxCache) {
      // Render flat strokes first (Canvas2D), then overlay the FX cache
      renderFlatElements(ctx, frame, width, height);
      ctx.drawImage(fxCache, 0, 0);
      hasFrameCache = true;
    }
  }

  if (!hasFrameCache) {
    // No frame cache -- render all elements normally (flat via Canvas2D, styled via p5.brush batch)
    renderPaintFrame(ctx, frame, width, height);
  }
}
```

3. Add a new internal function `renderFlatElements()` that renders ONLY the flat/eraser strokes and shapes (skipping FX-applied strokes):

```typescript
/**
 * Render only flat elements (non-FX strokes, shapes, fills, erasers).
 * Used when frame-level FX cache handles all FX-applied strokes separately.
 */
function renderFlatElements(
  ctx: CanvasRenderingContext2D,
  frame: PaintFrame,
  width: number,
  height: number,
): void {
  for (const el of frame.elements) {
    // Skip FX-applied strokes -- they're in the frame cache
    if (el.tool === 'brush') {
      const stroke = el as PaintStroke;
      if (stroke.fxState === 'fx-applied' || stroke.fxState === 'flattened') {
        continue;
      }
    }
    // Render flat strokes, erasers, shapes, fills via existing Canvas2D path
    renderElement(ctx, el);
  }
}
```

Note: `renderElement` is the existing internal function that handles individual element rendering (stroke via strokeToPath, shape via shape rendering, fill via flood fill). If it doesn't exist as a standalone function, extract the per-element rendering logic from `renderPaintFrame`'s loop into one.

Actually, look at the current `renderPaintFrame` structure. It likely iterates elements and handles flat vs styled differently. The `renderFlatElements` function should reuse the same per-element rendering logic but skip FX-applied strokes. Examine the existing code and:
- If `renderPaintFrame` has a simple per-element loop, extract the per-element body into a helper
- If it already uses `flushStyledStrokes`, the flat elements are what gets rendered outside of `flushStyledStrokes`

The key insight: when a frame cache exists, we DON'T need `flushStyledStrokes` at all -- the frame cache replaces it entirely. We only need to render the flat/eraser elements via Canvas2D.
