**Fix removeElement undo closure in `Application/src/stores/paintStore.ts` (lines 145-148):**

The undo closure currently only does `f.elements.splice(idx, 0, removed)` without calling `_notifyVisualChange`. This means `paintVersion` is not bumped after undo, so `StrokeList` (which subscribes to `paintVersion`) does not re-render.

All other element mutations (reorderElements, setElementVisibility, moveElements*) correctly call `_notifyVisualChange` in their undo closures.

**Change the undo closure (lines 145-148):**

Before:
```typescript
undo: () => {
  const f = _getOrCreateFrame(layerId, frame);
  f.elements.splice(idx, 0, removed);
},
```

After:
```typescript
undo: () => {
  const f = _getOrCreateFrame(layerId, frame);
  f.elements.splice(idx, 0, removed);
  _notifyVisualChange(layerId, frame);
  paintStore.invalidateFrameFxCache(layerId, frame);
  paintStore.refreshFrameFx(layerId, frame);
},
```

Note: Match the pattern used by `reorderElements` undo closure (lines 210-215) and `setElementVisibility` undo closure (lines 253-256) which both call `_notifyVisualChange` and `invalidateFrameFxCache` in their undo closures. The `refreshFrameFx` call is also needed to rebuild the FX cache after the undo.
</action>
