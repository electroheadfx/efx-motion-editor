**sequenceStore.ts -- Add sequence-aware layer mutation methods:**

Add two new methods that find the layer's owning sequence rather than using activeSequenceId:

1. `updateLayerInSequence(layerId: string, updates: Partial<Layer>)` -- Iterates all sequences to find which one contains layerId, then applies updates to that layer within its owning sequence. Uses snapshot/restore for undo. If layerId is not found in any sequence, no-op.

2. `removeLayerFromSequence(layerId: string)` -- Iterates all sequences to find which one contains layerId, then filters it out. Uses snapshot/restore for undo. Protects isBase layers. If the removed layer was the only layer in an FX sequence (kind === 'fx'), remove the entire FX sequence (empty FX sequences are meaningless).

Both methods follow the existing pattern: snapshot before, mutate, markDirty, snapshot after, pushAction.

Do NOT modify the existing updateLayer/removeLayer methods (they are used by content layer code that correctly targets the active sequence). The new methods are additions.
