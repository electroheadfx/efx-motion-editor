  <name>Task 1: Extend PreviewRenderer with FX rendering paths and in/out point filtering</name>
  <files>Application/src/lib/previewRenderer.ts</files>
  <action>
Modify `PreviewRenderer.renderFrame()` to support generator layers, adjustment layers, and in/out point temporal clipping.

1. Add imports at top of file:
   ```typescript
   import { isGeneratorLayer, isAdjustmentLayer } from '../types/layer';
   import { drawGrain, drawParticles, drawLines, drawDots, drawVignette } from './fxGenerators';
   import { applyColorGrade } from './fxColorGrade';
   import type { ColorGradeParams } from './fxColorGrade';
   ```

2. In the `renderFrame()` method, modify the layer iteration loop. Currently the loop is:
   ```
   for (const layer of layers) {
     if (!layer.visible) continue;
     const source = this.resolveLayerSource(layer, frame, frames, fps);
     ...
   ```

   Change to a three-path approach:
   ```
   for (const layer of layers) {
     if (!layer.visible) continue;
     // In/out point filtering (new for Phase 7)
     if (layer.inFrame != null && frame < layer.inFrame) continue;
     if (layer.outFrame != null && frame >= layer.outFrame) continue;

     if (isGeneratorLayer(layer)) {
       this.drawGeneratorLayer(layer, logicalW, logicalH, frame);
     } else if (isAdjustmentLayer(layer)) {
       this.drawAdjustmentLayer(layer, logicalW, logicalH);
     } else {
       // Existing content layer path (unchanged)
       const source = this.resolveLayerSource(layer, frame, frames, fps);
       if (source !== null) {
         resolved.push({layer, source});
       } else if (layer.source.type === 'video') {
         loadingVideoLayers.push(layer);
       }
     }
   ```

   IMPORTANT: The current pre-resolve loop builds a `resolved` array, then clears canvas and draws. This needs to change slightly: FX layers cannot be pre-resolved (they draw directly). Restructure to a single-pass draw loop:
   - Keep the canvas clear + DPI scale setup
   - Iterate layers once: for each layer, either drawLayer (content), drawGeneratorLayer, or drawAdjustmentLayer
   - Content layers still need source resolution, but do it inline in the loop
   - Keep the "no drawable sources" optimization: if ALL layers have no sources, keep previous frame. But FX generator layers always "have something" to draw.

   Restructured approach (preserving the flash-prevention logic):
   a. First pass: check if ANY layer will draw something (content layer has source, OR is a generator, OR is an adjustment with content below). If none, return early.
   b. Clear canvas, set DPI scaling
   c. Second pass: draw each layer in order

3. Add `drawGeneratorLayer(layer, logicalW, logicalH, frame)` private method:
   - `ctx.save()`
   - Set blend mode and opacity: `ctx.globalCompositeOperation = blendModeToCompositeOp(layer.blendMode); ctx.globalAlpha = layer.opacity;`
   - Switch on `layer.source.type`:
     - 'generator-grain': call `drawGrain(this.ctx, logicalW, logicalH, layer.source, frame)`
     - 'generator-particles': call `drawParticles(this.ctx, logicalW, logicalH, layer.source, frame)`
     - 'generator-lines': call `drawLines(this.ctx, logicalW, logicalH, layer.source, frame)`
     - 'generator-dots': call `drawDots(this.ctx, logicalW, logicalH, layer.source, frame)`
     - 'generator-vignette': call `drawVignette(this.ctx, logicalW, logicalH, layer.source)`
   - `ctx.restore()`

4. Add `drawAdjustmentLayer(layer, logicalW, logicalH)` private method:
   - For 'adjustment-color-grade':
     - The color grade needs to read ALL pixels currently on canvas (below this layer) and modify them
     - Since ctx has DPI scaling active (ctx.scale(dpr, dpr)), we must work in physical pixel coords for ImageData
     - `ctx.save()`; `ctx.setTransform(1, 0, 0, 1, 0, 0)` to reset to identity transform
     - Call `applyColorGrade(this.ctx, this.canvas.width, this.canvas.height, layer.source as ColorGradeParams)`
     - WAIT -- the layer also has opacity. If opacity < 1, the color grade should be partially applied. Approach: read pixels, apply full grade to a copy, then blend copy with original using layer.opacity as factor. OR: simpler -- scale all parameter values by opacity before passing. Choose the simpler approach: multiply brightness/contrast/fade by opacity, and interpolate saturation toward 1 by opacity (sat = 1 + (sat-1)*opacity), hue toward 0 by opacity.
     - Actually the cleanest approach: apply color grade at full strength, but only to a fraction determined by opacity. This means: getImageData original, clone it, apply grade to clone, then for each pixel blend between original and graded by opacity. This is correct but doubles the pixel work.
     - Simplest correct approach: just multiply the effect parameters by opacity. brightness *= opacity, contrast *= opacity, saturation = 1 + (saturation - 1) * opacity, hue *= opacity, fade *= opacity. This is an approximation but visually identical for typical values.
     - `ctx.restore()` to re-apply DPI scaling for subsequent layers

Note on "resolved" array refactoring: The current code builds `resolved[]` then draws. The refactor should process layers in a single draw pass. If a content layer has no source yet (async loading), skip it. If it's a video loading, show placeholder. Generators always draw. Adjustments operate on current canvas state.

Keep the flash-prevention: track whether anything was drawn. If nothing at all was drawn (no resolved content, no generators, no adjustments), return without clearing.
