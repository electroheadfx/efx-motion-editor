## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge internal padding, icon gaps |
| sm | 8px | Grid gap between thumbnails, popover item padding |
| md | 16px | Popover body padding, section spacing |
| lg | 24px | Popover max-width internal margin |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Badge minimum width is 18px (not a multiple of 4) to ensure single-digit counts remain legible at 9px font size. This matches the established SpeedBadge pattern.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 12px | 400 (regular) | 1.5 |
| Label | 9px | 600 (semibold) | 1.0 |
| Heading | 11px | 600 (semibold) | 1.2 |
| Badge count | 9px | 600 (semibold) | 1.0 |

Notes:
- Body (12px) is used for popover usage location text (e.g., "Seq 1 > Key Photo #3").
- Label (9px) matches existing ImportGrid section labels ("Videos", "Audio") and filename overlays (8px hover text rounds up to 9px for badge).
- Heading (11px) is used for popover section header ("Used in N locations").
- Badge count (9px semibold) displays the numeric usage count inside the colored circle. Semibold (600) renders visually distinct from regular at this size.
- All sizes align with existing codebase patterns: ImportGrid uses 8-10px for labels, 12px for list items.

## Interaction States

### Badge States

| State | Visual |
|-------|--------|
| Default | Colored pill with count number |
| Hover | Cursor pointer, slight brightness increase (filter: brightness(1.15)) |
| Active/Pressed | No visual change (opens popover immediately) |
| Popover open | Badge remains visible, popover appears adjacent |

### Popover States

| State | Visual |
|-------|--------|
| Empty (0 uses) | Header: "Not used anywhere", no location list, only "Remove Reference" and "Delete File" buttons |
| With locations | Full layout as described above |
| Many locations (>5) | Location list scrollable at max-height 120px |

### Removal Flows (from CONTEXT.md D-08 through D-13)

#### In-use asset -- "Remove Reference"
1. User clicks badge or right-click > "Remove Reference"
2. Native `confirm()` with message: `"[filename]" is used in N location(s):\n\n- Seq 1 > Key Photo #3\n- Seq 2 > Layer 'overlay'\n\nRemove all references and delete from library?`
3. On confirm: cascade removal (key photos become placeholders, layers removed, audio tracks removed)
4. On cancel: no action, popover/menu dismissed

#### In-use asset -- "Delete File"
1. User clicks "Delete File" in popover or context menu
2. First native `confirm()`: `"[filename]" is used in N location(s):\n\n- [locations]\n\nThis will remove all references first.`
3. On confirm: cascade removal executes
4. Second native `confirm()`: `Permanently delete "[filename]" from disk? This cannot be undone.`
5. On confirm: file deleted from disk
6. On cancel at either step: no further action

#### Unused asset -- "Remove Reference"
1. Native `confirm()`: `Remove "[filename]" from library?`
2. On confirm: asset removed from store

#### Unused asset -- "Delete File"
1. Native `confirm()`: `Delete "[filename]" from disk? This cannot be undone.`
2. On confirm: asset removed from store + file deleted from disk

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Remove Reference" (popover button) |
| Secondary CTA | "Delete File" (popover button, destructive) |
| Badge tooltip | "Used {N} time(s)" |
| Popover header (in use) | "Used in {N} location(s)" |
| Popover header (unused) | "Not used anywhere" |
| Location format (key photo) | "{SeqName} > Key Photo #{index}" |
| Location format (layer) | "{SeqName} > Layer '{layerName}'" |
| Location format (audio) | "Timeline > Track '{trackName}'" |
| Confirm: remove in-use | "{filename}" is used in {N} location(s):\n\n{location list}\n\nRemove all references and delete from library? |
| Confirm: delete in-use (step 1) | "{filename}" is used in {N} location(s):\n\n{location list}\n\nThis will remove all references first. |
| Confirm: delete in-use (step 2) | Permanently delete "{filename}" from disk? This cannot be undone. |
| Confirm: remove unused | Remove "{filename}" from library? |
| Confirm: delete unused | Delete "{filename}" from disk? This cannot be undone. |
| Empty state heading | Not applicable (ImportGrid already has empty states) |
| Empty state body | Not applicable |
| Error state | Not applicable (removal uses native confirm, no custom error UI) |
