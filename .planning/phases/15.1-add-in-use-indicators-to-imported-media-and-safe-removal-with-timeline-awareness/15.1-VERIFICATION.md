#### 3. Popover interaction flow

**Test:** Click a badge on an in-use asset.
**Expected:** Popover appears at click position (clamped to viewport), shows header "Used in N location(s)", lists each location as "SequenceName > detail string", has "Remove Reference" and "Delete File" buttons. Click outside dismisses it.
**Why human:** Portal rendering position, content accuracy, and dismiss behavior require runtime interaction.

#### 4. Cascade removal with undo

**Test:** Click badge on image used as key photo, click "Remove Reference", confirm the dialog.
**Expected:** Asset removed from library, key photo slots show placeholder. Press Ctrl+Z: everything restored in one undo step.
**Why human:** Runtime store mutations and undo stack behavior require the running app.

#### 5. Two-step delete confirmation flow

**Test:** Click badge on in-use asset, click "Delete File".
**Expected:** First dialog warns about reference removal (lists locations). Second dialog warns about permanent disk deletion. Both must be accepted for file to be deleted. Cancelling second still removes references.
**Why human:** Sequential dialog flow and file system interaction require runtime execution.

#### 6. v8 project backward compatibility
