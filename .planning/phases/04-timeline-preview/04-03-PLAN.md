<task type="auto">
  <name>Task 2: Add sequence reorder drag-and-drop on timeline track headers</name>
  <files>Application/src/components/timeline/TimelineRenderer.ts, Application/src/components/timeline/TimelineInteraction.ts</files>
  <action>
Add drag-and-drop capability to the timeline track headers for reordering sequences (TIME-06).

**Approach:** Since canvas doesn't have native drag events for drawn items, implement drag state manually in TimelineInteraction and visual feedback in TimelineRenderer.

**1. Update `TimelineInteraction.ts`:**

Add track header drag detection and handling:

**Hit testing for track headers:**
- On `mousedown`: check if click x-coordinate is within the track header column (x < TRACK_HEADER_WIDTH)
- If so, compute which track was clicked: `trackIndex = Math.floor((clientY - canvasRect.top - RULER_HEIGHT) / TRACK_HEIGHT)` (accounting for ruler height at top)
- Validate trackIndex is within bounds of `trackLayouts.peek().length`
- Start drag: `isDraggingTrack = true`, `dragTrackIndex = trackIndex`, `dragCurrentY = clientY`

**During drag:**
- On `mousemove` while `isDraggingTrack`:
  - Update `dragCurrentY`
  - Compute `dropIndex` from current Y position
  - Call renderer's `setDragState({ fromIndex, toIndex, currentY })` to show visual feedback
  - Change cursor to `grabbing`

**On drop:**
- On `mouseup` while `isDraggingTrack`:
  - Compute final `dropIndex`
  - If `dropIndex !== dragTrackIndex`: call `sequenceStore.reorderSequences(dragTrackIndex, dropIndex)`
  - Reset drag state: `isDraggingTrack = false`
  - Call renderer's `setDragState(null)` to clear visual feedback
  - Change cursor back to default

Import `sequenceStore` from `../../stores/sequenceStore`.
Import `trackLayouts` from `../../lib/frameMap` for track count.
Import `RULER_HEIGHT, TRACK_HEIGHT, TRACK_HEADER_WIDTH` from TimelineRenderer.

**2. Update `TimelineRenderer.ts`:**

Add drag visual feedback:

- Add `setDragState(state: DragState | null)` method and `private dragState` property
- Define `DragState = { fromIndex: number; toIndex: number; currentY: number } | null`
- In `draw()`, after drawing all tracks:
  - If `dragState` is not null:
    - Draw a horizontal blue line at the drop position (between tracks) to indicate where the dragged track will be inserted
    - Draw the dragged track row with slight transparency/offset at the current drag Y position (visual ghost)
    - The drop indicator line should be 2px high, full width, colored `#4488FF` (blue accent)

Export `RULER_HEIGHT` constant (if not already exported) so TimelineInteraction can use it for hit testing.

**Edge cases:**
- If there's only one sequence, drag should be a no-op (or simply not start)
- Drop at the same position as origin should be a no-op (don't call reorderSequences)
- Cursor should show `grab` when hovering over track headers, `grabbing` during drag
