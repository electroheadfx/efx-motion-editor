## Top 3 Priority Fixes

1. **Replace generic Roto save error copy with the UI-SPEC exact error** — users currently see “Could not apply physics paint output...” instead of Roto-specific recovery guidance — change all Phase 36.3 Roto save-failure paths in `PhysicsPaintStudio.tsx` to `Could not save the current Roto frame. Keep Physics Paint open, check the selected layer and frame, then click Save current again.`
2. **Remove broad timeline/interpolation remnants from the rendered Roto strip** — the contract requires a walking-skeleton surface, but `PhysicsPaintWorkflowStrip.tsx` still renders the timeline/ruler and interpolation connector elements — gate Phase 36.3 Roto to current-frame cell/navigation, `Save current`, and compact status/reference copy only.
3. **Normalize Phase 36.3 workflow CSS to the declared tokens** — off-scale typography/spacing/color weakens spec fidelity — update workflow-strip CSS to 11/13/16/20px and 400/600 only for touched UI, use 4/8/16px spacing rhythm, and use `var(--color-accent)` only for `Save current`/focus/active frame semantics.

## Detailed Findings

### Pillar 1: Copywriting (2/4)

- **WARNING:** Required primary and reference copy is present: `Save current` in `app/src/components/physic-paint/PhysicsPaintWorkflowStrip.tsx:343-345`, `Cached reference` and `Cached reference: repaintable, not stroke-editable.` in `PhysicsPaintWorkflowStrip.tsx:421-422`, and unsaved/save-in-flight copy in `app/src/components/physic-paint/physicsPaintWorkflowState.ts:116-118`.
- **WARNING:** Required exact Roto save error copy is absent. Grep found no `Could not save the current Roto frame...` string. Instead `PhysicsPaintStudio.tsx:1194`, `1239`, `1297`, `1533`, `1824`, and `1886` use generic `Could not apply physics paint output...`, which does not match the UI-SPEC recovery instructions.
- **WARNING:** Required empty-state copy is absent. Grep found no `No cached Roto reference` or `Paint this frame, then click Save current to cache it in EFX Motion.` in `app/src`.
- **WARNING:** Editor-side replacement warning diverges from UI-SPEC. `app/src/components/sidebar/PhysicPaintProperties.tsx:179` says `This frame already has physics paint output. Applying will replace it.` but the contract copy is `This frame already has physics paint output. Saving current will replace this frame only.`

### Pillar 2: Visuals (2/4)

- **WARNING:** The visible `Save current` action is a real text button with an accessible name at `PhysicsPaintWorkflowStrip.tsx:343-345`, satisfying the primary-action discoverability requirement.
- **WARNING:** The Roto workflow strip still renders a full timeline/ruler surface at `PhysicsPaintWorkflowStrip.tsx:352-414`, while the UI-SPEC says Phase 36.3 should show only minimal current-frame cell/navigation and compact status/reference copy, not a final Pencil timeline UI.
- **WARNING:** Interpolation connector elements are still computed and rendered in Roto mode at `PhysicsPaintWorkflowStrip.tsx:163` and `376-385`. Even if controls are not visible, generated/interpolation visuals remain in the rendered Roto strip surface, conflicting with the hidden-controls contract.
- **WARNING:** Broad conversion/destructive confirmation code remains wired in the component (`PhysicsPaintWorkflowStrip.tsx:293-299`, `444-455`) and is still passed from `PhysicsPaintStudio.tsx:2238-2239`. The modal is conditional, but Phase 36.3 requires no destructive confirmation surface in the Roto slice.

### Pillar 3: Color (2/4)

- **WARNING:** The Studio shell preserves the required dark editor palette: root/workflow backgrounds use `#0f0f0f`, `#3e3f41`, and `#343638` in `physicsPaintStudio.css:1035-1045` and related shell rules.
- **WARNING:** Accent is over-broadened beyond the declared `#2D5BE3`/`var(--color-accent)`. Workflow CSS uses `#6f90ff` for range controls and active states (`physicsPaintStudio.css:1158`, `1280`, `1311`) and `#365ed6` (`physicsPaintStudio.css:1312`), creating a second blue accent in the Phase 36.3 surface.
- **WARNING:** Hardcoded warning/destructive colors remain in touched surfaces: `#f59e0b` in `physicsPaintStudio.css:1419` and `PhysicPaintProperties.tsx:178/266`, plus hardcoded palette colors in Physics Paint components. The spec prefers semantic variables where available and reserves yellow for explicit unsaved/warning states.
- **WARNING:** The `Save current` active button uses `color-mix(... var(--color-accent) 42% ...)` in `physicsPaintStudio.css:1374-1379`; this is close to the contract but visually less decisive than the reserved primary accent role for the only Phase 36.3 CTA.

### Pillar 4: Typography (2/4)

- **WARNING:** The contract allows new/modified Phase 36.3 UI sizes 11px, 13px, 16px, 20px and weights 400/600. The workflow CSS includes out-of-contract values: 10px (`physicsPaintStudio.css:1163`), 12px (`1120`, `1325`, `1439`), 18px (`1179`), weight 700 (`1121`, `1139`, `1180`, `1203`, etc.), and weight 500 (`1346`).
- **WARNING:** `Save current` uses 13px/600 via `.physics-paint-render-action` (`physicsPaintStudio.css:1382-1385`), matching the action-button rule.
- **WARNING:** Compact reference/status copy is 12px in `physicsPaintStudio.css:1434-1440`, not the contract's 13px/400 explanatory text or 11px/600 compact label pattern.

### Pillar 5: Spacing (2/4)

- **WARNING:** The required existing structural dimensions are preserved: workflow strip min-height 146px (`physicsPaintStudio.css:1041`), top workflow row height 42px (`1075-1076`), current frame cell 48px by 26px (`1131-1133`), and 24px nav button width (`1172-1174`).
- **WARNING:** Several modified workflow-strip spacings are not multiples of 4 despite the declared scale: `gap: 6px` (`1040`, `1055`, `1118`, `1407`), `gap: 10px` (`1088`), `gap: 5px` (`1149`, `1256`), `padding: 5px 10px` (`1320`), and `gap: 1px`/negative margin (`1430-1431`).
- **WARNING:** The visible Roto strip retains a wide timeline (`rulerWidth = 1800` in `PhysicsPaintWorkflowStrip.tsx:181` and timeline surface at `352-414`) that creates a large scrolled region instead of the compact walking-skeleton layout requested for Phase 36.3.

### Pillar 6: Experience Design (3/4)

- **WARNING:** `Save current` correctly disables when not ready, when no unsaved Roto frames exist, or while save is in flight (`PhysicsPaintWorkflowStrip.tsx:343`), and in-flight copy is derived from `physicsPaintWorkflowState.ts:116-118`.
- **WARNING:** Successful save remains on the same frame when `saveRotoFrame(null)` is used (`PhysicsPaintStudio.tsx:2223`) and success copy `Saved current frame` is set when no advance target exists (`PhysicsPaintStudio.tsx:1329-1335`).
- **WARNING:** Save error handling exists, but the user-facing recovery copy is generic and not Roto-specific (`PhysicsPaintStudio.tsx:1194`, `1239`, `1297`), so failed-save UX partially misses the contract.
- **WARNING:** Human UAT approved Save current, preview, save/load, cached-reference reopen, navigation preservation, and saved-cache integrity, but `36.3-02-SUMMARY.md:75` records direct close behavior as not validated and deferred. This prevents a 4/4 on the close/discard interaction portion of the UI contract.
