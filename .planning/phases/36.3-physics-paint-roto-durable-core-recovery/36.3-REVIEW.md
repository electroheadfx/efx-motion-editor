## Summary

Reviewed the Phase 36.3 Physics Paint durable Roto cache changes across the listed studio, workflow, persistence, store, type, and test files. The implementation contains correctness and security defects around deleting durable Roto cache entries, trusting persisted cache paths, and classifying Play frames as real Roto keys.

## Narrative Findings (AI reviewer)

## Critical Issues

### CR-01: Clearing a saved Roto frame never deletes the durable cached frame

**File:** `app/src/components/physic-paint/PhysicsPaintStudio.tsx:994-1008`, `app/src/components/physic-paint/PhysicsPaintStudio.tsx:1173-1177`

**Issue:** The Roto clear path only clears local standalone state and marks the frame dirty. When the user then clicks Save current, `snapshotCurrentRotoFrame()` sees an empty/transparent frame and removes local snapshots, and `flushRotoFrame()` returns `null` for non-persistable state without sending any delete/update payload to the parent app. The old frame remains in `physicPaintStore` and on disk, so reopening the project resurrects a frame the user explicitly cleared.

**Fix:** Represent deletion as a durable operation. For example, send a delete payload or call a store method that removes the real-key cache and metadata for the frame, then mark the project dirty. A minimal store-side shape would be:

```ts
if (!editableState || !shouldPersistRotoFrame(editableState)) {
  await sendPhysicPaintApplyPayload({
    operationId,
    kind: 'delete-roto-frame',
    layerId: launchContext.layerId,
    startFrame: frame,
  }, bridgeMode);
  dirtyRotoFramesRef.current.delete(frame);
  syncPendingRotoFrames();
  return null;
}
```

Alternatively, add an explicit validated `removeRealRotoKeyFrame(layerId, frame)` bridge operation and invoke it from the clear/save path before reporting success.

### CR-02: Persisted cache paths are trusted, enabling path traversal reads

**File:** `app/src/lib/physicPaintPersistence.ts:114-119`

**Issue:** `loadPhysicPaintData()` reads `${projectDir}/${frame.cache_path}` directly from the project file. A crafted `.mce` can set `cache_path` to an absolute path or a `../` traversal path and cause the app to read arbitrary files accessible to the Tauri filesystem permissions, then wrap those bytes as a PNG data URL. This is a path traversal/security vulnerability in project loading.

**Fix:** Validate persisted cache paths before reading. Only accept relative paths under `cache/physic-paint/`, reject absolute paths, backslashes, empty segments, and `..` segments, and construct the final path from validated segments.

```ts
function isSafePhysicPaintCachePath(cachePath: unknown): cachePath is string {
  if (typeof cachePath !== 'string') return false;
  if (!cachePath.startsWith(`${PHYSIC_PAINT_CACHE_DIR}/`)) return false;
  if (cachePath.includes('\\')) return false;
  return cachePath.split('/').every((segment) => segment.length > 0 && segment !== '.' && segment !== '..');
}

// before readFile
if (!isSafePhysicPaintCachePath(frame.cache_path)) continue;
const bytes = await readFile(`${projectDir}/${frame.cache_path}`);
```

### CR-03: Play frames are treated as real Roto keys during interpolation

**File:** `app/src/stores/physicPaintStore.ts:174-179`

**Issue:** `_getRealRotoKeyFrames()` derives real Roto keys from `_frames` by accepting every frame whose `source` is not `'generated-interpolation'`. Cached Play output frames are stored in the same `_frames` map and typically have no `source`, so they are incorrectly classified as real Roto keys. Any Roto interpolation or generated-cache regeneration on a layer that also has Play output can create generated Roto frames from Play frames, corrupting the Roto cache and launch context.

**Fix:** Derive real Roto keys from `_rotoCacheMetadata` entries with `source === 'real-key'`, not from all rendered frames. Fall back only for explicit legacy/hydrated Roto outputs, not for arbitrary Play frames.

```ts
function _getRealRotoKeyFrames(layerId: string): number[] {
  const metadata = _rotoCacheMetadata.get(layerId);
  if (!metadata) return [];
  return Array.from(metadata.values())
    .filter((frame) => frame.source === 'real-key')
    .map((frame) => frame.appFrame)
    .sort((a, b) => a - b);
}
```
