# Phase 36.3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Boundary

Phase 36.3 is a minimal recovery UI slice for one durable Physics Paint Roto cache frame. It must preserve the existing Physics Paint Studio visual shell and narrow the visible Roto workflow surface to the walking skeleton only.

| Area | Contract |
|------|----------|
| Editor entry point | Existing physics-paint layer properties open action remains the route into Roto mode for the current layer/frame. |
| Standalone mode | Open directly in Roto mode; do not expose Roto/Play mode switching tabs for this recovery slice. |
| Workflow strip | Reuse `PhysicsPaintWorkflowStrip` in minimal Roto mode: mode label, current frame cell/navigation, `Save current`, compact saved/reference/unsaved status. |
| Hidden controls | Hide interpolation, in-between count, interpolation mode, Deform/Position, Duplicate/Insert/Delete/Copy/Paste, cached Roto playback, Play/Roto conversion, generated-frame connector UI, missing-background controls, and final Pencil timeline UI. |
| Durable truth | Cached PNG is the durable output. Reopened cache is a visual reference/background only; do not imply editable stroke restore. |
| Close behavior | Closing never auto-saves, never blocks, and never prompts in Phase 36.3. Unsaved edits after the last save are discarded. |

Sources: `36.3-CONTEXT.md` D-01 through D-15, `.planning/ROADMAP.md` Phase 36.3 success criteria and exclusions.

## Typography

Use exactly these four sizes and two weights for new or modified Phase 36.3 UI copy.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 600 | 1.2 |
| Heading | 16px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

Typography rules:

- Do not introduce additional font sizes in the Phase 36.3 surface.
- Use 11px/600 uppercase only for compact labels already following the Physics Paint style.
- Use 13px/400 for explanatory status/error text.
- Use 13px/600 for action buttons when matching existing workflow buttons.
- Use 16px/600 only for dialog headings or top-level panel headings if an existing heading is touched.
- Do not add a new large marketing-style title; this is an editor tool surface, not a landing page.

## Color

Use existing CSS variables and established Physics Paint color semantics. Hex values below are the dark-theme canonical values; theme-aware code should use the named variables when available.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F0F` / `var(--color-bg-root)` | Root background, canvas-adjacent dark surfaces, non-interactive surrounding shell |
| Secondary (30%) | `#3E3F41` and `#343638` | Physics Paint top bar, right panel, workflow strip, cards, compact control wells |
| Accent (10%) | `#2D5BE3` / `var(--color-accent)` | `Save current`, active focus ring, active Roto frame affordance only when blue is already the component state color |
| Destructive | `#FF6666` / `var(--color-error-text)` | Destructive actions only; Phase 36.3 should not surface new destructive actions |

Accent reserved for: `Save current` primary action, focus-visible outlines, active selected controls that already use `var(--color-accent)`, and success-related saved-cache emphasis only when paired with the established green saved/reference token.

Additional semantic colors:

| Semantic | Value | Usage |
|----------|-------|-------|
| Saved/reference | `#4CAF70` / `var(--sidebar-dot-green)` | Saved cache availability, cached Roto reference outline/status, successful save message |
| Unsaved/warning | `#F59E0B` | Unsaved changes status, current-frame outline, non-blocking warning copy |
| Error | `#FF6666` / `var(--color-error-text)` | Failed save/apply and bridge errors only |

Color rules:

- Cached reference must remain visually strong: full-opacity image reference with outline/status treatment; do not dim cached PNG opacity.
- Current Roto frame keeps the existing orange outline treatment.
- Empty/unpainted cells remain neutral gray.
- Saved/cached cells use green; editable-session/unsaved cells may use existing pink only if the implementation already classifies them, but Phase 36.3 copy must explain saving through `Save current` rather than relying on pink/green alone.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Save current |
| Empty state heading | No cached Roto reference |
| Empty state body | Paint this frame, then click Save current to cache it in EFX Motion. |
| Error state | Could not save the current Roto frame. Keep Physics Paint open, check the selected layer and frame, then click Save current again. |
| Destructive confirmation | None in Phase 36.3. Closing with unsaved edits is non-blocking and has no prompt; copy must warn before close, not confirm on close. |

Required status copy:

| State | Copy | Placement |
|-------|------|-----------|
| Unsaved edits present | Unsaved changes — click Save current to cache | Compact workflow strip status, visible before close |
| Save in flight | Saving current frame… | Replace or sit beside the primary CTA while disabled |
| Save succeeded | Saved current frame | Compact workflow strip status and/or transient status line |
| Reopened cached frame | Cached reference | Canvas/reference label and workflow strip status |
| Reopened cached frame helper | Cached reference: repaintable, not stroke-editable. | Compact workflow strip helper copy |
| Closed/discarded unsaved edits | No modal copy. On reopen show only the last saved `Cached reference`. | Behavior contract, not a prompt |
| Editor-side output available | Physics paint output is available for this layer. | Preserve existing sidebar copy if touched |
| Editor-side replace warning | This frame already has physics paint output. Saving current will replace this frame only. | Sidebar or workflow strip if replacement warning is shown |

Copy rules:

- Always spell the primary action exactly `Save current`; do not use `Save pending`, `Save all`, `Apply`, `Publish`, or `Save and advance` for Roto in this phase.
- Do not use auto-save language.
- Do not promise editable stroke persistence after reopen.
- Use `Cached reference`, not `Cached key`, `Generated`, or `Editable key`, for reopened PNG-only frames.
- Keep status copy compact and direct; the user is in a production tool while painting.

## Interaction Contract

| Interaction | Required behavior |
|-------------|-------------------|
| Open Roto | Editor opens Physics Paint in Roto mode for the selected physics-paint layer and current app frame. If cached PNG exists for that layer/frame, it appears as a reference. |
| Paint | User can paint normally on the live physics canvas. Painting creates unsaved state for the current frame. |
| Save current | Saves only the current Roto frame to the parent EFX Motion cache, for the exact source layer and app frame. Button is disabled while save is in flight. |
| After save success | Stay on the same frame; do not auto-advance. Show `Saved current frame` and saved/reference state. Parent preview must read and draw the cached PNG. |
| Reopen cached frame | Display cached PNG as a full-strength visual reference/background and label it `Cached reference`. New strokes are new edits over/with the reference, not edits to restored strokes. |
| Unsaved edits after save | Show `Unsaved changes — click Save current to cache`. Closing discards these edits without prompt and preserves the last saved cached PNG. |
| Close | Never auto-save, never block, never prompt. Existing saved cache remains intact. |
| Project save/load | Cache availability must survive save/load and reopen with the same `Cached reference` state. |
| Navigation | Minimal current-frame navigation may remain if already present. Navigation must not trigger save-on-leave in Phase 36.3. |

Disabled and loading states:

- Disable `Save current` when the engine is not ready, a save is in flight, or there are no unsaved changes to cache.
- During save-in-flight, show `Saving current frame…` and prevent duplicate save clicks.
- If save fails, keep unsaved edits visible and show the error copy from the copywriting contract.

Accessibility and input:

- `Save current` must be a real button with accessible name `Save current`.
- Cached reference status must be exposed in visible text, not only color.
- Unsaved state must be exposed in visible text, not only orange/pink cell styling.
- Focus-visible outlines remain `2px solid var(--color-accent)` with 2px offset per existing CSS.
- Do not add new icon-only controls; any visible new control must include text.
