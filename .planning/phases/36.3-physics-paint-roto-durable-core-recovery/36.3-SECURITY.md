# Phase 36.3 — Security

> Threat mitigation verification for the strict durable Physics Paint Roto cache MVP.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Standalone Physics Paint window → EFX Motion parent store | Rendered apply payload and editable state cross from child/editor surface into trusted project state. | Physics Paint apply payload, layer id, frame id, rendered PNG data URL |
| Project file/cache → runtime hydration | Saved `.mce` physics-paint outputs and PNG cache paths hydrate back through `loadPhysicPaintData` into runtime stores. | Local project metadata and local cache PNG paths/data |
| Runtime store → preview renderer | Cached PNG data URLs are decoded/rendered by browser image APIs in the preview canvas. | Local runtime `data:image/png` cache frame |
| Human-operated app → phase completion state | Human UAT result determines whether the phase can be considered complete. | User approval/failure evidence |
| Runtime behavior → planning summary | Observed UI/cache behavior is recorded into `.planning` as verification evidence. | Manual UAT notes and completion status |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-36.3-01 | Tampering | `applyPhysicPaintPayload` / `physicPaintStore.applyCanvas` | mitigate | Payload validation requires the existing apply-canvas shape and a known physics-paint layer/source; store apply writes only `payload.startFrame` with one applied frame. Evidence: `app/src/lib/physicPaintBridge.ts`, `app/src/stores/physicPaintStore.ts`, and `app/src/lib/physicPaintRotoDurableCore.test.ts`. | closed |
| T-36.3-02 | Spoofing | `createPhysicPaintLaunchContext` | mitigate | Launch context derives layer identity from the selected physics-paint layer source and exposes cached Roto references from store-backed real-key frames rather than arbitrary UI-provided reference lists. Evidence: `app/src/lib/physicPaintBridge.ts` and durable-core test reopen assertions. | closed |
| T-36.3-03 | Information Disclosure | `previewRenderer` image source | accept | Accepted local-only exposure: Phase 36.3 uses existing local runtime `data:image/png` cache data already stored in project outputs; no network fetch or external URL is introduced. Evidence: `app/src/lib/previewRenderer.ts` and project cache persistence coverage. | closed |
| T-36.3-04 | Denial of Service | Duplicate `Save current` clicks | mitigate | `Save current` is disabled while save/apply is in flight and operation-id handling is preserved for apply results. Evidence: `app/src/components/physic-paint/PhysicsPaintWorkflowStrip.tsx`, `app/src/components/physic-paint/PhysicsPaintStudio.tsx`, and durable-core test payload-count assertion. | closed |
| T-36.3-05 | Repudiation | RED/GREEN behavior proof | mitigate | Git history records RED before GREEN: `ab3c8101 test(36.3-01): add failing test for durable roto cache core` precedes `52677c0a feat(36.3-01): implement durable roto cache core`; summary records passing verification. | closed |
| T-36.3-06 | Repudiation | Manual UAT checkpoint | mitigate | User-run approval and the explicit close-behavior limitation are recorded in `36.3-02-SUMMARY.md` and the completed `36.3-UAT.md` shows 6/6 checkpoints passed. | closed |
| T-36.3-07 | Tampering | Phase scope during UAT | mitigate | Plan and UAT scope are limited to strict Phase 36.3 durable-core behavior and explicitly exclude deferred interpolation, generated frames, key utilities, cached playback, Play/Roto conversion, final timeline UI, and close-behavior follow-up scope. | closed |
| T-36.3-08 | Denial of Service | Agent starting duplicate dev server | mitigate | Plan forbids agent server/app commands and summary records the user owned the app run; this session did not start the dev server. | closed |
| T-36.3-SC | Tampering | npm/pip/cargo installs | mitigate | Phase plans and summaries declare no added packages; no package manifest or lockfile changes were introduced for Phase 36.3. | closed |
