# Phase 17.1: Adaptative sidebar's layer window - Context

**Gathered:** 2026-03-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the standalone LAYERS panel in the sidebar with an adaptive view inside the SEQUENCES panel. The sidebar changes from 3 panels (Sequences, Layers, Properties) to 2 panels (Sequences/Layers adaptive + Properties). Users click a layer icon button on a sequence header to switch the panel from the sequence list to that sequence's layer view, with a back button to return.

</domain>

<decisions>
## Implementation Decisions

### Adaptive behavior
- **D-01:** The standalone LAYERS panel (middle section) is removed entirely — sidebar becomes a 2-panel layout with one resizer
- **D-02:** A layer icon button is added to each content sequence header row, near the existing ellipsis button
- **D-03:** Clicking the layer icon replaces the entire SEQUENCES panel content with a layer view for that sequence
- **D-04:** The layer view shows a compact sequence header (thumbnail, name, key count) at the top, followed by the layer list below
- **D-05:** The panel header changes to "← [Sequence Name]" with a back arrow on the left and the AddLayerMenu button on the right
- **D-06:** Clicking the back arrow returns to the sequence list view

### Layer list behavior
- **D-07:** The existing LayerList component is reused as-is — flat list, topmost layer first, base layer locked at bottom, drag reorder via SortableJS
- **D-08:** The AddLayerMenu button (currently in the LAYERS panel header) is preserved in the adaptive layer view header

### Selection routing
- **D-09:** Clicking a layer in the adaptive view behaves identically to the current standalone panel — selects in layerStore, shows properties in PROPERTIES panel below
- **D-10:** No change to the PROPERTIES panel priority routing (transition > audio > FX > content > fallback)
- **D-11:** When the user switches to a different sequence (clicks another sequence header), the panel auto-closes the layer view and returns to the sequence list

### Panel layout
- **D-12:** The PanelResizer between SEQUENCES and LAYERS panels is removed (only one resizer remains, between the adaptive panel and PROPERTIES)
- **D-13:** The flex-based sizing system simplifies from 3-panel to 2-panel (seqFlex + propFlex, no more layFlex)

### Claude's Discretion
- Layer icon choice (Lucide icon) and placement relative to ellipsis button
- Transition animation between sequence list and layer view (if any)
- How the compact sequence header row looks in the layer view
- Whether the panel collapse/expand behavior needs adjustment for the 2-panel layout
- App config migration for persisted panel flex values (3-panel → 2-panel)

</decisions>

## Existing Code Insights

### Reusable Assets
- `LayerList` component: Fully functional with SortableJS drag reorder, visibility toggles, delete buttons — reuse as-is
- `AddLayerMenu` component: Dropdown for adding layers — just needs to move to adaptive header
- `CollapsibleSection`: Panel header with title + actions — needs a "back arrow" variant
- `SidebarScrollArea`: Scroll container — reusable for both sequence list and layer view

### Established Patterns
- Preact Signals for all UI state — layer view open/closed should be a signal (e.g., `layerViewSequenceId: Signal<string | null>`)
- `maxHeight` CSS transition used for key photo collapse — could inform view switching animation
- Portal rendering used for context menus — no portal needed here, simple conditional render

### Integration Points
- `SequenceItem` header row: Add layer icon button near ellipsis
- `LeftPanel`: Remove middle LAYERS panel, restructure to 2-panel flex layout
- `uiStore`: Add `layerViewSequenceId` signal, remove `layPanelFlex` and `layersSectionCollapsed`
- `handleSelect` in SequenceItem: When switching sequences, reset `layerViewSequenceId` to null (auto-close layer view)

</code_context>
