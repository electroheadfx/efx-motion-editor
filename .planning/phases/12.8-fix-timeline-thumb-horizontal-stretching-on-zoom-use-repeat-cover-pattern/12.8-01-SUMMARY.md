**Tile-pattern thumbnail rendering with natural aspect ratio, horizontal repeat, and center-crop for narrow frames**

## Accomplishments
- Replaced stretched drawImage with createPattern-based tile rendering that preserves natural image aspect ratio at any zoom level
- Added center-crop behavior when frame cell is narrower than one tile width (shows middle of image, not left edge)
- Implemented low-zoom solid color fallback when frame width drops below 4px threshold

## Task Commits

1. **Task 1: Replace stretched drawImage with tile-pattern rendering** - `137d975` (feat)

## Files Created/Modified
- `Application/src/components/timeline/TimelineRenderer.ts` - Tile-pattern thumbnail rendering with center-crop offset for narrow frames, createPattern per range, MIN_FRAME_WIDTH_FOR_THUMB fallback

## Decisions Made
- Scale factor derived from cell height divided by naturalHeight, making tile height exactly equal to cell height with natural width preserved
- Pattern created once per key photo range (outside frame loop) and reused for all frames, avoiding redundant createPattern calls
- Center-crop offset `(fw - tileWidth) / 2` applied via ctx.translate when frame cell narrower than one tile width, showing the middle of the image rather than the left edge
- MIN_FRAME_WIDTH_FOR_THUMB set to 4px -- below this threshold, solid placeholder colors are used instead of sub-pixel tile rendering

- **Issue:** At lower zoom levels where frame cell width is smaller than one tile width, the thumbnail showed from its left edge rather than being centered
- **Fix:** Calculated tileWidth from naturalWidth * scale, applied centering offset `(fw - tileWidth) / 2` to ctx.translate when fw < tileWidth
