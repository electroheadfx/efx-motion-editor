# Phase 12.8: Fix Timeline Thumb Horizontal Stretching on Zoom - Use Repeat Cover Pattern - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

When zooming the timeline, sequence thumbnails are stretched horizontally which looks bad. Replace the current `drawImage` stretch rendering with a repeat/tile pattern so thumbnails maintain their aspect ratio at any zoom level. No new timeline capabilities — this is a rendering fix only.

</domain>

<decisions>
## Implementation Decisions

### Fill strategy
- **Repeat/tile approach** — scale image to fill cell height, tile horizontally when cell is wider than one tile
- **Natural aspect ratio tiles** — tile width = `image.naturalWidth * (cellHeight / image.naturalHeight)`. Landscape photos get wider tiles, portrait photos get narrower tiles
- **Seamless tiling** — no separator between tile repetitions; existing frame border (0.5px stroke) is the only visual divider between frames

### Range rendering
- **Reset per frame** — each frame cell draws its tiles independently (not continuous across the key photo range)
- Every frame in the same key photo range looks identical — simpler, faster, smoother zoom transitions
- No need to calculate tile offset relative to range start

### Crop anchor
- **Horizontal: left-aligned** — tiles start from the left edge of the image. When frame cell is narrower than one tile, the left portion is shown
- **Vertical: center-aligned** — when image is taller than cell (after scaling to tile width), crop from the vertical center. Good general-purpose framing for diverse subjects

### Zoom extremes
- **Low zoom fallback** — below a minimum frame width threshold (Claude's discretion, e.g. 4-6px), skip `drawImage` and fill with existing placeholder colors (`PLACEHOLDER_BG_A` / `PLACEHOLDER_BG_B` alternating by range index). Avoids wasted draw calls on sub-pixel detail
- **High zoom** — unlimited tile repeats. Many tiles per frame cell is fine — the visual repetition makes it clear it's the same held photo (filmstrip look)

### Claude's Discretion
- Exact minimum frame width threshold for solid color fallback
- Whether to use `ctx.createPattern()` or manual tile loop for the repeat
- Any performance optimizations (e.g., caching pattern objects per image)

</decisions>

## Specific Ideas

- Filmstrip look: each frame cell shows the same tiled image independently (reset per frame, not continuous)
- Left-aligned horizontally so the tile starts at the image's left edge — consistent with timeline left-to-right reading direction
- Performance priority: user explicitly chose reset-per-frame for smoother, faster zoom responsiveness
