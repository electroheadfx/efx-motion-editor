if (cachedImg && pattern && fw >= MIN_FRAME_WIDTH_FOR_THUMB) {

**Key implementation notes:**
- Pattern is created ONCE per key photo range (outside frame loop), reused for all frames in that range -- avoids redundant `createPattern` calls
- `ctx.translate(fx, fy)` before `fillRect(0, 0, fw, fh)` ensures tiles start at the frame cell's left edge (pattern origin is canvas `(0,0)` by default)
- `ctx.save()/clip()/restore()` confines the pattern fill to the frame cell bounds
- The scale factor `cellH / cachedImg.naturalHeight` makes tile height exactly equal to cell height (no vertical crop needed -- the "vertical center-crop" in CONTEXT.md is a no-op since we scale to exact cell height)
- Tile width becomes `cachedImg.naturalWidth * scale` -- landscape photos get wider tiles, portrait photos get narrower tiles (natural aspect ratio preserved)
- Low-zoom fallback at `fw < 4px` uses existing `PLACEHOLDER_BG_A`/`PLACEHOLDER_BG_B` alternating by range index `ri`
- The frame border drawing (lines 280-283, `ctx.strokeRect`) remains UNCHANGED after the new block -- it overlays the tile pattern as before
- No DPR adjustment needed in the pattern transform -- `setupCanvas()` already applies `ctx.scale(this.dpr, this.dpr)` globally, and pattern `setTransform` operates in the pattern's own coordinate space
- The existing per-track clip region (lines 231-234) nests correctly with the per-frame clip -- nested clips are fully supported in Canvas 2D
