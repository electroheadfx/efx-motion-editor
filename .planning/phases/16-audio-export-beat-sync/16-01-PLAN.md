       ```typescript
       export interface BpmResult {
         bpm: number;        // BPM rounded to 1 decimal place
         confidence: number; // Raw autocorrelation peak value (higher = more confident)
       }

       export interface BpmOptions {
         minBPM?: number;  // default 60
         maxBPM?: number;  // default 200
       }

       export function detectBPM(
         channelData: Float32Array,
         sampleRate: number,
         options?: BpmOptions,
       ): BpmResult
       ```

       Algorithm (per D-05, from RESEARCH.md Pattern 2):
       a. Compute energy in 10ms windows (windowSize = floor(sampleRate * 0.01)), hop = windowSize / 2
       b. Compute onset detection function: half-wave rectified derivative of energy signal (max(0, energy[i] - energy[i-1]))
       c. Autocorrelate onset signal: for lags between minLag..maxLag (derived from maxBPM..minBPM), sum onsets[i] * onsets[i+lag] for up to 1000 samples
       d. Find lag with highest correlation, convert to BPM: 60 / (bestLag * hopSize / sampleRate)
       e. Octave correction (Pitfall 4): if bpm < 80, try doubling; if bpm > 160, try halving. Use heuristic: prefer the value closest to 120 BPM
       f. Round to 1 decimal: Math.round(bpm * 10) / 10
