- Provide `vec4 getToColor(vec2 uv) { return texture(u_to, uv); }`
- Declare `out vec4 out_fragColor;`
- Include the shader's `fragmentSource` (which contains `vec4 transition(vec2 uv)`)
- `void main() { vec2 uv = gl_FragCoord.xy / iResolution.xy; out_fragColor = transition(uv); }`

**3. Add computeTransitionProgress() to transitionEngine.ts (per D-07):**

```typescript
import type { GlTransition } from '../types/sequence';

/**
 * Compute the eased progress value for a GL transition at a given global frame.
 * progress = easing(framesIntoOverlap / overlapDuration)
 */
export function computeTransitionProgress(
  globalFrame: number,
  overlapStart: number,
  overlapDuration: number,
  curve: EasingType,
): number {
  if (overlapDuration <= 0) return 0;
  const framesIntoOverlap = globalFrame - overlapStart;
  const t = Math.max(0, Math.min(1, framesIntoOverlap / overlapDuration));
  return applyEasing(t, curve);
}
```

**CRITICAL PORTING RULES:**
1. The GLSL `fragmentSource` must contain ONLY the `vec4 transition(vec2 uv)` function and any helper functions. Do NOT include `uniform` declarations for `progress`, `from`, `to`, or `ratio` — the wrapper (`buildTransitionFragmentSource`) provides these.
2. Custom uniforms declared in gl-transitions source as `uniform float foo; // = default` must be REMOVED from the GLSL source (the wrapper generates `uniform float u_foo;`). Reference them as `u_foo` in the GLSL or let the wrapper's vec2 reconstruction handle it.
3. Actually, since the wrapper injects the source as-is and provides `getFromColor`/`getToColor` as functions, the transition code calls those directly. The gl-transitions convention already uses `getFromColor(uv)` / `getToColor(uv)`, so no changes needed there.
4. For shaders that declare `uniform vec2 direction;`, convert to two float params `directionX` and `directionY`. Add a line before the fragmentSource: the wrapper auto-generates `vec2 direction = vec2(u_directionX, u_directionY);` when it detects the pattern.
5. For shaders that use `texture2D()` in helper functions (not `getFromColor`/`getToColor`), replace with `texture()` for WebGL2 compatibility.
