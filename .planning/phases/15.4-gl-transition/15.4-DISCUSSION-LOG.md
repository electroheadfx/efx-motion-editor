## Transition Application Flow

### How should users apply a GL transition?

| Option | Description | Selected |
|--------|-------------|----------|
| Shader browser tab | Enable disabled Transition tab. User selects, configures, clicks Apply — replaces cross-dissolve | ✓ |
| AddTransitionMenu integration | Add GL transitions as options in existing AddTransitionMenu | |
| Both entry points | Browser tab + menu entries | |

**User's choice:** Shader browser tab
**Notes:** Follows Phase 15.3 pattern — browser is the primary entry point

### What happens to existing cross-dissolve?

| Option | Description | Selected |
|--------|-------------|----------|
| GL replaces cross-dissolve | One transition per sequence pair — either cross-dissolve OR GL shader | ✓ |
| GL is a cross-dissolve mode | New 'shader' mode alongside 'transparency' and 'solid' | |
| Separate system | GL and cross-dissolve coexist independently | |

**User's choice:** GL replaces cross-dissolve

### Which sequences does it target?

| Option | Description | Selected |
|--------|-------------|----------|
| Active + next sequence | Transition between active sequence and next adjacent one | ✓ |
| User picks the pair | User clicks two sequences after Apply | |
| Between all sequences | Bulk apply to all adjacent pairs | |

**User's choice:** Active + next sequence

### How to change or remove?

| Option | Description | Selected |
|--------|-------------|----------|
| TransitionProperties panel | Sidebar panel shows shader name, params, duration, Remove button | ✓ |
| Timeline context menu | Right-click transition zone for change/remove | |

## GL Rendering Pipeline

### How should GL transition render two sequences?

| Option | Description | Selected |
|--------|-------------|----------|
| Dual offscreen capture | Render outgoing to canvas A, incoming to canvas B, upload both as GL textures | ✓ |
| Single capture + live | Capture outgoing as texture, render incoming live, blend via GL | |

**User's choice:** Dual offscreen capture

### Which GLSL convention?

| Option | Description | Selected |
|--------|-------------|----------|
| gl-transitions format | `transition(vec2 uv)` with `progress`, `from`/`to` textures. 60+ open-source effects | ✓ |
| Shadertoy-style mainImage | `mainImage()` with iChannel0/iChannel1. Consistent with generators/FX | |
| Support both formats | Wrapper normalizes both formats. Maximum flexibility | |

**User's choice:** gl-transitions format

### How is progress driven?

| Option | Description | Selected |
|--------|-------------|----------|
| Frame-based from overlap | progress = framesIntoOverlap / overlapDuration, easing applied | ✓ |
| Time-based (iTime) | Real-time elapsed since transition start | |

**User's choice:** Frame-based from overlap

### Export support?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, preview + export | Identical rendering in both. exportRenderer runs GL shader | ✓ |
| Preview only first | Preview renders GL, export falls back to cross-dissolve | |

**User's choice:** Yes, preview + export
