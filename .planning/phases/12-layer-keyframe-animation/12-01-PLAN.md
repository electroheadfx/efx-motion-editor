  <name>Task 1: Define keyframe types and interpolation engine</name>
  <files>Application/src/types/layer.ts, Application/src/lib/keyframeEngine.ts, Application/src/lib/keyframeEngine.test.ts</files>
  <behavior>
    - interpolateAt([], 5) returns null (no keyframes)
    - interpolateAt([kf at frame 0], 0) returns kf values exactly (single keyframe, on frame)
    - interpolateAt([kf at frame 0], 10) returns kf values (single keyframe, after = hold last)
    - interpolateAt([kf0 at frame 0, kf1 at frame 10], -5) returns kf0 values (before first = hold first)
    - interpolateAt([kf0 at frame 0, kf1 at frame 10], 15) returns kf1 values (after last = hold last)
    - interpolateAt([kf0 at 0 with opacity=0, kf1 at 10 with opacity=1], 5, linear) returns opacity=0.5 (midpoint linear)
    - applyEasing(0, 'linear') returns 0
    - applyEasing(1, 'linear') returns 1
    - applyEasing(0.5, 'linear') returns 0.5
    - applyEasing(0, 'ease-in') returns 0
    - applyEasing(1, 'ease-in') returns 1
    - applyEasing(0.5, 'ease-in-out') returns 0.5 (symmetric midpoint)
    - lerpValues interpolates all 7 properties (opacity, x, y, scaleX, scaleY, rotation, blur)
    - extractKeyframeValues extracts values from a Layer correctly
  </behavior>
  <action>
1. In `types/layer.ts`, add these types AFTER the existing LayerTransform interface:

```typescript
export type EasingType = 'linear' | 'ease-in' | 'ease-out' | 'ease-in-out';

export interface KeyframeValues {
  opacity: number;
  x: number;
  y: number;
  scaleX: number;
  scaleY: number;
  rotation: number;
  blur: number;
}

export interface Keyframe {
  frame: number;          // Frame offset within the owning sequence (NOT global)
  easing: EasingType;     // Interpolation curve to the NEXT keyframe
  values: KeyframeValues; // Snapshot of all animatable properties
}
```

Add `keyframes?: Keyframe[]` to the Layer interface (optional for backward compat).

Add helper function:
```typescript
export function extractKeyframeValues(layer: Layer): KeyframeValues {
  return {
    opacity: layer.opacity,
    x: layer.transform.x,
    y: layer.transform.y,
    scaleX: layer.transform.scaleX,
    scaleY: layer.transform.scaleY,
    rotation: layer.transform.rotation,
    blur: layer.blur ?? 0,
  };
}
```

IMPORTANT: Store keyframe frames as OFFSETS from the sequence start (not global frame numbers). This makes keyframes immune to changes in other sequences. The research Open Question #1 recommends this approach. The conversion from global to sequence-local happens at the call site using `activeSequenceStartFrame`.

2. Create `lib/keyframeEngine.ts` with pure functions:
   - `applyEasing(t: number, easing: EasingType): number` -- polynomial cubic easing (ease-in: t^3, ease-out: (t-1)^3+1, ease-in-out: piecewise cubic). See RESEARCH.md code examples.
   - `lerp(a: number, b: number, t: number): number`
   - `lerpValues(a: KeyframeValues, b: KeyframeValues, t: number): KeyframeValues` -- interpolate all 7 properties
   - `interpolateAt(keyframes: Keyframe[], frame: number): KeyframeValues | null` -- the core function:
     - Empty array -> null
     - Single keyframe -> return its values
     - Before first -> hold first values
     - After last -> hold last values
     - Between two -> find surrounding pair, normalize t, apply easing from prev keyframe, lerp values
   - Use a reusable result object pattern to avoid garbage during playback (allocate once, update in place for the internal _mutable version; return a fresh object from the public API for safety)

3. Write tests in `lib/keyframeEngine.test.ts` covering the behavior list above. Use vitest/jest syntax matching project conventions.

  <name>Task 2: Create keyframeStore with CRUD and computed interpolation</name>
  <files>Application/src/stores/keyframeStore.ts</files>
  <action>
Create `stores/keyframeStore.ts` following the existing store pattern (signal-based, exported as single object).

Signals:
- `selectedKeyframeFrames: signal<Set<number>>(new Set())` -- frames of selected keyframe diamonds (using frame offsets, not global)

Computed signals:
- `interpolatedValues: computed<KeyframeValues | null>` -- calls `interpolateAt(layer.keyframes, localFrame)` for the selected layer at the current playhead position. Dependencies: `layerStore.selectedLayerId`, `timelineStore.currentFrame`, `sequenceStore.sequences` (to find the layer and compute local frame from global). Return null if no layer selected, layer has no keyframes, or layer is FX/base.
- `isOnKeyframe: computed<boolean>` -- true when the current frame exactly matches a keyframe frame for the selected layer
- `activeLayerKeyframes: computed<Keyframe[]>` -- keyframes array for the currently selected content layer (empty if none/FX/base)

Methods:
- `addKeyframe(layerId: string, globalFrame: number)`: Extract current property values from the layer, compute sequence-local frame offset, create Keyframe with default easing 'ease-in-out', upsert into layer.keyframes (update if frame exists, insert sorted if new). Use `layerStore.updateLayer()` which routes through sequenceStore with undo support. The layer values to snapshot come from the layer's current stored properties (opacity, transform.x/y/scaleX/scaleY/rotation, blur).
- `removeKeyframes(layerId: string, frames: number[])`: Remove keyframes at given frame offsets from layer.keyframes. Use `layerStore.updateLayer()`.
- `moveKeyframe(layerId: string, fromFrame: number, toFrame: number)`: Move a keyframe from one frame offset to another. Prevent moving to a frame that already has a keyframe. Re-sort after move. Use `layerStore.updateLayer()`.
- `setEasing(layerId: string, frame: number, easing: EasingType)`: Update the easing type on a specific keyframe. Use `layerStore.updateLayer()`.
- `clearSelection()`: Reset selectedKeyframeFrames to empty set.
- `selectKeyframe(frame: number, additive: boolean)`: If additive (shift-click), toggle frame in set. Otherwise, replace set with just this frame.

For computing the sequence-local frame from global frame:
- Find the sequence that owns the selected layer
- Find its start frame from trackLayouts (import from frameMap.ts)
- localFrame = globalFrame - sequenceStartFrame

Import `interpolateAt` from `../lib/keyframeEngine`. Import `layerStore`, `timelineStore`, `sequenceStore`, `trackLayouts` from their respective modules.

IMPORTANT: All keyframe frame values stored on Layer are sequence-local offsets. The store converts between global (from timelineStore.currentFrame) and local transparently.
