## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, sidebar row padding, Canvas 2D label padding |
| md | 16px | Default element spacing, section gaps in sidebar |

| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Canvas 2D track geometry uses existing named constants (`TRACK_HEIGHT=52`, `FX_TRACK_HEIGHT=28`, `TRACK_HEADER_WIDTH=80`) which are not on the 8-point scale but are established project constants exported from `TimelineRenderer.ts`. Transition overlay border and diagonal line endpoints use a 4px inset from track top/bottom edges (`trackY + 4` / `trackH - 8`), which is on the spacing grid and consistent with the FX bar inset pattern (`barY = y + 4`, `barH = FX_TRACK_HEIGHT - 8`).

## Typography

This phase has two rendering contexts: DOM (sidebar properties) and Canvas 2D (timeline overlay labels). Both use system-ui with Inter loaded.

### DOM (Sidebar TransitionProperties)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Section label | 11px | 600 (semibold) | 1.2 | CollapsibleSection/SectionLabel header text -- inherited from existing sidebar convention (`SectionLabel.tsx`, `CollapsibleSection.tsx`, `InlineInterpolation.tsx` all use 11px/600) |
| Control label | 10px | 400 (regular) | 1.4 | NumericInput labels, toggle labels, color picker labels -- inherited from existing `SidebarFxProperties.tsx` control labels |
| Input value | 11px | 400 (regular) | 1.4 | NumericInput value text, dropdown/select text -- inherited from existing select elements in `SidebarProperties.tsx` and `SidebarFxProperties.tsx` |
| Button text | 10px | 400 (regular) | 1.4 | "Add Fade In" / "Add Fade Out" button labels -- matches existing toggle button size in `SidebarFxProperties.tsx` |

**Note on 10px/11px coexistence:** The 1px gap between control labels (10px) and section headers/input values (11px) is an established project convention, not a new introduction. Section headers use 11px at weight 600 with 2px letter-spacing across all existing sidebar sections (`SectionLabel.tsx`, `CollapsibleSection.tsx`, `InlineInterpolation.tsx`). Control labels use 10px at weight 400 in `SidebarFxProperties.tsx`. Select/dropdown values use 11px in `SidebarProperties.tsx` and `SidebarFxProperties.tsx`. This phase inherits the existing pattern verbatim.

## Interaction Contract

### Timeline Transition Overlays

| Interaction | Behavior |
|-------------|----------|
| Hover over transition zone | Cursor changes to `pointer`. No visual hover state beyond cursor (overlay is already semi-transparent, adding hover would be noisy). |
| Click transition zone | Selects the transition. Border becomes full `#FFFFFF`. `uiStore.selectedTransition` updates. Sidebar shows TransitionProperties. Any previously selected layer/sequence is deselected. |
| Click away from transition | Deselects transition. `uiStore.selectedTransition` nulls. Sidebar returns to previous context. |
| Delete key (transition selected) | Removes the transition from the sequence. Sequence returns to no-fade state. |
| Sequence name shift | When fade-in is present, sequence name label at bottom-left shifts right by `fadeInWidth + 4px` to avoid overlapping the transition overlay. |
| Label visibility | Transition label ("In"/"Out"/"Cross Dissolve") is drawn centered in overlay only when overlay width > `measureText(label).width + 8px`. Otherwise label is hidden (overlay too narrow). |

### Sidebar TransitionProperties

| Interaction | Behavior |
|-------------|----------|
| Duration NumericInput | Change fade duration in frames. Min: 1. Max: half the sequence length (clamped). Step: 1. Supports pointer-drag scrub on label. |
| Mode toggle (Transparency / Solid) | Two-button toggle group. Only visible for fade-in/fade-out (not cross dissolve). Defaults to "Transparency". |
| Color picker | Native `<input type="color">`. Only visible when mode is "Solid". Default value: `#000000`. Inline with "Color" label. |
| Curve dropdown | Native `<select>` with 4 options: Linear, Ease In, Ease Out, Ease In-Out. Default: "Ease In-Out". |
| Remove Transition button | Text button at bottom of section. Styled with `--color-error-text`. Removes transition without confirmation (lightweight action, undoable via Cmd+Z). |

### Adding Transitions

| Interaction | Behavior |
|-------------|----------|
| Sidebar "Transitions" section | When a content sequence is selected (via `uiStore.selectedSequenceId`), a "TRANSITIONS" CollapsibleSection appears in the sidebar below existing sequence info. Contains "Add Fade In" and "Add Fade Out" buttons. Cross dissolve button appears only when a next sequence exists. |
| "Add Fade In" button | Creates a fade-in transition with default duration of 12 frames, mode "transparency", curve "ease-in-out". Transition overlay immediately appears on timeline. Button text changes to "Fade In" section showing properties. |
| "Add Fade Out" button | Same as above but for fade-out. Default duration 12 frames. |
| "Add Cross Dissolve" button | Creates cross dissolve centered on the boundary between current and next sequence. Default duration 24 frames (12 per side). Only available between adjacent content sequences. |

## State Contract

### Transition Selection State

| Signal | Type | Location | Purpose |
|--------|------|----------|---------|
| `selectedTransition` | `{ sequenceId: string; type: 'fade-in' \| 'fade-out' \| 'cross-dissolve' } \| null` | uiStore | Tracks which transition overlay is currently selected |

### Selection Mutual Exclusion

When a transition is selected:
- `selectedLayerId` is set to `null`
- `selectedTransition` holds the active selection
- Sidebar renders TransitionProperties instead of layer properties

When a layer is selected:
- `selectedTransition` is set to `null`
- Normal sidebar behavior resumes

## Canvas 2D Rendering Spec

### Fade In Overlay

```
Position: x = sequence start, w = fadeIn.duration * frameWidth
Gradient: horizontal linear, left-to-right
  Stop 0: rgba(0, 0, 0, 0.55)  -- dark (fully faded end)
  Stop 1: rgba(0, 0, 0, 0.05)  -- clear (fully visible end)
Border: 1px strokeRect, inset 4px from track top/bottom
  strokeRect(x, trackY + 4, w, trackH - 8)
  Color: rgba(255, 255, 255, 0.6) unselected / #FFFFFF selected
Diagonal: bottom-left to top-right, 1px
  moveTo(x, trackY + trackH - 4)
  lineTo(x + w, trackY + 4)
  Color: rgba(255, 255, 255, 0.6)
Label: "In", centered, 8px system-ui, rgba(255, 255, 255, 0.8)
  Hidden when overlay width < measureText("In").width + 8
```

### Fade Out Overlay

```
Position: x = sequence end - fadeOut.duration * frameWidth, w = fadeOut.duration * frameWidth
Gradient: horizontal linear, left-to-right
  Stop 0: rgba(0, 0, 0, 0.05)  -- clear (fully visible end)
  Stop 1: rgba(0, 0, 0, 0.55)  -- dark (fully faded end)
Border: 1px strokeRect, inset 4px from track top/bottom
  strokeRect(x, trackY + 4, w, trackH - 8)
  Color: rgba(255, 255, 255, 0.6) unselected / #FFFFFF selected
Diagonal: top-left to bottom-right, 1px
  moveTo(x, trackY + 4)
  lineTo(x + w, trackY + trackH - 4)
  Color: rgba(255, 255, 255, 0.6)
Label: "Out", centered, 8px system-ui, rgba(255, 255, 255, 0.8)
  Hidden when overlay width < measureText("Out").width + 8
```

### Cross Dissolve Overlay

```
Position: centered on pink boundary marker between two sequences
  x = boundary - (duration/2) * frameWidth
  w = duration * frameWidth
Gradient: single diagonal bottom-left to top-right creates two triangular zones
  Left triangle: rgba(0, 0, 0, 0.45) -- outgoing sequence (darker)
  Right triangle: rgba(0, 0, 0, 0.20) -- incoming sequence (lighter)
Border: 1px strokeRect, inset 4px from track top/bottom
  strokeRect(x, trackY + 4, w, trackH - 8)
  Color: rgba(255, 255, 255, 0.6) unselected / #FFFFFF selected
Diagonal: bottom-left to top-right, 1px
  moveTo(x, trackY + trackH - 4)
  lineTo(x + w, trackY + 4)
  Color: rgba(255, 255, 255, 0.6)
Pink marker: renders BEHIND the overlay but visible through transparency
Label: "Cross Dissolve", centered, 8px system-ui, rgba(255, 255, 255, 0.8)
  Hidden when overlay width < measureText("Cross Dissolve").width + 8
```

### Drawing Order (within drawLinearTrack)

1. Existing: track background fill
2. Existing: sequence selection highlight
3. Existing: thumbnail tiles
4. **NEW: transition overlay** (gradient + border + diagonal + label)
5. Existing: pink boundary markers (already drawn, visible through transition transparency)
6. Existing: sequence name overlay (shifted right when fade-in present)
7. Existing: isolation border and name label

## Sidebar Layout Spec

### TransitionProperties Panel (when transition selected)

**Primary visual anchor:** Duration row -- most frequently edited control; the NumericInput with scrub-on-label draws the eye and establishes the panel's purpose immediately.

```
CollapsibleSection "TRANSITIONS"
  padding: 8px horizontal, 0 vertical (matches existing sidebar sections)

  [If transition exists on this sequence:]

  Section: "FADE IN" or "FADE OUT" or "CROSS DISSOLVE"
    Row 1: Duration
      label "Duration" (10px, var(--color-text-muted))
      NumericInput (value in frames, step=1, min=1, max=totalFrames/2)

    Row 2: Mode (fade-in/fade-out only, hidden for cross dissolve)
      label "Mode" (10px, var(--color-text-muted))
      two-button toggle: [Transparency] [Solid Color]
        active: bg var(--color-accent), text #FFFFFF
        inactive: bg var(--color-bg-input), text var(--color-text-muted)
        button size: text-[10px] px-2 py-1 rounded

    Row 3: Color (visible only when mode = "Solid Color")
      label "Color" (10px, var(--color-text-muted))
      <input type="color"> (w-6 h-6 rounded, matches existing tint picker)

    Row 4: Curve
      label "Curve" (10px, var(--color-text-muted))
      <select> with 4 options (text-[11px], bg var(--color-bg-input), text var(--color-text-button))

    Row 5: Remove
      text button "Remove Transition" (text-[10px], var(--color-error-text))
      hover: bg var(--color-error-bg), rounded

  [If no transition of this type exists:]
    Buttons: "Add Fade In" / "Add Fade Out" / "Add Cross Dissolve"
      style: text-[10px] px-2 py-1 rounded
      bg: var(--color-bg-input)
      text: var(--sidebar-text-button)
      hover: bg var(--color-bg-hover-item), text #FFFFFF
```

### Row Layout Pattern

Each property row follows the existing 2-column pattern from SidebarFxProperties:

```
<div class="flex items-center gap-1">
  <span class="text-[10px] text-[var(--color-text-muted)] whitespace-nowrap">{label}</span>
  {control}
</div>
```

Gap between rows: 8px (matches existing sidebar property sections).
