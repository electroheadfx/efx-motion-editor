<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL/context -> standalone UI | Launch context parsed from URL/search/hash is untrusted local input. |
| Canvas capture -> app apply event/fallback | Rendered PNG data may be large and must stay bounded for sequence generation. |
| Standalone apply event -> app listener | Rendered-output payload leaves standalone and mutates editor state after validation. |
| JSON state load -> engine | User-selected state files can be malformed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-07 | Tampering | launch context parser | mitigate | Validate layerId/startFrame/operationId before reporting `Ready to apply`; otherwise show not-ready diagnostics. |
| T-35-08 | Denial of Service | `[apply play canvas]` | mitigate | Clamp frame count to 1..600, disable apply while running, and expose errors. |
| T-35-09 | Tampering | state file load | mitigate | Reuse serialized-project guard and show required invalid-state-file error copy. |
| T-35-10 | Information Disclosure | rendered canvas data URLs | accept | Local desktop/dev workflow with user-initiated apply only; no secrets are introduced. |
| T-35-18 | Tampering | standalone apply transport | mitigate | Emit rendered-output-only `physic-paint:apply` payloads with operationId/layerId/startFrame and no editable state; app-side Plan 35-04 validates before mutation. |
| T-35-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install tasks in this plan; RESEARCH.md Package Legitimacy Audit says no external package installation is recommended. |
</threat_model>
