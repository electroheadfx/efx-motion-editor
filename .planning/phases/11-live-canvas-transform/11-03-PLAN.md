<task type="auto">
  <name>Task 1: Create TransformOverlay component with bounding box, handles, and drag state machine</name>
  <files>Application/src/components/canvas/TransformOverlay.tsx</files>
  <action>
Create `TransformOverlay.tsx` as a Preact functional component that renders inside the same CSS-transformed div as `<Preview />` in CanvasArea. Since it shares the parent's CSS transform, all positioning is in project-resolution coordinates.

**Props:**
```typescript
interface TransformOverlayProps {
  containerRef: RefObject<HTMLDivElement>;  // The outer container (for getBoundingClientRect)
  getSourceDimensions: (layer: Layer) => { w: number; h: number } | null;  // From renderer image cache
}
```

**Rendering (when a content layer is selected and not playing):**

1. Read `layerStore.selectedLayerId.value` and `layerStore.layers.value` to find selected layer.
2. If no selection, or layer is FX, or `timelineStore.isPlaying.value` is true, render nothing (return null or empty div).
3. Call `getLayerBounds()` with the selected layer's transform and source dimensions to get the 4 corners and center.
4. Call `getHandlePositions()` to get handle coordinates.
5. Render an absolutely-positioned div covering the full project area (`position: absolute; inset: 0; pointer-events: none;`).

Inside this div, render:
- **Bounding box**: An SVG `<polygon>` or 4 `<line>` elements connecting the corners. Stroke: `#4A90D9` (blue), width: `${1.5 / zoom}px` (counter-scaled to maintain 1.5px screen width). Fill: none. Set `pointer-events: all` on the SVG.
- **Corner handles**: 4 `<div>` elements at corner positions. Each is `8px` screen size, rendered as `${8 / zoom}px` in project space. White fill, 1px blue border (also counter-scaled). `pointer-events: all`. Positioned with `transform: translate(-50%, -50%)` for centering.
- **Edge midpoint handles**: 4 `<div>` elements at edge midpoints (average of two adjacent corners). Same styling as corners but slightly smaller (6px screen size).
- **Rotation zones**: No visible element -- rotation is detected by hovering outside the bounding box near corners (handled in the pointer event logic, cursor changes to indicate rotation).

**Pointer event handling -- Drag State Machine (per RESEARCH.md Pattern 3):**

State:
```typescript
type DragMode = 'none' | 'pending' | 'move' | 'scale' | 'rotate' | 'pan';
interface DragState {
  mode: DragMode;
  startClientX: number;
  startClientY: number;
  startLayerTransform: LayerTransform;  // Snapshot at pointerdown
  handleType?: HandleType;
  layerId?: string;
}
```
Use `useRef` for drag state (not reactive -- avoid re-renders during drag).

**DRAG_THRESHOLD: 4 pixels** (per user decision: 3-5px before move begins).

The overlay div gets `pointer-events: none`, but individual handles and the bounding box area get `pointer-events: all`.

**onPointerDown on the overlay's parent (the full project-area div, with pointer-events: all):**
1. If middle-click (e.button === 1): forward to pan handler (call the pan start function passed as prop or use canvasStore directly). Return.
2. Convert clientX/clientY to project-space point via `clientToCanvas()`.
3. If a layer is selected, hit-test handles first via `hitTestHandles()`:
   - If handle hit: set drag state to `{ mode: 'pending', handleType, startClientX/Y, startLayerTransform: snapshot }`. Call `setPointerCapture(e.pointerId)`.
4. Check rotation zone via `getRotationZone()`:
   - If in rotation zone: set drag state to `{ mode: 'pending', handleType: 'rotate', ... }`. Capture.
5. Hit-test layers via `hitTestLayers()` (or `hitTestLayersCycle()` if Alt key held):
   - If layer hit: select it via `layerStore.setSelected(layerId)`. Set drag state to `{ mode: 'pending', handleType: undefined (move), layerId, ... }`. Capture.
6. If nothing hit: deselect via `layerStore.setSelected(null)`. Do NOT start drag.

**onPointerMove:**
1. If `mode === 'none'`: Update cursor based on what's under the pointer (handle, rotation zone, layer, or default). Use `getCursorForHandle()`.
2. If `mode === 'pending'`:
   - Calculate distance from start: `Math.hypot(e.clientX - state.startClientX, e.clientY - state.startClientY)`.
   - If distance < DRAG_THRESHOLD: do nothing (still pending).
   - If distance >= DRAG_THRESHOLD: Transition to actual mode based on handleType:
     - `handleType` is a corner -> `mode = 'scale'`
     - `handleType` is an edge -> `mode = 'scale'`
     - `handleType` is 'rotate' -> `mode = 'rotate'`
     - `handleType` is undefined -> `mode = 'move'`
     - Call `startCoalescing()` from history.ts.
3. If `mode === 'move'`: Calculate delta in project space. Apply:
   ```typescript
   const dx = (e.clientX - state.startClientX) / zoom;
   const dy = (e.clientY - state.startClientY) / zoom;
   layerStore.updateLayer(state.layerId!, {
     transform: { ...state.startLayerTransform, x: state.startLayerTransform.x + dx, y: state.startLayerTransform.y + dy },
   });
   ```
4. If `mode === 'scale'`: Per RESEARCH.md Pattern for scale.
   - For corner handles: compute distance ratio from center for uniform scaling. Apply same factor to both scaleX and scaleY.
   - For edge handles: compute distance change along the relevant axis (accounting for rotation) and apply to scaleX or scaleY only.
5. If `mode === 'rotate'`: Per RESEARCH.md Pattern for rotation.
   - Compute angle from center to current mouse vs angle from center to start mouse.
   - Apply delta angle to startLayerTransform.rotation.

**onPointerUp:**
1. If mode was 'move', 'scale', or 'rotate': call `stopCoalescing()`.
2. If mode was 'pending' (click without drag): selection already happened in pointerDown, nothing more needed.
3. Release pointer capture. Reset drag state to `{ mode: 'none', ... }`.

**Cursor management:**
- On pointermove when no drag: set cursor on the overlay container based on what's under pointer.
- Move cursor: `move`
- Corner handles: `nwse-resize` / `nesw-resize` (rotated based on layer rotation)
- Edge handles: `ns-resize` / `ew-resize` (rotated based on layer rotation)
- Rotation zone: Use CSS `cursor: url('data:image/svg+xml,...') 12 12, crosshair` with a curved arrow SVG, or simply use `crosshair` cursor.
- Default (no layer under cursor): `default`

     - For `video` layers: use a default of `projectStore.width x projectStore.height` (approximation).

   - The TransformOverlay handles forwarding middle-click to pan.
   - Update cursor logic: Remove the `grab`/`grabbing` cursor for left-click zoom scenarios. The cursor is now managed by TransformOverlay when a layer is hovered/selected, and defaults to `default` otherwise.

   - When Space is held and pointer is down, start pan mode (same as middle-click pan logic).
   - When Space+drag is detected, set a flag to suppress the play/pause toggle on space keyup.

5. **Expose containerRef** for coordinate mapping: The TransformOverlay needs the container's DOMRect. Pass `containerRef` as a prop.
