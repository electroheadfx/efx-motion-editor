Create `Application/src/components/canvas/` directory and `coordinateMapper.ts` with these exports:

```typescript
export interface Point { x: number; y: number; }

/**
 * Convert mouse clientX/clientY to project-resolution coordinates.
 *
 * Inverts the CSS transform chain applied to the canvas wrapper div:
 *   CSS: transform: scale(zoom) translate(panX, panY)
 *   transformOrigin: center center
 *
 * The wrapper div is centered in its container and sized to projectWidth x projectHeight.
 * The CSS applies scale first (around center), then translate (in scaled space).
 *
 * Inversion steps:
 * 1. Client -> container-center-relative (subtract container center point)
 * 2. Undo CSS scale: divide by zoom
 * 3. Undo CSS translate: subtract panX, panY
 * 4. Convert from center-relative to top-left-origin project coordinates
 */
export function clientToCanvas(
  clientX: number,
  clientY: number,
  containerRect: DOMRect,
  zoom: number,
  panX: number,
  panY: number,
  projectWidth: number,
  projectHeight: number,
): Point
```

Also export the inverse function:
```typescript
/**
 * Convert project-resolution coordinates to client coordinates.
 * Used for positioning HTML overlay handles in screen space.
 * Inverse of clientToCanvas.
 */
export function canvasToClient(
  projX: number,
  projY: number,
  containerRect: DOMRect,
  zoom: number,
  panX: number,
  panY: number,
  projectWidth: number,
  projectHeight: number,
): Point
```

Implementation matches the RESEARCH.md Pattern 1 exactly:
- clientToCanvas: `relX = clientX - containerCenterX`, `relY = clientY - containerCenterY`, then `canvasX = relX / zoom - panX`, `canvasY = relY / zoom - panY`, then `projX = canvasX + projectWidth / 2`, `projY = canvasY + projectHeight / 2`.
- canvasToClient: Inverse operations in reverse order.

Also export a helper:
```typescript
/**
 * Convert a distance in screen pixels to project-resolution pixels.
 * Used for drag deltas (move, scale calculations).
 */
export function screenToProjectDistance(screenDist: number, zoom: number): number {
  return screenDist / zoom;
}
```

**transformHandles.ts** -- Handle geometry and bounding box computation:

Define handle types and geometry functions:

```typescript
export type HandleType =
  | 'corner-tl' | 'corner-tr' | 'corner-br' | 'corner-bl'  // Corner handles (uniform scale)
  | 'edge-top' | 'edge-right' | 'edge-bottom' | 'edge-left' // Edge midpoint handles (axis scale)
  | 'rotate';  // Rotation zone (outside corners)

export interface HandlePosition {
  type: HandleType;
  x: number;  // Project-space X
  y: number;  // Project-space Y
}

export interface LayerBounds {
  corners: Point[];   // 4 corners in project space [TL, TR, BR, BL], rotated
  center: Point;      // Center point in project space
  drawW: number;      // Unscaled draw width (after aspect-fit + crop)
  drawH: number;      // Unscaled draw height
}
```

Key functions:

1. `getLayerBounds(layer, sourceWidth, sourceHeight, canvasW, canvasH): LayerBounds` -- Replicates the exact PreviewRenderer.drawLayer() transform pipeline:
   - Calculate effective source dimensions after crop (cropTop/cropRight/cropBottom/cropLeft)
   - Aspect-fit to canvas dimensions
   - Center at (transform.x + canvasW/2, transform.y + canvasH/2)
   - Apply scaleX/scaleY to half-dimensions
   - Rotate 4 corners around center by transform.rotation degrees
   - NOTE: Use scaleX/scaleY (not scale) since Plan 01 will have landed. If Plan 01 hasn't been applied yet, use `(layer.transform as any).scaleX ?? (layer.transform as any).scale ?? 1` as fallback. But prefer assuming scaleX/scaleY exist.

2. `getHandlePositions(bounds: LayerBounds, zoom: number): HandlePosition[]` -- Returns positions for all 8 handles (4 corners + 4 edge midpoints). Edge midpoints are the average of two adjacent corners.

3. `hitTestHandles(point: Point, handles: HandlePosition[], zoom: number, handleScreenSize?: number): HandleType | null` -- Check if a point (in project space) is within the hit area of any handle. Handle hit area is `handleScreenSize / zoom` pixels (counter-scaled). Returns the HandleType hit or null.

4. `getRotationZone(point: Point, bounds: LayerBounds, zoom: number): boolean` -- Returns true if the point is outside the bounding box but within a rotation hover zone (within ~15px screen distance of any corner). Uses point-to-line-segment distance checks.

5. `getCursorForHandle(handle: HandleType | null, isRotationZone: boolean, rotation: number): string` -- Returns CSS cursor value. Corner handles use ne-resize/nw-resize/etc rotated by the layer rotation. Edge handles use n-resize/e-resize/etc. Rotation zone uses a custom cursor or `url(data:...)` cursor.

6. `pointInPolygon(point: Point, polygon: Point[]): boolean` -- Standard ray-casting algorithm for testing if a point is inside a convex polygon (the rotated bounding box).

**hitTest.ts** -- Layer hit testing:

```typescript
import type { Layer } from '../../types/layer';
import type { Point } from './coordinateMapper';
import { getLayerBounds, pointInPolygon } from './transformHandles';
import { isFxLayer } from '../../types/layer';
```

Key functions:

1. `hitTestLayers(point: Point, layers: Layer[], canvasW: number, canvasH: number, getSourceDimensions: (layer: Layer) => {w: number, h: number} | null): string | null` -- Tests layers from top to bottom (reverse array order since layers[0] is bottom). Skips non-visible layers and FX layers (per user decision: only content layers selectable on canvas). For each content layer:
   a. Get source dimensions via callback (the overlay will provide this from the renderer's image cache)
   b. Compute bounds via getLayerBounds
   c. Test if point is inside the rotated bounding box via pointInPolygon
   d. Return the first (topmost) layer ID that passes
   e. If no layer hit, return null

2. `hitTestLayersCycle(point: Point, layers: Layer[], canvasW: number, canvasH: number, getSourceDimensions: ..., currentSelectedId: string | null): string | null` -- For Alt+click cycling: returns the next layer under the point below the currently selected one. If currentSelectedId is null or not under the point, returns topmost. Wraps around to top when reaching bottom.

NOTE: For this plan, use bounding-box-only hit testing (no pixel sampling). Pixel sampling can be added later as an enhancement if needed. The bounding box approach is fast and sufficient for most use cases.
