**ColorPickerModal.tsx changes:**

1. Add `import { createPortal } from 'preact/compat';` at the top of the file.

2. Add `onMouseDown={(e) => e.stopPropagation()}` to the outermost wrapper div (the `fixed inset-0` div at ~line 388). This is belt-and-suspenders to stop mousedown from reaching any parent containers.

3. Wrap the entire return JSX in `createPortal(...)` to render into `document.body`:
   - The component currently returns `<div class="fixed inset-0 ...">...</div>`
   - Change to: `return createPortal(<div class="fixed inset-0 ..." onMouseDown={(e) => e.stopPropagation()}>...</div>, document.body);`
   - This moves the modal's DOM tree out of the SortableJS container entirely.

**KeyPhotoStrip.tsx changes:**

4. Add `filter: '.color-picker-portal'` to the SortableJS config object (~line 100-115) as a secondary safety measure. Note: with portal rendering this filter may not match anything in the SortableJS container, but it's good practice as defense-in-depth. Alternatively, a more useful filter: add `filter: '.fixed'` to prevent any fixed-positioned element from initiating drag.

Actually, since the modal is now portaled out of the SortableJS container, the filter is not strictly needed. But add `handle: '.key-photo-card'` or a drag handle CSS class to the SortableJS config as proper practice. Specifically:
   - Add a CSS class `key-photo-drag` to each key photo card's outermost div in the card rendering (~line 356 area, the `group relative` div).
   - Add `handle: '.key-photo-drag'` to the SortableJS config. This is the same pattern used by LayerList.tsx (`handle: '.layer-drag-handle'`) and SequenceList.tsx (`handle: '.seq-drag-handle'`).

Wait -- adding a handle would change the drag UX (currently the whole card is draggable). The user may expect the entire card to remain draggable. The safer approach is:
   - Only apply the portal fix (step 3) and onMouseDown stopPropagation (step 2).
   - Do NOT add a handle constraint to SortableJS -- keep the current "drag anywhere on card" behavior.
   - The portal rendering alone fully resolves the drag propagation issue since the modal DOM is no longer inside the SortableJS container.

**Summary of changes:**
- ColorPickerModal.tsx: add `createPortal` import, wrap return in `createPortal(..., document.body)`, add `onMouseDown stopPropagation` on outer wrapper div
- KeyPhotoStrip.tsx: no changes needed (portal rendering removes the modal from the SortableJS tree)
