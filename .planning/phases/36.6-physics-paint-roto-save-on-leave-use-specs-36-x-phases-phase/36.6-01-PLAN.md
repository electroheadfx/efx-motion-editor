## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input to Roto navigation coordinator | Frame numbers from timeline cells and existing previous/next/first/last controls enter state-changing navigation/save logic. |
| Standalone Physics Paint window to parent apply bridge | Save-on-leave reuses apply-canvas/delete payloads and waits for parent apply results. |
| In-memory dirty state to durable cache state | Dirty unsaved engine state becomes cached PNG/editor timeline state only after successful apply result. |

| T-36.6-01 | Tampering | `requestRotoFrameNavigation` | mitigate | Validate target/source frame numbers as non-negative integers before snapshot, sync, or flush. |
| T-36.6-02 | Tampering | `flushRotoFrame` save-on-leave source selection | mitigate | Capture `sourceFrame = currentFrame` before target navigation and pass that source to the existing cache path. |
