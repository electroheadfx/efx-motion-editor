#### 1. Static Image Add-Layer Flow

**Test:** Open a project with at least one imported image. Click the "+ Add" button in the sidebar layers panel, then click "Static Image."
**Expected:** Imported view opens immediately with only images visible (no video section). Clicking one image closes the view, returns to editor mode, and the layer panel shows a new static-image layer with the image name.
**Why human:** Asset filter rendering, layer panel update, and auto-navigation require visual confirmation.

#### 2. Image Sequence Add-Layer Flow

**Test:** Open a project with at least 3 imported images. Click Add > Image Sequence. Select 3 images using checkmarks. Click "Confirm (3)."
**Expected:** Imported view opens with images-only filter and checkmark overlay on images. Header shows "Select images for sequence (3)" after selecting 3. Confirm button is active. After confirm, returns to editor with a new "Sequence (3 images)" layer.
**Why human:** Multi-select checkmark UI, header count update, Confirm button active/inactive states, and resulting layer name require visual verification.

#### 3. Video Add-Layer Flow

**Test:** Open a project with at least one imported video. Click Add > Video.
**Expected:** Imported view opens showing only video thumbnails (no images), each with a Film icon badge in the bottom-right corner. Clicking a video creates a video layer and returns to editor.
**Why human:** videos-only filter, Film badge visual rendering, and layer creation require visual confirmation.

#### 4. Cancel Without Sequence Deletion

**Test:** Start a new image sequence (which creates a pendingNewSequenceId). Before confirming key photos, cancel out. Then immediately click Add > Static Image. When the Imported view opens in static-image mode, click X to cancel.
**Expected:** No layers created, no sequences deleted. The pending sequence created in step 1 should still exist.
**Why human:** The cancel-delete collision guard (intent checked before pendingNewSequenceId) requires interactive state manipulation to confirm the guard works correctly.

#### 5. Stale Intent After Toolbar Toggle

**Test:** Click Add > Video (Imported view opens with videos-only filter). Instead of selecting a video, click the Toolbar "Imported" button to dismiss. Wait 1 second, then click the Toolbar "Imported" button again.
**Expected:** The Imported view reopens showing all assets (images and videos, with Videos section header) — not filtered to videos-only. The stale video intent has been cleared.
**Why human:** Requires visual inspection of the re-opened view's filter state to confirm intent was cleared.
