# Phase 12.9: Fix Sidebar Add-Layer Dialogs - Research

**Researched:** 2026-03-18
**Domain:** Preact UI refactoring -- replacing popover dialogs with full-window Imported view flows
**Confidence:** HIGH

## Summary

This phase replaces the three popover-based add-layer dialogs in `AddLayerMenu.tsx` (thumbnail picker for static images, OS directory picker for image sequences, and video list picker for videos) with proper Imported view flows. Each flow opens the existing full-window `ImportedView` with appropriate asset filtering (images-only or videos-only) and selection mode (single-select or multi-select), then creates the appropriate layer on selection/confirmation.

The existing `ImportedView.tsx` and `ImportGrid.tsx` already support single-select (key photo picking) and multi-select (new sequence key photo flow via `pendingNewSequenceId`). The work is extending these components with: (1) an asset filter mode to show images-only or videos-only, (2) a layer-creation callback mechanism separate from the existing key-photo flow, and (3) making video thumbnails in the grid clickable with a film icon badge.

**Primary recommendation:** Add a new `addLayerIntent` signal to `uiStore` that encodes the intent type (static-image, image-sequence, video). Derive asset filter and selection mode from this intent in `ImportedView`. Simplify `AddLayerMenu` to just set the intent and switch editor mode. Move layer creation logic from AddLayerMenu into ImportedView callback handlers.

## Architecture Patterns

### Signal-Based Intent Pattern

The core architectural decision is how `AddLayerMenu` communicates its intent to `ImportedView`. These components are not in a parent-child relationship -- `AddLayerMenu` lives inside `LeftPanel` while `ImportedView` is rendered by `EditorShell` based on `editorMode`. Signals are the established pattern for this (precedent: `pendingNewSequenceId`).

**Recommended: Single `addLayerIntent` signal on uiStore**

```typescript
// New type and signal in uiStore.ts
export type AddLayerIntent =
  | null                          // No add-layer flow active
  | { type: 'static-image' }     // Single-select, images-only
  | { type: 'image-sequence' }   // Multi-select, images-only
  | { type: 'video' }            // Single-select, videos-only

const addLayerIntent = signal<AddLayerIntent>(null);
```

**Why a single signal instead of two (filter + purpose):** The intent fully determines both the filter and the selection mode. A single signal avoids impossible state combinations (e.g., videos-only + multi-select, which no flow needs). ImportedView derives filter and selection mode from the intent:

```
intent null              -> filter: all,    mode: existing behavior
intent { type: 'static-image' }    -> filter: images-only, mode: single-select
intent { type: 'image-sequence' }  -> filter: images-only, mode: multi-select
intent { type: 'video' }           -> filter: videos-only,  mode: single-select
```

### ImportedView Mode Resolution

ImportedView derives all behavior from multiple signals with clear priority:

| Priority | Signal State | Selection | Filter | Header | On Select |
|----------|---|---|---|---|---|
| 1 | `addLayerIntent: static-image` | single | images | "Select image for layer" | Create static-image layer, close |
| 2 | `addLayerIntent: image-sequence` | multi | images | "Select images for sequence (N)" | Toggle, Confirm creates layer |
| 3 | `addLayerIntent: video` | single | videos | "Select video for layer" | Create video layer, close |
| 4 | `pendingNewSequenceId` set | multi | all | "Select key photos (N)" | Toggle, Confirm adds key photos |
| 5 | Active sequence exists | single | all | "Select a key photo" | Add key photo, close |
| 6 | None | -- | all | "Imported Assets" | -- |

**Critical:** `addLayerIntent` takes strict precedence over `pendingNewSequenceId`. These two flows should never be active simultaneously, but if they are, intent wins.

### Layer Creation in ImportedView

Layer creation logic moves from `AddLayerMenu.tsx` into `ImportedView.tsx` as handler functions:

**Static image (single-click):**
```typescript
const handleAddStaticImageLayer = (imageId: string) => {
  const img = imageStore.getById(imageId);
  if (!img) return;
  const layerId = crypto.randomUUID();
  const filename = img.original_path.split('/').pop() ?? 'image';
  layerStore.add({
    id: layerId, name: filename, type: 'static-image',
    visible: true, opacity: 1, blendMode: 'normal',
    transform: defaultTransform(),
    source: { type: 'static-image', imageId: img.id },
    isBase: false,
  });
  layerStore.setSelected(layerId);
  uiStore.selectLayer(layerId);
  uiStore.setAddLayerIntent(null);
  uiStore.setEditorMode('editor');
};
```

**Video (single-click):**
```typescript
const handleAddVideoLayer = (videoId: string) => {

    id: layerId, name: video.name, type: 'video',

    transform: defaultTransform(),
    source: { type: 'video', videoPath: video.path },
    isBase: false,

  uiStore.setAddLayerIntent(null);
```

```typescript
const handleConfirmImageSequence = () => {
```

```typescript
import { Film } from 'lucide-preact';

function VideoThumb({ video, onClick }: { video: VideoAsset; onClick?: () => void }) {
  const videoRef = useRef<HTMLVideoElement>(null);
  useEffect(() => {
    const el = videoRef.current;
    if (!el) return;
    const onMeta = () => {
      if (el.duration && isFinite(el.duration)) el.currentTime = el.duration / 2;

      class={`relative aspect-[4/3] rounded overflow-hidden bg-[var(--color-bg-input)] group${
        onClick ? ' cursor-pointer hover:ring-2 ring-[var(--color-accent)]' : ''
      }`}
      title={video.name}
      onClick={onClick}
    >
      <video ref={videoRef} src={assetUrl(video.path)}
             preload="metadata" muted
             class="w-full h-full object-cover pointer-events-none" />
      {/* Film icon badge -- bottom-right */}
      <div class="absolute bottom-1 right-1 w-4 h-4 rounded-sm flex items-center justify-center"
           style={{ backgroundColor: 'rgba(0,0,0,0.6)' }}>
        <Film size={10} class="text-white" />

      {/* Hover overlay with filename */}

      </div>
    </div>
  );
```

### Close/Cancel Handler

The close handler must be intent-aware to avoid cancel-delete collisions:

```typescript
const handleClose = useCallback(() => {
  const intent = uiStore.addLayerIntent.peek();
  if (intent) {
    // Add-layer flow: just clear intent, no side effects
    uiStore.setAddLayerIntent(null);
  } else {
    // Existing behavior: check for pending sequence cancel-delete
    const pending = uiStore.pendingNewSequenceId.peek();
    if (pending) {
      sequenceStore.remove(pending);
      uiStore.setPendingNewSequenceId(null);
    }
  }
  setSelectedIds([]);
  uiStore.setEditorMode('editor');
}, []);
```

### Video Import Handler (for "+ Import" in video mode)

```typescript
const handleImportVideo = async () => {
  const selected = await open({
    multiple: false,
    filters: [{ name: 'Video', extensions: ['mp4', 'mov', 'webm', 'avi'] }],
  });
  if (!selected) return;
  const dir = projectStore.dirPath.value ?? tempProjectDir.value;
  if (!dir) return;
  const filePath = typeof selected === 'string' ? selected : selected;
  const filename = filePath.replace(/\\/g, '/').split('/').pop() ?? 'video';
  const sep = dir.endsWith('/') ? '' : '/';
  const videosDir = `${dir}${sep}videos`;
  try { await mkdir(videosDir, { recursive: true }); } catch { /* exists */ }
  const destPath = `${videosDir}/${filename}`;
  try { await copyFile(filePath, destPath); } catch (err) { console.error(err); return; }
  imageStore.addVideoAsset({ id: crypto.randomUUID(), name: filename, path: destPath });
};
```

### Anti-Patterns to Avoid
- **Duplicating layer creation logic:** Do NOT keep layer creation in both AddLayerMenu and ImportedView. Move it entirely to ImportedView.
- **Adding new EditorMode values:** Do NOT add 'imported-images' or 'imported-videos' modes. Use the orthogonal intent signal instead.
- **Conflating pendingNewSequenceId with addLayerIntent:** These control different flows. Keep them independent with clear priority.
- **Prop drilling through EditorShell:** Do NOT thread intent props from EditorShell to ImportedView. Use signals (existing pattern).

## Common Pitfalls

### Pitfall 1: Stale Intent After Toolbar Toggle
**What goes wrong:** User opens Imported view via add-layer flow, then clicks the Toolbar "Imported" button to close. Reopening via Toolbar shows filtered/moded Imported view from the previous flow.
**Why it happens:** Toolbar toggles `editorMode` but does not clear `addLayerIntent`.
**How to avoid:** Clear `addLayerIntent` when switching away from 'imported' mode. Best approach: in ImportedView, clear intent on unmount (or when editorMode changes away). Alternative: have the Toolbar button handler clear it explicitly.
**Warning signs:** Opening Imported view from toolbar shows header text from a prior add-layer flow.

### Pitfall 2: Cancel-Delete on Wrong Flow
**What goes wrong:** Closing the Imported view during an add-layer flow accidentally deletes a pending sequence.
**Why it happens:** The existing close handler checks `pendingNewSequenceId` and removes the sequence. If both `addLayerIntent` and `pendingNewSequenceId` are set, the sequence gets deleted even though the user was in an add-layer flow.
**How to avoid:** In the close handler, check `addLayerIntent` first. If it is set, just clear it without touching sequences. Only perform cancel-delete when `addLayerIntent` is null.
**Warning signs:** User cancels video layer creation but their pending sequence disappears.

### Pitfall 3: Video Import Dialog Shows Image Extensions
**What goes wrong:** In video-only mode, the "+ Import" button opens a dialog filtered for images instead of videos.
**Why it happens:** The existing `handleImport` in ImportedView hardcodes image extensions.
**How to avoid:** Make the import handler aware of the current intent. When `addLayerIntent.type === 'video'`, use video extensions. Otherwise use image extensions.
**Warning signs:** User clicks "+ Import" in video mode but only sees image files in the dialog.

### Pitfall 4: Video Asset Not Registered After Import
**What goes wrong:** Video is copied to project directory but does not appear in the grid.
**Why it happens:** Forgetting to call `imageStore.addVideoAsset()` after file copy.
**How to avoid:** The video import flow must: (1) copy file to videos/, (2) call `addVideoAsset()` to register, (3) let user click to create layer. Step 2 is easy to miss.
**Warning signs:** Imported video not visible in the grid; requires app restart to appear.

### Pitfall 5: Multi-Select State Leak Between Flows
**What goes wrong:** Pre-selected images from a prior image-sequence flow appear when starting a new flow.
**Why it happens:** `selectedIds` state in ImportedView persists across intent changes.
**How to avoid:** Reset `selectedIds` whenever `addLayerIntent` changes. Use a `useEffect` watching the intent signal value.
**Warning signs:** Opening a new image-sequence flow shows images already checked from a prior session.

### Pitfall 6: Missing Bidirectional Layer Selection
**What goes wrong:** New layer is created but not visually selected in sidebar or canvas.
**Why it happens:** Only calling `layerStore.setSelected(id)` without `uiStore.selectLayer(id)`, or vice versa.
**How to avoid:** Always call BOTH, per the established Phase 11-04 pattern. All three layer creation handlers must include both calls.
**Warning signs:** Layer appears in sidebar but no transform handles on canvas; or canvas highlights wrong layer.

## Open Questions

1. **Toolbar "Imported" button during add-layer flow**
   - What we know: Toolbar toggles editorMode between 'imported' and 'editor'. Clicking it during add-layer flow will close the view.
   - What's unclear: Should the Toolbar visually indicate an add-layer flow is active?
   - Recommendation: No visual change needed on the Toolbar button. Clicking it simply cancels the flow (via intent cleanup). The header text inside ImportedView already communicates the mode.

2. **Video import: auto-select after import or manual click?**
   - What we know: For images, "+ Import" adds to grid and user then manually selects. For the current AddLayerMenu video flow, import directly creates the layer.
   - What's unclear: Should the new flow auto-create the video layer after import, or add to grid for manual selection?
   - Recommendation: Add to grid for manual selection. This is consistent with the image import behavior and gives users a chance to review/cancel. The imported video appears in the grid immediately thanks to reactive signals.
