**Step 2 -- Modify handleSelect in SequenceItem.** Replace the current `handleSelect` callback (lines 118-126) with:
```typescript
const handleSelect = useCallback(() => {
  if (!editing) {
    const wasActive = sequenceStore.activeSequenceId.peek();
    layerStore.setSelected(null);
    uiStore.selectLayer(null);
    uiStore.selectSequence(seq.id);
    sequenceStore.setActive(seq.id);
    sequenceStore.clearKeyPhotoSelection();
    // Auto-seek timeline playhead to sequence start (only when switching sequences)
    if (seq.id !== wasActive) {
      const layouts = trackLayouts.peek();
      const track = layouts.find(t => t.sequenceId === seq.id);
      if (track) {
        playbackEngine.seekToFrame(track.startFrame);
      }
    }
  }
}, [seq.id, editing]);
```

Key implementation details:
- `wasActive` is captured BEFORE `setActive(seq.id)` changes the signal value
- Guard `seq.id !== wasActive` prevents playhead jumping when re-clicking the already-active sequence
- `trackLayouts.peek()` reads without creating a signal subscription (correct for event handler context)
- `if (track)` guard handles empty sequences that have no entry in trackLayouts (graceful no-op)
- `playbackEngine.seekToFrame()` handles all downstream sync: displayFrame, activeSequence, player
- The dependency array `[seq.id, editing]` remains unchanged -- no new reactive dependencies added

**Do NOT:**
- Use `.value` instead of `.peek()` inside the callback (would create unwanted subscriptions)
- Use `activeSequenceStartFrame.value` after `setActive()` (signal may not update synchronously)
- Add playback-running guards (match timeline click-to-seek behavior which also works during playback)
