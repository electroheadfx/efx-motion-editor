    **Part A: Enhance projectStore**

    Rewrite `Application/src/stores/projectStore.ts` to add project file management capabilities:

    Add signals:
    - `filePath: signal<string | null>(null)` -- absolute path to .mce file (null = never saved)
    - `dirPath: signal<string | null>(null)` -- absolute path to project root directory
    - `isDirty: signal(false)` -- true when unsaved changes exist
    - `isSaving: signal(false)` -- true during save operation (prevents concurrent saves)

    Add methods:
    - `createProject(name: string, fps: number, dirPath: string)` -- calls `projectCreate` IPC, updates all store signals, sets dirPath, clears filePath (not yet saved to .mce)
    - `saveProject()` -- gathers state from all relevant stores (projectStore, sequenceStore, imageStore), builds an MceProject object, calls `projectSave` IPC. Sets isDirty to false on success. Uses isSaving mutex to prevent concurrent saves. Updates `modified_at` timestamp.
    - `openProject(filePath: string)` -- calls `projectOpen` IPC, hydrates all stores (projectStore, sequenceStore, imageStore) from the MceProject data. Sets filePath, dirPath, isDirty=false. Derives dirPath from filePath's parent.
    - `saveProjectAs(filePath: string)` -- sets filePath then delegates to saveProject(). Updates recent projects.
    - `markDirty()` -- sets isDirty to true.
    - `buildMceProject(): MceProject` -- assembles current state into MceProject struct for saving.
    - `hydrateFromMce(project: MceProject, projectRoot: string)` -- loads MceProject data into projectStore, sequenceStore, and imageStore. Converts relative paths to absolute for image store.
    - `closeProject()` -- resets all stores, clears filePath/dirPath.

    Keep all existing methods (setName, setFps, setResolution, loadFromData, reset) but have setName, setFps, setResolution also call `markDirty()`.

    **IMPORTANT: Store hydration on open:** When opening a project, the MceProject contains sequences, key photos, and image references with RELATIVE paths. The `hydrateFromMce` method must:
    1. Set projectStore signals (name, fps, width, height)
    2. Convert MceImageRef relative paths to absolute paths (join with projectRoot) and populate imageStore
    3. Convert MceSequence/MceKeyPhoto to frontend Sequence/KeyPhoto types and populate sequenceStore
    4. Set filePath and dirPath

    1. Read projectStore signals (name, fps, width, height)
    2. Convert imageStore images to MceImageRef with relative paths (strip dirPath prefix)
    3. Convert sequenceStore sequences to MceSequence with MceKeyPhoto

    **Part D: Create Auto-Save Module**

    Create `Application/src/lib/autoSave.ts`:

    ```typescript
    import { effect } from '@preact/signals';
    import { projectStore } from '../stores/projectStore';
    import { sequenceStore } from '../stores/sequenceStore';

    let saveTimeout: ReturnType<typeof setTimeout> | null = null;
    let intervalId: ReturnType<typeof setInterval> | null = null;
    let disposed = false;

    function scheduleSave() {
      if (disposed) return;
      if (saveTimeout) clearTimeout(saveTimeout);
      saveTimeout = setTimeout(() => {
        if (projectStore.filePath.value && projectStore.isDirty.value) {
          projectStore.saveProject();
        }
      }, 2000); // 2-second debounce
    }

    let disposeEffect: (() => void) | null = null;

    /** Start auto-save watchers. Call once after app initializes. */
    export function startAutoSave(): void {
      disposed = false;

      // Watch for meaningful store changes (effect auto-subscribes to accessed signals)
      disposeEffect = effect(() => {
        // Access signals to subscribe
        projectStore.name.value;
        projectStore.fps.value;
        projectStore.width.value;
        projectStore.height.value;
        sequenceStore.sequences.value;
        // Trigger debounced save
        scheduleSave();
      });

      // Periodic save every 60 seconds as safety net
      intervalId = setInterval(() => {
        if (projectStore.filePath.value && projectStore.isDirty.value) {
          projectStore.saveProject();
        }
      }, 60_000);
    }

    /** Stop auto-save watchers. Call on app shutdown or project close. */
    export function stopAutoSave(): void {
      disposed = true;
      if (disposeEffect) {
        disposeEffect();
        disposeEffect = null;
      }
      if (intervalId) {
        clearInterval(intervalId);
        intervalId = null;
      }
      if (saveTimeout) {
        clearTimeout(saveTimeout);
        saveTimeout = null;
      }
    }
    ```

    **CRITICAL: Auto-save only fires when filePath is set (project has been saved at least once).** Unsaved projects (filePath=null) are NOT auto-saved -- the temp dir serves as a recovery mechanism for those.
