## Deviations from Plan

### User-reported visual parity gaps fixed before closure

- Missing/background frames initially showed flat/tinted color instead of real paper texture.
- EFX Motion and standalone Physics Paint initially differed in paper texture tile scale.
- Painted Roto frames initially carried coarse embedded paper because output was composited instead of transparent strokes.
- Standalone Physics Paint initially had a background/canvas ratio mismatch with cropped height.
- Passing full project dimensions into the engine initially regressed save/render cache performance; this was fixed by separating project dimensions from bounded working engine dimensions.

These deviations were resolved before validation was marked complete.

### Package check caveat

`pnpm --filter @efxlab/efx-physic-paint check` was not used as a final gate because the package-local check has a pre-existing Vitest type-resolution issue in package tests. The package build passed, and the app-focused/full test suite covered the integrated behavior.
