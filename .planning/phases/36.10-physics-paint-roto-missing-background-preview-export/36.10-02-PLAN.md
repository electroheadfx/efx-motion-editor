## Phase Goal

**As a** stop-motion animator, **I want to** render missing Roto frames as transparent or paper-background-only frames, **so that** gaps between painted keys and trailing frames preview/export consistently without accidental paint content.

<objective>
Implement the bounded cache/store slice for Phase 36.10: interior background-only support can exist for preview/export stability, but edge spans stay dynamic and real-key alpha caches remain clean.

Purpose: Users need background-only interior gaps to be stable and visible without corrupting actual painted keys or creating infinite trailing cache entries.
Output: Tested store/type changes for bounded derived support, exact replacement by real key painting, recomputation after key layout changes, and no paper pixels baked into real-key paint cache data.
</objective>
