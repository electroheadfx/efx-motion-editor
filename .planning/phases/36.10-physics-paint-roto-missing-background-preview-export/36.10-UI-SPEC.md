## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#3e3f41` | Existing Physics Paint topbar, right panel, and workflow strip shell surfaces |
| Secondary (30%) | `#343638` | Existing cards, icon buttons, Roto transport, right sections, and compact control containers |
| Accent (10%) | `#2d5be3` | Active/primary existing controls only: Save current, active transport/control states, focus rings, selected swatches |
| Destructive | `#ff6666` | Existing destructive/error copy and destructive key utility treatments only |

Accent reserved for: existing active buttons, existing primary `Save current`, active mode/transport states, selected paper/color swatches, focus-visible outlines. Do not use accent to mark background-only cells, missing transparent frames, hover-only affordances, or all interactive controls.

**Roto state colors to preserve:**
- Empty: `#4d535a` / neutral gray.
- Cached real key: `#2d6f48` with green border `rgba(91, 216, 143, 0.78)`.
- Editable/current: `#a33d73` / pink, plus current orange outline.
- Generated/render-only: `#365ed6` / blue.
- Background only: existing striped gray treatment `repeating-linear-gradient(45deg, #505860 0 4px, #5b6269 4px 6px)` with border `#747980`.
- Dirty/pending: existing orange `#f59e0b` corner/pending animation.

**Missing-frame visual rule:** Transparent missing frames must contribute no visible pixels in preview/export. Background-only missing frames must show only configured paper/background pixels, never green cached-key styling on the canvas and never baked paper pixels inside real-key alpha caches.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Save current` |
| Empty state heading | `No Roto content on frame {frame}` |
| Empty state body | `This frame will preview/export transparent unless the current paper background setting resolves it as background only.` |
| Error state | `Missing-frame render failed for frame {frame}. Keep the current frame unchanged and try preview/export again.` |
| Destructive confirmation | `None for Phase 36.10. Reuse existing Delete/Clear confirmations if key utilities are touched; do not add a new destructive flow.` |

Additional required status/label copy:

| State | Copy |
|-------|------|
| Background-only cell title/aria | `Background only on frame {frame}` |
| Transparent missing frame compact status | `Frame {frame}: transparent missing Roto frame` |
| Interior background-only compact status | `Frame {frame}: background only between real Roto keys` |
| Leading/trailing background-only compact status | `Frame {frame}: background only from current paper setting` |
| Replacement success status | `Frame {frame} saved as a real Roto key` |
| Replacement pending status | `Saving frame {frame}…` |
| Alpha separation guard copy for developer-facing failure only | `Background-only support must not replace real-key paint alpha.` |

Copy constraints:
- Use `Roto`, `Background only`, `transparent`, `real Roto key`, and `Save current` exactly with this capitalization.
- Do not introduce `missing-frame mode`, `fallback mode`, `generated`, or `interpolation` copy as a new control concept.
- Status copy must be compact enough for the existing 66px Roto status stack. Prefer one line, max 80 characters where possible.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Preview missing transparent frame | Draw no Roto pixels for that frame. Do not snap to nearest cached key, do not show stale cached content, and do not create a cache entry. |
| Preview missing background-only interior frame | Show only paper/background result. If materialized, mark it as render support, not editable paint. |
| Preview leading/trailing missing frame | Resolve dynamically from current settings. Never persist leading/trailing background-only cache entries. |
| Export missing frame | Match the preview visual rule for the same frame/settings exactly. Export must use the same resolver path, not a duplicate branch. |
| Workflow strip cell state | Keep the existing striped `Background only` cell treatment and title/aria label. Do not add a new legend category unless the existing legend already renders the state list. |
| Opening/painting a background-only interior frame | The frame becomes a real Roto key after `Save current`; remove only that frame's derived background-only support marker/cache. |
| Key utility shift/delete/replace/paste | Recompute bounded interior background-only support from the current real-key layout; do not copy or paste background-only support as a real key. |
| Cached Roto playback | Preserve Phase 36.9 real-key-only playback. Play/Stop must not include transparent/background-only fallback frames. |
| Failure | Keep current frame/canvas unchanged, show compact error copy in existing status/log surface, and do not mutate real-key alpha caches. |

Keyboard/shortcut contract: No new shortcuts. Existing Space playback behavior remains Phase 36.9 behavior and must not be changed by missing-frame rendering.

## Visual QA / UAT Contract

The executor must leave the UI testable through existing visible surfaces:

1. Interior transparent gap: live preview shows no Roto pixels; exported PNG sequence matches.
2. Interior background-only gap: live preview shows paper/background only; exported PNG sequence matches.
3. Leading/trailing background-only frame: live preview/export resolve dynamically without new cache/cell growth beyond bounded export rendering.
4. Existing real cached keys beside gaps remain full-strength cached paint visuals.
5. Painting a background-only interior frame and pressing `Save current` changes that exact cell/frame into a real Roto key and keeps other support frames bounded.
6. Cached Roto playback still plays only real cached Roto keys and skips transparent/background-only fallback frames.
