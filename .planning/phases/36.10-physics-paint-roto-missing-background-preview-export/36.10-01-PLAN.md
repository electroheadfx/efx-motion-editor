## Phase Goal

**As a** stop-motion animator, **I want to** render missing Roto frames as transparent or paper-background-only frames, **so that** gaps between painted keys and trailing frames preview/export consistently without accidental paint content.

<objective>
Create the tested shared resolver slice for Phase 36.10: one pure missing-frame rule that classifies leading/interior/trailing spans, chooses transparent/background-only from existing settings, and remains the path used by preview and export.

Purpose: This is the vertical foundation for the user-facing preview/export behavior because both live preview and exported PNGs depend on the same draw instruction.
Output: Span-aware resolver tests, resolver implementation, preview call-site wiring, and export delegation contract tests.
</objective>
