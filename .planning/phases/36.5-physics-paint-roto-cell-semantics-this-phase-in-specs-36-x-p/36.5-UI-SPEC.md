# Phase 36.5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 36.5 makes Physics Paint Roto timeline cells trustworthy at a glance. The UI must distinguish five base frame meanings without hiding them behind dirty/current/pending overlays:

1. `empty`: no Roto content is available for the frame.
2. `cached`: a saved rendered Roto frame exists.
3. `editable/current`: the frame is currently open for Roto editing.
4. `generated`: the frame is derived output rather than a real edited key.
5. `background-only`: the frame intentionally shows only the background/reference result.

Dirty or pending work is an overlay on the base meaning, never a replacement base color.

Sources: `SPECS/36.x-phases/phase-36.5-cell-semantics/spec-36.5-cell-semantics.md`, `.planning/ROADMAP.md`, `.planning/STATE.md`, existing Physics Paint workflow CSS, and `SPECS/36.x-phases/timeline-ui/*`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, compact cell legend gaps |
| sm | 8px | Compact element spacing, status pill padding, timeline control gaps |
| md | 16px | Default element spacing, grouped controls, side padding where the strip has room |
| lg | 24px | Timeline cell row height, major control touch height where already established |
| xl | 32px | Major control clusters, status/legend grouping |
| 2xl | 48px | Major section breaks; avoid inside the compact workflow strip unless available width permits |
| 3xl | 64px | Page-level spacing; not used in the compact bottom workflow strip |

Exceptions: New or changed Phase 36.5 UI spacing must use only the approved spacing tokens above. Roto timeline cells may preserve the existing compact 13px width and 24px height, but any new or changed cell gaps must use the approved 4px `xs` token; icon-only navigation controls may remain 24px wide because this surface is desktop/editor-only and already follows the Physics Paint workflow strip density.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 600 | 1.2 |
| Heading | 16px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

Typography rules:

- Use exactly these four sizes in the Phase 36.5 cell-semantics UI.
- Use exactly two weights: regular `400` and semibold `600`.
- Frame numbers and status counts use tabular numerals.
- Cell state names in legends and status copy use sentence case: `Empty`, `Cached`, `Current`, `Generated`, `Background only`, `Unsaved`, `Saving`.
- Do not use all-caps for the new semantic labels except existing compact control labels already inherited from the surrounding UI.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#37393A` | Timeline lane background, cell row background, lower workflow strip body |
| Secondary (30%) | `#3E3F41` | Workflow strip shell, header/control surface, active tab backing |
| Accent (10%) | `#F5A623` | Current frame outline, playhead, primary semantic focus, `Save current` emphasis only |
| Destructive | `#FF6666` | Delete/discard/destructive confirmation copy and borders only |

Accent reserved for: current-frame outline, playhead marker, selected/current context capsule, primary `Save current` action, and focus ring where the active Roto cell receives keyboard focus. Never use accent as the base fill for cached, generated, empty, or background-only cells.

### Roto Cell Semantic Palette

| Base Meaning | Fill | Border | Required Non-Color Cue | Interaction Meaning |
|--------------|------|--------|------------------------|---------------------|
| Empty | `#4D535A` | `#3D454D` | Hollow/flat cell with no inner mark | No Roto cache or editable content exists for this frame. |
| Cached | `#2D6F48` | `rgba(91, 216, 143, 0.78)` | Inset saved mark: 1px inner highlight `rgba(91, 216, 143, 0.22)` | Saved rendered Roto frame exists and may reopen as cached visual reference. |
| Editable/current | `#A33D73` | `rgba(255, 126, 196, 0.82)` plus current outline when active | Real-key/editable dot: centered 4px white dot if editable content exists | This is a real edited key/current editable frame, not merely cached pixels. |
| Generated | `#365ED6` | `rgba(111, 144, 255, 0.82)` | 2px horizontal connector line through the cell | Derived/render-only output; not an editable target. |
| Background-only | `#505860` | `#747980` | Diagonal hatch at 45 degrees with low-contrast stripe | Intentional background/reference-only result; not missing content. |

### Overlay Palette

| Overlay | Visual Treatment | Rule |
|---------|------------------|------|
| Current | 2px outline `rgba(245, 166, 35, 0.9)`, 1px offset, subtle glow | Applies on top of any base meaning. Must not change base fill. |
| Dirty/unsaved | Top-right 5px amber corner wedge `#F59E0B` plus `Unsaved` status text | Overlay only; do not recolor the whole cell. |
| Pending/saving | Dashed border and low-frequency opacity pulse on border only | Base fill remains visible. Pulse duration 1.2s minimum; no rapid flashing. |
| Error | 1px destructive outer ring and inline status copy | Error ring may combine with base fill but must not use destructive as the base fill. |
| Disabled/read-only | 55% opacity on cell contents plus preserved base fill | Use only when interaction is unavailable; state must remain identifiable. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Save current |
| Empty state heading | No Roto frames yet |
| Empty state body | Paint this frame, then choose Save current to create the first cached Roto cell. |
| Error state | Roto cell state could not be updated. Save current again or reopen Physics Paint if the frame does not refresh. |
| Destructive confirmation | Delete Roto frame: Delete the cached Roto frame for frame {frame}? This cannot be undone. |

Additional visible copy:

- Legend label: `Roto cell states`
- Status summary: `Frame {frame}: {state}`
- Dirty summary: `Unsaved changes on frame {frame}`
- Pending summary: `Saving frame {frame}...`
- Cached summary: `Cached frame {frame}`
- Generated summary: `Generated frame {frame} (render-only)`
- Background-only summary: `Background only on frame {frame}`
- Empty summary: `No Roto content on frame {frame}`

Do not use ambiguous legacy labels such as `occupied`, `cached-only`, or `editable-session` in visible UI. Those names may remain internal only if tests assert the user-facing copy above.

## Interaction Contract

### Cell Interaction

| Interaction | Required Behavior |
|-------------|-------------------|
| Hover a cell | Show a compact tooltip or inline status update with `Frame {frame}: {state}`. If dirty, append `, unsaved`. |
| Focus a cell | Use the same current/accent outline treatment as hover plus keyboard-visible outline. |
| Click empty cell | Navigate/open that frame for Roto editing if allowed by existing workflow; status becomes `Frame {frame}: Empty`. |
| Click cached cell | Open as cached visual reference, preserving full-strength cached pixels per Phase 36.3 decision. |
| Click generated cell | Do not silently edit generated output. Redirect/open nearest real editable key only if existing behavior supports it; otherwise show `Generated frame {frame} is render-only.` |
| Click background-only cell | Show background/reference result without implying missing Roto content. |
| Save current | Preserve current base state semantics and add/remove dirty/pending overlays according to operation progress. |

### Legend and Trust Requirements

- A persistent or immediately visible legend must be available in the workflow strip or adjacent status area for all five base meanings plus dirty/pending overlays.
- The legend must fit the compact strip: use 11px labels, 13x24px swatches, 8px item gaps, and wrap if needed.
- The legend must include non-color cues: dot for editable/current, connector for generated, hatch for background-only, dashed border for pending, corner wedge for dirty.
- Color cannot be the only differentiator between cached, editable/current, generated, and background-only.

### State Precedence

Render order from bottom to top:

1. Timeline lane background.
2. Base cell fill and base border.
3. Non-color base cue (dot, connector, hatch, inset saved mark).
4. Dirty/pending/error overlays.
5. Current frame outline/playhead.
6. Focus-visible outline.

Current frame and dirty/pending are orthogonal. For example, a dirty current cached frame must remain visually cached, with dirty corner and current outline layered on top.
