## Decisions Made

- The current-frame affordance is an amber outline/glow so it does not overwrite cached, generated, or background-only base semantics.
- Dirty and saving states are overlay treatments: dirty uses a top-right amber wedge and pending uses a dashed border pulse.
- Generated/render-only, cached, editable/current, and background-only states remain visually distinct through both color and non-color cues.
- The visual checkpoint mismatch was treated as in-scope because cached/saved cells were visually untrustworthy when standalone navigation showed blank content.
- The fix stays MVP-only: it shows cached PNG references after navigation but does not restore editable strokes, autosave on navigation, save on leave, or add close behavior.

## Deviations from Plan

- The human visual checkpoint initially failed because saved Roto PNGs remained visible in the parent EFX Motion cache but disappeared inside the standalone Physics Paint window after navigating between saved frames.
- A narrow regression fix was added before approval: successful explicit Roto saves now update `launchContext.cachedRotoFrames` inside the standalone session, and delete/save-empty removes the local cached frame.

## Issues Encountered

- The visual surface showed a trustworthy saved/cached cell state while the standalone canvas could still go blank after navigation.
- The first regression test assertion scoped helper internals to the wrong source block; it was corrected to assert helper, callback, and flush responsibilities separately.
