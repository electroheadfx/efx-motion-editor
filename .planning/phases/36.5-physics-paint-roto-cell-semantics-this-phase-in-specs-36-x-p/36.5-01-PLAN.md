<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cache metadata→UI helper | Existing `PhysicPaintRotoCacheFrame` metadata is interpreted into user-visible state. |
| frame number→label text | Numeric frame data becomes visible `label`, `title`, and `ariaLabel` strings. |

## STRIDE Threat Register

ASVS L1 applies to this UI-state surface through input validation and output encoding expectations: frame numbers must be validated before they become labels, helper view-model copy must remain consistent with rendered classes, and no HTML injection APIs may be introduced. Use a HIGH blocking threshold for misleading Roto cell state risks: any ambiguity between editable/current, cached/saved, generated/render-only, background-only, dirty/unsaved, or pending/saving must block completion until tests or copy/class contracts make the state trustworthy, including non-color cues for later UI rendering.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36.5-01 | Spoofing | `getRotoCellViewModel` | mitigate | Derive one typed `baseMeaning` from validated metadata and test all five meanings. |
| T-36.5-02 | Tampering | dirty/pending overlay derivation | mitigate | Keep `dirty` and `pending` in `overlays`, never as replacement base meanings. |
| T-36.5-03 | Repudiation | status/label copy | mitigate | Generate exact state copy from the same helper used by UI rendering. |
| T-36.5-04 | Tampering | frame labels | mitigate | Render labels from numeric frame values and avoid HTML injection APIs. |
| T-36.5-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install tasks exist in this plan. |
</threat_model>
