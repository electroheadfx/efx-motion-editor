## Decisions Made
- FullscreenOverlay renders its own Preview instance rather than reparenting the existing canvas (simpler, avoids complex CSS/DOM manipulation)
- Capture-phase keydown listener scopes all keyboard input in fullscreen (ESC, arrows, Space pass through; everything else blocked)
- Defense-in-depth approach: isFullscreen.peek() guards added to all tinykeys handlers as backup for the capture-phase blocker
- Play in fullscreen always forces isFullSpeed=true before playbackEngine.start() per user requirement
- Entering fullscreen does NOT auto-start playback per user decision
