## Human Verification Required

### 1. ISO-03: Isolated Playback Skips Non-Isolated Sequences

**Test:** Open a project with 3+ content sequences laid out sequentially. Isolate sequences 1 and 3 (skip 2). Press play.
**Expected:** Playhead advances through sequence 1's frames, jumps directly to the start of sequence 3, advances through sequence 3's frames — never visits sequence 2's frame range.
**Why human:** Frame-skip timing and accurate playhead position during live rAF loop cannot be verified statically.

### 2. ISO-04: FX/Overlay Sequences Render During Isolated Playback

**Test:** Add an FX overlay sequence that overlaps an isolated content sequence's frame range. Play with isolation active.
**Expected:** The FX content composites correctly at every frame of the isolated sequence. No visual artifacts from the isolation mode.
**Why human:** Video compositing output is visual; architecture (isolation only gates playhead movement, not rendering) is correct per code review.

### 3. ISO-05: Loop ON Seamlessly Loops Through Isolated Sequences

**Test:** Enable loop toggle (Repeat1 icon, accent background). Isolate two non-adjacent sequences. Press play.
**Expected:** After the last isolated sequence ends, playback wraps seamlessly to the start of the first isolated sequence without stopping.
**Why human:** Loop timing and visual seamless-ness requires live observation.

### 4. ISO-06: Loop OFF Stops at End of Isolated Range

**Test:** Ensure loop is disabled (Repeat icon, muted background). Isolate a sequence. Press play.
**Expected:** Playback stops when the isolated range ends. Playhead seeks to the start of the first isolated sequence (not the global timeline start).
**Why human:** Stop-and-seek behavior must be observed; the code path (seek + stop + return) is verified but runtime timing needs confirmation.

### 5. ISO-08: Loop Preference Persists Across App Restarts

**Test:** Toggle loop ON. Quit and reopen the app.
**Expected:** Loop button shows ON state (Repeat1 icon, `var(--color-accent)` background). builder-config.yaml contains `loop_enabled: true`.
**Why human:** Tauri IPC persistence requires app quit/reopen cycle; the entire write/read chain is code-verified but filesystem persistence needs runtime confirmation.

### 6. ISO-09: Visual Sync Between Timeline and Sidebar

**Test 1:** Click a sequence name label on the timeline to isolate it. Without activating it in the sidebar, check the sidebar — if the sequence is the active sequence, the accent bar should be orange.
**Test 2:** Click a sidebar accent bar to isolate that sequence. On the timeline, the sequence name label background should be orange.
**Expected:** Both UIs respond to the same `isolatedSequenceIds` signal. Changes in one immediately appear in the other.
**Why human:** Cross-component reactive visual sync requires live observation with both panels visible.
