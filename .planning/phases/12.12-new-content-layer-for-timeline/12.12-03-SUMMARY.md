## Decisions Made
- CSS variable colors (--sidebar-dot-green, --sidebar-dot-blue) are resolved once in the theme color cache, not per-frame -- consistent with existing theme color pattern
- resolveTrackColor is a separate method rather than inline checks, keeping drawFxTrack readable
- clearFxLayerSelection now checks owning sequence kind instead of isFxLayer type -- correctly handles content-overlay layers which use content layer types (static-image etc) but live in non-content sequences
- Keyframe hover detection gate relaxed from "not FX area" to "not ruler" since content-overlay tracks in the FX area need diamond hover/click

- **Fix:** Changed to check owning sequence kind (`kind !== 'content'`) instead of layer type
- **Files modified:** Application/src/components/timeline/TimelineInteraction.ts
