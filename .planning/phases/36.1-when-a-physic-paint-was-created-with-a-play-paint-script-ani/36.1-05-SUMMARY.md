## Decisions Made

- Kept sequential playback as the only Play animation behavior for this phase, matching D-23 and avoiding a user-facing mode toggle.
- Used stroke point count as the allocation weight because it is already available from recorded stroke data and keeps distribution linear in stroke count.
- For frame counts smaller than stroke counts, compressed overflow strokes onto the final available frame so the final partial render includes the complete drawing before `renderAllStrokes()` finishes playback.
