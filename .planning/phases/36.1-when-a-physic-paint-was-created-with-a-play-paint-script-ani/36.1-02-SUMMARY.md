# Phase 36.1 Plan 02: Physics Paint Bridge Launch Selection Summary

Scrubber-aware Physics Paint launches now reopen only the saved Play script that contains the editor frame, while gaps launch Roto with no-overlap Play duration constraints.

## Objective

Make the editor-to-standalone launch bridge choose the correct saved Play script only when the editor scrubber is inside that range; otherwise launch Roto at the current frame with a max Play frame count that prevents overlap with the next saved script.

## Implementation Summary

### Saved Play Script Selection

- `createPhysicPaintLaunchContext` now truncates the editor scrubber frame and checks `physicPaintStore.findPlayScriptRangeAtFrame(layerId, frame)`.
- When the frame is inside a saved Play script range, the launch context opens Play mode with:
  - `startFrame` and `playStartFrame` set to the range start.
  - `playFrameCount` set to the saved range length.
  - `selectedPlayScriptId` set to the containing range id.
  - `previewFrame` set to the scrubber-relative frame offset.
  - `editableSource: 'play'` and the range-specific editable state.

### Gap/Roto Launch Constraints

- Frames outside all saved ranges now launch `workflowMode: 'roto'` at the current editor frame.
- Gap contexts omit `selectedPlayScriptId`, `playStartFrame`, and `playFrameCount`.
- Gap contexts include `maxPlayFrameCount` from `physicPaintStore.getMaxPlayFrameCountFromGap` and a user-facing `maxPlayFrameCountReason`.
- After the last saved range, the max duration falls back to `PHYSIC_PAINT_MAX_APPLY_FRAMES` instead of choosing the nearest saved range.

### Encoded Launch Payloads

- Existing `openPhysicPaintCanvas` validation path remains intact: invalid launch contexts return `{ ok: false, error: 'Invalid physics paint launch context' }` before any window opens.
- Browser fallback URL encoding carries the new saved-script and gap fields because it serializes the validated extended launch context.
- Tauri native command payloads continue to receive the same validated context object.
- Browser apply listener same-origin rejection (`event.origin !== window.location?.origin`) and typed apply payload validation remain unchanged.
