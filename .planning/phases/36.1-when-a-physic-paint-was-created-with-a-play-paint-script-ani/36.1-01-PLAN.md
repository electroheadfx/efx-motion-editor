<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project file -> store hydration | Saved `physic_paint_outputs` can contain malformed Play script ranges and cached frame metadata. |
| Standalone window -> editor apply payload | Cross-window `apply-play-canvas` payload can attempt overlap, invalid frames, or forbidden internals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36.1-01 | Tampering | `normalizePhysicPaintPlayScriptRanges` | mitigate | Validate id, startFrame, frameCount, sorted no-overlap ranges, serialized editable state, and reject malformed saved ranges before hydration or launch. |
| T-36.1-02 | Tampering | `physicPaintStore.applySequence` | mitigate | Enforce D-06 overlap rejection before writing cached frames; return a failed `PhysicPaintApplyResult` instead of partially mutating store state. |
| T-36.1-03 | Information Disclosure | `isPhysicPaintApplyPayload` | mitigate | Preserve forbidden `engine`/`internals` field rejection while adding range fields. |
| T-36.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned or required for this phase. |
</threat_model>
