- Inactive saved Play markers: subdued neutral `#B9BEC4` line with neutral endpoint dots at reduced opacity inside the Physic Paint layer bar.
- Warning/limit copy: `#F59E0B` amber for max-duration and non-destructive constraint messages.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Render` |
| Empty state heading | `No Play scripts in this layer` |
| Empty state body | `Move the editor scrubber to an open gap, choose Play paint, then Render and Save play to create a cached Play range.` |
| Error state | `This Play range would overlap another saved script. Move to an open gap or shorten the duration before rendering.` |
| Destructive confirmation | `Delete Play script`: `Delete this saved Play script and remove its cached frames from the Physics Paint layer? This cannot be undone.`; `Clear script`: `Clear this script and remove its cached line from EFX Motion until you Render and Save play again?`; `Replace cached range`: `Rendering this script will replace the cached frames for this range.` |

Additional required copy:
- Editor launch buttons: `Roto paint` and `Play paint`. Replace the single `[open fx paint canvas]` surface with these two clear buttons in the editor Physics Paint properties panel.
- Opening state copy: `Opening Roto paint...` and `Opening Play paint...`.
- Successful editor launch status: `Opened Roto paint at frame {frame}.` or `Opened Play paint range {startFrame}–{endFrame} at preview frame {relativeFrame}.`
- Gap duration limit message: `Maximum Play duration from this frame: {count} frame(s), until the next saved script.`
- Exact one-frame limit message: `Only frame {frame} is available before the next saved script.`
- Cached preview status: `Cached frames ready for preview.`
- Dirty/live preview status after Clear/remake: `Live preview active. Render and Save play to update the cached frames.`
- Render button tooltip/body copy: `Render cached frames for this script. Watch full playback in EFX Motion.`
- Do not use: `[open fx paint canvas]`, `Play` as an icon-only render action, `Sequential/Parallel`, ASCII marker examples, frame badges inside default nested timeline markers, or Roto/Play tabs inside a mode-specific opened Physics Paint window.

## Phase 36.1 Visual and Interaction Contract

### 1. Editor Physics Paint Opening Surface

| State | Contract |
|-------|----------|
| Default selected Physics Paint layer | The `Standalone Canvas` section shows two full-width or side-by-side text buttons: `Roto paint` and `Play paint`. The old single `[open fx paint canvas]` label must not appear. |
| Scrubber inside a saved Play range | `Play paint` opens that exact saved script range and starts the standalone preview at the scrubber-relative frame. `Roto paint` remains available only if the current frame can accept Roto work without confusing the selected Play script; if implementation keeps both buttons enabled, status copy must make the selected target explicit. |
| Scrubber in a gap | `Roto paint` opens Roto at the exact editor current frame. `Play paint` opens Play creation for the gap with duration clamped to the no-overlap space before the next saved script. |
| Gap before next saved script | The launch context and visible Physics Paint controls must show the current editor start frame, not frame 0. Example: opening at frame 11 before a saved script at frame 12 shows frame 11 as the first/only available Play frame and maximum duration 1. |
| Invalid context | Disable both launch buttons and show `Select a physics paint layer and frame first.` |

### 2. Standalone Roto/Play Mode Contract

| Requirement | Contract |
|-------------|----------|
| Mode selection | Roto and Play are distinct opening modes selected from the editor. Remove the Roto/Play tab switch inside EFX Physics when opened in a selected mode. |
| Header/mode label | Show the active mode as a non-tab label: `Roto paint` or `Play paint`. |
| No accidental switching | Do not let the user switch modes inside the standalone window for this phase; returning to the editor and choosing the other button is the mode switch. |
| Timeline origin | The standalone bottom timeline starts at the launch start frame for gap opens, not at frame 0 unless the launch frame is 0. |
| Local preview | Standalone scrubbing changes only local Physics Paint preview state and must not move the EFX Motion editor playhead. |

### 3. Main Editor Nested Timeline Markers

| Element | Contract |
|---------|----------|
| Container | Draw all saved Play scripts as nested mini-ranges inside the existing Physic Paint FX/layer bar. Do not add rows, badges, DOM overlays, or ASCII/text ranges. |
| Geometry | Marker x and width use the same frame-to-pixel math as the parent layer bar: frame scale, zoom, horizontal scroll, and clipping. Markers must remain aligned after zoom/scroll. |
| Shape | Each marker is a centered horizontal rounded line inside the layer block with circular start/end endpoint dots. Minimum visible line height 2px inactive, 3px active/cached. Endpoint dots 8–10px diameter. |
| Active state | The scrubber-contained saved Play range uses `var(--color-accent)` blue with a subtle glow. This is the only default accent marker state. |
| Inactive state | Inactive saved Play ranges use subdued neutral line/dots, around 55–70% opacity, still visible inside the layer bar. |
| Cached state | If cached frames exist and are valid, the marker may include a green inner line or glow, but active blue remains the selected/scrubber-contained signal. |
| Missing/stale cache | Stale or cleared scripts must not show as active green cached ranges. Use neutral/stale styling until Render + Save play regenerates cache. |
| Text | Do not show frame labels by default inside markers. The main timeline ruler provides frame context. Tooltips/status copy may include ranges if existing UI supports it. |

### 4. Play Duration and No-Overlap Contract

| Scenario | Contract |
|----------|----------|
| New Play from gap | Maximum duration equals available frames from current start frame up to before the next saved Play script on the same layer. |
| One-frame gap | Duration control is clamped to 1; next/previous/end controls must not imply extra available frames. Show `Only frame {frame} is available before the next saved script.` |
| Default duration too large | Clamp immediately on launch and show `Maximum Play duration from this frame: {count} frame(s), until the next saved script.` |
| Attempted overlap | Prevent the invalid save/render path and show overlap error copy. Do not partially mutate saved scripts, cached frames, or visible markers. |
| Existing saved script reopen | The saved range length is authoritative. Reopen with that range; do not recompute duration from the surrounding gap unless the user is creating a new script. |

### 5. Cached Preview, Render, Save, Clear

| Action/State | Contract |
|--------------|----------|
| Existing saved Play script opened | Cached rendered frames are active immediately. The spreadsheet/timeline frames for the cached range are green on first open, and next/previous/start/end/local scrub preview those cached frames without requiring a re-render. |
| Render action | Replace any icon-only play/render control with a text button labeled `Render` in both Roto and Play contexts where rendering is required. This button creates/refreshes cached frames; it is not full animation playback. |
| Full playback | Full animation playback is watched in EFX Motion from cached frames, not via a standalone Play icon button. |
| Save play | `Save play` publishes/replaces the saved cached Play range after Render. Preview and scrub remain local until this explicit action. |
| Clear script | Clearing a script removes the script line/cache from the EFX Motion Physics Paint layer immediately until the user renders and saves again. The standalone switches to live preview state and shows dirty/live preview copy. |
| Delete from editor | Deleting a saved script from EFX Motion removes its nested marker and cached frames after destructive confirmation. |
| Frame changes | Any clear/delete/save/render mutation must bump the Physics Paint version and update visible EFX Motion layer/marker state without requiring app reload. |

### 6. Sequential Playback Visibility

| Requirement | Contract |
|-------------|----------|
| User model | Play animation draws recorded strokes in order like one artist hand. |
| Controls | Do not expose a Sequential/Parallel toggle. Sequential is the only Play behavior. |
| Tight durations | Even when duration is short, final cached frame must contain the complete drawing within the chosen duration. |
| Visual expectation | Longer strokes receive visibly more of the duration than short strokes; multiple letters/strokes must not begin in parallel unless they were part of a single recorded stroke. |

### 7. Accessibility and Interaction Details

- Buttons must be text-labeled, not icon-only, for `Roto paint`, `Play paint`, and `Render`.
- Focus outline: 2px solid `var(--color-accent)` with 2px offset, matching existing CSS.
- Disabled state: opacity 0.45–0.5 and cursor not-allowed; disabled buttons must retain readable labels.
- Destructive confirmations use a modal/card over a dark overlay, 24px padding, 12px radius, and destructive copy in `var(--color-error-text)`.
- The current preview marker must be visible at all timeline zoom states and must not be confused with saved marker endpoints.
- Do not run the server as part of this UI work; user performs live visual UAT.
