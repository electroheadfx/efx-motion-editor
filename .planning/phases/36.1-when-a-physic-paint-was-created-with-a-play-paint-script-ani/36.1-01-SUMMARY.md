## Decisions Made

- Saved Play script IDs generated by `applySequence()` use `play-{startFrame}-{frameCount}` for deterministic replacement when saving the same range.
- Existing `getWorkflowMetadata()` output remains compatible with previous single-range callers; new multi-range consumers should use `getPlayScriptRanges()`.
- `getMaxPlayFrameCountFromGap()` returns `0` when the frame is inside an existing range, the distance to the next range when in a gap, and `PHYSIC_PAINT_MAX_APPLY_FRAMES` when no later range exists.
