# Phase 36.2 Plan 05: Roto Cached Playback UX Summary

Cached Roto playback now exposes a modest Play/Stop transport that previews already-cached PNG frames only, without all-frame rendering or editable stroke replay.

## What Changed

- Added `rotoCachedPlaybackAvailable`, `rotoCachedPlaybackStatus`, `onToggleRotoPlayback`, and `isRotoCachedPlaybackActive` workflow strip props.
- Rendered a compact Roto Play/Stop transport beside frame navigation with missing-frame status copy: missing frames play transparent/background.
- Added CSS for the Roto transport/status using existing dark UI styling, without adding new semantic Roto cell colors.
- Wired Studio cached Roto playback state and a timer loop that reads only local preview frames and `physicPaintStore.getFrame(...)` cached PNGs.
- Displayed cached Roto playback frames through the same canvas overlay surface used for cached preview imagery.
- Stopped cached Roto playback on explicit stop, manual Roto navigation, edit intent, and workflow mode changes.
