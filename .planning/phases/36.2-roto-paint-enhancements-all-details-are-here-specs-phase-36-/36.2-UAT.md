## Tests

### 1. Roto Cell Cache Colors and Save Copy
expected: In standalone Physics Paint Roto mode, the workflow strip shows empty Roto frames as gray, cached-only real-key frames as green, editable-session frames as pink, and the current frame as an outline over the existing fill. Dirty frames show subtle pending/saving status text and the action copy reads Save pending or Save current without introducing another cell color.
result: [pending]

### 2. Roto Save on Leave and Close
expected: After painting or editing a Roto frame, leaving the frame, clicking Save pending/current, or closing/unloading the standalone saves the pending Roto frame through the normal apply path. Returning to the frame shows the newly cached Roto image as a repaintable visual reference rather than losing it or compositing it twice.
result: [pending]

### 3. Main Editor Cached Roto Preview After Standalone Return
expected: After leaving or closing standalone Physics Paint with a saved Roto frame, the main EFX Motion editor shows the Roto cached frame preview/marker for the corresponding frame and uses that cached image for preview/export without requiring a manual refresh or replaying editable strokes.
result: [pending]

### 4. Generated-Only Frame Launch Redirect
expected: Opening standalone Physics Paint from a generated interpolation-only Roto frame targets the nearest real Roto key frame instead of opening the generated frame as an editable square.
result: [pending]

### 5. Cached Roto Playback Transport
expected: In Roto mode with cached frames available, the compact Roto Play control previews cached PNG frames only, changes to Stop while active, shows missing-frame status copy for transparent/background-only gaps, and stops when you stop playback, navigate manually, edit, or change workflow mode.
result: [pending]

### 6. Roto Interpolation Connector UI
expected: With at least two real cached Roto keys, enabling interpolation with one or more in-betweens shows connector lines between the real key squares. Generated interpolation frames do not appear as editable green or pink boxes in the standalone strip.
result: [pending]

### 7. Generated Roto In-Betweens in Preview and Export
expected: After interpolation regenerates, the main EFX Motion preview and export use the generated cached in-between Roto frames between real keys while still treating them as render-only cache frames.
result: [pending]

### 8. Roto Real-Key Utilities
expected: Duplicate key, Insert frame, Delete frame, Copy frame, and Paste frame operate on real Roto key squares only. Real keys shift, duplicate, replace, or delete as expected, generated interpolation cache updates afterward, and generated-only frames never become editable targets.
result: [pending]

### 9. Missing Roto Frame Background Behavior
expected: When a Roto playback/export frame is missing but the layer has active paper/background behavior, preview/export renders the background-only result virtually without creating a new green cached cell; missing transparent frames remain transparent.
result: [pending]
