## Editable persistence

| Option | Description | Selected |
|--------|-------------|----------|
| Cache-first only | Cached PNG frames are durable; editable strokes may only exist in the open session. Reopened cached-only frames show the PNG as reference and can be repainted cleanly. | ✓ |
| Persist strokes | Persist per-frame editable SerializedProject state now. Better reopen/edit fidelity, but expands store/project persistence beyond the current layer-level editable_state model. | |
| Hybrid minimum | Persist editable state only for frames touched in the current standalone session, but still treat cached PNG as playback truth when editable data is missing. | |

**User's choice:** Cache-first only.
**Notes:** Cached/no-editable frames should show as a reference/background and be repaintable from a clean editable state. UI should use subtle status text. Green indicates cached/no editable paint; pink indicates editable session paint. Painting over cached-only reference replaces the cache on save.

## Auto-save rules

| Option | Description | Selected |
|--------|-------------|----------|
| On frame leave | Paint marks the frame dirty; navigating away saves/renders the previous frame once, matching the spec’s natural boundary. | ✓ |
| Debounced idle | Auto-save after the user stops painting for a short delay; more live, but risks lag/race during drawing. | |
| Manual only | No automatic apply; user must click Save all frames. This conflicts with the phase expectation of visible frames after closing/navigating. | |

**User's choice:** On frame leave.
**Notes:** Save on close is also required. Navigation should block briefly while saving the previous dirty frame. The proposed Save all behavior was corrected: if save-on-leave is working, a bulk re-render-all button is unnecessary. Replace `Save roto frame` with a flush-dirty/pending-current action.

## Cache cell UX

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, 3 colors | Gray = empty/no cached output; green = cached PNG output only; pink = editable Roto session frame. Dirty/pending save is shown by subtle status or outline, not a fourth color. | ✓ |
| Add dirty mark | Keep gray/green/pink, but add a tiny pending indicator on pink frames that have unsaved changes. | |
| Other mapping | Use the same square spreadsheet cells, but define a different meaning for gray, green, or pink. | |

**User's choice:** Yes, 3 colors.
**Notes:** User clarified they want the same timeline spreadsheet with square gray, green, and pink cells. Pink means editable state exists in the currently open standalone session, not necessarily unsaved. Current frame uses a bright outline.

## Roto interpolation

| Option | Description | Selected |
|--------|-------------|----------|
| Real cache | Generate actual cached PNG frames for the in-between slots so EFX preview/export can use the existing physicPaintStore frame cache. | ✓ |
| Preview only | Only show interpolation during standalone playback; EFX Motion playback/export would not automatically get those in-between frames. | |
| Both modes | Preview virtually first, then allow committing generated interpolation frames to cache. More flexible but larger scope. | |

**User's choice:** Real cache.
**Notes:** User requested this feature in the last 36.2 plan. Roto frames are key frames; interpolation is not viewed as editable boxes in Physics Paint, only as connector lines like `[]-[]`. Generated in-between frames are visible only in EFX Motion. Settings are global for the layer/session. Changing interpolation settings regenerates all in-between cached frames. Opening from an interpolation-only EFX frame opens the nearest real Roto key.

## Roto key edit utilities

| Option | Description | Selected |
|--------|-------------|----------|
| Duplicate to next frame | Duplicate selected Roto key to the next frame for pauses. | ✓ |
| Ask frame | Prompt for target frame or let user choose a gray cell. | |
| After interpolation | Place duplicate after the current interpolation span. | |

**User's choice:** Duplicate to next frame.
**Notes:** User expanded this into duplicate, insert, delete, copy, and paste. Duplicate and insert ripple later real Roto keys right. Delete shifts later real Roto keys left. Copy/paste replaces the selected/current Roto key. Ripple operations affect real Roto keys only; generated interpolation cache is regenerated.

## Missing/background frames

| Option | Description | Selected |
|--------|-------------|----------|
| Remove cache | Empty/deleted transparent Roto keys remove cached output, so the cell returns gray unless still an editable pink session key. | ✓ |
| Background only | If any paper/white/photo background is active, save a background-only PNG for that key; useful but can create accidental solid/paper frames. | |
| Ask user | Prompt whether to remove cache or save background-only output each time, safer but interrupts editing. | |

**User's choice:** Remove cache.
**Notes:** For playback/export, missing frames should render transparent if no paper background is active, and render paper/background if paper is active. Paper/background-only output is playback/export behavior only, not green cached cells. Interpolation requires two real Roto keys; no interpolation is generated when one side is missing.
