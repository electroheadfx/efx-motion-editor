# Phase 36.2 Plan 06: Roto Interpolation and Key Utilities Summary

Implemented cache-first Roto interpolation with render-only generated frames, real-key edit utilities, and connector-line UI so in-betweens appear in EFX Motion cache/playback paths without becoming editable Physics Paint boxes.

## What Changed

- Added pure interpolation and real-key helper functions:
  - `getRotoInterpolationSpanFrames(realKeys, settings)`
  - `getNearestRealRotoKeyFrame(frame, realKeys)`
  - `duplicateRotoKeyFrame(realKeys, selectedFrame)`
  - `insertRotoKeyFrame(realKeys, selectedFrame)`
  - `deleteRotoKeyFrame(realKeys, selectedFrame)`
  - `replaceRotoKeyFrame(realKeys, targetFrame)`
- Added generated Roto cache source metadata to rendered frames via `source?: 'real-key' | 'generated-interpolation'`.
- Added store regeneration helpers and methods:
  - `renderDuplicateRotoInterpolationFrame(...)`
  - `renderBlendedRotoInterpolationFrame(...)`
  - `setRotoInterpolationSettings(...)`
  - `replaceGeneratedRotoCache(...)`
  - `regenerateRotoInterpolationCache(...)`
  - `getRealRotoKeyFrames(...)`
- Added Studio real-key utility wiring for Duplicate key, Insert frame, Delete frame, Copy frame, and Paste frame.
- Added generated-only frame redirect support by feeding the workflow strip an effective nearest-real-key current frame.
- Added connector-line rendering in the workflow strip using `physics-paint-roto-interpolation-connector` and `connector-count-*` classes.
- Filtered `source:'generated-interpolation'` metadata out of Roto square cell rendering so generated frames do not become editable cells.
