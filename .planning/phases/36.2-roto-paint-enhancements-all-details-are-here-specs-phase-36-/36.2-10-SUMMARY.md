# Phase 36.2 Plan 10: Missing Roto Background Playback/Export Summary

Persisted Roto paper/background metadata now drives transparent-vs-background missing-frame behavior in preview, export, and compact cached playback without creating fake cached cells.

## What Changed

### Task 1: Persist Roto paper/background metadata without creating cache cells

- Added typed `PhysicPaintRotoBackgroundMetadata` with validation for transparent, white, and canvas paper modes.
- Serialized/hydrated Roto background metadata through the existing physics paint output metadata path.
- Updated the standalone Physics Paint studio to persist Roto background metadata when paper/background settings change.
- Preserved the invariant that background-only missing gaps do not call `setFrame` and do not become green cached cells.

**Commit:** c3790a08

### Task 2: Draw virtual missing Roto backgrounds before no-draw exit

- Updated `PreviewRenderer` drawable detection so non-transparent missing Roto backgrounds bypass the no-draw early return.
- Extended missing Roto frame resolution to read persisted store metadata before falling back to legacy `paintBgColor`.
- Added paper/canvas grain draw instructions and deterministic background drawing for missing Roto frames.
- Kept export behavior delegated through PreviewRenderer; no export-only physics paint engine path was added.

**Commit:** 750c7e18

### Task 3: Synthesize transient background-only frames for cached Roto playback

- Updated compact Roto cached playback assembly to synthesize transient background-only frames from Roto background metadata.
- Transparent missing frames continue to return missing/null playback status.
- Transient playback frames are never written to `savedRotoFrames`, durable frame cache, or Roto cache metadata.

**Commit:** a10961f3
