  <action>
The root cause is a circular dependency between two useEffects in InlineColorPicker.tsx:

1. **Prop sync effect** (lines 44-50): When `color` prop changes, it sets internal HSV state (setHue, setSat, setVal)
2. **onChange effect** (lines 59-62): When HSV state changes (hue, sat, val, alpha), it calls `onChange(currentHex, alpha)` which goes to PaintProperties -> paintStore.setBrushColor -> brushColor signal updates -> PaintProperties re-renders with new `color` prop -> triggers prop sync effect -> loop

**Fix: Add a `useRef` guard to distinguish external prop changes from internal user interactions.**

Replace the two problematic useEffects with this pattern:

```typescript
// Track whether the current state change came from an external prop update
const isExternalUpdate = useRef(false);
// Track previous color/opacity to avoid unnecessary syncs
const prevColorRef = useRef(color);
const prevOpacityRef = useRef(opacity);

// Sync from external color prop changes (only when prop actually changes from outside)
useEffect(() => {
  if (prevColorRef.current === color && prevOpacityRef.current === opacity) return;
  prevColorRef.current = color;
  prevOpacityRef.current = opacity;
  
  isExternalUpdate.current = true;
  const r = hexToRgba(color);
  const h = rgbToHsv(r.r, r.g, r.b);
  setHue(h.h); setSat(h.s); setVal(h.v);
  setAlpha(opacity);
  setHexInput(color);
  // Reset flag after microtask (after React batches state updates)
  queueMicrotask(() => { isExternalUpdate.current = false; });
}, [color, opacity]);

// Fire onChange only on USER interactions (not external prop sync)
useEffect(() => {
  if (isExternalUpdate.current) return; // Skip if this was triggered by prop sync
  const hex = rgbaToHex(hsvToRgb(hue, sat, val).r, hsvToRgb(hue, sat, val).g, hsvToRgb(hue, sat, val).b);
  setHexInput(hex);
  onChange(hex, alpha);
}, [hue, sat, val, alpha]);
```

Also update `applyFromSwatch` to NOT go through the prop sync:
```typescript
const applyFromSwatch = useCallback((c: string) => {
  const rgb = hexToRgba(c);
  const hsv = rgbToHsv(rgb.r, rgb.g, rgb.b);
  // Set directly -- the onChange effect will fire and propagate to parent
  setHue(hsv.h); setSat(hsv.s); setVal(hsv.v);
}, []);
```
This is already correct -- it sets internal state which triggers the onChange effect. The fix is that the onChange effect won't loop back because the prop sync effect checks `prevColorRef`.

Keep all other code (slider handlers, canvas handlers, hex input, etc.) unchanged.
