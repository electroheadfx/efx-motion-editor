# Phase 12.7: Change Keyframe Icons and Improve Keyframe Click Targets - Research

**Researched:** 2026-03-18
**Domain:** Canvas 2D rendering, timeline interaction, keyframe visual identity
**Confidence:** HIGH

## Summary

This phase replaces uniform gold diamond keyframe markers with interpolation-aware icons (losange, full circle, half circles) and improves click target generosity plus hover feedback. The scope is confined to two files -- `TimelineRenderer.ts` (drawing) and `TimelineInteraction.ts` (hit testing/hover) -- with no data model changes required.

The existing architecture is well-suited for this change. The `drawKeyframeDiamonds()` method already receives `kf.easing` per keyframe but ignores it, always calling `drawDiamond()`. The fix is straightforward: replace the single `drawDiamond()` with a dispatcher that routes to shape-specific draw methods based on `kf.easing`. The hit test expansion is a constant change. The hover state requires new tracking in `TimelineInteraction` and a new field in `DrawState` to pass hover info to the renderer.

**Primary recommendation:** Implement four draw methods (`drawLosange`, `drawCircle`, `drawHalfCircleLeft`, `drawHalfCircleRight`), a dispatcher method, expand the hit area constant, and add hover state tracking with crosshair cursor.

### Pattern 1: Shape Dispatcher
**What:** Replace direct `drawDiamond()` call with a dispatcher that selects the draw method based on `kf.easing`
**When to use:** In `drawKeyframeDiamonds()` loop
**Example:**
```typescript
// In drawKeyframeDiamonds, replace:
//   this.drawDiamond(ctx, kfX, trackCenterY, diamondSize, isSelected);
// With:
this.drawKeyframeIcon(ctx, kfX, trackCenterY, iconSize, kf.easing, isSelected, isHovered);
```

```typescript
/** Route to shape-specific draw method based on easing type */
private drawKeyframeIcon(
  ctx: CanvasRenderingContext2D,
  x: number, y: number, size: number,
  easing: string,
  isSelected: boolean,
  isHovered: boolean,
): void {
  switch (easing) {
    case 'linear':
      this.drawLosange(ctx, x, y, size, isSelected, isHovered);
      break;
    case 'ease-in-out':
      this.drawCircle(ctx, x, y, size, isSelected, isHovered);
      break;
    case 'ease-in':
      this.drawHalfCircleLeft(ctx, x, y, size, isSelected, isHovered);
      break;
    case 'ease-out':
      this.drawHalfCircleRight(ctx, x, y, size, isSelected, isHovered);
      break;
    default:
      this.drawLosange(ctx, x, y, size, isSelected, isHovered);
  }
}
```

### Pattern 2: Canvas 2D Shape Drawing (save/restore sandwich)
**What:** Each shape method uses the established `ctx.save()`/`ctx.restore()` pattern
**When to use:** Every draw method
**Example:**
```typescript
private drawCircle(
  ctx: CanvasRenderingContext2D,
  x: number, y: number, size: number,
  isSelected: boolean, isHovered: boolean,
): void {
  ctx.save();
  ctx.beginPath();
  ctx.arc(x, y, size * 0.7, 0, Math.PI * 2);

  ctx.fillStyle = isSelected ? '#FFD700' : (isHovered ? '#F0B830' : '#E5A020');
  ctx.fill();

  if (isSelected) {
    ctx.shadowColor = '#FFD700';
    ctx.shadowBlur = 6;
    ctx.fill();
    ctx.shadowBlur = 0;
  } else if (isHovered) {
    ctx.shadowColor = '#E5A020';
    ctx.shadowBlur = 3;
    ctx.fill();
    ctx.shadowBlur = 0;
  }

  ctx.strokeStyle = isSelected ? '#FFFFFF' : 'rgba(136, 102, 0, 0.5)';
  ctx.lineWidth = 1;
  ctx.stroke();
  ctx.restore();
}
```

### Pattern 3: Half-Circle Rendering
**What:** Use `ctx.arc()` for semicircle + straight line for flat edge
**When to use:** Ease-in and ease-out icons
**Example:**
```typescript
// Ease-in: left half filled (easing INTO the next keyframe)
private drawHalfCircleLeft(
  ctx: CanvasRenderingContext2D,
  x: number, y: number, size: number,
  isSelected: boolean, isHovered: boolean,
): void {
  const r = size * 0.7;
  ctx.save();

  // Filled left half
  ctx.beginPath();
  ctx.arc(x, y, r, Math.PI * 0.5, Math.PI * 1.5);  // left semicircle arc
  ctx.closePath();
  ctx.fillStyle = isSelected ? '#FFD700' : (isHovered ? '#F0B830' : '#E5A020');
  ctx.fill();

  // Full circle outline
  ctx.beginPath();
  ctx.arc(x, y, r, 0, Math.PI * 2);
  ctx.strokeStyle = isSelected ? '#FFFFFF' : 'rgba(136, 102, 0, 0.5)';
  ctx.lineWidth = 1;
  ctx.stroke();

  // Glow effects (same pattern as other shapes)
  if (isSelected) {
    ctx.shadowColor = '#FFD700';
    ctx.shadowBlur = 6;
    ctx.beginPath();
    ctx.arc(x, y, r, Math.PI * 0.5, Math.PI * 1.5);
    ctx.closePath();
    ctx.fill();
    ctx.shadowBlur = 0;
  }

  ctx.restore();
}
```

## Common Pitfalls

### Pitfall 1: Forgetting save/restore for shadowBlur
**What goes wrong:** `shadowBlur` leaks to subsequent draw calls, causing all shapes after a selected keyframe to glow
**Why it happens:** Canvas shadow state is global and persistent
**How to avoid:** Every shape method MUST use `ctx.save()` at start and `ctx.restore()` at end. Reset `shadowBlur = 0` after the glow fill, before any stroke calls.
**Warning signs:** All keyframes after the first selected one appear to glow

### Pitfall 2: Half-circle arc direction confusion
**What goes wrong:** Left half-circle renders as right half-circle (or vice versa)
**Why it happens:** Canvas arc angles: 0 = 3 o'clock, PI/2 = 6 o'clock, PI = 9 o'clock, 3PI/2 = 12 o'clock. The direction is confusing.
**How to avoid:** Test visually. For ease-in (left half filled): arc from PI/2 to 3PI/2 (sweeps through 9 o'clock = left side). For ease-out (right half filled): arc from 3PI/2 to PI/2 (sweeps through 3 o'clock = right side).
**Warning signs:** Shape looks like a "D" when it should look like a reversed "D"

### Pitfall 3: Hit test not matching visual
**What goes wrong:** Click target doesn't align with the new larger icons or the expanded hit area
**Why it happens:** Hit test uses frame-based threshold while visuals use pixel-based sizes. If the icon grows but the hit threshold doesn't, there's a mismatch.
**How to avoid:** When increasing icon size from 6 to ~9px, increase the hit test threshold proportionally. The threshold formula is `Math.max(0.6, 8 / frameWidth)` -- the 8px constant should increase to match the new hit area (~16px).
**Warning signs:** Users click on a visible keyframe icon but nothing happens

### Pitfall 4: Hover redraw performance
**What goes wrong:** Every mouse move triggers a full canvas redraw, causing jank
**Why it happens:** Naive hover implementation tracks every pixel-level movement
**How to avoid:** Only trigger redraw when the hovered keyframe CHANGES (from null to a keyframe, from one keyframe to another, or from a keyframe to null). Compare the new hover target with the previous one; skip redraw if same.
**Warning signs:** CPU spikes when moving mouse over timeline

### Pitfall 5: Nearest-wins not implemented for overlapping keyframes
**What goes wrong:** When two keyframes are close together, the first one in the array always wins
**Why it happens:** Current `keyframeHitTest` returns on first match within threshold
**How to avoid:** Check ALL keyframes within the hit threshold, then return the one with minimum distance to the click point
**Warning signs:** Clicking between two close keyframes always selects the earlier one

### Pitfall 6: Hover cursor interfering with other cursor states
**What goes wrong:** Crosshair cursor shows when dragging or when hovering over track headers
**Why it happens:** Cursor logic in `onPointerMove` already handles FX area, track headers, etc. Adding keyframe hover cursor without proper priority causes conflicts.
**How to avoid:** Check keyframe hover AFTER ruling out active drags and BEFORE the default cursor fallback. Follow the existing cursor priority chain.
**Warning signs:** Crosshair cursor appears during drag operations or on track headers

## Code Examples

### Current drawDiamond (to be retained as drawLosange)
```typescript
// Source: TimelineRenderer.ts line 542
// The existing drawDiamond becomes drawLosange with added hover parameter
private drawLosange(
  ctx: CanvasRenderingContext2D,
  x: number, y: number, size: number,
  isSelected: boolean, isHovered: boolean,
): void {
  ctx.save();
  ctx.beginPath();
  ctx.moveTo(x, y - size);       // top
  ctx.lineTo(x + size, y);       // right
  ctx.lineTo(x, y + size);       // bottom
  ctx.lineTo(x - size, y);       // left
  ctx.closePath();

  ctx.fillStyle = isSelected ? '#FFD700' : (isHovered ? '#F0B830' : '#E5A020');
  ctx.fill();

  if (isSelected) {
    ctx.shadowColor = '#FFD700';
    ctx.shadowBlur = 6;
    ctx.fill();
    ctx.shadowBlur = 0;
  } else if (isHovered) {
    ctx.shadowColor = '#E5A020';
    ctx.shadowBlur = 3;
    ctx.fill();
    ctx.shadowBlur = 0;
  }

  ctx.strokeStyle = isSelected ? '#FFFFFF' : 'rgba(136, 102, 0, 0.5)';
  ctx.lineWidth = 1;
  ctx.stroke();
  ctx.restore();
}
```

### Nearest-wins hit test modification
```typescript
// Source: TimelineInteraction.ts keyframeHitTest (line 215)
// Currently returns first match. Modify to find nearest:
let bestHit: { frame: number; distance: number } | null = null;
for (const kf of keyframes) {
  const dist = Math.abs(localClickFrame - kf.frame);
  if (dist <= hitThresholdFrames) {
    if (!bestHit || dist < bestHit.distance) {
      bestHit = { frame: kf.frame, distance: dist };
    }
  }
}
if (bestHit) {
  return { frame: bestHit.frame, layerId: selectedId, sequenceStartFrame: track.startFrame };
}
```

### Hover state tracking in onPointerMove
```typescript
// In onPointerMove, after all drag checks and before cursor fallback:
// Run a lightweight keyframe proximity check
const kfHover = this.keyframeHitTest(e.clientX, e.clientY);
const newHoveredFrame = kfHover ? kfHover.frame : null;
if (newHoveredFrame !== this.hoveredKeyframeFrame) {
  this.hoveredKeyframeFrame = newHoveredFrame;
  // Trigger redraw with hover state
  if (this.renderer) {
    this.renderer.setHoveredKeyframe(newHoveredFrame);
  }
}
if (newHoveredFrame !== null) {
  this.canvas.style.cursor = 'crosshair';
  return;
}
```

## Open Questions

1. **Exact icon size within 8-10px range**
   - What we know: User wants increase from 6px to 8-10px range
   - Recommendation: Use 9px as the icon size parameter (good middle ground, odd number centers cleanly on pixel grid). Hit area at 18px (2x icon size).

2. **Half-circle rendering: arc+fill vs. clip path**
   - What we know: Both approaches work in Canvas 2D
   - Recommendation: Use arc+closePath+fill for the filled half, then full circle outline. Simpler, fewer state changes, matches existing code patterns. No clip needed.

3. **Hover highlight intensity**
   - What we know: User wants "subtle" highlight -- brighten slightly or faint glow
   - Recommendation: Use a slightly brighter gold (#F0B830) for fill + a 3px shadowBlur (half of the selected glow's 6px). Noticeable but clearly distinct from selected state.
