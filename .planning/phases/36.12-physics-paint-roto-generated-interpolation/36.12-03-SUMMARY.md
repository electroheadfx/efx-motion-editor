# Phase 36.12 Plan 03: Generated Preview-Only Roto Selection Summary

Generated interpolation frames are now clickable for preview/navigation selection while staying render-only across workflow strip controls, session-derived key action state, and bridge mutation seams.

## What Changed

- Added RED tests for generated Roto cell preview selection, generated-selected disabled key utilities, minimal interpolation controls, and generated bridge mutation guards.
- Updated the Roto workflow strip so generated cells call the same frame navigation/preview callback as ordinary timeline selection while preserving generated provenance.
- Added exact render-only copy for generated labels/status and disabled edit/key-tool messaging.
- Added one optional `Interpolation` toggle on the existing workflow strip surface and kept count/mode/motion controls absent.
- Updated session/key availability to use the long generated render-only disabled reason.
- Updated bridge behavior so generated interpolation provenance is rejected for editable apply/open seams instead of redirecting to a real-key editable target.
