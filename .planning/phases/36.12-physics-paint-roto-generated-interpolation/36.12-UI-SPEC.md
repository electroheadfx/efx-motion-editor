## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-bg-root, #0F0F0F)` | Overall app/background surface inherited from the current theme |
| Secondary (30%) | `#37393a` / `var(--color-bg-card, #1A1A1A)` | Workflow strip timeline lane, timeline track, compact control shells, status stack surfaces |
| Accent (10%) | `#6f90ff` and generated cell base `#365ed6` | Interpolation toggle checked state, generated/render-only cells, generated render-only affordance, and interpolation-on status only |
| Destructive | `#ff6666` / `#ffb0b8` | Existing destructive Roto key actions only; Phase 36.12 adds no destructive action |

Accent reserved for: interpolation toggle checked state, generated cell fill, generated-cell render-only stripe, interpolation connector/status emphasis, keyboard focus on the interpolation control when already using the workflow strip focus style. Accent must not be reused for Save current, real-key authority states, key utility actions, or background-only states.

**State colors:**
- Real cached keys stay green (`#2d6f48` / rgba green border) and remain authoritative.
- Current editable cells stay pink (`#a33d73`) with the existing white dot.
- Generated cells stay blue (`#365ed6`) with the existing horizontal white stripe, never pink or green.
- Background-only cells stay gray striped and are replaced visually by generated cells only for interior frames while interpolation is enabled.
- Current/selected outline remains amber (`#f5a623`) whether the selected preview cell is real or generated; render-only status copy disambiguates generated selection.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Interpolation` toggle label; checked means generated in-betweens are enabled for the whole Roto layer |
| Empty state heading | `No generated frames` |
| Empty state body | `Add at least two real Roto keys, then enable Interpolation to preview render-only in-betweens.` |
| Error state | `Interpolation could not regenerate. Real Roto keys were kept; disable and re-enable Interpolation after the source keys are saved.` |
| Destructive confirmation | None for Phase 36.12. Disabling Interpolation removes derived generated frames immediately without confirmation because real keys are preserved and generated frames are render-only. Existing Delete key confirmation/guard behavior remains unchanged. |

**Required status and label copy:**
- Generated cell label/title/aria: `Generated frame {frame} — render-only.`
- Generated preview status: `Generated frame {frame} is render-only.`
- Disabled edit/save/key-tools status while generated is selected: `Generated frame {frame} is render-only. Use timeline navigation or playback; edit a real Roto key to paint.`
- Interpolation enabled status: `Interpolation on — generated render-only frames refresh from real keys.`
- Interpolation disabled status: `Interpolation off — real Roto keys only.`
- Regeneration failure status: `Interpolation could not regenerate. Real keys were kept.`
- Real-key collision rule is not surfaced as an error; real keys silently win and stay editable.

## Phase 36.12 Interaction Contract

| Interaction | Required Behavior |
|-------------|-------------------|
| Enable Interpolation | Generate every integer frame strictly between every adjacent real Roto key in the layer. Show generated cells immediately in the existing Roto timeline lane. |
| Disable Interpolation | Remove generated interpolation cells immediately and leave only real keys plus existing transparent/background-only missing-frame rules. No confirmation. |
| Click generated cell | Preview/select that generated render-only frame and update the preview canvas from the generated cache. Do not open it for editing, redirect to a real key, or convert it to a real key. |
| Generated frame selected | Allow timeline navigation, previous/next/first/last, cached playback controls, and preview/export rendering. Disable Save current, paint editing, Duplicate, Insert, Copy, Paste, and Delete. |
| Real key selected | Preserve existing real-key behavior, including additive cached repaint from Phase 36.11 and key utilities from Phase 36.7. |
| Source real-key change while enabled | Auto-refresh generated frames immediately after the real-key mutation succeeds. If regeneration fails, keep the real-key change and show compact failure status. |
| Background-only interior frame while enabled | Generated interpolation may visually replace background-only support state for that interior frame, but generated alpha must compose over paper/background through the existing renderer. |
