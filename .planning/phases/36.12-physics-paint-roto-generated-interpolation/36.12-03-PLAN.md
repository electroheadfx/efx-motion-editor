## Artifacts this phase produces

- Changed component behavior: `handleRotoCellClick(frame, vm)` allows generated cell preview/navigation selection instead of returning early, while preserving generated provenance so the frame is never opened as editable engine state.
- Changed UI disabling boundary: generated frames are preview/selectable/renderable, but generated-selected state must disable Save current, paint editing/apply-canvas mutation, Duplicate, Insert, Copy, Paste, Delete, deletion-as-real-key behavior, generated-to-real conversion, and every source real-key utility operation; timeline navigation, previous/next/first/last, cached playback controls, preview canvas rendering, and preview/export rendering remain allowed.

Use timeline navigation or playback; edit a real Roto key to paint.`, `Interpolation on — generated render-only frames refresh from real keys.`, `Interpolation off — real Roto keys only.`, and `Interpolation could not regenerate. - Changed or new tests in `PhysicsPaintWorkflowStrip.test.ts`, `physicsPaintRotoSession.test.ts`, and `physicPaintBridge.test.ts` for generated preview-only, disabled actions, and source-contract mutation guards.
- No new package, schema, ORM, CLI flag, class, decorator, or dataclass/struct field is created by this plan.
