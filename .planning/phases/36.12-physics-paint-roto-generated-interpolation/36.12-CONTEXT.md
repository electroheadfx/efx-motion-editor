# Phase 36.12: Physics Paint Roto Generated Interpolation - Context

**Gathered:** 2026-06-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 36.12 delivers generated render-only in-between Physics Paint Roto frames between adjacent real cached Roto keys. The generated frames make Roto preview/export smoother while keeping real cached keys as the only editable authority.

Generated interpolation must follow the source spec at `SPECS/36.x-phases/phase-36.12-generated-interpolation/spec-36.12-generated-interpolation.md`. This phase builds on Phase 36.11's additive cached real-key repaint model: real keys are flattened alpha paint caches, not restored editable stroke scripts. Generated frames must remain derived alpha paint output, not editable state.

This phase must not add final Pencil timeline layout polish, convert-generated-to-real-key workflows, new key utilities, destructive cached-base editing, Play/Roto conversion scope, XState/state-machine work, or broad editor integration.

</domain>

<decisions>
## Implementation Decisions

### Frame Placement
- **D-01:** When interpolation is enabled, generate every integer frame strictly between each adjacent pair of real Roto keys. Do not use a partial in-between count for the visible MVP.
- **D-02:** Generate interpolation for the whole Roto layer at once: every interior gap between every adjacent pair of real keys should be filled while interpolation is enabled.
- **D-03:** Generated interpolation may replace derived background-only support state for interior gap frames, but must keep paper/background composition separate. Generated frames draw interpolated paint alpha over paper/background; they must not become real editable keys and must not bake background into alpha caches.
- **D-04:** Real keys always win. If a generated frame would collide with an existing real key after rounding, layout changes, or regeneration, skip generated output for that frame and treat the real key as authoritative.

### Generated Look
- **D-05:** The MVP generated look is proportional alpha blending between neighboring real-key paint alpha caches.
- **D-06:** At each generated gap frame, blend the previous and next real-key alpha by normalized position within the gap. Do not use stepped hold behavior for this phase.
- **D-07:** Phase 36.12 exposes only the proportional alpha-blend interpolation behavior. Do not expose duplicate/hold, Move, Deform, or multiple interpolation modes in the visible MVP.
- **D-08:** Blend alpha only. Paper/background composition follows the existing Roto background and missing-frame rules from Phases 36.10/36.11, and generated cache output must not contain baked paper/background pixels.

### Refresh Timing
- **D-09:** While interpolation is enabled, source real-key changes auto-refresh generated frames immediately. Saves, repaint merges, paste, duplicate, insert, delete, replace, or any real-key layout/cache change should clear and regenerate dependent generated output from the current real-key layout.
- **D-10:** Interpolation is a persistent toggle. Once enabled, generated frames keep auto-refreshing until the user disables interpolation.
- **D-11:** If generated regeneration fails after a real-key change, preserve the real-key change, remove or ignore generated frames, and show compact status that interpolation could not regenerate. Never roll back or risk real-key edits because generated output failed.
- **D-12:** When interpolation is disabled, remove generated interpolation frames immediately and leave only real keys plus existing background/missing-frame rules.

### Controls and Feedback
- **D-13:** Use a simple visible Generate/Interpolation toggle in the existing Physics Paint workflow strip for UAT. Do not expose count, mode, or one-shot Generate controls in this phase.
- **D-14:** Generated cells must remain visually distinct and explicitly labeled as generated/render-only, with copy such as `Generated frame N — render-only.` Reuse existing generated-cell semantics where possible.
- **D-15:** Clicking a generated frame cell should preview/select that generated render-only frame without opening it for editing and without converting it into a real key.
- **D-16:** While a generated frame is preview-selected, allow navigation/timeline/playback only. Save current, paint editing, and key utilities must be disabled with render-only status copy.
- **D-17:** Keep UI changes minimal and inside the existing workflow strip/status surfaces. Final Pencil timeline layout and visual polish remain Phase 36.13.

### UAT-Driven Gap Closure Scope Change
- **D-18:** Supersedes D-01 only for Phase 36.12 gap-closure work after UAT Test 8: the visible in-between count is required and means generated in-betweens per adjacent source real-key pair. Generated display positions may expand while interpolation is enabled, but stored real-key source frames must remain unchanged and disabling interpolation must restore the original real-key layout exactly.
- **D-19:** Supersedes D-05/D-06/D-07 only for Phase 36.12 gap-closure work after UAT Test 8: duplicate/hold and alpha-blend are both required visible interpolation modes. Duplicate/hold copies the previous source real key into generated in-between display frames; alpha-blend uses the existing alpha-only generated-cache contract. Move, Deform, and other interpolation modes remain deferred.
- **D-20:** Supersedes D-13 only for Phase 36.12 gap-closure work after UAT Test 8: the existing workflow strip must visibly expose interpolation enabled state, in-between count, and selected mode so the user can switch between duplicate/hold and alpha-blend behavior.
- **D-21:** The reason for D-18/D-19/D-20 is the user-owned UAT failure in Test 8: the user explicitly required selected interpolation mode to be visible and functional, current frame-copy/hold behavior to be selectable, alpha-blend mode to remain available, and disabling interpolation to restore original source keys. These decisions are a narrow blocker-closure correction, not permission to add conversion workflows, Move/Deform, final Pencil timeline polish, or unrelated editor integration.

### Established Patterns
- Real Roto keys are the only editable authoritative frames. Generated interpolation and background-only support frames are render support and must not become editable sources.
- Cached real Roto keys are alpha paint caches; paper/background remains a separate composition concern.
- Roto cache changes must bump `physicPaintVersion` / notify visual change so preview, export, and project dirty state observe current output.
- Preact/Signals plus pure helper/controller tests are preferred over new broad `useEffect` orchestration.
- Normal UI feedback stays compact in the Physics Paint workflow strip/status surface; no modal, toast, tutorial, or final timeline redesign belongs in this phase.
- Motion behavior for Physics Paint should be deterministic held stop-motion behavior, not random jitter. This phase selected alpha-only blend, so no random Move/Deform behavior should be introduced.

### Integration Points
- Store regeneration should replace generated interpolation and derived background-only support for interior gap frames while preserving real keys and paper/background separation.
- Real-key save/repaint/key-utility paths should trigger generated refresh when interpolation is enabled.
- Generated preview selection should connect to workflow-strip cell clicks, current frame/status state, preview renderer/cache lookup, and control disabling without opening the generated frame as editable engine state.
- Tests should cover integer-gap placement, background-only replacement, real-key collision/authority, alpha-only blending, persistence/toggle behavior, auto-refresh on real-key changes, disable clearing generated frames, generated click preview-only behavior, disabled editing/key utilities on generated preview, preview/export parity, and `physicPaintVersion` invalidation.

<specifics>
## Specific Ideas

- Think of generated frames as derived alpha caches: `generated alpha = proportional blend(previous real-key alpha, next real-key alpha)`.
- Think of final visible output as: `paper/background composition + generated alpha`, not a full-image blend.
- Interpolation is an enabled layer behavior, not a one-shot command.
- Generated cells should be previewable for UAT, but preview selection must not imply editability.

</specifics>

<deferred>
## Deferred Ideas

- Move/Deform and interpolation modes beyond duplicate/hold and alpha-blend are deferred. Duplicate/hold and alpha-blend are now required only because D-18/D-19/D-20 supersede the earlier MVP limitation for UAT Test 8 gap closure.
- Convert-generated-to-real-key behavior is a new capability and is out of scope for Phase 36.12.
- Final Pencil timeline layout and visual polish remain Phase 36.13.
