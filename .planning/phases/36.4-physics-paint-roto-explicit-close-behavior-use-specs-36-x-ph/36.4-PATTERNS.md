## Shared Patterns

### Dirty-close guard

**Source:** `app/src/components/physic-paint/PhysicsPaintStudio.tsx`

**Apply to:** Tauri `onCloseRequested` only.

Pattern:

1. If not Roto, do nothing special.
2. Snapshot current Roto frame.
3. Check only `dirtyRotoFramesRef.current.has(currentFrame)`.
4. Clean current frame: allow normal close.
5. Dirty current frame: prevent default, show prompt.

### Explicit three-choice branching

**Sources:** `app/src/lib/unsavedGuard.ts`, Phase 36.4 spec

**Apply to:** close prompt state machine in `PhysicsPaintStudio.tsx`.

Required labels exactly:

- `Close without saving`
- `Cancel`
- `Close saving`

Required outcomes:

- `Close without saving`: bypass close guard and close; do not send any apply/delete payload.
- `Cancel`: dismiss prompt; preserve canvas/engine state.
- `Close saving`: show saving state, call existing Save current path, wait for matching successful apply result, then bypass close guard and close.

### Save/apply completion

**Source:** `app/src/components/physic-paint/PhysicsPaintStudio.tsx` `saveRotoFrame`, `flushRotoFrame`, `handleApplyResult`

**Apply to:** `Close saving` only.

Use existing operation-id based apply confirmation. Do not close immediately after `sendPhysicPaintApplyPayload` resolves.

### UI/modal accessibility

**Source:** `app/src/components/physic-paint/PhysicsPaintWorkflowStrip.tsx` confirmation modal and `physicsPaintStudio.css` modal styles

**Apply to:** close prompt UI.

Use `role="dialog"`, `aria-modal="true"`, and `aria-labelledby`. Keep styling close to `.physics-paint-confirmation*`.

### Error handling / non-stuck behavior

**Source:** `startApplyTimeout` and `handleApplyResult` in `PhysicsPaintStudio.tsx`

**Apply to:** close-save state.

On send error, apply result error, or timeout, keep the window open and restore actionable prompt state. Clear close-after-save refs so later close attempts are not stale.

## Anti-Patterns to Avoid

- Do not autosave on close without showing the three choices.
- Do not save on `beforeunload`; async bridge persistence belongs in the Tauri close-request path.
- Do not check all pending frames for the MVP prompt; use current Roto frame only.
- Do not send `delete-roto-frame` or `apply-canvas` for `Close without saving`.
- Do not close after `saveRotoFrame()` returns unless `handleApplyResult()` has confirmed the matching successful operation.
- Do not change `physicPaintStore`, serialization schema, Play cache behavior, or engine internals.
