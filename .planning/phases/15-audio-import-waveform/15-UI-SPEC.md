## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, waveform center-line offset |
| sm | 8px | Compact element spacing, button padding, track label margin |
| md | 16px | Default element spacing, properties panel section gaps |
| lg | 24px | Section padding in properties panel |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Audio track default height: 44px (per D-01, midpoint of 40-48px range; matches existing TRACK_HEIGHT pattern at 52px for content tracks)
- FX track height reference: 28px (existing FX_TRACK_HEIGHT constant)
- Track header width: 80px (existing TRACK_HEADER_WIDTH constant)
- Resize handle hit-zone: 6px (3px above + 3px below the track edge border)
- Range bar vertical padding: 4px (inside the 44px track, waveform area = 36px)
- Fade overlay triangle height: matches waveform rendering area (36px)

1. D-02 specifies "teal/cyan" accent for waveforms
2. It contrasts with existing FX track colors: grain (#A0522D brown), particles (#6A5ACD purple), lines (#20B2AA -- similar but lighter, audio is slightly more saturated), dots (#DA70D6 pink), vignette (#708090 gray), color-grade (#CD853F tan)
3. It is distinct from the UI accent (#2D5BE3 blue) so waveform color and selection color do not conflict

## Interaction Contracts

### INT-01: Add Audio Button

- **Trigger:** Click "Add Audio" button in timeline controls bar
- **Location:** Next to existing AddLayerMenu button, separated by a divider
- **Visual:** Same button style as existing controls: `rounded bg-[var(--color-bg-input)] px-2 py-[5px]` with lucide `Music` icon (14px)
- **Hover:** `bg-[var(--color-bg-hover-item)]` + text brightens to white
- **Action:** Opens native file picker via Tauri dialog (filtered to WAV, MP3, AAC, FLAC, M4A)
- **Loading state:** Button shows spinner icon while decoding (replace Music icon with `Loader2` icon with `animate-spin`)
- **Success:** New audio track appears at bottom of timeline, auto-selected
- **Error:** Toast/inline error if decode fails (see Copywriting Contract)

### INT-02: Audio Track on Timeline (Canvas 2D)

- **Track row:** Full-width row at 44px height, positioned below all FX/content-overlay tracks (per D-06)
- **Header area (0 to 80px):** Track name in 10px, click toggles mute (per D-15)
- **Waveform area (80px to canvas width):** Range bar with clear start/end edges (per D-08)
- **Range bar fill:** Waveform peaks rendered as vertical lines from center outward, filled with --color-audio-waveform
- **Center line:** 1px hairline at vertical center in --color-audio-centerline (per D-05)
- **Muted state:** Waveform opacity drops to 30% using --color-audio-waveform-muted (per D-11)
- **Selected state:** 2px left accent border in --color-accent, same pattern as FX track selection (per D-12)
- **Click anywhere on track:** Selects track, opens AudioProperties in sidebar (per D-16)
- **Fade overlays:** Triangular gradient overlays at track start (fade-in) and end (fade-out), same visual as FX/sequence fade indicators (per D-20)

### INT-03: Audio Track Drag-to-Offset

- **Trigger:** Mouse down on waveform body (not on edges), then drag left/right
- **Cursor:** `grab` on hover, `grabbing` while dragging
- **Constraint:** Horizontal only; snaps to frame boundaries
- **Visual feedback:** Track position updates in real-time during drag
- **Release:** Commits new offsetFrame via audioStore with undo support
- **Properties panel:** Offset numeric field updates live during drag

### INT-04: Audio Track Edge Resize (In/Out Points)

- **Hit zone:** 6px around each edge (3px inside + 3px outside the visual edge)
- **Cursor:** `col-resize` when hovering within hit zone
- **Drag behavior:** Adjusts inFrame (left edge) or outFrame (right edge); minimum 1 frame width
- **Visual:** Edge renders as 2px vertical line in --color-audio-waveform at full opacity
- **Properties panel:** In/Out numeric fields update live during drag

### INT-05: Audio Track Slip Editing (per D-09)

- **Trigger:** Hold Option/Alt + drag on waveform body
- **Behavior:** Audio content slides within the defined edges without moving the edges
- **Cursor:** `ew-resize`
- **Visual:** Waveform shifts within the range bar boundaries
- **Constraint:** Cannot slip beyond audio duration limits

### INT-06: Audio Track Reorder (per D-07)

- **Trigger:** Drag from track header area (0-80px) vertically
- **Constraint:** Only among other audio tracks (cannot move above FX/content tracks)
- **Visual:** Ghost row at 30% opacity follows cursor, drop indicator line between tracks (same pattern as FX track reorder)
- **Release:** Updates track order, commits via audioStore with undo

### INT-07: Audio Track Height Resize (per D-01)

- **Trigger:** Drag the bottom edge of an audio track row (6px hit zone)
- **Cursor:** `row-resize`
- **Constraint:** Minimum 28px (matches FX_TRACK_HEIGHT), maximum 120px
- **Visual:** Track height updates in real-time; waveform re-renders to fill new height
- **Default:** 44px

### INT-08: Audio Properties Panel

- **Trigger:** Audio track selected on timeline (per D-13, D-16)
- **Location:** Right sidebar (LeftPanel.tsx), replaces current properties view
- **Layout:** Vertical stack with sections separated by SectionLabel components
- **Sections (top to bottom):**
  1. **Track Name** -- Editable text input, 12px, full width
  2. **File** -- Read-only filename in --color-text-secondary, "Replace..." button (text-button style, 10px)
  3. **Volume** -- Vertical slider (120px tall) + NumericInput (0-100%, step 1), mute toggle button (lucide `Volume2` / `VolumeX` icons)
  4. **Fades** -- NumericInput for Fade In (frames), NumericInput for Fade Out (frames), curve selector dropdown (Linear / Exponential / Logarithmic, default Exponential per D-21)
  5. **Position** -- NumericInput for Offset (frames), NumericInput for In Point (frames), NumericInput for Out Point (frames)
- **All numeric controls:** Use existing NumericInput component with drag-to-adjust behavior

### INT-09: Mute Toggle

- **On timeline:** Click track name in header area toggles mute (per D-15)
- **In properties:** Toggle button with Volume2 (unmuted) or VolumeX (muted) icon
- **Visual change:** Track waveform fades to 30% opacity when muted (per D-11)
- **Undo-able:** Mute toggle is undoable via audioStore snapshot

## Waveform Rendering Contract

### Resolution Tiers (per D-04)

| Tier | Sample Count | Use When |
|------|-------------|----------|
| Tier 1 (peak envelope) | ~100 peaks | Extreme zoom-out: pixelsPerPeak < 1 |
| Tier 2 (standard) | ~2000 peaks | Normal zoom: 1 <= pixelsPerPeak <= 4 |
| Tier 3 (detailed) | ~8000 peaks | Zoomed-in: pixelsPerPeak > 4 |

Selection formula: `pixelsPerPeak = (trackDurationFrames * currentFrameWidth) / peakCount`. Choose the tier where pixelsPerPeak is closest to the 1-4px range.

### Rendering Method

- Render on the existing Canvas 2D context in TimelineRenderer.draw()
- For each visible peak pair (min, max): draw a vertical line from center-min to center+max
- Fill style: solid fill (per D-02 default), using --color-audio-waveform
- Center line: 1px line across full range bar width at vertical center
- Clip waveform rendering to range bar boundaries (inFrame to outFrame)

### Fade Overlay Rendering

- Fade-in: Linear gradient from --color-audio-track-bg (100% opacity at bar start) to transparent (at fadeInFrames boundary)
- Fade-out: Linear gradient from transparent (at fadeOutFrames boundary from end) to --color-audio-track-bg (100% opacity at bar end)
- Matches existing transition fade overlay visual pattern
