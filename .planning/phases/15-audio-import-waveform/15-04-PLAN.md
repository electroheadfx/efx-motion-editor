After the main hydration completes, schedule async audio decode for re-creating AudioBuffers and peaks (needed for playback and waveform rendering):
```typescript
// Async: re-decode audio files for playback and waveform peaks
(async () => {
  for (const track of audioStore.tracks.peek()) {
    try {
      const fileBytes = await readFile(track.filePath);
      const arrayBuffer = fileBytes.buffer;
      const audioBuffer = await audioEngine.decode(track.id, arrayBuffer);
      const peaks = computeWaveformPeaks(audioBuffer);
      audioPeaksCache.set(track.id, peaks);
    } catch (err) {
      console.error(`Failed to decode audio track "${track.name}":`, err);
    }
  }
})();
```

Note: This async decode happens after hydration returns. The tracks appear on the timeline immediately (from store data) but waveform rendering will show empty until peaks arrive. This is acceptable -- peaks arrive within 1-2 seconds per file.
