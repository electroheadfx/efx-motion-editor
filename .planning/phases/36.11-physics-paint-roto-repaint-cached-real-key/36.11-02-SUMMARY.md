**Cached real-key repaint sessions now distinguish immutable cached base from editable live overlay, preserving no-change saves/navigation and keeping Clear/Undo limited to post-reopen paint.**

## Accomplishments

- Added session-level cached-base and live-overlay actions so clean reopen, first live edit, and empty-overlay restoration are explicit Signals-backed transitions.
- Added Studio no-new-paint Save current preservation before delete/apply paths, with compact existing status feedback.
- Protected cached-base Clear and Undo paths so they clear only new live overlay work and never load old cached pixels/history into editable stroke state.

## Decisions Made

- Cached-base open feedback uses the existing `applyMessage` status surface with copy that says the base is visible and non-editable.

- Clean cached-base Save current returns `null` after feedback instead of emitting a bridge payload, preserving cache without delete/rewrite.
