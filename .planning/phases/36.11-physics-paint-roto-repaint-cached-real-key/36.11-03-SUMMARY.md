## Decisions Made

- DOM/CSS underlay was rejected because it could not preserve the engine-rendered paper/background proportions.
- Cached repaint preview base is a visual-only engine layer; it is not editable state, not exported output, and not a saved alpha-cache input.
- Same-session navigation uses a confirmed cached frame ref so source lookup does not depend on stale React `launchContext` timing or transient `rotoPreviewFramesRef` cleanup.

## Deviations from Plan

### User-reported UAT fixes during Plan 03

**1. Additive save initially replaced prior cache**
- **Observed during:** Live UAT
- **Issue:** Painting over a cached real key replaced the old cached paint instead of adding to it.
- **Fix:** Preserved the cached repaint base after dirty edit and ensured dirty cached-key save merges the prior cached alpha plus live transparent alpha.
- **Verification:** Focused tests/typecheck passed; user later confirmed additive save worked.

**2. Live preview initially drew new strokes behind cached base**
- **Observed during:** Live UAT
- **Issue:** Saved output was correct, but live stroke preview appeared behind old cached paint.
- **Fix:** Introduced engine-side preview-base layer below live dry/display paint.
- **Verification:** Focused tests/typecheck passed; user confirmed new paint was added over old cache.

**3. CSS underlay candidate broke paper/background proportions**
- **Observed during:** Live UAT screenshot feedback
- **Issue:** The CSS/DOM underlay preserved layer order but distorted paper/background proportions.
- **Fix:** Removed CSS underlay approach and made preview-base draw engine paper/background directly.
- **Verification:** Durable-core and Studio contracts passed; user confirmed behavior improved.

**4. Fresh saved cache was lost when navigating away/back in the open Physics Paint session**
- **Observed during:** Live UAT
- **Issue:** The latest saved cache was persisted and visible after close/reopen, but same-session timeline navigation could reload stale/missing cache.
- **Fix:** Added `confirmedCachedRotoFramesRef` and made cached-reference lookup prefer accepted in-session saves before preview refs, launch context, or store fallback.
- **Verification:** `PhysicsPaintStudio.test.ts`, durable-core test, focused matrix, typecheck, and full app suite passed; user typed `approved`.
