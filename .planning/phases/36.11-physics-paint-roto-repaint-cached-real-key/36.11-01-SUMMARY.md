# Phase 36.11 Plan 01: Cached Real-Key Repaint Merge Summary

**Physics Paint Roto now saves dirty cached real-key repaints by merging prior cached alpha with new live alpha, without restoring prior pixels into editable strokes or baking paper/background.**

## Accomplishments

- Added `mergeCachedRotoAlphaFrame`, a Canvas 2D helper that draws the cached real-key alpha PNG first, then the live transparent stroke canvas, and returns a real-key frame payload.
- Wired `PhysicsPaintStudio` to preserve a cached repaint base after first dirty edit and merge that base into dirty cached-key saves through the existing `apply-canvas` payload.
- Added source-contract coverage for full-strength cached repaint base display, alpha-only merge inputs, retry-oriented failure copy, and preservation of dirty edits/base after merge failure.

## Decisions Made

- Dirty cached real-key saves now use a merge helper instead of `buildRotoOutputFrame` when a matching cached repaint base exists.
- Cached repaint base visibility is tracked separately from playback/onion/reference overlays so editing can begin without losing the prior cached alpha base.
- Merge/apply failure copy is specific (`Could not merge frame ... edits are still open`) and the frame remains dirty for retry.
