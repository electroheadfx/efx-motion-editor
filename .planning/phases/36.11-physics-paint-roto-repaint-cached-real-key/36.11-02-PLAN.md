<objective>
Deliver the no-change and live-overlay boundary slice: a user can reopen a cached real key, inspect it, leave or Save current without losing it, and only live overlay actions affect dirty state.

Purpose: Additive repaint is safe only if cached-base display is not mistaken for editable engine content. This plan hardens the clean/dirty/no-op semantics and prevents no-change saves or Clear from destroying prior cached work.
Output: Session and Studio tests plus implementation for cached-base clean reopen, first live edit dirty, clean restoration when live overlay empties, and compact no-new-paint feedback.
</objective>
