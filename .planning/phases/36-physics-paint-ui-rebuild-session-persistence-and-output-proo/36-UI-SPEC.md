> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — use existing Preact components and EFX Motion CSS-variable UI |
| Icon library | provided SVG assets from `SPECS/physics-paint-ui/icons/` for the physics paint tool rail; `lucide-preact` only for generic app/status affordances already using it |

**Sources:** `36-CONTEXT.md` D-01 through D-11, `36-RESEARCH.md` Standard Stack, `app/src/index.css`, `SPECS/physics-paint-ui/ui-connection-phase-36.md`, and `SPECS/physics-paint-ui/physics-paint-ui-after.png`.

**shadcn gate:** `components.json` was not found. The project is Preact/Vite with an established custom EFX Motion design language and the phase explicitly rejects adding a heavy UI library. Do not initialize shadcn for Phase 36.

**Layout contract:** Replace the old toolbar and diagnostics-heavy layout with exactly five regions:

1. Top bar: brush size, opacity, background, paper grain, `Grain strength`, compact readiness/status/error area, collapsed dev export in development only.
2. Left sidebar/tool rail: vertical icon buttons in the confirmed order below.
3. Main paint canvas: the largest surface, visually centered, paper/canvas background preserved.
4. Right sidebar: brush color, palette/lightweight blend preview, and tool options.
5. Bottom workflow strip: Roto canvas and Play canvas workflow tabs/segments, save/load state, and two-row physics-paint-specific timeline.

**Left rail tool order:**

| Order | Tool label | Icon asset | Interaction contract |
|-------|------------|------------|----------------------|
| 1 | Paint | `paint-mode-normal.svg` | Select paint tool without automatic physics. |
| 2 | Paint with physics | `paint-mode-physics.svg` | Select paint tool with local physics behavior. |
| 3 | Erase | `LineiconsEraser.svg` | Select erase tool and reveal erase strength in right options. |
| 4 | Undo | `MaterialSymbolsUndo.svg` | Invoke undo immediately; disabled when unavailable. |
| 5 | Clear frame | `clear-canvas-pencil.svg` | Clear active source; confirm only when clearing Play canvas/script range. |
| 6 | Apply physics to last stroke | `physics-last-stroke.svg` | Press/hold or click action mapped to existing last-stroke physics behavior. |
| 7 | Apply physics to all strokes | `physics-all-active-paint.svg` | Press/hold or click action mapped to existing all-strokes physics behavior. |
| 8 | Dry / freeze paint | `physics-dry-paint.svg` | Freeze/dry active paint via existing engine behavior. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, slider tick gaps, timeline cell gutters, compact inline padding |
| sm | 8px | Tool rail gaps, top-bar control gaps, compact panel padding, status badge padding |
| md | 16px | Default group spacing, right-panel sections, bottom-strip section separation |
| lg | 24px | Canvas-to-panel breathing room, major panel padding, destructive dialog content padding |
| xl | 32px | Width/height of compact EFX icon buttons and tool rail buttons |
| 2xl | 48px | Minimum height for top bar and bottom workflow strip rows |
| 3xl | 64px | Major layout reserve for the left tool rail and right-sidebar section group breaks |

Exceptions: icon-only interactive targets may be 32px where matching existing EFX toolbar buttons, but every icon-only control must keep at least an apparent 32px square hit target and a visible focus ring. Canvas pointer targets are governed by brush size, not layout spacing. Timeline cells may be 12px or 20px wide for frame density, but their gaps and padding must remain multiples of 4.

## Typography

Use exactly these four sizes and two weights in the rebuilt Physics Paint UI.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 600 | 1.2 |
| Heading | 16px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

**Usage rules:**

- Body is for status text, publication summaries, error explanations, and dialog body copy.
- Label is for top-bar labels, right-panel option labels, timeline row names, button microcopy, and badges. Labels may use uppercase only where existing EFX toolbar styling already does.
- Heading is for the Physics Paint title, workflow section labels, and dialog titles.
- Display is reserved for prominent current-frame or range identifiers only; do not use Display for decorative text.
- Numeric timeline/frame/status values should use tabular numerals when possible.

## Color

Use existing CSS variables from `app/src/index.css`. Values below are the dark-theme canonical values; medium/light themes should inherit via the same variables rather than hardcoded overrides.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-bg-root)` / dark `#0F0F0F` | Window background, canvas surrounding shell, dominant empty space |
| Secondary (30%) | `var(--color-bg-toolbar)` / dark `#1C1C1C`; `var(--color-bg-card)` / dark `#1A1A1A` | Top bar, left rail, right sidebar, workflow strip, cards, collapsed status panels |
| Accent (10%) | `var(--color-accent)` / dark `#2D5BE3` | Active workflow tab, active tool, primary publish action, current-frame marker, selected timeline cell/range, focus outline |
| Destructive | `var(--color-error-text)` / dark `#FF6666` | Destructive confirmation text, destructive button border/text, blocking error messages only |

Accent reserved for: active left-rail tool, `Save roto frame`, `Save play`, active `Roto canvas`/`Play canvas` tab, current timeline frame marker, selected script-play range, keyboard focus outline, and successful selection/active states. Do not use accent for every hover state or decorative borders.

Additional semantic colors allowed:

| Semantic | Value | Usage |
|----------|-------|-------|
| Success | `var(--sidebar-dot-green)` / dark `#4CAF70` | Engine ready badge, successful save/publication summary, physics-active indication |
| Warning | `var(--color-dot-yellow)` / dark `#CCAA22` or existing orange `#F59E0B` | Unsaved changes, preview-only warning, large dev export caution |
| Roto lane | `var(--color-dot-blue)` / dark `#5588FF` at reduced opacity | Roto frame cells and saved roto markers |
| Play lane | `var(--color-dot-orange)` / dark `#FF6633` at reduced opacity | Play canvas/script range bar and ghost range preview |
| Onion previous | light cyan/blue with decreasing opacity | Previous onion frames only, preview-only and not editable |
| Onion next | light orange/yellow with decreasing opacity | Next onion frames only, preview-only and not editable |

## Interaction Contract

### Workflow modes

| Mode | Active controls | Inactive controls | Primary action |
|------|-----------------|-------------------|----------------|
| Roto canvas | Paint/erase, frame navigation, onion skin, `Save roto frame`, clear current roto frame | Play lane remains visible but subdued and not editable | `Save roto frame` saves current frame and advances to the next synced frame |
| Play canvas | Paint/erase/script preview, frame count, `Play`/`Stop`, `Save play`, clear play range | Roto lane remains visible but subdued and not editable | `Save play` publishes the rendered range and keeps the window open |

Switching workflow tabs changes the editable mode. Casual timeline clicks must never convert or delete source data. Conversion between Play and Roto is explicit and destructive confirmations are mandatory.

### Bottom timeline / workflow strip

The bottom strip must include two visible rows:

| Row | Visual contract | Interaction contract |
|-----|-----------------|----------------------|
| `Roto frames` | Per-frame cells; saved frames use blue markers; current frame has accent outline/marker | Clicking a cell moves to that synced frame. Immediate previous/next frame buttons move both EFX current frame and Physics Paint frame. |
| `Play canvas` | Start square plus orange range bar; current-position marker inside the range, e.g. `[0]----•---[50]`; ghost range before save when feasible | Clicking inside a range moves the current frame/marker for inspection or preview only. It must not switch modes or convert data. |

Frame-count field defines the Play canvas range length. Custom FPS must not appear; preview uses project FPS from launch/sequence context, with 24 fps only as an internal fallback if no project FPS is available.

### Onion skin

- Roto canvas must show onion skin controls: global on/off, previous toggle, next toggle, and count.
- Default count is `1`; cap at `3` for Phase 36 unless implementation discovers a safer lower engine limit.
- Previous frames render cyan/blue with opacity decreasing by distance.
- Next frames render orange/yellow with opacity decreasing by distance.
- Onion frames are preview-only and not editable.
- Live Play preview temporarily disables onion skin and should show `Onion skin hidden during preview` only if the user needs explanation.

### Save/load/export

- `Save state` exports `engine.save()` editable JSON and is manual.
- `Load state` accepts only validated `SerializedProject` JSON and keeps the user in the rebuilt UI.
- `Save roto frame` maps to existing current-frame publish behavior and includes rendered PNG plus editable state when available.
- `Play` is non-committing preview only.
- `Save play` maps to existing play-range publish behavior, includes rendered frames and editable state, shows the success summary, and does not auto-close the window.
- Dev PNG+manifest export is visible only in development mode, collapsed in top status, and generated from the same live engine/captured frames used for proof output.

### Destructive actions

| Action | Confirmation required | Confirmation behavior |
|--------|-----------------------|-----------------------|
| Clear current Roto frame | No | Perform immediately and show `Cleared roto frame {frame}.` |
| Clear Play canvas range | Yes | Modal/dialog with affected frame span; default focus on Cancel; destructive action styled with Destructive color. |
| Save over same-mode output | No modal | Replace same-mode output and show clear success feedback. |
| Play→Roto conversion | Yes | State frame count/range and that editable Play script source will be deleted. |
| Roto→Play conversion | Yes | State affected frame span/count and that roto images in that range will be removed. |

### Keyboard shortcuts

Shortcuts are contextual to the Physics Paint window/surface only and must not trigger when focus is outside Physics Paint or inside an input.

| Context | Shortcut | Behavior |
|---------|----------|----------|
| General | `Cmd+Z` | Undo |
| General | `Cmd+Shift+Z` | Redo if available |
| General | `Esc` | Stop preview first; otherwise close dialog/cancel active interaction |
| General | `Cmd+S` | Save active workflow: `Save roto frame` in Roto mode, `Save play` in Play mode |
| General | `?` | Show shortcuts/help |
| Roto | `←` / `→` | Previous/next synced frame |
| Roto | `Shift+←` / `Shift+→` | Previous/next saved roto frame if implemented |
| Roto | `G` | Focus/go-to-frame |
| Roto | `O` | Onion skin on/off |
| Roto | `[` / `]` | Onion count down/up within 1–3 |
| Roto | `Cmd+Enter` | Save roto frame and go next |
| Roto | `Delete` | Clear current active source using warning rules |
| Play | `Space` | Play/stop preview |
| Play | `Enter` | Play/stop preview when not inside an input |
| Play | `Cmd+S` | Save play |
| Play | `Esc` | Stop preview before cancel/close behavior |
