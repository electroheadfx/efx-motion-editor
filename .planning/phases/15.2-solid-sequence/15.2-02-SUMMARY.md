## Decisions Made
- ColorPickerPopover uses dual callback pattern (onLiveChange/onCommit) to prevent undo stack flooding from continuous color picker drag events (50+ onInput events per gesture)
- updateKeySolidColorLive skips snapshot/pushAction, only updates signal and marks dirty -- single undo action created by updateKeySolidColor on commit
- Escape key in ColorPickerPopover reverts to initial color (captured via useRef at open time)
